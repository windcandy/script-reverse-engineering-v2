# 분석 가이드 — 지식해적단 대본 정보 분리·말맛 추출

> **목적**: 지식해적단 대본 40편을 분석해 "같은 정보를 어떻게 재미있게 전달하는가"의 메커니즘을 추출, 이거봐봐 채널 집필 지침에 반영.
>
> **현재 단계**: 1단계 — 정보 단위 분리 + OCR 정정 (블록 마킹만, 표기 작업 없음)

---

## 1. 작업 흐름 (5단계)

```text
원본 .txt (보존)
   ↓ 복사
[추출] 대본 - md/NN_엔진명/대본 NN.md   ← 본 작업 대상
   ↓
1단계 — 정보 분리 + OCR 정정 (블록 마킹)
   ↓
2단계 — 팩트 추출 + 출처 카테고리 (객관 모드)
        · 권위 자료 4 (통계·학술·역사·시사)
        · 비권위 자료 3 (대중·커뮤니티·일상)
   ↓
3단계 — 정보 배치 분석 (구조 모드)                ← 거시 분석
        "왜 이 정보를 + 이 순서로 + 이 위치에 배치했는가"
        · 순서·위치·밀도·연결·누적·대비 6요소
   ↓
4단계 — 말맛 표현 분석 (수사학 모드)              ← 미시 분석
        팩트가 어떤 어휘·비유·치환·슬랭으로 옷 입혀졌는가
   ↓
5단계 — 지침 변환
        "권위 팩트 어디서 / 어떤 배치로 / 비권위 옷 어디서" 가이드 추출
        → 집필 지침 반영
```

### 단계 분리 이유

- **2단계 객관 모드**: 사실만 추출, 표현·배치 무시
- **3단계 구조 모드**: 거시 — 영상 전체 흐름·블록 간 관계·A극↔B극 대비
- **4단계 수사학 모드**: 미시 — 한 블록 안의 어휘·비유·치환
- **분리 이유**: 사고 모드가 달라, 같이 하면 혼입·왜곡 발생
- **목적 부합**: "어떤 정보를 어떻게 배치해서 어떤 말맛으로 표현했나"를 비교 분석하려면 세 차원을 분리해야 정확

### 3.1 파일명 규칙

분석 작업 완료 시 파일명에 **주제 키워드 병기**.

- 형식: `대본 NN_주제키워드.md` (대본·번호는 공백, 번호·주제는 언더바)
- 예: `대본 01_감자.md`, `대본 02_입식좌식.md`, `대본 05_락스.md`
- 주제 키워드는 가급적 짧게 (2~5음절) — 영상의 핵심 주제 압축
- 기존 분석 폴더(`[추출] 대본 - 분석/`)의 명명 규칙 참고 가능 (예: `대본_05_락스의역사_분석.md` → `대본 05_락스`로 단축)
- 대기 상태 파일은 원래 이름(`대본 NN.md`) 유지, 완료 시 동시 리네임

## 4. 파일 양식 (필수 준수)

### 4.1 헤더

```markdown
# 대본 NN

> 제목: (원본의 제목 줄 그대로)
> 썸네일 문구: (원본의 썸네일 문구 줄 그대로)
> URL: <원본 URL을 꺾쇠로 감쌈>
> 서사 유형: (폴더명에 따라 — 역설엔진 / 영역확장엔진 / 인과추적엔진 / 역설+인과추적_결합형 / 다층원인수렴엔진 / 사례비교엔진 / 사례나열엔진 / 다층분기수렴엔진)
> 베이스 구조: 기승전결 (내용 기반 분할)
> 부가 표기: (해당 엔진 특유의 영역 구분이 있다면 표기, 없으면 줄 생략)

---
```

### 4.2 본문 구조 — 기승전결 4단계 헤더

```markdown
## [기]

### 토픽 블록 제목 (시작 타임)

(클린 텍스트)

### 다음 토픽 블록 제목 (시작 타임)

...

---

## [승] — (부가 표기가 있으면 병기, 없으면 생략)

### 토픽 블록들...

---

## [전] — (부가 표기 있으면 병기)

### 토픽 블록들...

---

## [결]

### 토픽 블록들...
```

### 4.3 토픽 블록 입자 — 내용 기반 분할 (필수)

- **분할 기준 = 의미 단위** (시간 단위 ❌)
- 한 타임스탬프 블록 안에 두 의미가 있으면 두 토픽으로 분리
- 두 타임스탬프 블록이 같은 의미면 한 토픽으로 묶기
- **1블록 = 1 토픽** (한 가지 개념·사건·인과·사례 묶음)
- 블록 제목은 그 블록의 핵심을 압축한 짧은 구절
- **편당 평균 15~20블록 권장** (편 길이에 따라 가변)

### 4.4 타임스탬프 표기 — 시작 시점만

- 토픽 블록 제목 뒤 괄호에 **시작 시점만** 표기 (예: `(00:00)`)
- 범위 표기(`(00:00~00:43)`) ❌ — 분할 기준이 시간이 아니므로 부정확
- 같은 타임스탬프 블록을 두 토픽으로 나눈 경우 `(00:43 초)`, `(11:17 끝)` 같이 보조 표기 허용
- 본문 안에 타임스탬프 별도 표기 불필요

### 4.5 기승전결 4단계 — 판정 기준

기승전결은 **내용 흐름**으로 분할. 시간 비율이 아님.

#### 기(起) — 발단·문제 제시

- 화제 던지기 (후킹)
- 주제 정체 소개 (필요 시)
- **메인 오픈루프** — 영상이 답할 핵심 질문 (호명 `지식해적단과 함께 알아봅니다` 포함)
- 끝점: 메인 오픈루프 직후

#### 승(承) — 전개·정보 빌드업

- 본격 정보 영역
- 주제의 한 측면을 다각도로 빌드업
- (역설엔진의 경우 "장점/위력" 영역, 영역확장엔진의 경우 "첫 영역" 등)
- 끝점: 다음 영역으로의 반전·전환 직전

#### 전(轉) — 반전·전환

- 새 정보 추가 (반전 정보)
- 주제의 "다른 면" 드러냄
- (역설엔진의 경우 "단점/함정" 영역, 영역확장엔진의 경우 "두 번째 영역" 등)
- 첫 토픽은 [전] 진입 — 전환 신호 명시 ("그런데", "결론부터 말하면" 등). 엔진별 명칭: 역설엔진 = '피벗' / 영역확장엔진 = '영역 전환점' / 인과추적엔진 = '[전] 진입 = 인과 2 확장·심화'. '피벗'은 역설엔진 한정 용어 (§4.6 부가 표기 표 정합)

#### 결(結) — 마무리·종합

- 종합 평가 + 미래 전망 + 메시지
- **새 팩트 추가 없음** (있어도 종합 평가의 보조)
- 시간 점프 (먼 과거 → 현대/미래)
- 엔딩 시그니처 ("지금까지 지식해적단의 ~였습니다")

#### 전 vs 결 — 실용 판정 테스트

해당 블록이…

1. **새 팩트를 추가하는가?** → [전]
2. **이미 나온 정보를 종합·평가하거나 미래 전망인가?** → [결]
3. **화자가 "마무리 모드"인가?** (현대 회귀 / 미래 / 일상 / 엔딩) → [결]

→ 모호하면 1번 우선. 새 정보가 들어오면 [전], 정보가 닫히면 [결].

**전환 신호 (결 시작 표지):**

- "결국", "그러니까", "결론적으로"
- "오늘날에는", "현대에 와서는", "근세를 거치며"
- "앞으로는", "미래에는"
- 시간 점프 (먼 과거 → 현대)
- 친근한 일상 회귀 ("햄버거 감자 튀김 먹으러", "강아지 발 씻겨")
- 엔딩 시그니처 등장

### 4.6 부가 표기 — 엔진별 구조 패턴

부가 표기는 **베이스 기승전결을 넘어 그 엔진의 특유 영역 구분이 있을 때만** 표기. [기]·[결]에는 미표기, [승]·[전]의 영역에만 병기.

#### "같은 표기"의 의미 — **구조 패턴 동일 / 단어 변주 가능**

엔진별 **구조 패턴**은 고정이되 **단어는 대본 주제에 맞춰 변주**한다.

| 엔진 | 구조 패턴 ([승] / [전]) | 변주 단어 예시 |
|---|---|---|
| 역설엔진 | 긍정 영역 / 부정 영역 (대칭 반전) | 위력/함정, 장점/단점, 빛/그늘, 효과/부작용 |
| 영역확장엔진 | 영역 A / 영역 B (영역 이동) | 좌식/입식, 동양/서양, 첫 지역/다른 지역 |
| 인과추적엔진 | 인과 1 / 인과 2 (확장·심화) | 표백 발명/살균 확장, 발견/확산, 원형/진화 |
| 역설+인과추적_결합형 | 빌드업 / 그림자 (인과+반전 결합) | 글로벌화/그림자, 발전/대가 |
| 다층원인수렴엔진 | 국가·구조 원인 / 개인·보완 원인+예외 | 외교 원인/한국인·예외, 시스템/문화 |
| 사례비교엔진 | 사례 A / 사례 B (병렬 비교) | 돼지 금기/소 금기, 동양/서양 사례 |
| 사례나열엔진 | 배경·원리 / 다중 사례 | 배경·원리/각국 사례, 메커니즘/실제 사례 |
| 다층분기수렴엔진 | 분기 / 수렴 (다층→하나) | 세계 분기/한반도 수렴, 다양한 시도/현재 표준 |

#### ⚠️ 부가 표기 적용 절차 — **패턴 우선 ❌, 내용 우선 ✓**

**중요: 구조 패턴은 시작점이지 강제가 아님.** 패턴이 분석을 강제하면 분석 오염의 시작.

**1단계: 내용 우선 분석 (먼저)**

대본을 읽고 **패턴 표를 잊은 채** 자연스럽게 [기]·[승]·[전]·[결] 흐름을 식별. 이때 [승]·[전]의 영역이 어떤 의미인지 자기 단어로 표현.

**2단계: 패턴 대조 (나중)**

1단계 결과를 위 구조 패턴 표와 대조.

**3단계: 판정**

- **부합**: 변주 단어로 부가 표기 작성
- **부분 부합**: 표기 작성하되 비고에 "표준 패턴 변주" 명시
- **불일치**: 4단계로

**4단계: 패턴 이탈 처리**

구조 패턴과 다르면 **억지로 끼워 맞추지 말 것**. 다음 중 하나로 처리:

- **이 대본 고유 패턴** — 헤더 비고에 명시
  ```text
  > 부가 표기: (대본 고유 패턴 — 예: "어두운 도입 / 반전된 의미")
  > 비고: 표준 역설엔진 패턴(긍정/부정) 이탈 — 부정→긍정 순서
  ```
- **엔진 분류 의심** — 유저에게 보고 후 분류 재검토
- **새 변주 패턴 등록** — 같은 엔진에서 3편 이상 동일 변주 발견 시 가이드에 변주 패턴으로 추가

#### 적용 원칙

- **내용이 패턴보다 우선** (내용이 맞춰가는 게 아니라 패턴이 따라옴)
- **이탈 보고는 미덕** (감추지 말고 보고)
- **억지 합치는 분석 오염의 시작**

### 4.7 피벗 표시 (역설엔진 등)

**피벗(모순 전환점)은 [전]의 첫 토픽 블록 제목 뒤에 `— 피벗` 병기.** 상위 헤더에는 표시하지 않음 (중복 방지).

```markdown
## [전] — 함정 빌드업

### 피벗 — 감자만 기르면 큰일 난다 (07:19)
### 단점① 전염병 취약 (07:19 중)
```

### 4.8 메인 오픈루프 표시

**메인 오픈루프**(영상 전체가 답할 핵심 질문 — 일반적으로 [기] 후반)는 **별도 토픽 블록으로 분리 + 인라인 굵게 마커**로 표시.

```markdown
### 메인 오픈루프 — 핵심 질문 압축 (00:43 초)

**[메인 오픈루프]** 왜 누군가는 X일까요? 왜 누군가는 Y일까요? 지식해적단과 함께 간단히 알아봅니다.
```

- 별도 토픽 블록으로 분리 (이전 안과 변경)
- 마커 `**[메인 오픈루프]**` 인라인 굵게
- [기]의 마지막 블록으로 위치

### 4.9 명시적 메인 오픈루프 부재 처리

일부 영상은 "지식해적단과 함께 알아봅니다" 식 호명 없이 묵시적으로 [기]를 마무리한다 (예: 대본 10_종교고기금기). 이 경우:

1. **마커 사용 안 함** — `**[메인 오픈루프]**` 표기 생략
2. **[기]의 끝점 = 묵시적 보편 원리 제시 직후**
   - 예 (대본 10): "이런 일종의 안전수칙들은 ... 종교에 녹아 전해지기 시작했습니다" — 영상이 답할 보편 원리(종교 = 사회 안전장치) 제시
   - 이 보편 원리는 영상 전체가 사례로 증명할 화두 역할
3. **헤더 비고 줄에 명시**: `> 비고: 명시적 메인 오픈루프 부재 (보편 원리 제시로 [기] 마무리)`

#### 묵시적 [기] 끝점 식별 신호

- "그래서", "이런 식으로", "결국" 등의 마무리 부사 + 보편 원리 진술
- 그 직후 첫 사례·분석으로 진입
- 사례 진입 신호: "대표적으로", "그 중에서", "특히" 등

### 4.11 팩트 추출 + 수집 가능 출처 매핑 (2단계)

#### 전체 분석의 본 목적 (1~5단계 전체)

**"어떤 팩트를 어디서 어디까지 수집해서 어떻게 어떤 방법으로 말맛을 살려 집필했는가"** 를 분석하고, 그것을 **이거봐봐 채널이 재현·상위 퀄리티 도달하기 위한 지침**으로 변환하는 것이 본 분석 전체의 목적.

| 단계 | 분석 영역 | 본 단계와의 관계 |
|---|---|---|
| 2단계 (본 §4.11) | **어떤 팩트 + 어디서 + 어디까지** (수집 범위·깊이) | 본 단계 |
| 3단계 (§4.14) | **어떻게 배치** (정보 배치·아하 모먼트) | 후속 |
| 4단계 (§4.12) | **어떤 방법·말맛으로 가공** (어휘·비유·치환·슬랭) | 후속 |
| 5단계 (§7.5) | **재현 지침 변환** (이거봐봐가 동일·상위 퀄리티 재현) | 최종 |

#### 본 단계 (2단계) 의 핵심 목적

(1) **동일·상위 수준의 정보를 수집 가능한지** (2) **그 정보를 어디서 어디까지 수집해야 하는지** 매핑한다.

지식해적단이 실제로 본 출처를 추정하는 것이 아니라, **이거봐봐가 재현·상위 퀄리티 도달에 활용할 수 있는 권위·비권위 출처를 매핑**하는 것이 본질.

#### 카테고리 7가지 + 화자 의견 (확인 불가)

**권위 자료 (4):**

| 카테고리 | 예시 |
|---|---|
| **통계** | FAO·WHO·KOSIS·WTO 등 국제·국가 통계 |
| **학술** | 논문(KCI·구글 학술)·과학 백과·전문서 |
| **역사** | 1차 사료·역사 학술서 |
| **시사** | 언론사·정부 발표·뉴스 |

**비권위 자료 (3) + 확인 불가 (1):**

| 카테고리 | 예시 |
|---|---|
| **대중** | 위키백과·나무위키·블로그(티스토리·네이버·브런치) |
| **커뮤니티** | 디시·트위터·SNS·게임·영화·드라마·인터넷 밈·예능 |
| **일상** | 한국 사회·문화 관습·속담·생활 정보·자체 영상 콜백 |
| **화자 의견 (확인 불가)** | 즉흥적 주관 평가·재치·해석 (검증 불가) |

객관 사실은 화자 발화 여부와 관계없이 권위·비권위 출처 둘 다 매핑 대상. "화자 의견"은 정말로 즉흥 주관에만 한정.

#### 출처 매핑 — 거짓 단정 금지 원칙

- **본문 명시:** 영상이 직접 언급한 출처만 단정 표기 (영화 제목·인용된 통계명 등) — 검증 가능
- **수집 가능 출처:** 이거봐봐 재현 시 활용 가능한 권위·비권위 후보 — **지식해적단이 봤다는 단정·암시 금지**
- **URL 의무:** 후보 출처마다 URL 첨부로 검증 가능성 확보
- **재현 난이도 평가:** 쉬움 / 보통 / 어려움 / 추적 불가
- **권위 깊이 평가:** 학술 1차 / 정부 통계 / 위키류 / 일반 상식

#### 표기 표준

블록 본문 뒤에 `**[팩트]**` 섹션 추가. 팩트별 표준 형식:

```markdown
- F1. (팩트 내용)
  - 영역: {권위 또는 비권위 카테고리}
  - 본문 명시: {영상 본문 직접 인용 출처 or 없음}
  - 수집 가능 출처 (이거봐봐 재현용 — 지식해적단 본 것 아님):
    - 권위: {학술·1차 사료·정부 통계 — URL}
    - 비권위 경유: {위키·블로그·신문·커뮤니티 — URL}
  - 메모: {재현 난이도·권위 깊이·검색 한계·재현 시 주의 사항}
```

##### 예시

```markdown
- F1. 면적당 칼로리: 쌀 30,000 / 밀 18,000 / 감자 54,000 kcal/ha
  - 영역: 농업 통계 (권위)
  - 본문 명시: 없음
  - 수집 가능 출처:
    - 권위: FAO Statistical Yearbook https://www.fao.org/...
    - 권위 2차: KREI "세계 감자산업 동향" https://repository.krei.re.kr/...
    - 비권위 경유: 나무위키 "감자" https://namu.wiki/w/감자
  - 메모: 난이도 보통 (FAO 영문 가공 필요), 깊이 통계 1차. 영상의 정확 수치(30000/18000/54000)는 검색 미일치 — 자체 재가공 가능성.

- F2. 화성 식민지 감자 재배
  - 영역: 영화 콜백 (비권위·커뮤니티)
  - 본문 명시: 영화 「마션」
  - 수집 가능 출처: 위키백과 "마션 (영화)" https://...
  - 메모: 난이도 쉬움 (대중적 영화).

- F3. "와 진짜 사기적인 작물이죠"
  - 영역: 화자 의견 (확인 불가)
  - 본문 명시: 없음
  - 수집 가능 출처: N/A
  - 메모: 화자의 즉흥 평가. 검증 대상 아님.
```

#### 진행 방식 — 팩트출처추적 에이전트 위임

대본 1편 단위로 `팩트출처추적` 에이전트 호출 (참조: `.claude/agents/팩트출처추적.md`). 메인은 [팩트] 추출까지만 담당하고 출처 매핑은 에이전트가 수행. 세부 절차는 §7.2 참조.

### 4.12 말맛 표현 분석 (3단계)

각 팩트가 대본에서 어떤 **어휘·비유·치환·슬랭·인용**으로 표현됐는지 추출. **권위 팩트가 비권위 옷을 어떻게 입었나** 분석.

#### 표기 방식

`**[팩트]**` 다음에 `**[말맛 표현]**` 섹션.

```markdown
**[말맛 표현]**
- "압도적인 1황" — 게임 슬랭 [커뮤니티]
- "그 효율 좋다는 쌀을 가볍게 씹어 먹을 정도" — 의인화·도발 비유 [일상]
- "오로지 덩이 줄기에만 몰빵하기 때문" — 게임 슬랭 [커뮤니티]
- "여차하면 다 자라기 전에 캐서 먹거나" — 친근한 묘사 [일상]
```

#### 핵심 발견 — "권위 팩트 + 비권위 옷 입힘"

지식해적단의 말맛 공식:

```
원자료 (FAO 통계 — 권위)
    "1ha당 54,000kcal"
        ↓
지식해적단 가공 = 권위 팩트 + 비권위 옷 입힘
    "감자는 면적당 생산 칼로리 면에서 압도적인 1황인데요"
                                          ↑
                                   게임 슬랭 [커뮤니티]
```

본 작업의 궁극 목표 = 이거봐봐 채널이 같은 공식을 적용할 수 있도록 "권위 팩트 어디서 / 비권위 옷 어디서" 가이드 추출.

### 4.13 단계별 분리 작업 원칙

**1·2·3·4단계는 별도 라운드로 진행.** 같이 하면 모드 혼입·정확도 손실.

| 단계 | 사고 모드 | 작업 대상 |
|---|---|---|
| 1단계 | 구조 식별 | 블록 마킹 (내용 기반 분할) |
| 2단계 | 객관 모드 (팩트 추출) | 팩트·출처 카테고리 |
| 3단계 | **구조 모드 (정보 배치)** | **순서·위치·밀도·연결·누적·대비** |
| 4단계 | 수사학 모드 (말맛 분석) | 어휘·비유·치환·슬랭 |

→ 1단계 40편 완료 → 2단계 40편 진입 → 3단계 40편 진입 → 4단계 40편 진입.

### 4.14 정보 배치 분석 (3단계 작업) — 거시 분석

각 영상의 정보 배치 의도를 분석. **왜 이 정보를 + 이 순서로 + 이 위치에 배치했는가** 답을 추출. 표기·팩트만으로는 안 보이는 **서사 구조·아하 모먼트 극대화 메커니즘** 분석.

#### 배치 분석 6요소

| 요소 | 분석 질문 |
|---|---|
| **1. 순서 (Order)** | 시간순? 친숙도순? 충격 강도순? 빌드업순? |
| **2. 위치 (Position)** | [기]에 무엇? [승] 첫·중간·끝(절정)? [전] 진입? [결] 마무리? (※ [전] 진입 명칭 엔진별 차이 — 역설엔진 = '피벗' / 영역확장엔진 = '영역 전환점' / 인과추적엔진 = '[전] 진입 = 인과 2 확장·심화' — §4.6 정합. '피벗'은 역설엔진 한정 용어) |
| **3. 밀도 (Density)** | 한 블록 정보량? 빠른 진행 vs 느린 빌드업? |
| **4. 연결 (Connection)** | 블록 간 연결어 ("그런데", "심지어", "결정적으로", "여기다") |
| **5. 누적 (Cumulation)** | 점진 빌드업? 한 번에 폭발? 정점이 어디인가? |
| **6. 대비 (Contrast)** | A극 vs B극 어떻게 대비해서 아하 모먼트 극대화? |

#### 표기 방식

블록 본문 + `**[팩트]**` + `**[배치]**` 섹션 추가.

```markdown
### 인구 폭증 + 휴경지 활용 [인과] (06:39)

(본문)

**[팩트]**
- F1. 100년 만에 유럽 인구 1.5~2배·아일랜드 6배 [통계]
- F2. 3포제 윤작 + 감자 휴경지 활용 [학술]

**[배치]**
- 위치: [승]의 8번째 = 마지막 블록 (절정)
- 순서 의도: 안데스 기원(시작) → 칼로리·전쟁·전래·강제 보급(빌드업) → 인구 폭증(정점)
- 누적: 점진 빌드업의 정점 → A극(긍정) 절정 형성
- 대비: 이 정점이 있어야 다음 [전] 피벗("그런데 큰일 난다")의 충격 극대화
```

#### 분석 단위 — 3계층

배치 분석은 다음 3계층에서 수행:

| 계층 | 분석 대상 |
|---|---|
| **블록 내부** | 블록 안 정보 나열 순서 |
| **블록 간** | 블록 순서·연결어·누적 흐름 |
| **극 간** | [기]↔[승]↔[전]↔[결] 흐름, A극↔B극 대비, 아하 모먼트 위치 |

#### 엔진별 배치 패턴 (예측)

| 엔진 | 특유 배치 패턴 |
|---|---|
| 역설엔진 | A극 점진 빌드업 → 정점 형성 → 피벗 폭발 → B극 빌드업 → 종합 |
| 영역확장엔진 | 좁은 영역 → 넓은 영역 확장 또는 가까운 영역 → 먼 영역 |
| 인과추적엔진 | 시간순 인과 체인 (옛날 → 현재) |
| 사례비교엔진 | 사례 A 깊이 → 사례 B 깊이 → 공통 원리 |
| 사례나열엔진 | 친숙도순 또는 충격 강도순 사례 나열 |
| 다층원인수렴 | 다층 원인 → 수렴 결과 |
| 다층분기수렴 | 분기 다양성 → 수렴 |
| 결합형 | 두 엔진 패턴 혼합 |

→ 40편 분석으로 패턴 검증·정교화.

### 4.14.1 추가 분석 4축 (배치 메커니즘 추출)

§4.14 6요소(순서·위치·밀도·연결·누적·대비)는 표면 관찰. 본 분석 목적("어떤 정보를 어떻게 배치할지 지침 만들기") 달성을 위해 다음 4축을 동시 분석.

**축 A — 정보 유형 표기 (9유형):**

| 유형 | 정의 |
|---|---|
| 기원·배경 | 대상이 시작된 시점·맥락 |
| 정의·개념 | 대상이 무엇인가 |
| 통계·수치 | 정량 데이터 |
| 인과 | 원인→결과 사슬 |
| 사례·일화 | 구체 케이스 |
| 비교·대비 | A vs B |
| 인용·증언 | 권위 발언·역사 기록 |
| 메커니즘 | 작동 원리 |
| 해석·결산 | 화자의 종합 판정 |

**축 B — 인지 부담 등급 (3등급):**

- 낮음: 직관·친숙·일반 상식
- 중간: 낯선 개념 1개
- 높음: 다층 인과·전문 통계

**축 C — 레고 이론 작동 (시청자 추리·보상 사이클, 4역할):**

> **레고 이론:** 시청자는 직접 추리하고 맞췄을 때 성취감을 느낀다. "내가 영리하다"고 느끼게 하는 콘텐츠가 몰입을 만든다.
>
> - **의도적 정보 지연:** 예상을 살짝 비틀거나 더 깊은 정보 던지기
> - **보상 불균형 금지:** 루프가 긴 만큼 보상 정보 밀도가 충분해야 함

각 블록 4역할 분류:

- 오픈루프 — 지적 공백 생성·질문 제시 (답을 의도적으로 차단·지연해 시청자 궁금증 유발)
- 의도적 지연 — 예상 비틀기·추가 정보 투입 (단서 제공해 시청자 추측 자극)
- 보상 투하 — Aha-moment·역전 팩트·통찰 회수
- 충전·빌드업 — 다음 보상을 위한 사전 누적

**끌리는 이유 옵션 (전략 결정 지점 한정):** [전] 진입(역설엔진 = 피벗 / 영역확장엔진 = 영역 전환점 / 인과추적엔진 = [전] 진입)·Aha-moment·도입부·엔딩 등 핵심 블록에서는 "끌리는 이유 (이문호공 N번 + 작동 메커니즘 부연)" 한 줄을 추가 표기. 일반 블록은 생략. 이문호공 10요인 정의는 `00_지식파일/03 이문호공.md` 참조.

**작동 메커니즘 부연 형식:** 이문호공 요인 명칭 직후 괄호로 "왜 이 블록에 이 요인을 매칭했는가" 한 줄 명시.

- 예: `3 호기심형 ("왜 지평선이 특산물?" 정보 공백)`
- 예: `5 상식 파괴형 ("축제 = 평범" 통념 정정)`

**축 D — 보상-루프 균형 평가:**

루프 길이 vs 보상 밀도 매칭:

- ✓ 짧은 루프 + 강한 보상
- ✓ 보상 직후 다음 루프 즉시 개방
- ❌ 긴 루프(2분+) + 약한 보상

### 4.14.2 표기 양식 (확장)

블록 본문 + 5섹션 동시 표기.

```markdown
### 블록 제목 (시작 타임)

(본문)

**[팩트]** (2단계 결과 유지)
- F1. ...

**[배치]** (6요소)
- 위치: ...
- 순서 의도: ...
- 누적: ...
- 대비: ...

**[정보유형]** (축 A)
- 주 유형: 통계·수치
- 부 유형: 인과

**[인지부담]** (축 B)
- 등급: 중간
- 근거: 낯선 개념 "3포제 윤작" 등장

**[레고작동]** (축 C)

- 회수 대상: [기]에서 던진 "왜 인구가 폭증?" 오픈루프

**[보상균형]** (축 D)
- 루프 길이: 약 3분
- 보상 밀도: 강 (1.5배·6배 수치 + 인과 명시)
- 평가: ✓
```

## 5. OCR 정정 원칙

원본 .txt는 PDF에서 변환된 텍스트라 OCR 오류가 많음.

### 5.1 정정 (FIX)

- 단어 깨짐: `미를→밀을`, `5,000년을 지음해→즈음해`, `1타르당→1헥타르당`, `선늘하고→선선하고`
- 불완전 단어: `k칼로리→킬로칼로리`, `kc리→킬로칼로리`
- 중복 글자: `이 이 동네→이 동네`, `심지어이→심지어 이`, `감자는이→감자는 이`
- 띄어쓰기 깨짐: `그런데이→그런데 이`, `사실이→사실 이`
- 의미상 명백한 오류: `남자→감자`, `미래→밀의`, `함유되 있기→함유돼 있기`, `창고→창궐`, `복재→복제`, `씨눈(신혼)`, `잉카(인카)`, `케추아(캐추아)`, `구세계(9세계·두 세계)`, `자의로(자이로)`, `자라니까(자니까)`, `기른(길은)`, `폭우(포구)`
- 추정 단어: 영상 맥락상 명확한 단어 (예: `최중→[삭제하여 원문 의미만 살림]`)

### 5.2 보존 (KEEP)

- 음성 톤·강조어: `오지게`, `미친 생존력`, `엥?`, `똥땅`, `4대장`, `1황`
- 채널 시그니처: `지식해적단의 키드였습니다`
- 슬랭·신조어: `뇌정지`, `쉴드 쳐`, `좌식러`, `입식러`, `원툴`, `크리`
- 신체적 묘사어: `골로 가는`, `삥 뜯어가는`
- 음악 표기: `[음악]` 그대로 둠

### 5.3 화자 모순 처리 — 원본 보존 원칙

화자가 같은 수치를 두 번 다르게 말한 경우 (예: 대본 01에서 아일랜드 인구 증가율을 06:39 `여섯 배` / 08:10 `여덟 배`로 다르게 언급) → **원본 그대로 보존**. 일관성 통일 금지.

이유: 화자의 실수도 분석 대상이며, 원본 보존이 원칙.

### 5.4 줄바꿈 처리

- 원본의 PDF 줄바꿈(중간 단어가 끊긴 부분) 제거 → 토픽 블록 단위로 한 단락
- 페이지 번호 (`01    1`, `02    3` 등) 삭제
- 영상 끝의 `>> [음악] [음악]` 같은 의미 없는 잡음 표기 삭제

| 엔진 | [전] 진입 용어 |
|---|---|

| 사례비교엔진 | 사례 전환점 |
