# 대화 히스토리 — 지식해적단 대본 분석 작업 시작

> 일자: 2026-05-30
> 주제: 지식해적단 대본 25편(현재 40편 분류 확인)을 분석해 "같은 정보를 어떻게 재미있게 전달하는가"의 메커니즘을 추출하여 이거봐봐 채널 집필 지침에 반영하는 작업 시작

---

## 1. 문제 제기 (유저)

> "지식해적단 대본을 분석하여 팩트리포트를 어떻게 말맛을 살려 재미있게 전달하는가를 분석하여 이것을 바탕으로 현 지침을 전면 수정 또는 보완 등을 하려고한다. 같은 정보를 전달해도 지식해적단은 매우 지루하지 않게 스토리텔링을 재미있게 전달한다. 그 이유를 찾아 지침으로 만드는 것이 목적이다. 어떤 방식과 순서로 분석하는 것이 의도에 맞는 결과물을 만들어 내는데 적합한지 제안해줘"

## 2. AI 1차 제안 — 4단계 귀납 분석

1. **대상·축 확정**: 재미 상위 5~7편 + 평작 2편 대비 / 6개 분석 축(정보→말맛 변환, 문장 리듬, 화자 스탠스, 정보 배치, 장면 전환, 긴장-이완 사이클)
2. **팩트 원문 ↔ 대본 1:1 대조** (핵심 — 변환 규칙 추출)
3. **메커니즘 명명·빈도 집계**
4. **지침 변환** — 현 지침 차분 비교 후 신설/보완/강화 판정

## 3. 유저 수정 — 정보 단위 마킹 선행

> "내 생각에는 우선 대본을 md 파일로 복사하여 대본 안에 정보 단위로 구분하여 표시를 먼저 하는 것이 어떠한가?"

→ AI 동의. **정보 단위 분리가 선행되어야** "이 단위를 어떻게 말맛으로 변환했는가"를 분리해 볼 수 있다.

## 4. 폴더 구조 확인

유저가 IDE 스크린샷으로 실제 폴더 구조를 보여줌:
- 기존 `[대본] 모순형 / 이미지강화형 / 과정추적형 / 추가 (미분류)` 분류가 아니라
- **8개 엔진 분류**로 재편됨: 01_역설엔진 ~ 08_다층분기수렴엔진

```
[추출] 대본/
  01_역설엔진/        대본 04, 01, 03, 14, 19
  02_영역확장엔진/    대본 02, 07, 08, 11, 17, 24(광고), 30
  03_인과추적엔진/    대본 05, 06, 12(광고)~41 (18편)
  04_역설+인과추적_결합형/ 대본 23, 29
  05_다층원인수렴엔진/ 대본 09, 26, 27(광고)
  06_사례비교엔진/    대본 10, 32, 40
  07_사례나열엔진/    대본 20
  08_다층분기수렴엔진/ 대본 31
  지식브런치/         (참고)
```

→ 총 40편 (광고/중복 포함).

## 5. txt → md 일괄 복사

유저 승인 → 40개 파일을 `[추출] 대본 - md/` 폴더에 원본 그대로 .md 확장자로 복사 완료. 원본 .txt는 보존.

## 6. 대본 01 시범 마킹 (1차)

유저: "대본 01 먼저 진행해줘"

AI: 11종 마킹 기호 양식 제안 (`[F#]`, `[훅]`, `[질문]`, `[여담]`, `[전환]`, `[모순]`, `[피벗]`, `[위트]`, `[청자호명]`, `[비유]`, `[인용]`).

유저 승인 → **HTML 주석 + 4단계 구조 라벨([도입부]/[본문—장점]/[피벗]/[본문—단점]/[결론부]) + F1~F50 50개 팩트 마킹** 적용.

## 7. 유저 수정 — 토픽 단위 정보 분리 + OCR 정정

> "아래와 같이 정보 분리를 해야지"
> '기원·생존력 (01:34~02:19)' (클린 텍스트)
> '칼로리·생장 속도 (02:19~03:04)' (클린 텍스트)
> '아일랜드 대기근 (08:10~09:39)' (클린 텍스트)

→ 문장 단위 F# 마킹이 아니라 **토픽 단위 정보 블록**으로 재분리 + OCR 깨짐 정정 필요. 음성 톤(`오지게`, `미친 생존력`, `엥?`)은 유지.

유저: "기존 주석본 파일은 무시하고 진행해"

AI 재작업 → 대본 01을 **17개 토픽 블록** 4단계 구조로 재구성:
- [도입부]: 인트로 / 감자 정체
- [본문—장점]: 기원·생존력 / 칼로리·생장 / 조리·전쟁 / 유럽 거부 / 솔라닌 / 맛 문제 / 강제 보급 / 인구 폭증
- [피벗]: 감자 원툴의 위험
- [본문—단점]: 전염병 / 아일랜드 대기근 / 재배 난이도 / 환경 제약 / 보관·운송 / 추뇨
- [결론부]: 정착·냉동 기술 / 미래·엔딩

OCR 정정: `미를→밀을`, `5,000년을 지음해→즈음해`, `1타르당→1헥타르당`, `k칼로리→킬로칼로리`, `시감자→씨감자`, `창고→창궐`, `나마시아→나미비아` 등.

## 8. 기승전결 베이스 + 부가 라벨 병기 도입

유저: "다른 대본들은 현 지침 구조에 안맞을 수도 있으니 기본은 기승전결로 나누고 현 지침에 구조가 맞으면 옆에 추가로 정보를 기재하는 것은 어떤가?"

→ 보편 4단 구조(기승전결)를 베이스로, 엔진 특유 라벨(역설엔진의 장점/단점 빌드업 등)이 있으면 병기.

```
## [기] (00:00~01:34)
## [승] — 장점 빌드업 (01:34~07:19)
## [전] — 단점 빌드업 (07:19~12:49)
## [결] (12:49~14:21)
```

유저 승인 → 대본 01 헤더 반영.

## 9. 헤더 라벨 간소화 — 도입부/본문/결론부 제거

유저: "도입부/본문/결론부 표시는 없어도 될 것 같다"

→ 기·결의 부가 라벨 제거. 부가 라벨은 [승]·[전]에 영역명만 표기.

```
> 부가 라벨: 장점 빌드업 / 단점 빌드업

## [기] (00:00~01:34)
## [승] — 장점 빌드업 (01:34~07:19)
## [전] — 단점 빌드업 (07:19~12:49)
## [결] (12:49~14:21)
```

## 10. 헤더 라벨 추가 간소화 — 피벗 제거

유저: "'감자 원툴의 위험 신호 (07:19)' 여기에는 피벗 표시가 되어 있으니 '[전] — 단점 빌드업 (07:19~12:49)' 여기에는 피벗 표시를 하지 않아도 될 것 같다"

→ 상위 헤더에서 "피벗" 제거. 토픽 블록 제목 `### 감자 원툴의 위험 신호 — 피벗 (07:19)`에만 표시.

## 12. 대본 02 (영역확장엔진) 마킹 완료

14개 토픽 블록 / 4단계 구조:
- [기] 미국 영화 충격 / 입식·좌식 정의
- [승] — 좌식 영역: 한중일 디딤돌 / 다용도 공간 / 군대 침상 / 일본군 유래 / 동아시아 우기 / 유목민 / 이란·아랍 양탄자
- [전] — 입식 영역: 유럽 입식 / 가축 동거 / 난방 가성비 / 입식 내 스펙트럼 / 영국 슬리퍼 / 19세기말 위생
- [결]: 미국 PPL 논리 / 팬데믹 후 변화

OCR 정정: `가혹→가옥`, `포구→폭우`, `뒤르크→튀르크`, `배두인→베두인`, `신뢰화→실내화`, `벽날로→벽난로`, `천참한 별→천차만별`, `남아→남하`, `입직→입식` 등.

## 합의된 핵심 원칙

1. **베이스 구조**: 기승전결 (모든 대본 공통)
2. **부가 라벨**: 엔진 특유 영역 구분이 있을 때만 [승]·[전] 헤더에 병기. [기]·[결]에는 미표기
3. **피벗 등 진입 신호**: 토픽 블록 제목 뒤 `— 피벗` 식으로만 표시. 상위 헤더 중복 금지
4. **OCR 정정**: 단어 깨짐 정정하되 음성 톤·강조어·슬랭은 보존
5. **줄바꿈**: 타임스탬프 단위로 한 단락. 페이지 번호 삭제
6. **토픽 블록 입자**: 편당 12~17블록 권장
7. **검증 순서**: 폴더별 첫 파일 1편씩 8편 → 양식 검증 → 32편 확장
8. **단계**: 1단계 정보 분리 (현재 진행) → 2단계 말맛 추출 → 3단계 지침 변환
