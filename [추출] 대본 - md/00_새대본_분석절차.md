# 새 대본 분석 절차 (자기완결 마스터)

> **용도:** 지식해적단 새 대본이 `09_미분류`에 추가될 때마다 **본 파일 하나로 분석 전 과정을 수행**한다.
> **자기완결 원칙:** 절차·판정 기준·색인 갱신은 본 파일 안에서 완결한다. 단 **말맛 4단(에이전트 실행 도구)·이문호공 10요인·9유형 상세 정의**만 원 출처를 참조한다.

---

## 0단계 — 사전 준비

1. txt에서 YouTube URL 추출 (없으면 유저에게 요청)
2. 썸네일 다운로드 `https://img.youtube.com/vi/{영상 ID}/maxresdefault.jpg` → **썸네일 문구는 이미지에서 추출**(제목 ≠ 썸네일)
3. txt 통독 → 8엔진 1차 분류 → `09_미분류` 임시 배치
4. md 생성 + `09_미분류 대본/` 폴더 배치 → 원본 txt 삭제
5. OCR 정정 (의미 명백한 것만, 화자 실수는 보존)
6. 헤더 작성 (제목·썸네일 문구·URL·서사 유형 잠정·부가 표기 잠정)

---

## 1단계 — 블록 마킹 + 엔진 확정

- **[기]·[승]·[전]·[결] 4단 구분** (내용 기반)
- 토픽 블록 단위 분할
- **메인 오픈루프(ML) 블록 표시** — 화자 직접 질문 형태 ("왜 ~?·어떻게 ~?"). 슬로건·화제 선언 금지
- 1단계 완료 후 **8엔진 판정** → 확정 시 엔진 폴더로 이동 + 헤더 갱신 / 불일치 시 09 유지·유저 보고

**서사 유형 ↔ 8엔진 매칭:** 모순형 = 역설·결합형·다층원인수렴 / 이미지강화형 = 영역확장·사례비교 / 과정추적형 = 인과추적 (보류 2: 사례나열·다층분기수렴)

**엔진별 [전] 진입 용어 (본 파일 자기완결):**

| 엔진 | [전] 진입 용어 |
| --- | --- |
| 역설엔진 | 피벗 |
| 영역확장엔진 | 영역 전환점 |
| 인과추적엔진 | [전] 진입 |
| 결합형(역설+인과추적) | 역설 요소=피벗 / 인과 영역=[전] 진입 |
| 다층원인수렴엔진 | [전] 진입 |
| 사례비교엔진 | 사례 전환점 |
| 사례나열엔진 | [전] 진입 — 사례 나열 진입 |
| 다층분기수렴엔진 | 수렴 진입 |

> "피벗"은 역설엔진(속성 대립 폭발) 한정 — 다른 엔진 부적합.
> 블록 4단 판정·엔진별 부가 표기 상세: `00_가이드_분석_방법.md` §4.5·§4.6

## 2단계 — 팩트 추출 + 출처 매핑

- 각 블록 `**[팩트]**` 섹션 (F1·F2…)
- **카테고리:** 권위 4(정부·국제기구 / 학술·연구기관 / 권위 언론 1차 / 전문가·칼럼) + 비권위 3(위키·민백 / 나무위키 / 블로그·언론) + 화자 의견
- 권위 1차(원자료) + 비권위 경유 출처 매핑
- 상세: 가이드 §4.11

## 3단계 — 정보 배치 분석 (블록별 5섹션)

각 블록 [팩트] 직후에 5섹션 추가:

- **[배치]** — 6요소(위치·순서·누적·대비·밀도·연결)
- **[정보유형]** — 9유형 중 주 유형·부 유형 (정의·개념 / 사실·결과 / 인과 / 메커니즘 / 사례·일화 / 기원·배경 / 통찰·해석 / 결산 / 부가). 다중 주 유형 금지
- **[인지부담]** — 3등급(낮음·중간·높음) + 근거
- **[레고작동]** — 4역할(정보 / 오픈루프 / 회수 / 보상 투하) + 회수 대상 + **전략 지점만** "끌리는 이유"(이문호공)
- **[보상균형]** — 루프 길이·보상 밀도·평가

> 전략 지점 5종: 도입 후킹·메인 오픈루프·[전] 진입·Aha-moment·엔딩
> 4축 상세: 가이드 §4.14·§7.3 / **이문호공 10요인**(1이익·2문제제기·3호기심·4공감·5상식파괴·6반전·7경고·8비교·9최신정보·10도덕공분) 정의: `00_지식파일/03 이문호공.md`
