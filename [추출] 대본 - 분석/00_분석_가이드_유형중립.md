# 분석 가이드 — 끼워맞춤 사고 금지

> 작성일: 2026-05-29
> 적용 대상: 미분류 14편 주석본 분석
> 핵심 목적: 현행 모순형·이미지강화형·과정추적형 지침에 본문을 억지로 끼워맞추지 않고, 본문이 보여주는 작동 방식 그대로 묘사 후 신유형 발견

---

## 핵심 원칙

**금지: 끼워맞춤 사고**

본문 작동과 다른 분류 라벨을 본문에 강제 부여하지 않는다.

**허용: 기존 용어 사용 (단, 본문 작동에 실제로 부합할 때만)**

용어 자체는 자유롭게 사용. 단어 바꿔치기는 혼란만 유발. 핵심은 단어가 아니라 **분류 라벨을 본문에 강제하느냐**.

---

## 원칙 1 — 본문 단위 표기

본문 단락 단위 이름은 그 단락이 **실제로 어떤 작동을 하느냐**에 따라 부여.

| 단락 작동 방식 | 단위 이름 |
|---|---|
| 서브루프 작동 (작은 갈증→해소) | SL |
| 원인 누적 (서브루프 없음) | 원인 레이어 |
| 두 사례 평행 비교 | 사례 N — 단계 N |
| 시간 흐름 전환점 | 전환점 (연도·인물·사건) |
| 단순 메커니즘 설명 (갈증 없음) | 메커니즘 단락 |
| 핵심 논지 외 보강·부록 | 부록 케이스 |
| 통념 깨기 | 통념 교정 단락 |
| 다층 분기 후 한 점 수렴 | 분기 → 수렴 |

→ 한 대본 안에 여러 단위가 섞여도 무방 (예: 09에서 SL6은 SL, SL3~5는 원인 레이어, SL8~9는 부록 케이스)

→ 위 표에 없는 작동이면 새 이름 만들기 + 가이드에 추가

**❌ 금지:** 모든 단락을 SL로 일괄 표기 (서브루프 작동 안 하는 단락도 SL 표기)

## 원칙 1-1 — SL 표기 세부 원칙 (필수, 2026-05-29 신설)

SL은 **서브 오픈 루프 = 지적 공백 유발 단위**다. 본 원칙은 SL 판정의 텍스트 근거를 의무화하여 같은 오류(사실 누적 단락을 SL로 라벨링)를 차단한다.

### SL 판정 기준 (3축, 2026-05-29 강화)

SL은 시청자가 갈증을 안고 본문을 따라가게 만들어 **이탈 방지·몰입 강화**하는 단위. 다음 3축을 적용:

| 판정 축 | SL 부합 | 브릿지 부합 | 자문자답 어법 부합 |
|---|---|---|---|
| **① 시청자 갈증 강도** | 강 (시청자가 "정말 왜?·어떻게?·뭐?" 직접 의문) | 약 (정보 환기 수준, "또 뭐가 있지?" 정도) | 강 또는 중 (화자가 시청자 의문 대리 발화) |
| **② 본문 응답 길이** | 단락 이상 이연 | 1~2문장 환기 (응답은 다음 단락이나 동력 약함) | 1~2문장 안 즉시 답 |
| **③ 화자 어법 형태** | 직접형 (직접 질문·선언·반전·미스터리 등 시청자 갈증을 본격 여는 어법) | 환기형/접속형 ("X만이 아니다"·"또한"·"심지어"·"물론") | 자문자답형 ("엥? X 아니냐?" + 즉시 답) |

**판정 우선 순위:**

- **SL**: ①·③ 부합 + ② 부합 (3축 모두 부합)
  - ① 강함만으로는 부족 — 응답이 즉시면 자문자답
  - ② 길이만으로는 부족 — 갈증 약하면 브릿지
- **자문자답 어법**: ③ 부합 + ② 즉시 (갈증 강도와 무관)
- **브릿지**: ① 약함 + ③ 환기·접속형

→ 사실을 단순 누적·증명·결산하는 단락은 SL 아님. 누적 단위명 부여 (A극 누적 단계 N / B극 누적 단계 N / 정점 결산 / 증명 케이스 등).

### SL 표기 필수 3요소 (2026-05-30 본질 회귀)

모든 SL은 다음 3요소를 명시한다.

```markdown
#### SL{N} — 키워드 (시간 범위)

> [지적 공백을 연 본문 문장]: "본문에서 지적 공백을 연 실제 문장(들) 직접 인용"
> [지적 공백 열림]: 시청자가 갖게 되는 궁금증을 한 문장으로 명시
> [해소 위치]: 본 SL 안 / 다음 단락 / 본문 후반 (단순 표기)
```

→ "즉시 / 점진 / 분산" 분류는 SL 정의(응답 단락 이상)와 중복이므로 폐기. "해소 완료 지점" 시간 + 인용 정밀 표기는 정량 분석으로 본질 작동에 과잉이므로 폐기.

**[지적 공백을 연 본문 문장] 식별 자기 점검 질문 (2026-05-29 신설):**

본 문장이 SL을 여는 본격 문장인가, 단순 매개 브릿지인가? 다음 질문으로 점검:

1. 시청자가 본 문장 자체를 읽고 "**정말 왜?**", "**어떻게?**", "**뭐?**" 직접 의문 발생하는가? → **SL 본격 문장**
2. 본 문장은 단순 매개일 뿐, 다음 문장(들)에서 본격 갈증 유발 → **브릿지** (SL 본격 문장 아님)

```markdown
❌ 잘못된 SL 본격 문장 식별 예시:
> [지적 공백을 연 본문 문장]: "하지만 가뭄 앞에서는 장사 없지요"

  → 본 문장은 격언/환기 어법, 시청자가 본 문장 자체에서 의문 발생 안 함.
  → 다음 문장("어떻게든 감자를 먹이기 위해 강제적인 수단부터 고도의 심리전까지 온갖 방법을 동원하기 시작합니다")에서 본격 갈증 유발.
  → 본 문장은 SL을 여는 본격 문장 아닌 브릿지.

✅ 올바른 SL 본격 문장 식별:
> [지적 공백을 연 본문 문장]: "이런 아사리판의 와중에도 차라리 굶어 죽으면 죽었지 안 먹겠다는 사람들이 대부분이라서 유럽 각국에 높으신 분들은 백성들한테 어떻게든 감자를 먹이기 위해 강제적인 수단부터 고도의 심리전까지 온갖 방법을 동원하기 시작합니다"

  → 시청자가 "어떤 수단? 어떤 심리전?" 직접 의문 발생.
  → 본 문장이 본격 SL을 여는 문장.
```

**격언·환기·접속 어법은 보통 브릿지** — "하지만 X 앞에서는 Y 없지요", "X만이 아니다", "물론 X지만 Y" 등은 매개 어법으로 작용. 시청자 갈증을 본격 여는 직접 질문·선언·반전·미스터리 어법이 SL을 여는 본격 문장.

**SL 판정 3축 점검 라벨 HTML 주석 처리 (2026-05-29 강화):**

[SL 판정 3축 점검]·[SL 아님 판정 근거 — 3축 점검] 라벨은 검증용 정보이므로 마크다운 렌더링 시 가독성 위해 **HTML 주석 `<!-- ... -->`으로 처리**한다.

- 소스 파일에서는 보존되어 검증 흔적·향후 재분석 시 참조 가능
- 마크다운 렌더링 시 본문 흐름과 분리되어 가독성 향상
- SL/자문자답 어법/브릿지/B극 메커니즘 단락 등 모든 분류 판정 근거에 동일 적용

```markdown
✓ 권장 형식:

<!--
[SL 판정 3축 점검]:
  ① 갈증 강도: 강
  ② 응답 길이: 단락 이상
  ③ 화자 어법: 직접형
  → 3축 모두 SL 부합
-->
```

**Aha 발견 ≠ SL 분류 조건 (2026-05-29 신설):**

Aha 발견(역설 완성·핵심 통찰 도달)을 담은 단락이라도 **새 갈증을 여는 단위가 아니면 SL 아님**. Aha는 단락의 **효과**이지 SL **분류 조건** 아님.

- 피벗/직전 SL의 응답 단락이 Aha를 담을 수 있음 → 이때는 **응답 단락(메커니즘 단락)으로 분류**, SL로 분류 X
- 본 단락이 새 갈증("정말 왜?·어떻게?·뭐?")을 여는가? → 그렇다면 SL
- 본 단락이 직전 갈증의 응답인가? Aha를 담더라도 응답 단락 → SL 아님

```markdown
❌ 잘못된 분류 예시:
[피벗]: "감자만 기르면 안 되냐? → 큰일 날 수도"
[옛 SL3 = Aha 단락]: "덩이줄기 = 유전 다양성 X = 전염병 취약" (Aha 발견)

  → 옛 SL3은 피벗 "왜 큰일?" 갈증의 1차 응답 단락이므로 SL 아님.
  → Aha를 담더라도 새 갈증 영역을 여는 게 아니라 응답 단락.

✅ 올바른 분류:
[피벗 응답 단락 (Aha 발견 포함)]: 피벗 "왜 큰일?" 갈증의 1차 응답 메커니즘 + Aha 효과
```

### 누적 단계 표기 형식 (SL이 아닐 때)

```markdown
#### {영역} 누적 단계 N — 키워드 (시간 범위)

> [지적 공백 유발 없음 — X 누적/증명/결산]
> [본 단계 사실]: 단락이 전달하는 핵심 사실
> [어법]: 직전 단계와 연결 어법 (수치 비교·점층·대조 등)
```

→ "[작동 방식]" 라벨은 폐기 (가이드 §라벨 표기 원칙 참조). 헤더에 "누적 단계"가 명시되어 있으므로 본문 구조 역할은 중복 라벨 불필요.

### 피벗 (역설 엔진 특수 단위 — ML 재오픈)

피벗 단락처럼 본문 메인 루프가 재오픈되는 큰 갈증 단위는 "피벗"으로 표기. SL 표기 필수 3요소 동일 적용. 피벗은 일반 SL 번호 순서와 분리하여 별도 라벨 부여 (예: 역설 엔진 모순형 표준).

**피벗 예외 조항**: 피벗은 형식상 자문자답 어법(예: "감자만 기르면 안 되냐? → 큰일 날 수도")처럼 보일 수 있으나, 응답이 본문 거대 영역(B극 전체 등)으로 이연되어 시청자 갈증을 분 단위로 끌고 가면 **SL 본질 우선** — 피벗으로 분류한다.

### 자문자답 어법 (SL과 분리, 2026-05-29 신설)

자문자답 어법은 화자가 시청자가 가질 법한 질문·반박을 대리 발화하고 **1~2문장 안 즉시 응답**하는 어법. 시청자 호기심 점화·호흡 조절·친밀감 형성 기능. SL 아님.

**자문자답 어법 vs SL 핵심 차이:**

- SL = 시청자가 갈증을 안고 본문을 분 단위로 따라가게 만드는 **이탈 방지·몰입 강화** 단위
- 자문자답 어법 = 화자가 즉시 답 주는 어법. 시청자가 갈증 안고 따라가지 않음

**자문자답 어법 표기 형식:**

```markdown
"본문 자문자답 문장 인용 (질문 + 즉시 답)"

> [자문자답 어법]: 본 문장은 SL이 아닌 화자 자문자답 어법
> [어법 묘사]: 본문 어법 그대로 묘사 (분류 라벨 강제 X — 예: "엥? X 아니냐? → 즉시 답" 형태)
> [SL 아님 판정 근거]: 응답이 1~2문장 안 즉시 해소되어 갈증 끌고 가지 않음
```

**자문자답 어법 발현 위치:**

- 독립 단락 (SL1·누적 단계와 분리된 자문자답 단락)
- SL/누적 단계 안 부속 (단락 내부에서 화자 호흡 조절 어법)

→ 본 어법은 위치와 무관하게 작동 방식(1~2문장 안 즉시 답)으로 판정.

**❌ 금지**: 자문자답 어법 유형을 사전 정의하여 본문에 끼워맞추기 — 분석 시 본문 어법 그대로 묘사하고, 누적 패턴이 도출되면 사후 유형화 진행. 원칙 1(끼워맞춤 사고 금지)과 정합.

### SL 번호 부여 원칙

- **SL 번호는 진짜 SL의 발생 순서로 부여** (SL1·SL2·SL3 … 순차)
  - ❌ 금지: 본문 단락 순서(정보 레이어 순서)로 SL 번호 부여 — 누적 단계와 섞이면 SL이 비순차(SL1·SL5·SL8…)가 되어 SL 본래 의미와 어긋남
  - 각 SL 헤더에 시간 범위와 정보 레이어 번호를 병기하여 본문 위치 추적 보완 (예: `SL2 — 키워드 (03:48~04:29) [정보 레이어 5]`)
- **누적 단계 번호는 영역별 별도 정수** (예: A극 누적 1~5, B극 누적 1~2)
- **피벗은 SL 번호 순서와 분리** (별도 라벨)

### 브릿지 (SL과 분리, 2026-05-29 신설)

브릿지는 단락 간 전환 어법 단위. 다음 단락 진입 동력 제공하나 시청자에게 새 큰 갈증 영역을 열지 않음 → SL과 구분.

**SL vs 브릿지 판정 기준:**

| 항목 | SL (서브 오픈 루프) | 브릿지 |
|---|---|---|
| 시청자 새 큰 궁금증 발생 | ✓ | ✕ |
| 본문이 응답에 할애 | 수 분 (전용 단락·메커니즘) | 1~2문장 안 즉시 해소 또는 단순 환기 |
| 기능 | 새 정보 영역 열림 | 직전 단락 → 다음 단락 전환 매끄럽게 |

**브릿지 표기 형식:**

```markdown
"본문 브릿지 문장 인용"

> [브릿지]: 본 문장은 SL이 아닌 단락 간 전환 어법
> [어법 묘사]: 본문 어법 그대로 묘사 (분류 라벨 강제 X — 예: "X만이 아니다 형태")
> [SL 아님 판정 근거]: 다음 단락이 새 큰 갈증 영역 아님 (단순 누적·환기)
```

**❌ 금지: 브릿지 유형을 사전 정의하여 본문에 끼워맞추기** — 분석 시 본문 어법 그대로 묘사하고, 누적 패턴이 도출되면 사후 유형화 진행. 원칙 1(끼워맞춤 사고 금지)과 정합.

### 브릿지 분류 범위 (2026-05-29 강화 — 단순 접속 부사 제외)

브릿지로 분류 ✓ — **본문 구조 전환을 매개하는 어법** (시청자 호흡 조절·관점 전환·역설 환기):

- 정보 레이어 환기형: "X만이 아니다"
- 관점 전환형: "물론 X지만 Y"
- 작은 역설 전환형: "다행히 X지만 그러고 나니깐 이제 또 Y"
- SL을 여는 진입형: "그런데 ~ 오래되지 않았습니다" 등 반전 어법

브릿지로 분류 ✗ — **단순 연결 부사** (자연 한국어 흐름, 분석 가치 없음):

- 단순 접속 부사: "여기다", "또한", "심지어", "한편", "그런데"(단독), "또", "또 이제"
- 정보 레이어 전환 시 일반적으로 자연 발생하는 연결 부사는 분석 표기 제외

→ 브릿지 표기는 **본 대본 특유 구조 전환 어법**에만 한정. 비대화 방지·효과 없는 신설 회피 원칙(CLAUDE.md)과 정합.

**자기 점검 질문**: 본 어법이 다른 일반 대본·글에서도 자연 발생하는 단순 접속인가? 그렇다면 브릿지 표기 제외. 본 대본 특유 구조 전환 어법인가? 그렇다면 브릿지 표기.

### 라벨 표기 원칙 (2026-05-29 강화)

**1. "작동 방식" 라벨 폐기 → "본문 구조 역할"로 통일**

"작동 방식"이라는 모호 표현은 폐기. 4분류(SL / 누적 단계 / 피벗 / 자문자답·브릿지) 중 어디에 속하는가를 명시하는 라벨은 **[본문 구조 역할]**로 통일.

**2. 헤더와 라벨 중복 금지**

단락 헤더에 이미 본문 구조 역할이 명시돼 있으면 (예: `#### A극 진입 단락 — ...`), 라벨에서 같은 정보 중복하지 않음. 헤더에 없는 추가 정보(어법·기능)만 라벨로 명시.

```markdown
❌ 중복:
#### A극 진입 단락 — 식물학적 특이점 (00:43~01:34)
> [본문 구조 역할]: A극 진입 단락  ← 헤더와 중복

✅ 개선:
#### A극 진입 단락 — 식물학적 특이점 (00:43~01:34)
> [어법]: 자문자답 어법 결합 (시청자 친숙 오해 대리 발화 후 즉시 해소)
```

**3. 누적 카운트 표기 금지 — 본 대본 외 정보**

분석 시점 누적 카운트("누적 13건"·"누적 14건" 등)는 시점에 따라 변동하여 새 창 AI가 봐도 이해 불가. 본 대본 분석 결과 외 다른 대본 누적 정보·시점 수치는 표기하지 않음.

```markdown
❌ 금지:
> [시청자 의문 대리 발화: "감자를 그렇게 많이 먹나?" — 누적 13건]
> [지침 없음 — 32와 동일 패턴 누적 2건]

✅ 허용:
> [시청자 의문 대리 발화: "감자를 그렇게 많이 먹나?" — 본 대본 어법]
> [지침 없음 — 본문 후반이 결산 도출 시 결론부 위트만으로 마무리]
```

본 대본 안 어법 카운트(SL 5개·자문자답 3건·브릿지 5건 등)는 본 대본 분석 결과이므로 유지. 다른 대본 비교·누적은 제외.

### 본문 인용 원칙 (2026-05-29 신설)

본문에서 SL/누적/브릿지/자문자답 어법을 인용할 때:

- **원문 그대로 유지** — 임의 생략 절대 금지
- ❌ 금지: "(...)"로 본문 일부 생략 — 정보 누락·분석 정확도 저하·새 창 AI 이해 불가
- 분석에 활용한 모든 본문은 원문 그대로 인용
- 길이가 길어도 원문 보존 우선 (가독성보다 정확성 우선)

```markdown
❌ 금지:
> "그런만큼 감자는 춥고 척박한 땅에서도 잘 자라는 미친 생존력의 특징으로 (...) 오늘날에는 (...) 동유럽·북유럽·북아메리카·동북아시아 내륙 산간 지방 등 (...) 주력 작물로 재배되고 있지요."

✓ 허용:
> "그런만큼 감자는 춥고 척박한 땅에서도 잘 자라는 미친 생존력의 특징으로 오늘날에는 원산지인 안데스 고산지대 뿐 아니라 동유럽과 북유럽 북아메리카 동북아시아 내륙 산간 지방 등 기후가 선늘하고 강수량이 적은 지역들에서 주력 작물로 재배되고 있지요."
```

### 본문 vs 분석 메모 시각적 구분 (2026-05-29 강화)

대본 원본 본문은 **일반 텍스트**로 표시, 분석자 요약·키워드 메모는 **`>` 인용 박스** 안에 표시하여 대본과 시각적으로 구분.

```markdown
❌ 금지: 분석자 요약 "(X — Y + Z)"를 본문 텍스트 영역에 일반 텍스트로 표기 (대본과 시각적 구분 안 됨)

또한 그런만큼 기존에 전통적으로 감자를 잘 재배하지 않던 나라들에서도 최근 수십 년간 감자 생산량이 엄청나게 늘었습니다.

(중국 — 2015년 4대 주식 작물 격상 + 동남아·아프리카 — 냉장 시설 확충)  ← 본문 텍스트 영역, 대본과 섞임

실제로 감자는 앞으로 인류의 식량 안보를 책임질 핵심 작물 중 하나로 평가받고 있습니다...

✓ 권장: 분석자 요약은 `>` 인용 박스 안에 표시 (대본 흐름과 구분)

또한 그런만큼 기존에 전통적으로 감자를 잘 재배하지 않던 나라들에서도 최근 수십 년간 감자 생산량이 엄청나게 늘었습니다.

> (중국 — 2015년 4대 주식 작물 격상 + 동남아·아프리카 — 냉장 시설 확충)  ← `>` 박스, 대본과 시각적 구분

실제로 감자는 앞으로 인류의 식량 안보를 책임질 핵심 작물 중 하나로 평가받고 있습니다...
```

→ 모든 분석자 키워드 요약·압축 표기 "(X — Y + Z)"는 `>` 인용 박스 안에 표시한다. 본 원칙은 본문 인용 원칙(원문 그대로 유지)과 별개 — 본문은 일반 텍스트로, 분석 메모는 박스로 분리.

### ❌ 금지

- 지적 공백을 연 본문 문장 인용 없이 SL 라벨 부여
- 사실 누적 단락에 SL 라벨 부여
- "SL 내부 작은 서브루프" 같은 모호 표현으로 SL 정당화 (작동 단위 단위로 명확히 분리)

## 원칙 2 — 도입부·결론부 분류 라벨

도입부 전략 A~H / 결산 양식 A·B·C·D / 결론 브릿지 어법 / 엔딩 8유형 / 통찰 격상 차원 등 분류 라벨은 분석 어디서든 자유롭게 사용 가능.

**단:**
- 본문 묘사가 라벨 강제로 왜곡되면 안 됨
- 본문에 라벨이 안 맞으면 "라벨 안 맞음 — 신유형 후보"로 표기
- 라벨이 잘 맞으면 그대로 사용

**예시:**
- ✓ "엔딩 유형 ③ 미래 희망형 + ⑥ 통찰형 결합" — 잘 맞으면 사용
- ✓ "기존 엔딩 8유형 어디에도 안 맞음 — 신규 엔딩 유형 후보" — 안 맞으면 신유형 표기
- ❌ "엔딩 ③ 미래 희망형의 변주" — 안 맞는데 변주로 분류 (끼워맞춤)

## 원칙 3 — 신유형 명명

신유형 후보 이름은 본문 작동 방식에서 도출.

- 본문이 "여러 원인을 누적해서 한 결과로 수렴"한다 → "다층 원인 수렴형"
- 본문이 "두 사례를 평행 구조로 비교"한다 → "2사례 병렬 비교형"
- 본문이 "통념을 부드럽게 깨고 새 사실 누적"한다 → "통념 교정형"

**❌ 금지:** 기존 3유형 변주로 분류 후 라벨 붙이기 (예: "모순형 변주 — 약점→강점 역전")

**확정 임계값** (기획안 정합):

1. 기존 3엔진 어디에도 해당하지 않음
2. 최소 2~3편 이상 동일 작동 구조 확인
3. 구조적 작동 원리가 기존과 명확히 다름

→ 단일 1편은 "신유형 후보" 표기, 누적 2건 이상 도달 시 확정 판정

## 원칙 4 — 서사 엔진 명명 (2026-05-29 갱신)

기존 3유형의 엔진 명명을 본문 작동 방식 기준으로 재정의:

| 유형 | 엔진 명명 | 핵심 동력 |
|---|---|---|
| **모순형** | **역설 엔진** | 양극 공존·역설·모순 발견. A극과 B극이 같은 대상의 두 측면으로 동시 존재 (단순 반전이 아닌 역설 작동) |
| **이미지강화형** | **영역 확장 엔진** | 단일 대상이 시청자가 인식 못한 다층 영역까지 영향. 영역 확장 자체가 체감 격상 (단순 규모 확장이 아닌 다층 영향 발견) |
| **과정추적형** | **인과 추적 엔진** | 현재 상태의 원인을 시간 거슬러 추적. 시간/인과 흐름으로 "왜 지금 이렇게 됐나" 답 (단순 역사 나열이 아닌 현재 도달 추적) |

**엔진 변경 사유:**

- "반전 엔진" → "역설 엔진": 모순형은 단순 반전이 아니라 양극 공존·역설·모순을 포괄하는 작동. 14(단일 속성 양면 작동)도 역설 엔진의 변주
- "스케일 엔진" → "영역 확장 엔진": 한국어 직관성 + 단순 규모가 아닌 다층 영역 영향 본질 반영
- "시간/메커니즘 엔진" → "인과 추적 엔진": 두 단어 결합 제거 + 시간 추적이 인과 추적 자연 포함

**엔진 결합형 가능:**

한 대본이 두 엔진 결합 작동 가능 (예: 23 = 인과 추적 + 역설 결합). 결합형은 신유형이 아닌 기존 엔진 결합 변주로 표기.

**보조 엔진 작동 영역 점검 (2026-05-29 신설):**

엔진 결합형에서 보조 엔진은 척추 엔진의 일부 영역만이 아닌 **본문 전 영역**(도입부·A극·B극·피벗·현대·미래·결론부)에서 작동 가능. 분석 시 보조 엔진 작동 영역을 전수 점검 후 표기한다.

- ❌ 금지: 보조 엔진 작동 영역을 일부만 표기 (예: "인과 추적 엔진 (보조 — A극 누적 안 역사 진행)" → A극만 명시한 부분적 표기)
- ✓ 권장: 본문 전 영역 점검 후 표기 (예: "인과 추적 엔진 (보조 — A극·B극·현대·미래 전 영역)")

**점검 영역 (모순형 기준)**: A극(도입~정점) / 피벗(피벗) / B극(Aha~증명 케이스~누적) / 현대 영향 / 미래 확장 / 결론부

→ 영역 확장 엔진·인과 추적 엔진·사례 비교 엔진 등 모든 보조 엔진에 동일 적용.

**자기 점검 질문**: 보조 엔진이 본문 어느 영역에서 작동하는가? 전 영역 점검했는가? 일부만 표기하지 않았는가?

**신 엔진 후보 (분석 누적 중):**

| 엔진 후보 | 작동 방식 | 누적 |
|---|---|---|
| 다층 원인 수렴 엔진 | 여러 원인이 한 결과로 수렴 | 09 (1건) |
| 사례 비교 엔진 | 다수 사례 평행 비교 → 공통 원리 발견 | 10 (1건) |
| 사례 나열 엔진 | 다수 사례 동일 패턴 망라 | 20 (1건) |

→ 신 엔진 확정 임계값: 2~3건 누적

## 원칙 5 — 정량 수치 기준 회피 (2026-05-29 신설)

분석 결과의 정량 수치(예: "정보 레이어 3개", "누적 단계 5개")는 그 대본의 결과일 뿐 일반 지침의 기준 아님.

**❌ 금지 — 정량 수치 기반 지침**:

- "정보 레이어 3개 이상이면 X 구조 써라"
- "A극 누적 단계 5개 필수"
- "SL 4개 이상 배치"
- "본문 단락 9개 권장"

→ 본질(왜 그 구조가 필요한가)을 가리는 형식적 틀. 집필 에이전트가 본질 모르고 수치만 맞추면 의도와 어긋남.

**✓ 권장 — 본질·작동 원리 기반 지침**:

- "기초 레이어가 충분히 쌓인 후 다음 단계로 진입"
- "시청자가 갈증을 안고 따라갈 만큼 누적된 후 전환"
- "본 대상의 다층 영향이 충분히 인지된 후 정점 결산"
- "피벗 직전 A극 정점이 시청자에게 확신되도록 충분히 누적"

→ 본질(시청자 인지·갈증 상태)을 기준으로 작성. 정확한 갯수는 주제·강도·맥락에 따라 자유.

**자기 점검 질문**:

- 본 지침이 "왜 그 구조가 필요한가"를 설명하는가? 아니면 "몇 개여야 한다"는 형식만 명시하는가?
- 집필 에이전트가 본 지침으로 본질 이해하고 맥락 맞게 적용 가능한가? 아니면 수치만 맞추게 되는가?

→ 후자라면 정량 기준 → 정성 기준으로 재작성.

**적용 영역**:

## 원칙 7 — 시청자 추리·성취감 + 이문호공 + 레고이론 (2026-05-30 신설)

본 채널 본문 이탈 방지의 **핵심 작동 원리**. 본 분석에서 도출된 모든 어법은 이 원리들의 작동 결과.

### A. 시청자 추리·예상·성취감 메커니즘

- 본문은 **일방 정보 제공이 아닌 시청자 추리 유도**
- 시청자가 직접 추론·예상 → 맞췄을 때 성취감 ("내가 영리하다")
- 본문 = 작은 퍼즐 구성 → 시청 시간·몰입도 증가

### B. 의도적 정보 지연 (Strategic Information Delay)

- 시청자가 "알 것 같아!" 순간 → 예상 살짝 비틀기 또는 더 깊은 정보 투입
- 시청자 추리에 맞춰주기만 하면 단순 → 고차원 성취감 형성 위해 한 단계 더 깊이

### C. 보상 불균형 원칙

- 루프(SL·피벗) 길이 = 응답 정보 밀도에 비례
- 갈증을 오래 끌었으면 응답에서 그에 합당한 통찰 제공
- 갈증 길이 > 응답 가치 → 시청자 이탈

### D. 이문호공 10요인 (지식채널 단순화 체계)

본 채널 본문 전 영역(도입부~결론부)에서 작동하는 시청자 자극 요인:

| # | 요인 | 시청자 효과 |
|---|---|---|
| 1 | 이익 제시형 | 통찰 프레임 전환 → 지적 우월감 |
| 2 | 문제 제기형 | 지적 허점 자극 → 인지 부조화 |
| 3 | 호기심형 | 정보 공백 → 해결 욕구 극대화 |
| 4 | 공감형 | 공통 경험·일상 → 지적 유대감 |
| 5 | 상식 파괴형 | "과연 그럴까?" → 인지 부조화 |
| 6 | 반전형 | 예측 불가 전환 → 경탄 |
| 7 | 경고형 | 디버프 자극 → 위기의식 |
| 8 | 비교형 | A vs B → 판단 욕구 |
| 9 | 최신 정보형 | 정보 선점 + FOMO + 시간 후회 |
| 10 | 도덕적 공분 | 부당함·억울함 (썸네일 후킹용) |

### E. 본 분석 어법과 이문호공 매핑

| 본 분석 어법 | 작동 이문호공 요인 |
|---|---|
| 시청자 친숙 분류 호명 후킹 (밀·쌀 → 감자 4위) | 5 상식 파괴형 + 1 이익 제시형 |

### F. 본 원리 작동 영역

- **도입부**: 3·5·9 위주 (호기심·상식 파괴·정보 선점)

## 분석 흐름

각 대본 분석 시 다음 순서:

1. **원문 통독** — 작동 방식 파악 (어떤 동력으로 끌고 가는가)
2. **단위 식별** — 본문 단락이 무엇으로 작동하는지 보고 단위 이름 부여 (원칙 1)
3. **도입부 묘사** — 무엇으로 시작해서 무엇으로 끝나는가 + 라벨 부합 여부 (원칙 2)
4. **본문 묘사** — 각 단위가 무엇을 하는가, 다음 단위로 어떻게 넘어가는가
5. **결론부 묘사** — 진입·마무리 방식 + 서브루프 점검(도입부 미해결 루프) + 라벨 부합 여부 (원칙 2)
6. **서사 엔진 식별** — 무엇이 시청자를 끌어당기는가 (반전·스케일·시간/메커니즘·다층 원인 수렴·평행 비교·통념 교정 등)
7. **유형 판정** — 기존 3유형 부합 / 신유형 후보 (원칙 3)
8. **지침 갭 요약** — 현행 지침과 대조하여 ❌ AI 자연 생성 / △ 방향만 필요 / ✓ 보강·수정·신설 5종 판정
