> **이 블록은 항상 최상단에 위치한다.** 새 세션 시작 시 AI는 본 블록을 먼저 읽고, "⏱️ 현재 진행 상태"의 재개 지점을 파악하여 **유저 추가 지시 없이 작업을 이어간다.**

### ① 분석 기준 우선 (기존 지침 절충 금지)

지침 수정·작성 시 **기존 지침 연속성이나 절충안을 우선하지 않는다.**

- 분석 파일(`[추출] 대본 - 분석/01~11번 + 오픈 루프 v2`)이 도출한 25편 실측 결과가 **유일한 기준**
- 분석 결과와 기존 지침 충돌 시 → 기존 지침 삭제·교체
- **절충안 만들기 금지**

**적용 규칙:**

- **실측 근거가 있는 항목:** 그대로 반영
- **실측 근거와 충돌하는 기존 지침:** 삭제 (절충 금지)
- **실측 근거 외 항목** (품질 관리·AI 주의사항·실무 도구 등): "분석 기준 외 카테고리" 주석 부착 후 유지

### ② 모호 분석 처리 — 대본 재분석으로 명확화

분석 파일에 모호한 내용 발견 시:

1. **출처 대본 재분석** — `[추출] 대본/` 폴더의 지식해적단 대본 25편
2. 재분석 결과로 분석 파일 갱신
3. 갱신된 분석 → 지침 반영

> **출처 대본 위치:** `[추출] 대본/` (지식해적단 대본 25편). 현재 폴더 일부가 git history에 보존되어 있으면 복원해 활용.

### ⑤ 새 세션 자동 진행 절차

새 세션 AI는 다음 절차를 자동 수행:

1. **본 계획 파일 최상단 (이 블록) 먼저 읽기**
2. **"⏱️ 현재 진행 상태"에서 재개 지점 파악**
3. **다음 단계 자동 착수** (유저 추가 지시 불필요)
4. 모호 분석 발견 시 → 대본 재분석 (위 ②)
5. 작업 완료 시 → "수정 이력" 표 행 추가 + "현재 위치" 갱신
6. 본 원칙은 지침·에이전트·스킬·체크리스트 등 모든 산출물에 일관 적용

   - 마일스톤 달성 시 (🏁 표기)

3. **세션 관리**
   - 세션 시작 시 본 섹션 확인 → 재개 지점 파악
   - 세션 종료 전 "최종 업데이트" 날짜 갱신 + 다음 재개 지점 명시

4. **보존 정책**
   - "수정 이력" 표는 최상단 누적 (삭제 금지)
   - 완료된 단계는 "단계 전환 로그"에 기록 후 유지

5. **용어 통일 원칙 (필수)**
   - 지침 수정 시 **반드시 `지침/00_용어정리.md`를 먼저 확인**하여 해당 용어의 표준 표기를 따른다
   - 용어정리.md에 등록된 표준 용어가 있으면 그 용어로 통일 (축약·풀네임 병용 포함)
   - 같은 개념을 다른 표기로 쓰지 않는다 (예: "SL" / "서브 OL" / "서브 오픈루프" 혼용 금지 → SL 또는 풀네임 "서브 오픈루프" 중 문맥에 맞게 선택)
   - 신규 용어 발생 시 용어정리.md에 먼저 경계 정의 등록 후 지침 본문에 반영
   - 용어 충돌·혼재 발견 시 용어정리.md 기준으로 본문 일괄 정비

## 배경
현 지침(P1~P6)은 지식해적단 분석 기반으로 작성됐으나 잘못된 분석이 다수 포함됨.
이에 따라 현 지침을 배제하고, 검증된 분석 파일 8개만을 기반으로 신규 지침을
먼저 작성한 뒤, 기존 지침을 신규 지침에 맞춰 전면 재구성함.

## 작업 프로세스 (5단계 + 중복 체크)
1. **변환** — 분석 파일 8개를 보고서 → 지침서 문법으로 변환 + 불필요 내용 삭제
2. **분류** — 변환된 내용을 ⒜공통 ⒝유형별 2분류
   - **2-1. 중복 체크** — 01 레고 이론 v2 내용이 분석 파일들과 중복되는지 확인

     - (b) 전면 중복 → 01 레고 이론 v2 제외
3. **신규 지침 작성** — 공통 지침 + 유형별 지침 파일 생성 (기존 지침과 독립)
4. **현 지침 재구성** — 신규 지침에 맞춰 기존 P1~P6 구조·내용 전면 수정
5. **검증** — CLAUDE.md 연동 검증 규칙 적용, 전체 흐름 재점검

2. **변환 계획 수립** → 승인
3. **실제 변환 진행** → 승인
4. **백업 대조 검수** → 승인

8개 파일 × 4단계 = 최대 32회 승인 포인트 (1단계 변환 內)

작업 순서:
1. 오픈 루프 (2중첩) v2 (통합 지침, 변환 기준 수립)
2. 07 오픈루프 4단계 접착제 분석 (v2 우선으로 중복 제거)
3. 01~03 유형별 도입부 분석
4. 04~06 유형별 본문 분석

### 사전 진단 항목 (5가지)

변환 계획 수립 전 파일 내부를 사전 진단한다. 누락·변질·왜곡 방지 + 2단계 분류 준비의 핵심 장치.

#### 1. 지침 모호성 체크 (핵심)

지침이 존재하지만 실행 기준이 없어 에이전트가 판단 불가한 지점을 식별한다.

**4가지 패턴:**

- **선택 기준 부재:** "A~H 중 선택", "유형 중 택1" 등 옵션만 있고 선택 조건 없음
- **정량 기준 부재:** "짧게", "많이", "충분히" 같은 정성 표현으로 수치·임계값 없음
- **판단 체인 단절:** 상위 판단이 선행되어야 하는데 그 판단 기준이 없음
  (예: "A극 성격별 편차 허용"인데 A극 성격 판정 기준 없음)
- **실행 방법 미제시:** 규칙은 있으나 "구체적으로 어떻게 수행하는가" 없음

발견 시 → 대본 원본 재분석으로 기준 도출 → 신규 지침에 반영

#### 2. 용어 일관성 체크

- 동일 개념을 다른 용어로 혼용하는지 (예: v2 "강화 변주" vs 01 "실질적 전략")
- 약어·코드(A~H 등)의 정의 명시 여부
- 파일 간 용어 불일치 (변환 대상 파일 ↔ v2 ↔ 예시 파일 ↔ 00_지식파일)

발견 시 → "용어 대조표"에 누적 기록 → 3단계 신규 지침 작성 시 통일

#### 3. 예시 적합성 체크

- 참조 예시 파일(`00_지식파일/인트로_예시_전략별.md` 등)이 지침과 부합하는지
- 부합하지 않는 예시는 수정 대상으로 기록

발견 시 → 예시 파일 품질 개선 (현 단계 또는 해당 파일 변환 시)

#### 4. 보고서 → 지침서 변환 가능성 분석

각 섹션이 지침서로 변환 가능한지, 삭제 대상인지 섹션별로 판정한다.

**판정 유형:**

- **✅ 유지:** 이미 지침 형태 (규범적 문법) — 그대로 또는 미세 변환
- **✅ 변환:** 보고서 문법 → 지침 문법 변환 가능 (서술형 → 명령형)
- **⚠️ 부분:** 혼합 섹션 (일부 유지 + 일부 변환 + 일부 삭제)
- **❌ 삭제:** 지침서에 부적합 (시사점·지침 vs 실제 대조·유형 간 비교 등)
- **📊 원자료:** 편별 데이터는 원자료로 유지 (지침 아니나 참조 가치)

발견 시 → 변환 계획에 섹션별 판정 명시

#### 5. 공통 vs 유형별 분류 판정 (2단계 분류 준비)

각 섹션 내용이 해당 유형 고유인지 전 유형 공통인지 판정한다. 2단계 분류 시 중복 분석 방지.

**판정 유형:**

- **공통:** 전 유형에 동일 적용되는 원칙 (ex: 썸네일 판정 기준 자체)
- **유형별:** 해당 유형 고유의 규칙·패턴 (ex: 모순형 4단계 구조)
- **혼합:** 공통 원리 + 유형별 적용 사례 혼재 (분리 방법 명시 필요)

발견 시 → 파일 내 분류 태그로 표시 (해당 섹션 시작점에 인라인 주석)

### 모호성 표시 원칙 (핵심)

1단계 변환 시 모호성을 발견하면 **해결하지 않고 표시만** 한다. 3단계 신규 지침 작성 시 일괄 수정한다.

**표시 형식 (파일 내 인라인):**

```text
> 🚨 **[모호성-{파일번호}-{일련번호}]** **{패턴명}**
> {간단 설명}
> **일괄 수정 대기** (3단계 신규 지침 작성 시 반영)
```

**태그 명명:** 파일번호는 01~07, v2는 `v2`. 일련번호는 01, 02...

**인덱스 관리:** 발견 즉시 계획서 말미 "모호성 인덱스" 섹션에 누적 기록. 3단계에서 이 인덱스를 전량 검토하여 일괄 해결.

**예외 — 1단계 즉시 처리 대상 (표시하지 않고 해결):**

- 용어 불일치 → 용어 표준화 원칙으로 즉시 통일
- 불필요 섹션(시사점·유형 간 비교·지침 vs 실제) → 변환 원칙으로 즉시 삭제
- 보고서 → 명령형 변환 → 변환 원칙으로 즉시 변환

이 외 "지침 내용 자체의 모호성"은 모두 표시만 하고 일괄 대기.

### 분류 태그 원칙 (2단계 분류 준비)

1단계 변환 시 사전 진단 5번(공통 vs 유형별 분류 판정) 결과를 파일 내 인라인 주석으로 표시한다. 2단계 분류 시 일괄 추출하여 공통/유형별 지침 파일에 배치.

**태그 형식 (인라인 HTML 주석):**

```text
<!-- [분류-공통] 사유 -->
<!-- [분류-유형별-{유형명}] 사유 -->
<!-- [분류-혼합] 사유 + 분리 방법 -->
```

**유형명:** `모순형` / `이미지강화형` / `과정추적형`

**적용 위치:** 해당 섹션 제목 직후 또는 관련 문단 직전

**2단계 분류 활용:** `grep "\[분류-"` 으로 일괄 추출 → 공통 지침/유형별 지침 파일에 배치

## 원본 파일 처리 방식 (C안 — 백업 후 수정)
- 원본 분석 파일은 유저가 `분석 백업/` 폴더에 백업 완료 (8개 파일 전체 확인)
- 변환 작업은 원 위치에서 원본 파일을 직접 수정
- 단일 진실 원칙 유지

## 변환 원칙 (보고서 → 지침서 문법)

| 보고서 문법 (삭제·변환 대상) | 지침서 문법 (유지·변환) |
|---|---|
| "지식해적단은 ~한다" (서술) | "~하라" / "~해야 한다" (명령) |
| "~편 중 ~편(%)에서 발견" (빈도 통계) | "~할 때는 반드시 ~한다" (규칙화) 또는 삭제 |
| 대본 원문 인용 (근거 제시) | 집필 예시로 필요한 것만 1~2개 유지 |
| "해석", "시사점", "현재 지침 vs 실제" | 삭제 (지침화 완료 후 근거 불필요) |
| "~편이 특이 사례" (예외 분석) | "예외: ~인 경우는 ~" (조건부 규칙) |
| 편별 데이터 나열 | 패턴만 추출, 편별 상세 삭제 |
| 유형 간 비교표 | 해당 유형 파일에 귀속, 비교 삭제 |

## 불필요 내용 판단 기준
- 분석 근거(대본 원문 인용) → 원리 설명에 꼭 필요한 1~2개만 유지, 나머지 삭제
- 수치 통계(퍼센트, 빈도) → 판단 기준 전환 가능하면 유지, 아니면 삭제
- 현 지침과의 비교·시사점 → 전면 삭제
- 유형 간 비교 섹션 → 삭제 (유형별 파일 분리 예정)
- 중복 서술 → 1회로 통합

## 변환 계획 포맷 (파일별 공통 양식)

각 파일의 변환 계획은 아래 포맷으로 작성한다. 섹션별로 반복 기재.

### 섹션별 계획

#### 현 상태
- 형식: [서술형 / 규칙형 / 표 / 혼합]
- 내용 구성: [섹션 하위 항목 요약]
- 분량: [N줄, 라인 범위]

#### 변환 방향

| 요소 | 처리 |
|---|---|
| [항목] | 유지 / 변환 / 삭제 / 이관 + 사유 |

#### 의도된 수정 목록 (예외 처리 대상)

| 수정 항목 | 수정 유형 | 이유 | 의도 |
|---|---|---|---|
| 예: 유형 간 비교표 | 삭제 | 유형별 파일 분리 예정 | 중복 제거 |

#### 예상 결과
- 변환 후 섹션 제목
- 주요 내용 개요
- 예상 분량: N줄

## 검수 방법 (3단계 백업 대조)

### 검수 체크리스트

| 항목 | 체크 방법 | 판정 기준 |
|---|---|---|
| 누락 | 백업 파일의 모든 섹션·하위섹션·표·규칙 항목 리스트업 → 변환본에서 위치 추적 | 이관 표시 없이 사라진 내용 발견 시 위반 |
| 변질 | 백업 핵심 주장 vs 변환본 규칙 문장 의미 대조 | 서술→명령 외 의미 변경 발견 시 위반 |
| 왜곡 | 백업 조건·예외·적용 범위 vs 변환본 대조 | 조건 삭제·확장·축소 발견 시 위반 |
| 축약 | 섹션별 분량 비율 체크 | 핵심 규칙 압축으로 의미 전달 불가 시 위반 |
| 축소 | 백업 적용 대상·사례 수 vs 변환본 | 정당한 이유 없이 사례·조건 감소 시 위반 |

### 의도된 수정 예외 처리
- 변환 계획에 명시된 "의도된 수정"은 검수 시 예외 처리
- 예외 처리 조건: 수정 항목·유형·이유·의도 4가지 명시
- 계획에 없던 수정 발견 시: "계획 누락"으로 사후 기록. 원칙적으로 불허

## 공통 vs 유형별 경계 기준
- **공통**: 모든 서사 유형에 동일하게 적용되는 원칙
  (4단계 구조, 접착제 툴킷 기본, 박스 이미지 공식, ML 구조 기본)
- **유형별**: 서사 유형마다 다르게 작동하는 규칙
  (강화 변주, 피벗/Scale Jump/전환점 설계, 엔딩 궁합, ML 변형)

## 단계별 유저 승인 포인트

**1단계 (파일별, 8개 × 4단계):**
- 사전 진단 결과 보고 → 승인
- 변환 계획 보고 → 승인
- 변환 진행 완료 보고 → 승인
- 백업 대조 검수 결과 보고 → 승인

**2단계 이후:**
- 2단계(분류) 종료 시: 분류 결과 → 승인
- 2-1단계(중복 체크) 종료 시: 처리 방향 → 승인
- 3단계(신규 지침 작성) 시작 전: 구조·목차 → 승인
- 3단계 각 파일 작성 시: 개별 승인
- 4단계(현 지침 재구성) 시작 전: 재구성 계획 → 승인
- 5단계(검증) 종료 시: 전체 결과 보고

## 3단계 신규 지침 작성 시 필수 포함 사항

### 다단계 비율 선정 기준 체인 — 신규 추가 필요

비율은 단일 판정으로 결정되지 않고, **주제 → 팩트 리포트 → 서사 유형 → A극 성격 → 비율 적용**의 5단계 체인으로 결정된다. 각 단계의 판정 기준을 신규 지침에 포함한다.

| # | 판정 단계 | 내용 | 배치 지침 |
|---|---|---|---|
| 1 | **팩트 성격 판정 기준** | 수집된 팩트가 시간축/속성/스케일/메커니즘 중 어느 성격인가 | `P2_딥리서치` 또는 `P3_공통_구조원칙` |
| 2 | **서사 유형 판정 기준** | 팩트 성격 → 모순형/이미지/과정 중 어느 유형이 맞는가 | `P3_공통_구조원칙` |
| 3 | **A극 성격 판정 기준** | 서사 유형 내에서 A극의 구체 성격 (7조건 정의·지표·예시) | `P3_유형별_*` |
| 4 | **비율 적용 기준** | A극 성격에 따라 기본 범위 + 편차 허용 적용 | `P4_유형별_*` |
| 5 | **복합형 처리 원칙** | 2개 이상 조건 동시 해당 시 주 조건 선택 기준 | `P3_공통_구조원칙` |

### 단계별 검증 고리

- P2 팩트 리포트 확정 시 → **팩트 성격 ↔ 서사 유형 일치 검증**
- P3 A극 성격 확정 시 → **팩트 성격 ↔ A극 성격 일치 검증**
- 불일치 시 팩트 리포트 재정리 또는 서사 유형 재검토

### 배경

현 `오픈 루프 (2중첩) v2` §6에는 A극 성격별 편차 허용 조건명만 있고, "내 주제가 어느 조건에 해당하는가?"의 판정 기준이 부재. 또한 팩트 리포트의 성격이 비율에 직접 영향을 미침에도 이 체인이 명시되지 않음. 같은 주제라도 팩트 리포트 구성에 따라 비율이 달라지므로, 주제·팩트 리포트 단계부터 판정 기준을 연결해야 함.
