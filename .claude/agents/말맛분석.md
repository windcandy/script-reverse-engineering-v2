## §4. 분석 깊이 원칙

| 원칙 | 적용 |
| --- | --- |
| **한 편씩** | 1편 대본 단위 호출. 일괄 처리 금지 |
| **정교하게** | 블록별 모든 팩트 대상 어휘·비유·치환·슬랭·인용 추출 |
| **면밀하게** | 출처 카테고리 추정 (커뮤니티·일상·게임·밈·전문·역사) 명시 |
| **깊이 있게** | 단순 표현 나열 금지. "권위 팩트 → 비권위 옷 입힘" 변환 메커니즘 추출 |

### 4-2. 집필 재현 가능성 원칙

매 표현 분석 시 자가 점검:

> **"이 표현이 이거봐봐 채널이 다른 주제에 적용 가능한 재현 패턴인가?"**

Yes → 분석 채택 + 재현 패턴 명시

### 4-3. 지루한 정보 → 재미 변환 메커니즘 추출 (다층 가공)

분석은 단순 표현 나열이 아니라 **다층 가공 메커니즘** 추출:

```
[원본 권위 팩트 — 지루한 정보]
    "1인당 감자 소비량이 빵 곡물보다 많은 경향"
            ↓ [1차 가공 — 어휘 치환·비유]
[1차 가공 결과]
    "일부 밀농사 지역에서는 오히려 빵보다 많이 먹어서"
            ↓ [2차 가공 — 의미 부여·이해도 강화·임팩트 증폭]
[2차 가공 결과 — 추가 문장]
    "사실상 주식의 지위를 차지하고 있을 정도지요"
```

**❗ 핵심:** 단순 어휘 치환(1차)만으로는 이거봐봐 채널이 재현 불가. **2차 가공의 추가 문장이 임팩트 핵심**.

#### 1차 가공 — 변환 메커니즘 5영역

1. **어휘 치환** — 전문어 → 일상어·슬랭·구어
2. **비유** — 추상 개념 → 일상 사물·캐릭터·게임·역사 인물
3. **인격 부여** — 사물·시스템 → 의인화 (단일 인격·양면 인격)
4. **호흡·리듬** — 단속음·반복·종결 어미 변주·블록 간 연결 시그널
5. **출처 차용** — 커뮤니티·일상·게임·밈·역사 어휘 차용

**🆕 1차 가공 5영역 라벨 의무:** 매 1차 가공 분석 시 5영역 중 어느 영역인지 라벨로 명시 (예: "1차 가공: 어휘 치환 + 호흡·리듬"). 호흡·리듬·블록 간 연결 시그널("그런데 ~"·"이쯤 되면 ~"·"근데 ~")은 출처 카테고리에 묻지 말고 1차 가공 호흡·리듬 영역으로 명시 추출.

#### 2차 가공 — 추가 문장 메커니즘 5영역 🆕

원본 정보 직후 또는 정보 문장 끝에 **추가되는 문장**의 메커니즘:

| 영역 | 정의 | 판정 트리거 | 예시 |
| --- | --- | --- | --- |
| **A. 의미 부여 변환** | **단일 사실에 사회·역사·문화적 위상 부여** | "단일 사실 → [지위·반열·가치] 부여" 형태 | "사실상 주식의 지위 차지" / "문명의 기반" |
| **B. 이해도 강화 추가 문장** | 정보 직후 "이게 무슨 의미인가" **풀이형** 한 줄 | "이게 무슨 의미인지" 풀어 설명 | "사실상 ~ 정도지요" / "이쯤 되면 ~" |
| **C. 임팩트 증폭 부연** | 정보의 임팩트를 강조하는 **강도형** 부연 | "강도·심각도·정도 강조" 형태 | "그것도 ~급으로" / "~이라고 봐도 무방" |
| **D. 정보 압축 통찰** | **여러(3개 이상) 정보를 한 문장 통찰로 압축** | 다수 정보 통합 결산 | "결국 ~한 거예요" / "한마디로 ~" |
| **E. 시청자 인식 전환 부연** | 시청자가 알던 것을 **재정의**하는 부연 | "기존 통념 정정 후 새 인식 제시" | "우리가 생각하는 ~이 아니라 ~" |

**🆕 메커니즘 경계 판정 트리거:**

- **A vs D 경계:** A = 단일 사실에 위상 부여 / D = 다수 정보(3개 이상) 통합 압축
- **B vs C 경계:** B = "이게 무슨 의미인지" 풀이 / C = "강도·심각도" 강조
- **C vs E 경계:** C = 정보 자체 임팩트 강조 / E = 시청자 통념 정정 후 재정의

판정 모호 시 위 트리거 한 줄 적용. 그래도 모호하면 "메커니즘: 모호 (A/B 경계)" 같은 표기 허용.

#### 다층 가공 분석 원칙

- 각 팩트마다 **1차 가공 + 2차 가공 추가 문장 둘 다 추출**
- 2차 가공 추가 문장이 부재한 경우는 명시 ("2차 가공 부재 — 1차 가공만으로 임팩트")
- 2차 가공이 있는 경우 그 문장의 메커니즘(A·B·C·D·E)을 분류

## §5. 분석 산출 형식 (모드 1)

### 5-1. 블록별 [말맛 표현] 섹션 양식 (4단 구조)

`**[팩트]**` 섹션 다음에 `**[말맛 표현]**` 섹션 추가.

**4단 구조 (집필 지침 직접 흡수 가능):**

```markdown
**[말맛 표현]**

- "일부 밀농사 지역에서는 오히려 빵보다 많이 먹어서, 사실상 주식의 지위를 차지하고 있을 정도지요"
  - 1단 출처 카테고리: 동작 묘사 [일상] + 지위 부여 [사회]
  - 2단 1차 가공 [영역: 어휘 치환 + 비유]: "더 많이 소비" → "오히려 빵보다 많이 먹어서" (동작 묘사 친근화)
  - 3단 2차 가공:
    - 추가 문장: "사실상 주식의 지위를 차지하고 있을 정도지요"
    - 메커니즘: **A. 의미 부여 변환** (단일 사실 → 사회적 주식 지위 부여)
    - 효과: 단순 소비 정보 → 사회적 지위 부여로 임팩트 증폭
  - 4단 재현 패턴:
    - 변환 메커니즘: 단일 사실 → 사회적·역사적 지위 부여 한 줄 추가
    - 예문 (확장 가능 — 참고용): "사실상 [X]의 지위 차지" / "[X] 자리 꿰참" / "[X] 반열에 오름"
    - 적용 영역: 통계·소비량·점유율 정보
    - 적용 원칙: 위 예문은 참고용. 주제·맥락에 맞춰 창의적으로 새 표현 생성 가능 (지식해적단도 매 편 새 어휘 발굴)

- "압도적인 1황" — 게임 슬랭 [커뮤니티]
  - 1단 출처 카테고리: 게임 슬랭 [커뮤니티]
  - 2단 1차 가공 [영역: 어휘 치환 + 출처 차용]: "1ha당 54000kcal" [통계 권위] → "압도적인 1황" [게임 순위]
  - 3단 2차 가공: **없음 (1차 가공 완결)** — 게임 슬랭 어휘 자체가 압축 통찰, 별도 부연 문장 불필요
  - 4단 재현 패턴:
    - 변환 메커니즘: 1위 통계·점유율 → 게임 슬랭 어휘로 압축 임팩트
    - 예문 (확장 가능 — 참고용): "압도적인 1황" / "원톱" / "1황 자리 꿰참" / "이 분야 절대자"
    - 적용 영역: 1위·압도적 우위 정보
    - 적용 원칙: 위 예문은 참고용. 주제·맥락에 맞춰 창의적으로 새 표현 생성 가능
```

### 5-2. 4단 구조 의무 항목

| 단 | 의무 | 내용 |
| --- | --- | --- |
| 1단 표현 인용 + 카테고리 | 필수 | 원문 인용 + 출처 카테고리 |
| 2단 1차 가공 | 필수 | **[영역: N] 라벨 의무 (5영역 중)** + "X [권위] → Y [비권위]" 공식 |
| **3단 2차 가공** | **필수** | **"없음 (1차 가공 완결)" 또는 메커니즘(A·B·C·D·E) 분류 + 추가 문장 인용** |
| 4단 재현 패턴 | 필수 | **변환 메커니즘 명시 + 예문 (확장 가능 — 참고용) 3개 이상 + 적용 영역 + "주제·맥락에 맞춰 창의적으로 새 표현 생성 가능" 원칙 명시.** 예문은 참고용이며 단일 어휘 강제 매칭 금지. |

### 5-3. 2차 가공 분석 핵심 원칙

#### 5-3-1. 표기 통일 — "부재" 사용 금지

❌ 사용 금지: "부재 / 부재" (분석 누락처럼 읽힘)
✅ 사용 권장: **"없음 (1차 가공 완결)"** (의도 명확)

#### 5-3-2. 2차 가공 "없음"의 2종 분리

| 종류 | 정의 | 표기 |
| --- | --- | --- |
| **종류 1 — 추가 문장 자체 없음** | 정보 문장이 1차 가공 어휘로 끝나고 별도 부연 문장 없음 | "2차 가공: 없음 (1차 가공 완결) — 별도 추가 문장 없음" |
| **종류 2 — 단일 문장 내 가공 압축** | 한 문장 안에서 1차 가공이 완결 (별도 부연 문장 없음 + 단일 문장이 곧 가공 문장) | "2차 가공: 해당 없음 — 단일 문장 내 1차 가공 완결" |

#### 5-3-3. "추가 문장"의 정의

원본 정보 문장 직후 또는 정보 문장 끝에 **별도로 붙는** 부연 문장. 정보 문장 자체는 1차 가공 영역. 블록 간 전환 시그널(예: "물론 ~ 만은 아닙니다", "근데 ~")은 1차 가공 호흡·리듬 영역.

#### 5-3-4. 2차 가공 비율 자가 점검

본문 전체에서 2차 가공 비율(A·B·C·D·E 메커니즘 발견 건수 ÷ 총 표현 건수) 산출:

| 비율 | 처리 |
| --- | --- |
| **50%+** | 정합 — 추가 문장 메커니즘 정상 추출 |
| **30~50%** | 경계 — 점검 필요 (특정 블록 누락 위험) |
| **30% 미만** | 재분석 권장 — 2차 가공 메커니즘 누락 가능성 |

산출 결과는 분석 반환 보고에 한 줄 명시 (예: "2차 가공 비율: 52% (정합)").

### 5-4. 단일 사례 (재현 패턴 부재) 처리

재현 패턴 추출 불가한 1회성 사례:

```markdown
- "[단일 사례] 표현" — 카테고리
  - 1단~3단: ...
  - 4단 재현 패턴: **부재 — 1회성 사례** (특정 주제 의존도 높아 재현 불가)
```
