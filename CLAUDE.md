# CLAUDE.md — 유튜브 대본 제작 프로젝트

## 기본 규칙

- **응답 언어:** 항상 한국어로 응답한다. 기술 용어·코드는 원어 유지.
- **파일 수정 금지:** 유저 승인 없이 대본·지침 파일을 임의로 수정하지 않는다.
- **파일 삭제 금지:** 어떤 파일도 유저 명시 지시 없이 삭제하지 않는다.

## 워크플로우 — Phase별 로드 원칙

**한 번에 한 Phase 지침만 읽는다.** 전체 지침을 동시에 로드하지 않는다.

| Phase | 로드 파일 | 진입 조건 |
|---|---|---|
| P1 | `P1_퀵서치_제목썸네일.md` | 키워드 입력 시 |
| P2 | `P2_딥리서치.md` | 유저가 제목/썸네일 후보 선택 후 |
| P3 | `P3_구조설계.md` | GATE 2 통과 후 |
| P4 | `P4_집필.md` | 유저 구조 보고 승인 후 — **집필 직전 반드시 재로드** |
| P5 | `P5_퇴고.md` | GATE 4 통과 후 |
| P6 | `P6_장면소스.md` | P5 완료 후 |

## 핵심 금지 원칙

1. **지침 내부 용어를 대본에 노출하지 않는다** — SL, BR, ML, A극/B극, 피벗, 이문호공, 박스 이미지, 역설엔진 등
2. **지침의 예시 문구를 대본에 그대로 복사하지 않는다** — 원리를 이해하고 새 표현을 생성
3. **확인되지 않은 수치·출처를 단정하지 않는다** — 웹 검색으로 팩트 확인 후 사용
4. **GATE를 건너뛰지 않는다** — 각 GATE 체크리스트 전 항목을 실제로 검증
5. **욕설 사용 금지** — "미쳤다", "대박" 수준의 구어 감탄만 허용

## 대본 파일 저장 규칙

```
대본/[NN_주제명]/[버전명]/[대본] 주제명_YYYYMMDD_HHMMSS.md
```

- 수정본은 같은 폴더에 타임스탬프 갱신하여 저장
- 히스토리 파일은 `00_히스토리/` 하위에 보관

## 유저 인터랙션 필수 포인트

Phase를 넘어가기 전 반드시 유저 확인을 받는다:

- P1 완료 → 제목/썸네일 후보 선택 대기
- P2 완료 → 팩트 리포트 검토 대기
- P3 완료 → 구조 보고(조감도) 승인 대기
- P4 완료 → 초안 리뷰 대기
- P5 완료 → 최종본 승인 대기
