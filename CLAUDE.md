# CLAUDE.md — 유튜브 대본 제작 프로젝트

## 기본 규칙

- **응답 언어:** 항상 한국어로 응답한다. 기술 용어·코드는 원어 유지.
- **파일 수정 금지:** 유저 승인 없이 대본·지침 파일을 임의로 수정하지 않는다.
- **파일 삭제 금지:** 어떤 파일도 유저 명시 지시 없이 삭제하지 않는다.

| --- | --- | --- |

## 핵심 금지 원칙

1. **지침 내부 용어를 대본에 노출하지 않는다** — SL, BR, ML, A극/B극, 피벗, 이문호공, 해소 비유, 역설엔진 등
2. **지침의 예시 문구를 대본에 그대로 복사하지 않는다** — 원리를 이해하고 새 표현을 생성
3. **확인되지 않은 수치·출처를 단정하지 않는다** — 웹 검색으로 팩트 확인 후 사용
4. **GATE를 건너뛰지 않는다** — 각 GATE 체크리스트 전 항목을 실제로 검증
5. **욕설 사용 금지** — "미쳤다", "대박" 수준의 구어 감탄만 허용

## 대본 파일 헤더 규칙

모든 대본 파일은 최상단에 아래 형식의 헤더를 반드시 포함한다.

```markdown
# 대본: [주제명]

> 제목: [최종 확정 제목]
> 썸네일: "[썸네일 메인 카피]"
> 서사 유형: [이미지 강화형 / 반전형 / 공감형 등]
> 목표 길이: [X~Y분]
```

- `목표 길이`는 **분(分)으로 표기**한다 (공백 포함 분당 약 300자 기준)
  - 예: `목표 길이: 12~13분`
- 헤더 다음에 `---` 구분선을 넣고 본문을 시작한다
- 제목·썸네일은 P1에서 유저가 선택한 확정안을 그대로 기재한다

---

## 대본 파일 저장 규칙

**폴더 생성 시점:** P1에서 주제가 확정되면(GATE 1 통과 시) `대본/NNN_주제명/` 폴더를 즉시 생성한다.

- **NNN은 3자리 번호** (001, 002, … 099, 100, …) — 100개 이상 확장을 전제한다

```text
대본/[NNN_주제명]/
  [퀵서치] 주제명.md                  ← P1 완료 시 저장
  [팩트리포트] 주제명.md              ← P2 완료 시 저장
  [조감도] 주제명.md                  ← P3 완료 시 저장
  [대본] 주제명_YYYYMMDD_HHMMSS.md   ← P4 완료 시 저장 (최신본 1개만 유지)
  [낭독대본] 주제명.md               ← P5 완료 시 저장
  히스토리/
    [대본] 주제명_YYYYMMDD_HHMMSS.md  ← 대본 수정 전 이전 버전들
```

### Phase별 저장 파일

| Phase | 파일명 | 내용 |
|---|---|---|
| P1 | `[퀵서치] 주제명.md` | 핵심 팩트 리포트 + 제목/썸네일 후보 + 확정안 |
| P2 | `[팩트리포트] 주제명.md` | 딥리서치 팩트 리포트 (출처·마킹 포함) |
| P3 | `[조감도] 주제명.md` | 구조 보고 전문 |
| P4 | `[대본] 주제명_YYYYMMDD_HHMMSS.md` | 대본 본문 |
| P5 | `[낭독대본] 주제명.md` | 발음 변환 목록 + 대본 원문 |

**수정 시 반드시 지켜야 할 순서:**

1. 기존 대본 파일을 `히스토리/` 폴더로 이동 (덮어쓰기 금지)
2. 수정된 내용을 새 타임스탬프로 `대본/[NNN_주제명]/` 에 저장
3. `대본/[NNN_주제명]/` 폴더 루트에는 최신본 1개만 존재해야 함
4. **새 파일 최상단에 수정 내역 주석을 반드시 삽입한다**

**수정 내역 주석 형식 (파일 맨 첫 줄):**

```markdown
<!-- [수정] YYYY-MM-DD HH:MM | 수정 내용 한 줄 요약 (예: 3장 팩트 보강, 오프닝 훅 교체, 전체 퇴고) -->
```

- 수정 항목이 여러 개면 쉼표로 구분하여 한 줄에 작성
- 이전 수정 주석은 히스토리 파일에 보존되므로 새 파일에는 최신 주석 1개만 유지

- 히스토리 폴더가 없으면 첫 수정 시 자동으로 생성
- 파일명 형식: `[대본] 주제명_YYYYMMDD_HHMMSS.md`

## 유저 인터랙션 필수 포인트

Phase를 넘어가기 전 반드시 유저 확인을 받는다:

- P1 완료 → 제목/썸네일 후보 선택 대기
- P2 완료 → 팩트 리포트 검토 대기
- P3 완료 → 구조 보고(조감도) 승인 대기
- P4 완료 → 초안 리뷰 대기
- P5 완료 → 최종본 승인 대기
