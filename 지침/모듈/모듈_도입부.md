# 모듈_도입부 — 썸네일·도입부 카탈로그

> **역할:** 본 채널 도입부 영역 카탈로그. 썸네일 판정 3분류 · 약속·도입부·본문 역할 분담 원리 · 자발적 발견 단일 정의 · 도입부 블록 골격 + ML 봉인.

---

## §1. 썸네일 판정 기준 — 3분류

썸네일 문구의 형태에 따라 3가지 전략으로 구분한다.

| 전략 | 판정 기준 | 문형 신호 | 참고 매핑 |
| --- | --- | --- | --- |
| **결과 제시형** | 결론·지위·단정을 직접 선언 | `~이다`, `~나뉜다`, `~하는 X`, `~할 수 없다` | 과정추적형, 이미지강화형 일부 |
| **조각 제시형** | 질문 형태 또는 특정 부품 던지기 | `~이유`, `~일까?`, `~라고?`, `얼마나 ~?`, `어떤 ~?` | 모순형, 이미지강화형 |
| **모순 공존형** | 썸네일 문구 내에 A극 속성과 B극 속성이 동시 제시 | `[긍정 수식] + [부정 명사]`, `[신적·절대적 표현] + [위험·문제 표현]` 조합 | 모순형 우세 |

**참고 매핑 운영:** 위 매핑은 빈도 경향이며 강제 아님. 본 채널 3유형 모두 3종 분류 자유 활용 가능.

### 모순 공존형 예시

- "악마의 풀" — 마성의 이미지 + 식물 속성
- "축복의 저주" 식 대조 — A극·B극 속성 충돌

## §2. 썸네일 약속·도입부·본문 역할 분담 원리

용어 구분은 모듈_레고이론 §0.2 참조.

- **썸네일·제목 = 썸네일 약속** — 최종 통찰 암시
- **도입부 = 썸네일 약속에 대한 궁금증을 생성하는 전제 조건 세팅** (의외형=상식 뒤집기 / 호기심형=순수 궁금증) + (도입부 신뢰 상환 전략일 경우) 신뢰 상환 초기 증거 제공
- **본문 = 단서들이 최종 통찰을 향해 수렴** + 아하 모먼트 발생 지점

썸네일 문구를 반복하면 "이미 알고 있는 걸 또 말하네"가 되어 이탈한다. 전제 조건만 깔면 시청자가 스스로 궁금증을 생성하여 자발적 몰입이 일어난다.

### 자발적 발견 (썸네일 약속 보호) — 직접 인용 금지 단일 정의

**썸네일 약속 메인 문구 자체**를 그대로 인용·반복하지 않는다. 동의어 치환·간접 확인으로 처리하여 시청자가 본문에서 아하 도약을 통해 최종 통찰을 자발 발견하도록 전제 조건만 세팅한다 (노출 강도 기준 모듈_집필기법 §4 간접 확인 원리 — 약·중 강도 허용 / 강 강도 회피).

- **적용 대상:** 썸네일 **메인 문구** 자체 + 제목 (예: "인류를 지키고 있는 물질")
- **예외 — 썸네일 서브 문구:** 서브 문구는 본문/결론부에서 **직접 인용 허용**. 서브는 메인 미끼의 결과·보충 선언이라 본문 노출 시 후킹 약화 영향이 미미하고, 본문 통찰 강화 효과가 더 크다
- **목적:** 시청자 능동 추리 유도 (모듈_레고이론 §0.1 능동 추리자 원리·§0.6 지적 유능감 자극 정합). "이미 아는 걸 또 듣는다" 이탈 방지
- **우회 기법 참조:** 직접 인용 없이 우회해서 확인시키는 3가지 방법(우회 기법 — 동의어 치환·속성 대체·반대 방향 진입)의 구체 정의는 모듈_집필기법 §4 참조. (본 §2는 "직접 인용 금지" 원칙만 정의)

### 도입부 공통 골격 — 인트로 블록 + ML 블록

| 블록 | 공통 원칙 |
| --- | --- |
| **인트로** | ML 이전 진입 영역 — 정보 블록 1~3개 (대부분 1개), 구성은 §3.1 참조 |
| **ML (메인 루프)** | 제목·썸네일의 약속을 질문·선언으로 문장화 (도입부 말미) |

**신뢰 상환 = "영상이 약속을 지킬 것"이라는 초기 증거 제공 기능** (완전한 답이 아닌 미끼 진정성 인증). **별도 블록이 아니라** 인트로·ML 블록에 얹히며, **수행 위치가 아래 2전략을 가른다:**

| 전략 | 신뢰 상환 위치 |
| --- | --- |
| **도입부 신뢰 상환** | 전제 조건 세팅 + 도입부에서 초기 증거 제공 |
| **본문 이연 신뢰 상환** | 전제 조건 세팅만 (초기 증거 제공은 본문 초반으로 이연) |

※ **두 전략 모두 완전한 답은 본문에 있다.** 차이는 "**도입부에서 초기 증거를 주느냐**"뿐 — "도입부 신뢰 상환"이 도입부에서 답을 끝낸다는 뜻이 **아니다**.

## §3. 도입부 정보 블록 골격 + ML 봉인

도입부는 **정보 블록**으로 구성한다. 각 블록을 **두 축**(축① 이문호공 요인 + 축② 정보 배치 방식)으로 설계한다.

### 3.1 블록 골격

- 도입부 = **총 2~4개 블록** (진입 정보 블록 1~3개 + 마지막 **ML 봉인 블록** 1개). **대부분 2개**(진입 1 + ML 1).
- **최소 골격 = 진입 정보 블록 + ML 봉인.** 진입 블록 내부 구성은 주제마다 다르다.
- 약속 확인을 **뒤 블록까지 미루면**(균열을 여러 겹 쌓는 경우) 진입 블록이 늘어 3~4개가 된다.

### 3.2 축① 이문호공 요인 (심리 자극)

각 블록이 자극하는 심리. 10요인에서 선택: 1 이익 / 2 문제제기 / 3 호기심 / 4 공감 / 5 상식파괴 / 6 반전 / 7 경고 / 8 비교 / 9 최신정보 / 10 도덕공분. 한 문장에 2개 이상 압축하면 중첩.

- **요인을 촘촘히 쌓는다:** 진입 블록은 이문호공 요인을 **여러 개(실측 3~5개) 중첩·순차 배치**로 쌓는다. 한두 개로 빈약하게 끝내지 않는다.
- **고정 레시피 없음:** 요인 **조합·순서·개수는 주제마다 다르게** 짠다. 첫 요인은 4 공감·9 최신정보가 다수이나 **4에 편중하지 말 것** — 3 호기심·5 상식파괴로 시작하는 것도 적극 고려한다. 후속은 **10요인 전체에서 자유 조합**한다 (특정 요인 세트로 고정 금지).

### 3.3 축② 정보 배치 방식

각 블록이 정보를 배치하는 방식. 두 역할로 나뉜다.

- **빌드업 블록:** 약속 확인 전 정보를 쌓는 방향 — **반대 정보 빌드업**(약속 반대극을 먼저 폭로해 낙차 생성) / 사례 빌드업 / 위상 빌드업 등.
- **약속 확인 블록:** 썸네일 약속을 확인시키는 방식 — 우회 원칙(모듈_집필기법 §4.1): (가) 동의어 치환 / (나) 속성 대체 / (다) 반대 방향 진입 중 택1. 강 강도만 **직접 노출** 예외. 표에 ★표기.
- **약속 확인 블록 위치 자유** (블록1~마지막). 균열을 여러 겹 쌓을수록 뒤로 미뤄진다.
- **한 블록이 여러 역할 겸직 가능** (블록당 1역할 아님): 진입·빌드업·약속 확인을 한 블록이 동시에 할 수 있다 (예: §4.5 당뇨 블록1 — 진입하며 동시에 동의어로 약속 확인). 역할별로 블록을 억지로 쪼개지 않는다.

### 3.4 ML 봉인 2종

**ML(메인 오픈루프)** = 썸네일 약속을 질문·선언으로 문장화(도입부 말미).

| 봉인 | 질문 | 결정 |
| --- | --- | --- |
| **단면** | 1개 | 본문이 단일 갈래 |
| **양면** | 2개 이상 | 본문이 여러 갈래 |

**질문 수는 본문에서 다룰 갈래에 맞춰 정한다** (단일 인과→단면 / 여러 갈래→양면). 유형별 경향·변주는 모듈_본문구조 §1.4.

### 3.5 진입 공감 출발 단위 (4 공감 진입 시)

시청자 일상 경험에서 출발. 출발점은 이 주제에만 성립하는 구체적 **공통 인지·상식·경험·일상 장면·일상 행동·감각**이어야 한다. 주제 이름만 바꾸면 다른 대본에도 쓸 수 있는 문장은 금지. — **동료 호명 톤 사용** (예: "우리 다 아는 ~"·"~ 한 번쯤 들어봤죠"·"다들 ~ 경험 있죠").

## §4. 도입부 예시 카탈로그 (두 축 골격 유형별 실사례)

> 지식해적단 실제 도입부 발췌. 각 블록의 두 축(요인 / 정보 배치)과 ML 봉인을 표기.

### 4.1 감자 — 반대 정보 빌드업 · 3블록

- 블록1(진입·빌드업): "밀·쌀 문명" → "의외의 4위 감자·3억 8천만 톤" → [(5 상식파괴+3 호기심)중첩 → 9 최신정보 → 3 호기심 · **반대 정보 빌드업**]
- 블록2: "곡물 아닌 채소·덩이줄기" → [5 상식파괴 → 6 반전 · **속성 대체 ★약속 확인**]
- 블록3(ML): "언제부터? 밀·쌀과 장단점?" → [3 호기심 → 8 비교 · **양면 봉인**]

### 4.2 옥수수 — 속성 대체

- 블록1(진입): "콘벨트·한반도보다 넓은 밭·12억톤 2위" → [3 호기심 → (5 상식파괴+9 최신정보)중첩 → 1 이익 · **속성 대체**(위상 폭로)]
- 블록2(ML): "사람도 안 먹는데 어디 쓰려고? 왜 옥수수?" → [(4 공감+5 상식파괴)중첩 → (6 반전+8 비교) → (3 호기심+8 비교) · **동의어 ★약속 확인** · **양면 봉인**]

### 4.3 온돌 — 4 공감 진입

- 블록1(진입): "소파 있으시죠? → 소파 아래 바닥에 앉기 → 왜? 바닥 따뜻 → 바닥 난방 국룰" → [(4 공감+5 상식파괴)중첩 → 3 호기심 → (8 비교+9 최신정보)중첩 · **반대 방향**]
- 블록2(ML): "바닥 난방=전통 온돌 → 언제부터? 어떻게?" → [9 최신정보 → 3 호기심 · **양면 봉인** · 약속(귀신)은 본문 사례로 이연]

### 4.4 루프탑코리안 — 단면 봉인

- 블록1(진입): "수정헌법 2조 권위 → 200년 된 법이 요즘 왜?" → [9 최신정보 → (2 문제제기+8 비교)중첩 → 3 호기심 · **반대 방향**]
- 블록2(ML): "루프탑 코리안 → 모범 소수가 어쩌다 총을?" → [(5 상식파괴+6 반전)중첩 → 9 최신정보 → 3 호기심 · **동의어 ★약속 확인** · **단면 봉인**]

### 4.5 당뇨 — 요인 촘촘 5개 · 역할 겸직

- 블록1(진입): "당뇨=현대 대사질환 → 오줌 당분이 뭐가 문제? → 혈관=초콜릿 공장·합병증" → [4 공감 → 9 최신정보 → 3 호기심 → (5 상식파괴+7 경고)중첩 · **동의어 ★약속 확인**]
- 블록2(ML): "천천히 망가짐 + 왜 걸리나? 예방?" → [(7 경고+2 문제제기)중첩 → 3 호기심 · **양면 봉인**]

### 4.6 참외 — 반대 정보 빌드업 · 블록2 직접 확인

- 블록1(진입·빌드업): "케이푸드 인기 → 근데 한국 농산물 경쟁력 낮음(김만 강함)" → [9 최신정보 → 5 상식파괴 · **반대 정보 빌드업**(농산물 약한데 참외 떡상 = 낙차)]
- 블록2: "일본이 한국참외 수입 급증(대일 수출 40%↑)" → [(8 비교+9 최신정보)중첩 → 3 호기심 · **직접 확인 ★약속 확인**]
- 블록3(ML): "왜 하필 참외? 외국에도 있나?" → [(3 호기심+5 상식파괴)중첩 · **양면 봉인**]

### 4.7 UN군 — 직접 노출(강 예외)

- 블록1(진입): "7월 27일 유엔군 참전의 날 → 22개국 200만 피와 땀 → 결성은 기적·처음이자 마지막" → [4 공감 → (9 최신정보+10 도덕공분)중첩 → 1 이익 → 5 상식파괴 · **직접 노출 ★약속 확인** (광고·감정형 강 예외)]
- 블록2(ML): "변방 나라 위해 전 세계(모순) + 왜 도왔나? 활약?" → [(5 상식파괴+6 반전)중첩 → 3 호기심 · **양면 봉인**]

### 4.8 서울요새도시 — 4블록 다층 반전

- 블록1(진입): "서울 소개·한강의 기적" → [9 최신정보 → 1 이익 → 4 공감 · **반대 방향**(위상 폭로)]
- 블록2: "외국인 '전쟁 안 무섭냐?' 북한 40km" → [4 공감 → (5 상식파괴+7 경고)중첩 → 3 호기심 · **반대 방향**]
- 블록3: "왜 수도 못 옮기나 — 통일·정통성" → [(3 호기심+9 최신정보)중첩 → 6 반전 · **반대 방향**]
- 블록4(ML): "뼛속 요새 도시 + 방어력? 어떻게?" → [(5 상식파괴+6 반전)중첩 → (3 호기심+8 비교) · **동의어 ★약속 확인** · **양면 봉인** · 약속 확인을 블록4까지 미룸]

### 4.9 사용 규칙

1. **압축 예시만 흡수** — 완성 문장 그대로 차용 금지. 매 주제 창의적 새 표현 생성.
2. **신뢰 상환 위치 명시** — 본문 이연 / 도입부 수행 중 P3 조감도에 기술.
