## §0. 서사 구조 프레임

> **레고 이론 (능동 추리자 메커니즘·L1·L2·L3·자발적 발견·아하·단서 수렴·지적 유능감)** → `지침/모듈/모듈_레고이론.md` §0 위임

### 0.1 결론 보류 + 간접 회수

#### ① 결론 보류

**최종 통찰**은 본문 후반부까지 보류한다. **아하 모먼트 자체**는 완결 조건 충족 시 본문 어느 지점에서도 발생 가능하다.

#### ② 간접 회수 (수미상관 구현)

**도입부에서 던진 소재·에피소드·비유·진입 장치**를 결론부에서 돌려받을 때 명시적 문구 반복이 아닌 동의어 치환·맥락 이동·간접 연결로 처리한다.

- **적용 대상:** 도입부 장치 (썸네일 약속이 아닌 도입부 소재)
- **목적:** 완결감·위트 부여 ("아, 이게 돌아왔네!" 효과)
- **주의:** 본 회수는 도입부 소재 **재등장 허용** (방식 제한). 모듈_레고이론 §0.3 자발적 발견과 적용 대상이 다르다 — §0.3은 썸네일 메인 자체 금지, 본 ②는 도입부 소재 재등장 허용

> **회수 처리 방식:** 동의어 치환·맥락 이동·간접 연결로 회수한다. 도입부 소재 그대로 반복은 회피.

### 0.2 서사 4단계 프레임

| 단계 | 정의 | 집필 지시 |
| --- | --- | --- |
| 1단계 | 약속의 선언과 공백 형성 | 제목+썸네일+ML을 결합하여 시청자에게 의외성을 약속한다 |
| 2단계 | 단서의 연쇄적 수렴 | 서사 유형별 구조에 따라 단서를 연쇄 배치한다 |
| 3단계 | 아하 모먼트와 발견 | 시청자가 스스로 최종 통찰을 발견하는 순간(아하 모먼트)을 설계한다 |
| 4단계 | 가치 확장 (결론부 착지) | 도달한 최종 통찰을 현대 사회·일상·미래·정서·실천 등으로 확장하여 여운을 남긴다 (§B.5 결론부 8유형 참조) |

**결론부 5단락 영역 (4단계 세부 분할):** 본문 정점 SL → 결론부 직전 단락 → 결론부 진입 단락 → 결론부 본체 단락 → 시그니처. 세부 운영은 P4 본체에서 정의한다.

### 0.3 약속의 완결성 + 신뢰 상환

#### ① 약속의 완결성

썸네일·제목이 제시한 "의외성"은 **2단계로 완결**된다.

1. **신뢰 상환 (1차 단계):** 신뢰 상환 전략(도입부 / 본문 이연)에 따라 도입부 또는 본문 내에서 초기 증거 제시. 시청자 이탈 방지 기능
2. **최종 통찰 도달 (2차 단계):** 본문 후반부에서 완성된 결론. 영상 전체의 지적 전리품

**결론부에서는 신뢰 상환·최종 통찰을 수행하지 않는다.** 결론부는 §B.5 엔딩 8유형으로 **여운 착지**한다.

#### ② 신뢰 상환

서사 유형별로 신뢰 상환 위치가 다르다. 세부 위치·이유는 모듈_엔진 해당 엔진 절 참조, 유형별 궁합 표는 §B.5 참조. 신뢰 상환 전략 결정은 P2 본체에서 정의한다.

### 0.4 인과관계의 접착

단서와 단서 사이에 논리적 유격이 없어야 한다. 연결 문장으로 인과를 명시한다.

### 0.5 집필 흐름 순서

집필 흐름: **서사 유형 확정 → 썸네일 → 도입부 → 본문 → 간접 확인 → 아하 → 해소 은유(사용 시) → 원칙 검증**. 세부 운영은 각 § 본문과 P4 본체 참조.

## §A. 썸네일·도입부 공통

### A.1 썸네일 판정 기준 — 3분류

썸네일 문구의 형태에 따라 3가지 전략으로 구분한다.

| 전략 | 판정 기준 | 문형 신호 | 참고 매핑 |
| --- | --- | --- | --- |
| **결과 제시형** | 결론·지위·단정을 직접 선언 | `~이다`, `~나뉜다`, `~하는 X`, `~할 수 없다` | 과정추적형, 이미지강화형 일부 |
| **조각 제시형** | 질문 형태 또는 특정 부품 던지기 | `~이유`, `~일까?`, `~라고?`, `얼마나 ~?`, `어떤 ~?` | 모순형, 이미지강화형 |
| **모순 공존형** | 썸네일 문구 내에 A극 속성과 B극 속성이 동시 제시 | `[긍정 수식] + [부정 명사]`, `[신적·절대적 표현] + [위험·문제 표현]` 조합 | 모순형 우세 |

**참고 매핑 운영:** 위 매핑은 빈도 경향이며 강제 아님. 본 채널 3유형 모두 3종 분류 자유 활용 가능 (39편 분석 기준 약 36% 자유 매핑 발생).

#### 모순 공존형 예시

- "악마의 풀" — 마성의 이미지 + 식물 속성
- "축복의 저주" 식 대조 — A극·B극 속성 충돌

#### 경계 사례 판정 원칙

문형의 **주도적 구조**를 본다.

- 질문 문형이 포함되면 조각 제시형
- 단정·지위 선언이 포함되면 결과 제시형
- A극·B극 속성 충돌이 명시적이면 모순 공존형

### A.2 썸네일 약속·도입부·본문 역할 분담 원리

용어 구분은 모듈_레고이론 §0.2 참조.

- **썸네일·제목 = 썸네일 약속** — 최종 통찰 암시
- **도입부 = 썸네일 약속이 왜 의외인지의 전제 조건 세팅** + (도입부 신뢰 상환 전략일 경우) 신뢰 상환 초기 증거 제공
- **본문 = 단서들이 최종 통찰을 향해 수렴** + 아하 모먼트 발생 지점

썸네일 문구를 반복하면 "이미 알고 있는 걸 또 말하네"가 되어 이탈한다. 전제 조건만 깔면 시청자가 스스로 궁금증을 생성하여 자발적 몰입이 일어난다.

#### 자발적 발견 (썸네일 약속 보호) — 직접 인용 금지 단일 정의

**썸네일 약속 메인 문구 자체**를 그대로 인용·반복하지 않는다. 동의어 치환·간접 확인으로 처리하여 시청자가 본문에서 아하 도약을 통해 최종 통찰을 자발 발견하도록 전제 조건만 세팅한다 (노출 강도 기준 모듈_집필기법 §4 — 약·중 강도 허용 / 강 강도 회피).

- **적용 대상:** 썸네일 **메인 문구** 자체 + 제목 (예: "인류를 지키고 있는 물질")
- **예외 — 썸네일 서브 문구:** 서브 문구는 본문/결론부에서 **직접 인용 허용**. 서브는 메인 미끼의 결과·보충 선언이라 본문 노출 시 후킹 약화 영향이 미미하고, 본문 통찰 강화 효과가 더 크다
- **목적:** 시청자 능동 추리 유도 (모듈_레고이론 §0.1 능동 추리자 원리·§0.6 지적 유능감 자극 정합). "이미 아는 걸 또 듣는다" 이탈 방지
- **단일 정의:** 본 §A.2에서만 정의한다. 모듈_집필기법 §4는 본 §A.2에 종속되며 우회 기법(동의어 치환·속성 대체·반대 방향 진입)은 모듈_집필기법 §4에서만 정의한다

### A.3 도입부 전략 A~H 정의 + 유형별 주력

도입부 첫 블록에서 사용하는 진입 전략. 서사 유형 + 주제 성격에 따라 1개 주력 전략을 선택한다.

#### A~H 각 전략 정의

| 코드 | 명칭 | 정의 |
| --- | --- | --- |
| A | 규모폭로 | 압도적 수치·스케일로 대상의 거대함을 즉시 증명 |
| B | 공감적이질감 | 시청자가 당연히 여기는 것에 이질감을 심어 균열 생성 |
| C | 자조적체감 | 대상의 결함·단점을 자조적으로 먼저 제시 |
| E | 밈진입 | 밈·패러디·농담으로 가볍게 진입 |
| F | 맥락재설정 | 시청자의 상식적 맥락을 뒤집는 새 맥락 제시 |
| G | 현재불안소환 | 현재 진행형 위기·불안을 소환 |
| H | 감정직접소환 | 감정·분노·감사를 직접 소환 |

> D 결번 — 과거 통계 진입형 D 코드가 A 규모폭로와 기능 중복으로 폐기, 번호는 안정성 유지.
>
> 각 전략 사용 예시는 모듈_어법 도입부 예시 카탈로그 참조.

#### 본 채널 3유형별 주력 전략

- **모순형**: B·F 우선 (A·C·E·G·H 회피 권장 — 부수 작동 허용)
- **이미지강화형**: B 우선 고려, 주제 적합 시 A·C·E·G·H 선택 (F 회피 권장)
- **과정추적형**: B 우선 고려, 주제 적합 시 A·F·G·H 선택 (C·E 회피 권장)

**범용 전략 안내:** B(공감적이질감)는 3유형 모두에서 우선 고려. 상식 전복 구조로 서사 유형 불문 범용성이 높다.

**B 전략 공감 진입 출발 단위 (3유형 공통):** 시청자 일상 경험에서 출발. 출발점은 이 주제에만 성립하는 구체적 **공통 인지·상식·경험·일상 장면·일상 행동·감각**이어야 한다. 주제 이름만 바꾸면 다른 대본에도 쓸 수 있는 문장은 금지. — **동료 호명 톤 사용** (예: "우리 다 아는 ~"·"~ 한 번쯤 들어봤죠"·"다들 ~ 경험 있죠").

## §B. 오픈 루프·구조 공통

### B.1 보상의 비례 원칙

공백의 크기(궁금증의 강도)에 비례해 해소 시 보상(지적 쾌감·스케일 확인·인과 완성)의 강도가 커야 한다. 시청자가 "기다린 만큼 값어치가 있었다"고 느끼게 한다.

**유형별 보상의 성격:**

- 모순형: 역설 해소 (A극·B극이 동일 원천에서 나온 양면성이었음의 발견 — "어? 같은 거였네?")
- 이미지강화형: 스케일 확인 ("이것까지 영향이 미쳤다고?")
- 과정추적형: 인과 체인 완성 ("이래서 이렇게 됐구나!")

**단서 — 모순형 비율 편차:** 모순형에서 A극이 역사적 서사나 공감 소재로 "읽는 재미"를 제공하는 경우, A극/B극 비율이 2:1~3:1까지 불균형해도 허용된다.

> **어조 변주 운영:** 위치별 어조 변주(유지=부드러움 / 전환·강화=유형별 단호함·경외·감정 이입 / 결론부=§B.5 참조)는 모듈_집필기법 본체에서 자연 결합. 별도 원칙 영역으로 분리하지 않는다.

### B.2 단일 대상 조건

ML 선언부에 도입부 비유를 삽입하려면 대상이 **단일 사물·개체**여야 한다. 현상·시스템·비교 구조에서는 비유 삽입을 피한다.

- **사용 가능 예시:** 옥수수·서울·여권·에어컨·고래잡이 등 단일 대상
- **사용 회피 예시:** 입식 vs 좌식 / 락스 원리 등 현상·시스템·비교 구조

### B.4 ML 구조 공통 원리

ML(메인 루프)은 **도입부 말미에서 본문이 답할 핵심 질문을 공식화하는 단위**이다. 서사 유형에 따라 변주하며, 유형별 구체 구조와 자체 형태 변주는 모듈_엔진 ML 영역 참조.

#### ML 3대 실용 기능

1. **본문 로드맵 제시** — ML 질문이 본문 파트를 예고 (예: "언제부터? 장단점?" → 역사 + 비교분석 2파트)
2. **본문 진입 선언 (선택)** — ML 질문 직후 본문으로 자연스럽게 넘어가면 생략 가능. 삽입 시 "~알아봅니다" / "한번 보시죠" / "지금부터 보시죠" 등 진입 표현만 허용. 채널명·화자 소개 삽입 금지
3. **서사 방향 설정** — 유형별 고유 기능

**ML 봉인 형태 (질문 개수):** 양면 봉인(2개 질문 — "왜? 어떻게?")이 기본. 단면 봉인(1개 질문)도 본문 단일 방향 추적 영상에서 활용 가능.

#### 이문호공 요인 참고 기준

이문호공 10요인 + 4역할 카탈로그는 모듈_이문호공.md 참조. ML 영역에서 이문호공 요인을 자연 결합한다. **자연스러운 궁금증 유발이 요인 카운팅보다 우선**한다.

- 본문 ML·해소 = **9 기본 요인** 중심
- 썸네일·제목 후킹 = **10요인 모두 + 부정·감정 변주** 활용

#### 본문 운영 원칙

본문은 메인 루프(ML) 1개를 축으로, 서브 오픈 루프 6~10개를 **연쇄적으로 배치**하여 ML 해소 동력을 유지한다. 각 SL은 독립적 '문제-해결'이 아니라 ML의 Q1·Q2 분산 해소, 피벗 직전 확신 극대화 등으로 **ML에 수렴**시킨다.

#### 본 채널 3유형별 ML 유지·강화·닫기 구현

| 단계 | 모순형 | 이미지강화형 | 과정추적형 |
| --- | --- | --- | --- |
| **유지** | Q2를 보류하여 공백 유지 (Q1은 A극에서 분산 해소) | How에서 Q1·Q2 해소 후 Scale Jump가 새 공백 생성 | 시간순 맥락을 축적하며 핵심 사건·인과를 보류 |
| **강화** | 피벗 직전 A극 확신 극대화 | Scale Jump 진입 시 새 루프 열기 | 인과 폭로로 절정 직전 텐션 극대화 |
| **닫기** | B극에서 3~4회 분할 제공. Q1·Q2 해소 위치 5패턴 허용 | Why에서 스케일 각인 — Scale Jump 2~5회 | 절정의 사건 자체가 해소 + 결론부에서 현재 의미 부여 — 전환점 2~6회 |

### B.5 결론부 8유형

결론부 전략은 아래 8유형 중 1~2개를 조합한다.

| # | 유형 | 착지 방식 |
| --- | --- | --- |
| ① | 일상행위연결 | "오늘 ~해 보세요" 식 실천 제안 |
| ② | 감정적여운 | "~한번쯤 생각해 보는 것도" 식 감정 착지 |
| ③ | 미래희망 | "~평화가 계속되길" 식 미래 기원 |
| ④ | 시의성마무리 | 현재 시점 이슈로 연결 |
| ⑤ | 위트반전 | 위트 있는 반전·유머로 착지 |
| ⑥ | 통찰형 | 본문 전체를 관통하는 한 문장 통찰 제시 |
| ⑦ | 경고형 | 대상의 한계·문제·위험 경고로 착지 |
| ⑧ | 광고·협찬형 | 협찬·제품 홍보와 결합한 착지 |

**⑦ 경고형 보편성 원칙 (필수 조건):** 경고 대상자가 시청자 대다수와 일치할 때만 채택한다. 일부 소수에게만 적용되는 경고는 ⑦ 채택 금지.

#### 본 채널 3유형별 결론부 궁합 요약

- **모순형:** 미래희망(③) + 위트(⑤) 혼합 주류. 심각 주제는 경고형(⑦) 단독 허용
- **이미지강화형:** 미래희망(③) 주력. 감정적여운(②) 비사용
- **과정추적형:** 감정적여운(②) + 시의성(④) 주류. 위트(⑤) 비사용

## §할루시네이션 방지

> **단일 지침 위치.** 모든 Phase·에이전트가 본 절을 참조한다. 마스터·P2·P3·P4·P5 단계 파일은 1줄 인용으로 본 절을 가리킨다.

### 원칙 — 전 Phase 공통

#### ① 지침 예시 활용 방식은 종류에 따라 다르다

- **패턴형 예시** (`~`·`[N]`·`OOO` 등 자리채움 마커 포함): 마커 자리를 주제에 맞게 채워 그대로 사용 가능. 다만 한 대본 내 동일 패턴 2회 이상 사용 시 다른 패턴 또는 새 표현으로 교체한다
- **완성 문장형 예시** (자리채움 마커 없는 완성 문장·문단): 구조·어조·분량 감각만 참고하고, 실제 단어·문장은 해당 주제와 맥락에 맞게 새로 작성한다

#### ② 지침 내부 용어 노출 금지

SL·ML·BR·A극·B극·역설엔진·이문호공·해소 비유·피벗 등 지침 전용 용어를 대본에 노출하지 않는다.

#### ③ 같은 비유·밈·감탄사 한 편 내 반복 금지

같은 표현을 한 영상에서 두 번 이상 사용하지 않는다.

#### ④ 확인되지 않은 수치·사실 단정 금지

웹 검색으로 팩트 확인 후 사용한다.

### P2 딥리서치 단계 적용

- 존재하지 않는 통계나 연구를 만들어내지 않는다
- 상관관계를 인과관계로 단정하지 않는다
- 확인되지 않은 팩트를 아하 모먼트 후보로 채택하지 않는다
- 팩트가 부족한 경우 추가 검색을 수행한다. "팩트가 없는 구조"를 만들지 않는다

### P3 구조 단계 적용

- SL 개수를 억지로 맞추기 위해 불필요한 정보를 넣지 않는다
- 감정 곡선을 도식적으로 따르느라 부자연스러운 전환을 만들지 않는다
- 팩트 리포트에 없는 내용으로 구조를 채우지 않는다

### P4 집필 단계 적용

- **화자가 자기 정보에 놀라는 척 금지** — 감탄·감상을 제거했을 때 정보나 대비가 남지 않는 문장은 제거. 대비를 명시하는 문장으로 교체한다
- **비유의 메커니즘 일치 확인** — 표면적 유사성이 아니라 작동 원리가 일치하는지 검증
- **수정 후 정합성 확인 (일반):** 문장 수정 시 같은 단락 전체를 다시 읽는다
- **수정 후 정합성 확인 (구조 요소):** ML·피벗·해소 은유를 수정할 경우 해당 요소를 참조하는 모든 문장을 검색하여 재검증한다
- **수치 최신성 확인** — 현재 시점 기준 최신인지, 주체·범위·조건이 보존되었는지 확인
- **집필 중 추가 팩트 출처 기록** — 팩트리포트에 없는 정보는 반드시 웹 검색 확인 후 상태 태그 "추가 팩트"에 내용·출처 기록. 미확인 정보 단정형 서술 금지

### P5 퇴고 단계 적용

- 퇴고 시 "통과"를 위해 실제로 고치지 않고 체크만 하는 행위 금지
- 문제를 발견하면 반드시 수정 후 해당 항목 재검증
- "이 정도면 괜찮다"는 판단 금지 — 기준에 미달하면 수정
