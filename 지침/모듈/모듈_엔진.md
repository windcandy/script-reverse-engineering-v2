## §0. 본 채널 3유형 ↔ 8엔진 매칭 + 결정 흐름

### 0.1 본 채널 3유형 ↔ 8엔진 매칭표

| 본 채널 3유형 | 매칭 엔진 (주) | 매칭 엔진 (잠정) | 본질 |
| --- | --- | --- | --- |
| **모순형** | §1 역설 엔진 | §4 결합형 엔진 (역설+인과추적) · §5 다층원인수렴 엔진 | 속성·의미 반전 폭발 |
| **이미지 강화형** | §2 영역확장 엔진 | §6 사례비교 엔진 | 영역 확장 / 사례 비교 → 1점 강화 |
| **과정 추적형** | §3 인과추적 엔진 | (없음 — 인과추적 단일) | 시간·원인 누적 추적 |

**용어 정합:**

- **본 채널 3유형 (서사 유형)** = 본 채널 대본의 거시 서사 골격. P1 후보 결정 + P3 확정 단위.
- **8엔진 (서사 엔진)** = 본 채널 3유형의 세부 작동 양식. P3 구조설계 진입 단위.
- **9유형 (정보 유형)** = 본문 단락별 정보 분류. 본 모듈에서는 엔진별 "강세 정보유형 권장 범위" 컬럼에서 사용. 정의 본체는 00_용어정리 §6 위임.
- **블록** = 본문·도입부·결론부의 단락 단위. **한 화제·한 정점·한 정보 묶음을 다루는 최소 의미 단위**(분량은 30초~3분 전후로 자연 결정, 정량 강제 없음). 본 모듈에서 "[승] 블록 수·정점 블록·8블록 이상" 등 모든 블록 표기는 본 정의를 따른다. 도입부 블록 골격은 모듈_도입부 §2 도입부 공통 골격 정합.

### 0.2 주 엔진 vs 잠정 엔진

- **주 엔진 (역설·영역확장·인과추적)** — 운영 시 채택 우선. 핵심 패턴 강제 / 변주 ±5%p.
- **잠정 엔진 (결합형·다층원인수렴·사례비교)** — 운영 시 사용 가능. 핵심 패턴 강제 유지하되 세부 비중 변주 ±10%p 확대 허용. 후속 분석 누적 시 주 엔진으로 격상.
- **사용 보류 (사례나열·다층분기수렴)** — 분석 1편으로 일반화 근거 부족. 본 모듈 §7·§8에 보존하되 운영(집필) 사용 안 함. 후속 분석 2편+ 누적 시 잠정 엔진으로 복귀.

### 0.3 엔진 결정 흐름 (P3 진입 시)

```
1. 주제 → 본 채널 3유형 확정 (P1·P3 결정)
2. 본 채널 3유형 → 매칭 엔진 후보 결정 (§0.1 표)
   - 모순형: 역설 (주) 또는 결합형·다층원인수렴 (잠정)
   - 이미지 강화형: 영역확장 (주) 또는 사례비교 (잠정)
   - 과정 추적형: 인과추적 (주, 단일)
3. 매칭 엔진 후보 → 해당 §N 결정 트리 진입
4. 결정 트리 통과 → 본문 구조 + 부가 표기 영역 + 정보 배치 확정
5. 변주 허용 영역 → 새 주제 특성에 맞춰 자유 조정
```

### 0.4 엔진 선택 시 자가 점검 (혼동 방지)

- 두 측면이 충돌하는 주제 (긍정·부정 양면) → **역설 엔진** 우선
- 인과 빌드업 끝에 속성 반전이 폭발 (그림자) → **결합형 엔진**
- 하나의 대상을 두 영역으로 가를 수 있음 (사용 방식·용도·공간·시대·평가·기술·유형) → **영역확장 엔진**
- 옛 시기 → 현 시기 인과 체인 명확 → **인과추적 엔진**
- 다층 N층 원인이 1점으로 수렴 (구조적·국가 단위 원인) → **다층원인수렴 엔진**
- 두 사례 깊이 비교 → 공통 원리 수렴 → **사례비교 엔진**

## §1. 역설 엔진 (모순형 주)

### 1.1 엔진 정의

대상의 한 측면(A극)을 단조 누적으로 빌드업한 뒤 피벗을 통해 정반대 측면(B극)으로 흐름을 전복하여 속성·의미 반전을 폭발시키는 엔진. **단조 빌드업 → 피벗 → 반전 빌드업** 구조 자체가 본질.

### 1.2 본문 구조 골격

```
본문 = [A극 누적 ([승])] → [피벗 (보상+속성 반전)] → [B극 분할 ([전])]
```

- **[승] A극 빌드업 영역** — 한 측면 단조 누적. 정점 형성으로 피벗 직전 최고점.
- **[전] 첫 블록 = 피벗** — 보상 투하 + 속성 반전 이중 역할.
- **[전] B극 분할 영역** — 반전된 측면을 사례·메커니즘으로 회수.

### 1.3 부가 표기 영역

| 위치 | 표기 영역 | 명명 가이드 |
| --- | --- | --- |
| [승] | A극 빌드업 | 주제 톤에 맞춰 자유 명명 (위력·마법·광범위·장점 등). 긍정/부정 방향성은 자유. |
| [전] | B극 빌드업 | A극과 짝을 이루는 반대 측면 명명 (함정·저주·파괴·단점 등). |

### 1.4 강세 정보유형 권장 범위

> **9유형 정의 본체 → 00_용어정리 §6 참조.**

| 영역 | 강세 정보유형 (1~2위) | 권장 범위 |
| --- | --- | --- |
| [승] A극 | 메커니즘·기원·비교·사례 4종 분산 동률 권장 | 단일 유형 25% 미만 / 4종 합산 60% 이상 |
| [전] B극 | 메커니즘·사례 양강 | 합산 40% 이상 |

**해석:** A극은 단조 빌드업 회피를 위한 4종 정보유형 분산. B극은 함정 작동 원리(메커니즘) + 충격 사례 콤보가 표준.

### 1.5 핵심 패턴 (강제)

| 항목 | 권장 범위 | 강제 사유 |
| --- | --- | --- |
| [승] 블록 수 | 전체의 35~50% | A극 누적이 부족하면 피벗 진폭 약화 |
| [전] 블록 수 | 전체의 25~35% | B극 회수가 부족하면 역설 미완성 |
| [승] 정보유형 분산 | 4종 이상 혼용 | 단조 빌드업 방지 |
| 피벗 1블록 | 필수 — [전] 시작 위치 | 반전형 + 호기심형 또는 경고형 결합 |
| [승] 끝 = A극 정점 | 필수 | 피벗 직전 최고점 형성 |
| [전] 끝 = 반전 회복 또는 함정 결산 | 필수 | 회수 완결 |
| A극 ↔ B극 같은 메커니즘 양면 | 권장 — 역설 핵심 장치 | 같은 정보가 위력이자 함정 |

### 1.6 변주 허용 영역 (가변 ±5%p)

| 항목 | 가변 범위 | 사유 |
| --- | --- | --- |
| [승]·[전] 명칭 | 주제 톤에 맞춰 자유 명명 | 분석 표본 5편 모두 다름 |
| [승] 첫 항목 긍정/부정 방향 | 자유 — A극 = 긍정이 강제 아님 | 단조 → 반전 구조 자체가 핵심 |
| [승] 안의 의도적 지연 1~2블록 | A극 길이 6분 이상일 때 권장 | 짧으면 불필요 |
| 9유형 비율 | ±5%p 가변 | 주제별 정보 가용성 |
| 결론부 블록 수 | 1~3블록 자유 | 본 채널 결론부 글자수 제거 정책 정합 |

### 1.7 결정 트리

```
1. 주제의 "단조 인상" 확인 — 일면적으로 강해/약해 보이는 대상인가?
   YES → 역설 엔진 채택. NO → 다른 엔진 검토.

2. [승] 방향 결정 — A극을 긍정으로 갈지 부정으로 갈지
   주제 통념이 긍정 → [승] = 긍정 (B극 단점 폭로 진폭 강화)
   주제 통념이 부정 → [승] = 부정 (B극 반전 발견 진폭 강화)

3. [승] 길이 결정 — 6분 이상이면 의도적 지연 1~2블록 삽입
   6분 미만이면 단조 빌드업 그대로 진행

4. 피벗 1블록 작성 — 반전형 + (호기심형 또는 경고형) 결합어

5. [전] 사다리 설계 — 단점/회복 항목 3~5개 적층
   각 항목마다 메커니즘(원리) + 사례(증거) 1세트 권장

6. 결론부 — 양 극 결산 + 미래 전망 또는 위트 반전
```

### 1.8 아하 완결 조건

**아하 본질:** "A극의 바로 그 속성이 B극에서 역전됨"을 시청자가 발견하는 순간.

**완결 4조건 (모두 충족 시 아하 자연 발생):**

1. A극 속성 X가 최소 2회 이상 누적되었는가?
2. 피벗 이후 B극에서 X가 반대 결과의 원인으로 재해석되었는가?
3. 시청자가 "A극 X = B극 재앙의 원인"을 스스로 연결할 단서가 제공되었는가?
4. A극 누적 정점 직후 피벗에서 즉시 작동했는가?

**해소 위치:** % 강제 아님. 4조건 충족 시점에서 자연 발생.

### 1.9 엔딩 궁합

> **엔딩 8유형 정의 본체 → 모듈_본문구조.md §2 참조.**

- **주력:** 미래희망(③) + 위트반전(⑤) 혼합 — 양면성 발견 후 균형 회귀에 자연 정합.
- **허용:** 경고형(⑦) 단독 — 심각 주제 한정. 단 경고 대상자가 시청자 대다수와 일치해야 채택.
- **회피:** 감정적여운(②) 단독 — 양극 결산에 정합 약함.

### 1.10 다른 엔진과의 차이 (혼동 방지)

- vs **결합형 엔진** — 역설 엔진은 속성 반전 단일 폭발 / 결합형은 인과 빌드업 + 속성 반전 이중 폭발 (인과 Aha + 역설 Aha 동시).
- vs **영역확장 엔진** — 역설은 같은 대상의 양면 / 영역확장은 두 영역으로 분할 확장.

## §2. 영역확장 엔진 (이미지 강화형 주)

### 2.1 엔진 정의

하나의 주제를 두 영역으로 분할하고 각 영역을 단계별로 누적 빌드업하여 영역 A의 작동 원리가 영역 B로 확장·반전되는 양상을 보여주는 엔진. **같은 축의 영역 확장** 자체가 본질.

### 2.2 본문 구조 골격

```
본문 = [영역 A 빌드업 ([승])] → [영역 전환점 (보상+오픈루프)] → [영역 B 확장 ([전])]
```

- **[승] 영역 A 빌드업** — 영역 A 작동 원리 사다리식 누적. 미니 회수 반복.
- **[전] 첫 블록 = 영역 전환점** — 영역 A 누적 회수 + 영역 B 사다리 점화 이중 역할.
- **[전] 영역 B 확장** — 영역 B의 다측면 종합 결산.

### 2.3 부가 표기 영역

| 위치 | 표기 영역 | 명명 가이드 |
| --- | --- | --- |
| [승] | 영역 A | 시청자가 더 친숙한 쪽(원형) 권장 (좌식·식용·강북·이전·혜택·내연기관·2형 등). |
| [전] | 영역 B | 영역 A로부터 확장·반전되는 쪽(확장) 권장 (입식·산업·강남·이후·한계·전기차·1형 등). |

**분류축 종류:** 사용 방식·용도·공간·시대·평가·기술·유형 등 자유 선택.

### 2.4 강세 정보유형 권장 범위

| 영역 | 강세 정보유형 (1~2위) | 권장 범위 |
| --- | --- | --- |
| [승] 영역 A | 메커니즘·인과 양강 | 합산 50% 이상 |
| [전] 영역 B | 인과 + 사례·해석·비교 5종 분산 | 단일 유형 30% 미만 / 인과 25% 이상 |

**해석:** [승]은 영역 A 작동 원리 사다리식 누적. [전]은 영역 B의 다양한 측면을 종합 결산.

### 2.5 핵심 패턴 (강제)

| 항목 | 권장 범위 | 강제 사유 |
| --- | --- | --- |
| [승] 블록 수 | 전체의 40~65% | 영역 A 작동 원리 누적이 부족하면 영역 B 확장 의미 약화 |
| [전] 블록 수 | 전체의 20~40% | 영역 B 회수가 부족하면 확장 미완성 |
| [승] 메커니즘·인과 비중 | 합산 50% 이상 | A극 원리축 형성 — 단순 사례 나열 회피 |
| [전] 첫 블록 = 영역 전환점 | 필수 — [전] 시작 위치 | 보상+오픈루프 이중 역할, 반전형 + 비교형 결합 |
| [승] 끝 = 영역 A 정점 | 필수 | 영역 전환 직전 최고점 형성 |
| [전] 끝 = 영역 B 결산 | 필수 | 두 영역 통합 회수 완결 |
| A극 ↔ B극 같은 축의 영역 확장 | 권장 | 영역확장 핵심 장치 |

### 2.6 변주 허용 영역 (가변 ±5%p)

| 항목 | 가변 범위 | 사유 |
| --- | --- | --- |
| 영역 A·B 명칭 | 주제 분류축에 맞춰 자유 명명 | 분석 표본 다양 |
| 분류축 자체 (사용 방식·용도·공간·시대·평가·기술·유형) | 자유 선택 | 주제 특성 |
| [승] 안의 미니 영역 확장 | [승] 길이 5분 이상일 때 권장 | 짧으면 불필요 |
| 9유형 비율 | ±5%p 가변 | 주제별 정보 가용성 |
| 결론부 블록 수 | 1~2블록 자유 | 본 채널 결론부 글자수 제거 정책 정합 |

### 2.7 결정 트리

```
1. 주제의 "두 영역 분할 가능성" 확인 — 하나의 대상을 어떤 축으로 두 영역으로 가를 수 있는가?
   YES → 영역확장 엔진 채택. NO → 다른 엔진 검토.

2. 영역 A·B 명칭 결정 — 분류축에 맞춰 두 영역 명명
   영역 A = 시청자가 더 친숙한 쪽 (원형) 권장
   영역 B = 영역 A로부터 확장·반전되는 쪽 (확장) 권장

3. [승] 길이 결정 — 5분 이상이면 영역 A 내부 미니 영역 확장 1~2단 삽입
   5분 미만이면 단조 빌드업 그대로 진행

4. [승] 사다리 설계 — 메커니즘·인과 50% 이상 + 미니 회수 반복

5. 영역 전환점 1블록 — [전] 첫 블록에 보상+오픈루프 이중 역할
   반전형 + (상식 파괴형 또는 비교형) 결합어

6. [전] 사다리 설계 — 영역 B 다측면 3~5블록
   인과 1건 + 사례·해석·비교 등 다양화

7. 결론부 — 두 영역 통합 결산 + 미래 전망 또는 일상 회귀
```

### 2.8 아하 완결 조건

**아하 본질:** "이것까지 영향이 미쳤다고?"의 스케일·영역 발견.

**완결 4조건:**

1. 영역 A 작동 원리가 누적되어 시청자 기존 이미지의 마지막 저지선(일상·상식·경험)이 뚫렸는가?
2. 영역 전환점에서 "이것까지?" 반응을 유발할 구체 연결이 제공되었는가?
3. 영역 B에서 같은 축의 확장·반전이 실증되었는가?
4. 단순 "스케일 수치"가 아니라 "보이지 않던 연결"을 가시화했는가?

**해소 위치:** % 강제 아님. 4조건 충족 시점에서 자연 발생.

### 2.9 엔딩 궁합

- **주력:** 미래희망(③) — 두 영역 통합 후 "앞으로도 계속" 착지에 자연 정합.
- **허용:** 통찰형(⑥) — 두 영역 본질 압축 한 문장.
- **회피:** 감정적여운(②) — 스케일 주제와 궁합 약함.

### 2.10 다른 엔진과의 차이

- vs **다층분기수렴 엔진** — 영역확장은 2영역 균형 확장 / 다층분기수렴은 N분기(3+) → 1점 수렴.
- vs **인과추적 엔진** — 영역확장은 분류축 이동 (공간·용도·시대 등) / 인과추적은 시간 축 단절.

## §3. 인과추적 엔진 (과정 추적형 주)

### 3.1 엔진 정의

옛 시기의 인과 체인을 사다리식으로 추적하고 시기 단절점([전] 진입)을 통해 현 시기 결과·결산으로 도달하는 엔진. **시간 축의 단절·전환점** 자체가 본질.

### 3.2 본문 구조 골격

```
본문 = [옛 시기 인과 1 ([승])] → [[전] 진입 (시기 단절)] → [현 시기 인과 2 ([전])]
```

- **[승] 옛 시기 인과 빌드업** — 인과 노드 미니 빌드업·미니 회수 반복 사다리. 각 시기마다 "원인 → 결과" 작은 사이클.
- **[전] 첫 블록 = [전] 진입** — 옛 시기 인과 회수 + 현 시기 사다리 점화 이중 역할 (시기 단절점).
- **[전] 현 시기 결과·결산** — 현재 상태·통계·해석 통합.

### 3.3 부가 표기 영역

| 위치 | 표기 영역 | 명명 가이드 |
| --- | --- | --- |
| [승] | 인과 1 (옛 시기) | 옛 시기·기원·발견·발명·확장 명칭 (발명·확장·흥망·건설·전성·파병·전공 등). |
| [전] | 인과 2 (현 시기) | 현 시기·진화·규제·부활·결과 명칭 (확장·현재·규제·변형·회복·발굴 등). |

**의미 방향:** 발전 / 쇠퇴 / 반전 / 확장 / 갈등→수교 등 자유 — 시기 단절 자체가 본질.

### 3.4 강세 정보유형 권장 범위

| 영역 | 강세 정보유형 (1~2위) | 권장 범위 |
| --- | --- | --- |
| [승] 인과 1 | 인과 + 사례·일화 양강 | 합산 60% 이상 |
| [전] 인과 2 | 인과 + 사례·일화 + 통계·해석 | 인과 35% 이상 |

**해석:** 양 영역 모두 인과 + 사례 양강 (시간 축 인과 체인 + 구체 사건). [승]은 기원·메커니즘 보조, [전]은 통계·해석 보조.

### 3.5 핵심 패턴 (강제)

| 항목 | 권장 범위 | 강제 사유 |
| --- | --- | --- |
| [승] 블록 수 | 전체의 45~65% | 옛 시기 인과 체인이 부족하면 [전] 시기 전환 진폭 약화 |
| [전] 블록 수 | 전체의 15~35% | 현 시기 결과 회수가 부족하면 시기 추적 미완성 |
| [승] 인과·사례 비중 | 합산 60% 이상 | 시간 축 인과 노드 + 구체 사건 빌드업 |
| [전] 첫 블록 = [전] 진입 | 필수 — [전] 시작 위치 | 보상+오픈루프 이중 역할, 반전형 + 상식 파괴형 결합 |
| [승] 끝 = 옛 시기 정점 | 필수 | 시기 전환 직전 최고점 형성 |
| [전] 끝 = 현 시기 결과·결산 | 필수 | 시기 추적 완결 |
| [승]→[전] 시기 단절 표시 | 필수 | 인과추적 핵심 장치 |

### 3.6 변주 허용 영역 (가변 ±5%p)

| 항목 | 가변 범위 | 사유 |
| --- | --- | --- |
| [승]·[전] 명칭 | 주제 시간 축에 맞춰 자유 명명 | 주제별 다양 |
| 의미 방향 ([승]→[전] 발전/쇠퇴/반전/확장) | 자유 — 주제 특성 | — |
| [승] 안의 의도적 지연 1~2블록 | [승] 8블록 이상일 때 권장 | 짧으면 불필요 |
| 9유형 비율 | ±5%p 가변 | 주제별 정보 가용성 |
| 결론부 블록 수 | 1~4블록 자유 | 본 채널 결론부 글자수 제거 정책 정합 |

### 3.7 결정 트리

```
1. 주제의 "시간 축 인과 체인 가능성" 확인 — 옛 시기에서 현 시기로 가는 인과 체인이 명확한가?
   YES → 인과추적 엔진 채택. NO → 다른 엔진 검토.

2. [승]·[전] 시기 단절점 결정 — 어디서 옛 시기와 현 시기를 가를 것인가?
   (특정 연도·사건·정책·기술 도약·정치 변혁 중 1종)
   결정점은 [전] 첫 블록("[전] 진입")으로 표기

3. [승] 길이 결정 — 인과 노드 수에 따라 가변
   8블록 이상이면 의도적 지연 1~2블록 삽입

4. [승] 사다리 설계 — 인과·사례 60% 이상 + 미니 회수 반복
   각 블록 = 시간순 인과 노드(원인→결과) 또는 미니 보상

5. [전] 진입 1블록 — 시기 단절을 표시하는 블록 + 보상+오픈루프 이중 역할
   "그런데 [현 시기 변수] 등장 → 모든 게 바뀐다" 어법
   반전형 + (상식 파괴형 또는 최신 정보형) 결합어

6. [전] 사다리 설계 — 현 시기 결과·현황
   인과 1건 + 통계·사례·해석 등 다양화

7. 결론부 — 시기 통찰 + 미래 전망 또는 일상 회귀
```

### 3.8 아하 완결 조건

**아하 본질:** "이래서 이렇게 됐구나"의 인과 발견 또는 "이런 사람들이 있었구나"의 감정 결집.

**완결 4조건:**

1. 인과 체인의 모든 필수 링크가 연결되었는가?
2. 시청자가 "그래서 이렇게 됐구나"를 인지할 최종 고리가 공개되었는가?
3. (감정 결집형) 개별 사례들이 감정적 통합 수준에 도달했는가?
4. [전] 진입 시점에서 시기 단절이 명확히 표시되었는가?

**해소 위치:** % 강제 아님. 4조건 충족 시점에서 자연 발생.

### 3.9 엔딩 궁합

- **주력:** 감정적여운(②) + 시의성마무리(④) — 시기 통찰 후 잔향 또는 현재 시점 연결.
- **허용:** 통찰형(⑥) — 시기 본질 압축 한 문장.
- **회피:** 위트반전(⑤) — 인과 추적 본질과 궁합 약함.

### 3.10 다른 엔진과의 차이

- vs **다층원인수렴 엔진** — 인과추적은 시간 축 1차원 인과 체인 / 다층원인수렴은 다차원 원인 병렬 누적.
- vs **사례비교 엔진** — 인과추적은 옛→현 시기 단절 / 사례비교는 사례 A↔B 분기.
- vs **사례나열 엔진** — 인과추적은 시간 축 단일 노드 추적 / 사례나열은 N≥5건 균등 나열.

## §4. 결합형 엔진 (모순형 잠정)

### 4.1 엔진 정의

인과추적의 인과 빌드업 끝에 역설의 속성 반전이 폭발하여 인과 Aha와 역설 Aha를 동시에 작동시키는 엔진. **인과 빌드업 + 속성 반전 결합** 자체가 본질. **본 채널 3유형 매칭:** 모순형 (잠정) — 역설 본질 포함.

### 4.2 본문 구조 골격

```
본문 = [인과 빌드업 ([승])] → [피벗 (보상+속성 반전)] → [그림자 ([전])]
```

- **[승] 인과·사상 빌드업** — 시간 축 또는 사상 축의 인과 사다리 누적. 충전 압도 (~55%).
- **[전] 첫 블록 = 피벗** — 역설 엔진과 동일 용어 사용. 인과 결과물이 사실은 그림자였다는 속성 반전.
- **[전] 그림자 영역** — 인과 누적의 어두운 면을 사례·메커니즘·결산으로 회수.

### 4.3 부가 표기 영역

| 위치 | 표기 영역 | 명명 가이드 |
| --- | --- | --- |
| [승] | 빌드업 (인과 축) | 인과 빌드업 축 명명 (글로벌화·종말론 발전·기술 발전 등). |
| [전] | 그림자 | 인과 결과물의 어두운 면 명명 ("그림자" 표기 권장). |

### 4.4 강세 정보유형 권장 범위

| 영역 | 강세 정보유형 (1~2위) | 권장 범위 |
| --- | --- | --- |
| [승] 빌드업 | 인과 + 기원·배경 양강 | 합산 50% 이상 |
| [전] 그림자 | 메커니즘 + 인과 + 비교·대비 | 합산 60% 이상 |

**해석:** [승]은 인과·사상 빌드업이 본질. [전]은 그림자 작동 원리(메커니즘) + 양면 대비(비교) 결합. [승]→[전] 이행에서 통계·수치·기원 → 0%, 사례·해석 증가 (결산·반전 모드 전환).

### 4.5 핵심 패턴 (강제)

| 항목 | 권장 범위 | 강제 사유 |
| --- | --- | --- |
| [승] 블록 수 | 전체의 40~60% | 인과/사상 빌드업이 부족하면 피벗 진폭 약화 |
| [전] 블록 수 | 전체의 15~40% | 그림자 회수가 부족하면 역설 반전 미완성 |
| [승] 인과·기원·정의 비중 | 합산 55% 이상 | 시간/사상 인과 축 형성 |
| [전] 첫 블록 = 피벗 | 필수 — [전] 시작 위치 | 보상 투하 + 속성 반전 이중 역할, 반전형 + 상식 파괴형 + (경고형 또는 호기심형) 결합 |
| [승] 끝 = 인과 빌드업 정점 | 필수 | 피벗 직전 최고점 형성 |
| [전] 끝 = 그림자 결산 + 이중 Aha 완성 | 필수 | 역설 + 인과 양면 회수 완결 |
| 이중 Aha 구조 | 권장 — 양식 본질 | 역설 Aha (속성 반전) + 인과 Aha (시간/사상 추적) 둘 다 발동 |

### 4.6 변주 허용 영역 (잠정 가변 ±10%p)

| 항목 | 가변 범위 | 사유 |
| --- | --- | --- |
| [승]·[전] 명칭 | 주제 인과 축에 맞춰 자유 명명 | — |
| 인과 축 방향 (시간 축 / 사상 축 / 기술 축) | 자유 선택 | 잠정 엔진 — 표본 작음 |

### 4.7 결정 트리

```
1. 주제의 "역설 + 인과 결합 가능성" 확인 — 인과 빌드업 끝에 속성·의미 반전이 폭발하는가?
   YES → 결합형 엔진 채택 (모순형으로 작성).
   NO → 단일 엔진 검토 (역설 엔진 또는 인과추적 엔진).

2. 인과 축 결정 — 시간 축인가 사상 축인가 기술 축인가
   주제 인과 노드가 시간순이면 시간 축
   주제 인과 노드가 사상·이론 발전이면 사상 축

3. [승] 길이 결정 — 인과 노드 수에 따라 가변
   8블록 이상이면 의도적 지연 1~2블록 삽입

4. [승] 사다리 설계 — 인과·기원·정의 55% 이상 + 미니 회수 반복

5. 피벗 1블록 — [전] 첫 블록 = 보상 투하 + 속성 반전
   "[승] 인과 결과물이 사실은 [그림자]였다" 어법
   반전형 + 상식 파괴형 + (경고형 또는 호기심형) 결합어

6. [전] 사다리 설계 — 그림자 사례·메커니즘
   메커니즘 + 비교·대비 + 사례·일화 + 해석·결산 다양화
   이중 Aha 권장 — 역설 Aha + 인과 Aha 둘 다 작동

7. 결론부 — 균형 회귀 + 일상 톤 + 화자 합류 권장
```

### 4.8 아하 완결 조건

**이중 Aha 본질:**

- **역설 Aha** = 속성·의미 반전 폭발 (인과 결과물의 그림자 폭로).
- **인과 Aha** = 시간/사상 인과 체인 도달 폭발 (인과 추적 정점).

**완결 4조건:**

1. [승] 인과 빌드업이 정점에 도달했는가?
2. 피벗에서 속성 반전이 작동했는가? (역설 Aha)
3. [전] 그림자 영역에서 인과 추적이 완결되었는가? (인과 Aha)
4. 양 Aha가 한 영상 안에서 모두 폭발했는가?

**해소 위치:** % 강제 아님. 4조건 충족 시점에서 자연 발생.

### 4.9 엔딩 궁합

- **주력:** 통찰형(⑥) + 위트반전(⑤) 혼합 — 양면 발견 후 균형 회귀.
- **허용:** 미래희망(③) — 그림자 결산 후 균형 회귀.
- **회피:** 단순 감정적여운(②) — 이중 Aha 결산에 정합 약함.

### 4.10 다른 엔진과의 차이

- vs **역설 엔진** — 결합형은 인과 빌드업 + 속성 반전 이중 Aha / 역설은 속성 반전 단일 Aha.
- vs **인과추적 엔진** — 결합형은 [전]에서 속성 반전 폭발 / 인과추적은 시기 결과 결산.

## §5. 다층원인수렴 엔진 (모순형 잠정)

### 5.1 엔진 정의

다층 N층 원인을 1점(메인 오픈루프의 답)으로 수렴시키는 엔진. 원인 단위가 차원(역사·기술·정책·자연·생물·문화 등)이며 각 원인 차원이 동시에 작동한다. **N층 원인 → 1점 수렴** 자체가 본질. **본 채널 3유형 매칭:** 모순형 (잠정) — 다층 원인 수렴이 모순 폭로로 작동.

### 5.2 본문 구조 골격

```
본문 = [국가·구조 원인 ([승])] → [[전] 진입 (차원 추가)] → [개인·예외·보완 원인 ([전])]
```

- **[승] N층 원인 병렬 누적** — 다층 원인 노드마다 미니 빌드업·미니 회수 사이클.
- **[전] 첫 블록 = [전] 진입** — 인과추적 엔진과 동일 용어 사용. [승]만으로는 부족 + 새 원인 차원 추가.
- **[전] 보완 원인·예외** — 개인 차원·예외 사례·국제 비교 영역 확장.

### 5.3 부가 표기 영역

| 위치 | 표기 영역 | 명명 가이드 |
| --- | --- | --- |
| [승] | 국가·구조 원인 | 주제별 자유 (외교 원인·석유 수입 원인·발생 원인 등). |
| [전] | 개인·보완 원인+예외 | 차원 추가 영역 (개인 차원·예외 사례·국제 비교 등). |

### 5.4 강세 정보유형 권장 범위

| 영역 | 강세 정보유형 (1~2위) | 권장 범위 |
| --- | --- | --- |
| [승] 다층 원인 | 인과 + 메커니즘 양강 | 합산 50% 이상 |
| [전] 보완 원인 | 인과 + 사례·일화 양강 | 합산 60% 이상 |

**해석:** [승]은 인과·메커니즘 우세 (자연·과학·생물 메커니즘 다수 포함 가능). [전]은 인과 + 사례 결합.

### 5.5 핵심 패턴 (강제)

| 항목 | 권장 범위 | 강제 사유 |
| --- | --- | --- |
| [승] 블록 수 | 전체의 50~65% | N층 원인 누적이 부족하면 1점 수렴 진폭 약화 |
| [전] 블록 수 | 전체의 15~30% | 보완 원인·예외가 부족하면 다층 본질 미완성 |
| [승] 인과·메커니즘 비중 | 합산 50% 이상 | 원인 추적 본질 |
| [전] 첫 블록 = [전] 진입 | 필수 — [전] 시작 위치 | 보상+오픈루프 또는 의도적 지연 역할, 반전형 + 상식 파괴형 결합 |
| [승] 끝 = 다층 원인 결산 또는 정점 | 필수 | [전] 진입 직전 최고점 형성 |
| [전] 끝 = 통합 회수 + 예외 결산 | 필수 | 다층 원인 분산 회수 완결 |
| 메인 오픈루프 = 단일 "Why" 질문 | 필수 | 분산 회수 N단계의 구심점 |
| [승] 안의 의도적 지연 + 오픈루프 | 권장 — [승] 7블록 이상 시 각 1~2건 | 단조 원인 누적 방지 |

### 5.6 변주 허용 영역 (잠정 가변 ±10%p)

| 항목 | 가변 범위 | 사유 |
| --- | --- | --- |
| [승]·[전] 명칭 | 주제 원인 영역에 맞춰 자유 명명 | — |
| 원인 차원 수 ([승] 3~7+층) | 자유 — 주제 인과 복잡도에 맞춤 | — |
| [전] 차원 (개인 / 예외 / 국제 비교 등) | 자유 선택 | — |
| 9유형 비율 | ±10%p 가변 (잠정 확대) | 표본 작음 |
| 결론부 블록 수 | 1~2블록 자유 | — |

### 5.7 결정 트리

```
1. 주제의 "다층 원인 누적 가능성" 확인 — 하나의 결과를 만드는 원인이 3+층 이상 병렬 누적되는가?
   (역사·기술·정책·자연·생물·문화 등 다차원 원인이 동시에 작동)
   YES → 다층원인수렴 엔진 채택.
   NO → 단일 인과 → 인과추적 / 양극 반전 → 역설 / 두 사례 비교 → 사례비교 검토.

2. 메인 오픈루프 단일 질문 결정 — "왜 X 인가?" 형태로 명시화

3. [승] 원인 차원 수 결정 — 3~7+층
   주제 인과 복잡도가 높으면 7+층, 단순하면 3~5층
   각 원인을 1블록씩 배정

4. [승] 사다리 설계 — 인과·메커니즘 50% 이상 + 미니 회수 반복
   7블록 이상이면 의도적 지연 + 오픈루프 각 1~2건 삽입

5. [전] 진입 1블록 — 보상+오픈루프 또는 의도적 지연
   "[승]만으로는 부족 — 새 원인 차원 추가" 어법
   반전형 + 상식 파괴형 결합어

6. [전] 사다리 설계 — 보완 원인·예외·국제 비교
   [전] = 개인·예외·최신 변수

7. 결론부 — 다층 원인 통합 결산 + 위트 엔딩
```

### 5.8 아하 완결 조건

**아하 본질:** N층 원인이 1점으로 수렴되는 통찰 — "그래서 이 모든 게 X 때문이구나".

**완결 4조건:**

1. 메인 오픈루프 단일 "Why" 질문이 [기]에 명시되었는가?
2. [승]에서 N층 원인이 부분 회수 사이클로 누적되었는가?
3. [전]에서 보완 원인 차원이 추가되었는가?
4. 모든 원인이 1점(메인 오픈루프의 답)으로 수렴되었는가?

**해소 위치:** 분산 회수 — 한 블록 단발이 아니라 N블록 누적 후 1점 수렴.

### 5.9 엔딩 궁합

- **주력:** 통찰형(⑥) — N층 원인 통합 압축에 정합.
- **허용:** 위트반전(⑤) — 정보성 주제에서 가볍게 착지.
- **회피:** 무거운 주제에서 위트반전(⑤) 정합 약함.

### 5.10 다른 엔진과의 차이

- vs **인과추적 엔진** — 다층원인수렴은 N차원 원인 병렬 / 인과추적은 1차원 시간 축.
- vs **다층분기수렴 엔진** — 다층원인수렴은 원인 차원 단위 / 다층분기수렴은 사례·국가·환경 분기 단위.

## §6. 사례비교 엔진 (이미지 강화형 잠정)

### 6.1 엔진 정의

두 사례를 각각 깊이 있게 파고든 뒤 공통 원리(보편 명제)로 수렴시키는 엔진. 각 사례는 환경·시기·구조 깊이까지 파고들고, 두 사례가 함께 보편 명제를 증명한다. **사례 A 깊이 + 사례 B 깊이 → 공통 원리 수렴** 자체가 본질. **본 채널 3유형 매칭:** 이미지 강화형 (잠정) — 두 사례 비교가 일상 환기 + 통념 균열로 작동.

### 6.2 본문 구조 골격

```
본문 = [사례 A 깊이 ([승])] → [사례 전환점 (보상+오픈루프)] → [사례 B 깊이 ([전])] → [공통 원리 수렴]
```

- **[승] 사례 A 깊이** — 사례 A 발생 배경·인과 사슬·구체 사건 누적.
- **[전] 첫 블록 = 사례 전환점** — 사례 A 결산 회수 + 사례 B 사다리 점화 이중 역할.
- **[전] 사례 B 깊이** — 사례 B 인과·해석·결산 통합.
- **공통 원리** — [기]에 선제시 또는 [결]에 결산.

### 6.3 부가 표기 영역

| 위치 | 표기 영역 | 명명 가이드 |
| --- | --- | --- |
| [승] | 사례 A 깊이 | 시기 더 길거나 친숙한 사례 권장 (돼지 금기·중국·일본·동남아 화교 등). |
| [전] | 사례 B 깊이 | 후발·예외·특수성 강한 사례 (소 금기·한국·19세기 각국 등). |

### 6.4 강세 정보유형 권장 범위

| 영역 | 강세 정보유형 (1~2위) | 권장 범위 |
| --- | --- | --- |
| [승] 사례 A | 인과 + 기원·배경 양강 | 합산 50% 이상 |
| [전] 사례 B | 인과 + 사례·일화 + 해석·결산 | 인과 25% 이상 |

**해석:** 양 영역 모두 인과 + 기원·배경 + 사례·일화 3종 우세. 두 사례를 시간 배경·인과 사슬·구체 사건으로 깊이 파고들기.

### 6.5 핵심 패턴 (강제)

| 항목 | 권장 범위 | 강제 사유 |
| --- | --- | --- |
| [승] 블록 수 | 전체의 35~50% | 사례 A 깊이가 부족하면 공통 원리 수렴 진폭 약화 |
| [전] 블록 수 | 전체의 30~45% | 사례 B 깊이가 부족하면 비교 본질 미완성 |
| [승]·[전] 길이 균형 | 한쪽이 다른 쪽의 1.5배 이내 권장 | 한쪽 과다하면 사례비교 본질 약화 |
| [승] 인과·기원·배경 비중 | 합산 50% 이상 | 사례 A 발생 배경·인과 누적 |
| [전] 첫 블록 = 사례 전환점 | 필수 — [전] 시작 위치 | 보상+오픈루프 이중 역할, 반전형 + 호기심형 + 비교형 결합 |
| [승] 끝 = 사례 A 결산 또는 정점 | 필수 | 사례 전환점 직전 최고점 형성 |
| [전] 끝 = 사례 B 결산 + 공통 원리 수렴 | 필수 | 두 사례 통합 회수 완결 |
| 공통 원리 = [기] 선제시 또는 [결] 결산 | 필수 | 사례비교 본질 — 두 사례가 함께 증명할 명제 |

### 6.6 변주 허용 영역 (잠정 가변 ±10%p)

| 항목 | 가변 범위 | 사유 |
| --- | --- | --- |
| [승]·[전] 명칭 | 주제 사례에 맞춰 자유 명명 | — |
| 사례 A·B 대비축 (환경·시기·인과·구조·지리 등) | 자유 선택 | — |
| 사례 깊이 정점 블록 "높음" 인지부담 | [승] 5블록 이상 시 1~2블록 권장 | 짧으면 불필요 |
| 9유형 비율 | ±10%p 가변 (잠정 확대) | 표본 작음 |
| 결론부 블록 수 | 1~2블록 자유 | — |
| 메인 오픈루프 위치 | [기] 선제시 또는 [기]·[결] 분산 | — |

### 6.7 결정 트리

```
1. 주제의 "두 사례 깊이 비교 가능성" 확인 — 두 사례가 공통 원리를 함께 증명하는가?
   YES → 사례비교 엔진 채택. NO → 다른 엔진 검토.

2. 사례 A·B 결정 — 대비축에 맞춰 두 사례 선택
   사례 A = 시기 더 길거나 친숙한 쪽 권장
   사례 B = 후발·예외·특수성 강한 쪽 권장

3. 공통 원리(보편 명제) 결정 — 두 사례가 함께 증명할 명제 명시
   [기] 선제시 또는 [기]에 질문 형태

4. [승]·[전] 길이 결정 — 균형형 권장
   한쪽이 2배 이상이면 다른 엔진 검토

5. [승] 사례 A 깊이 설계 — 인과·기원·배경 50% 이상
   각 블록 = 사례 A의 한 측면 (환경·시기·구조)
   5블록 이상 시 정점 블록 "높음" 인지부담 1~2건 권장

6. 사례 전환점 1블록 — [전] 첫 블록에 보상+오픈루프
   "같은 뿌리인데 정반대 결과" 또는 "어 근데 한쪽은" 어법
   반전형 + 호기심형 + 비교형 결합어

7. [전] 사례 B 깊이 설계 — 인과·사례·해석 결합
   각 블록 = 사례 B의 한 측면

8. 결론부 — 두 사례 통합 결산 + 공통 원리 봉인 + 일상 회귀 또는 위트
```

### 6.8 아하 완결 조건

**아하 본질:** 두 사례가 같은 원리로 수렴된다는 통찰 — "두 사례가 결국 같은 원리구나".

**완결 4조건:**

1. 공통 원리(보편 명제)가 [기]에 선제시되거나 [기]에 질문 형태로 명시되었는가?
2. [승]에서 사례 A 깊이가 누적되었는가?
3. 사례 전환점에서 사례 분기가 명확히 표시되었는가?
4. [전]에서 사례 B 깊이가 공통 원리를 증명했는가?

**해소 위치:** 사례 전환점 + [결] 공통 원리 수렴.

### 6.9 엔딩 궁합

- **주력:** 통찰형(⑥) — 공통 원리 압축에 정합.
- **허용:** 일상행위연결(①) — 일상 영역 사례 비교 시.
- **회피:** 위트반전(⑤) — 깊이 비교 결산에 정합 약함.

### 6.10 다른 엔진과의 차이

- vs **사례나열 엔진** — 사례비교는 2영역 깊이 비교 + 공통 원리 수렴 / 사례나열은 N영역 균등 나열.
- vs **다층원인수렴 엔진** — 사례비교는 사례 단위 / 다층원인수렴은 원인 차원 단위.
- vs **영역확장 엔진** — 사례비교는 균형 깊이 / 영역확장은 [승] 우세 (영역 A 사다리 누적).

## §7. 사례나열 엔진 (⚠️ 사용 보류 — 분석 1편)

> **⚠️ 사용 보류:** 분석 1편(해수면상승)으로 일반화 근거 부족 → 운영(집필) 사용 안 함(6엔진 외). 본문은 후속 분석 2편+ 누적 시 잠정 엔진 복귀 대비 보존. §0.2 참조.

### 7.1 엔진 정의

[승]에서 작동 원리·기준 수치를 누적한 후 [전]에서 그 기준이 적용되는 다중 사례를 충격 강도·친숙도순으로 N≥5건 균등 나열하는 엔진. 각 사례는 단일 블록 단위로 압축. **배경·원리 → 사례 N건 균등 나열** 자체가 본질. **본 채널 3유형 매칭:** 과정 추적형 (잠정) — 인과추적 본질 공유 + 다중 사례 균등 나열.

### 7.2 본문 구조 골격

```
본문 = [배경·원리 ([승])] → [[전] 진입 — 사례 나열 진입] → [다중 사례 N건 ([전])]
```

- **[승] 배경·원리** — 통계·메커니즘 누적. 무거운 기준 수치·작동 원리 한 번에 깔기.
- **[전] 첫 블록 = [전] 진입 — 사례 나열 진입** — [승] 기준 수치 회수 + 메인 오픈루프 "어디?" 첫 답 동시.
- **[전] 다중 사례** — N≥5건 사례 균등 나열. 단일 블록 단위로 압축.

### 7.3 부가 표기 영역

| 위치 | 표기 영역 | 명명 가이드 |
| --- | --- | --- |
| [승] | 배경·원리 | 작동 원리·기준 수치 누적 영역. |
| [전] | 다중 사례 나열 | 사례 적용 영역 (각국 사례·각 도시·각 산업 등). |

### 7.4 강세 정보유형 권장 범위

| 영역 | 강세 정보유형 (1~2위) | 권장 범위 |
| --- | --- | --- |
| [승] 배경·원리 | 통계·수치 + 메커니즘 | 합산 80% 이상 |
| [전] 다중 사례 | 사례·일화 | 80% 이상 (단일 사례마다 인과·통계·기원·비교 1~3개 동반) |

**해석:** [승]은 수치·메커니즘만, [전]은 사례만. 8엔진 중 가장 극단적 영역 분리.

### 7.5 핵심 패턴 (강제)

| 항목 | 권장 범위 | 강제 사유 |
| --- | --- | --- |
| [승] 블록 수 | 전체의 15~30% | 배경·원리가 부족하면 [전] 사례 충격 기준 약화 |
| [전] 블록 수 | 전체의 50~70% | 사례 N≥5건 부족하면 사례나열 본질 미완성 |
| [승] 통계·수치·메커니즘 비중 | 합산 80% 이상 | 기준 수치·작동 원리 누적 |
| [전] 첫 블록 = [전] 진입 — 사례 나열 진입 | 필수 — [전] 시작 위치 | 보상+오픈루프 이중 역할, 경고형 + 상식 파괴형 결합 |
| [승] 끝 = 기준 수치 확정 | 필수 | [전] 모든 사례 충격의 기준점 형성 |
| [전] 사례 = N≥5건 | 필수 — 양식 본질 | 4건 이하는 사례비교 또는 인과추적 검토 |
| 사례 폭발 절정 1~2블록 | 권장 — [전] N≥7블록 시 | 5건 이상 동시 투하 — 아하 형성 |
| 메인 오픈루프 = N중 질문 | 권장 | N단계 분산 회수의 구심점 |

### 7.6 변주 허용 영역 (잠정 가변 ±10%p)

| 항목 | 가변 범위 | 사유 |
| --- | --- | --- |
| [승]·[전] 명칭 | 주제 사례 영역에 맞춰 자유 명명 | — |
| 사례 정렬축 (충격 강도순 / 친숙도순 / 시간순 / 지리순) | 자유 선택 | — |
| 9유형 비율 | ±10%p 가변 (잠정 확대) | 표본 작음 |
| 결론부 블록 수 | 1~2블록 자유 | — |

### 7.7 결정 트리

```
1. 주제의 "사례 N≥5건 가능성" 확인 — 같은 작동 원리·기준 수치가 적용되는 사례를 5건 이상 N건 균등 나열 가능한가?
   (각국·각 도시·각 시대·각 산업 등)
   YES → 사례나열 엔진 채택.
   NO → 사례비교(2건 깊이) 또는 다른 엔진 검토.
   사례 4건 이하는 사례나열 엔진 채택 금지.

2. 배경·원리 결정 — 모든 사례에 적용될 작동 원리·기준 수치 명시

3. [승] 길이 결정 — 2~4블록 (기준 수치·메커니즘 누적)
   [승] 마지막 블록 = 기준 수치 확정 강제

4. 메인 오픈루프 N중 질문 결정 — "어디?·부작용?·대처?" 형태

5. [전] 사례 정렬축 결정 — 충격 강도순 / 친숙도순 / 시간순 / 지리순 중 1종 (또는 이중 정렬)

6. [전] 진입 1블록 — 보상+오픈루프 이중 역할
   "[승] 기준 수치로 측정하면 첫 사례가 X" 어법
   경고형 + 상식 파괴형 결합어

7. [전] 사례 사다리 설계 — N건 사례 균등 나열
   각 블록 = 1~3 사례 동시 투하
   N≥7블록 시 정점 블록 1~2건 권장 (5건 이상 동시 투하)

8. 결론부 — N건 통합 결산 + 미래 전망 또는 위트
```

### 7.8 아하 완결 조건

**아하 본질:** 다중 사례가 동일 기준 수치로 측정될 때의 충격 — "이미 일어나고 있구나·다 적용되는구나".

**완결 4조건:**

1. [승] 마지막 블록에서 기준 수치가 확정되었는가?
2. [전] 진입에서 메인 오픈루프의 첫 답이 동시 점화되었는가?
3. [전] N≥5건 사례가 균등 나열되었는가?
4. 사례 폭발 절정(N건 동시 투하) 블록이 작동했는가? ([전] N≥7블록 시)

**해소 위치:** 분산 회수 — 각 사례 블록이 부분 회수.

### 7.9 엔딩 궁합

- **주력:** 시의성마무리(④) — 진행형 사례의 시점 연결.
- **허용:** 경고형(⑦) — 글로벌 재해·기후·국제 위기 주제 + 경고 대상자가 시청자 대다수와 일치할 때.
- **회피:** 위트반전(⑤) — 무거운 사례 누적 결산에 정합 약함.

### 7.10 다른 엔진과의 차이

- vs **사례비교 엔진** — 사례나열은 N≥5건 균등 나열 / 사례비교는 2건 깊이 비교.
- vs **다층분기수렴 엔진** — 사례나열은 균등 나열 (수렴 1점 없음) / 다층분기수렴은 분기 → 1점 결정체 수렴.

## §8. 다층분기수렴 엔진 (⚠️ 사용 보류 — 분석 1편)

> **⚠️ 사용 보류:** 분석 1편(흑각궁)으로 일반화 근거 부족 → 운영(집필) 사용 안 함(6엔진 외). 본문은 후속 분석 2편+ 누적 시 잠정 엔진 복귀 대비 보존. §0.2 참조.

### 8.1 엔진 정의

N개 분기 변수(사례·국가·환경 변주)가 한 점 결정체로 수렴되는 엔진. 분기 단위가 사례·국가·환경 등 적용 영역 차원이며, 모든 분기 변수가 한 점에 압축되어 이미지 강화로 작동한다. **N개 분기 → 1점 수렴** 자체가 본질. **본 채널 3유형 매칭:** 이미지 강화형 (잠정) — 영역확장 본질 공유 + N분기 → 1점 수렴.

### 8.2 본문 구조 골격

```
본문 = [분기 N개 ([승])] → [수렴 진입 (보상+오픈루프)] → [수렴 결정체 ([전])] → [양면성 (이중 Aha 권장)]
```

- **[승] 분기 N편** — 각 분기 1블록씩 빌드업. 빌드업 압도 (~72%).
- **[전] 첫 블록 = 수렴 진입** — 분기 4편 누적의 한 점 압축 회수 + 수렴 사다리 점화.
- **[전] 수렴 결정체** — 정체 공개 + 보너스 강점 + 양면성/그림자 (이중 Aha 권장).

### 8.3 부가 표기 영역

| 위치 | 표기 영역 | 명명 가이드 |
| --- | --- | --- |
| [승] | 분기 (N개) | 분기 빌드업 영역 (세계 활 분기·다양한 시도 등). |
| [전] | 수렴 (1점) | 수렴 결정체 영역 (한반도 수렴·한 산물 수렴 등). |

### 8.4 강세 정보유형 권장 범위

| 영역 | 강세 정보유형 (1~2위) | 권장 범위 |
| --- | --- | --- |
| [승] 분기 | 사례·일화 | 50% 이상 |
| [전] 수렴 | 인과 + 메커니즘 + 정의·개념 + 사례·일화 균등 | 단일 유형 30% 미만 / 인과 35% 이상 |

**해석:** [승]은 분기 4편 각각 다른 사례로 균등 나열 (사례 우세). [전]은 수렴 영역에서 인과 + 메커니즘 + 정의 + 사례 결합.

### 8.5 핵심 패턴 (강제)

| 항목 | 권장 범위 | 강제 사유 |
| --- | --- | --- |
| [승] 블록 수 | 전체의 35~50% | 분기 N≥3건 누적이 부족하면 수렴 진폭 약화 |
| [전] 블록 수 | 전체의 25~40% | 수렴 정체+양면성이 부족하면 수렴 본질 미완성 |
| [승] 사례·일화 비중 | 50% 이상 | 분기 단위 명료화 |
| [전] 첫 블록 = 수렴 진입 | 필수 — [전] 시작 위치 | 보상+오픈루프 이중 역할, 반전형 + 경고형 결합 |
| [승] 끝 = 분기 정점 (가장 강한 분기) | 필수 | 수렴 진입 직전 최고점 형성 |
| [전] 끝 = 수렴 결정체 결산 또는 양면성 | 필수 | 수렴 완결 |
| 분기 N ≥ 3건 | 필수 — 양식 본질 | 2건 이하는 영역확장 또는 사례비교 검토 |
| 수렴 1점 명확성 | 필수 | 모든 분기 변수가 한 점에 압축되는 구심점 (단일 결정체) |
| 이중 Aha 구조 (수렴 정체 + 양면성 재해석) | 권장 | 결합형과 유사한 이중 폭발 |

### 8.6 변주 허용 영역 (잠정 가변 ±10%p)

| 항목 | 가변 범위 | 사유 |
| --- | --- | --- |
| [승]·[전] 명칭 | 주제 분기·수렴 영역에 맞춰 자유 명명 | — |
| 분기 단위 (사례·국가·환경·시기·기술 등) | 자유 선택 | — |
| 분기 수 (N=3~5건) | 자유 — 주제 복잡도 | — |
| 9유형 비율 | ±10%p 가변 (잠정 확대) | 표본 작음 |
| 결론부 블록 수 | 1~3블록 자유 | — |
| 아하 횟수 | 단일 또는 이중 권장 | — |

### 8.7 결정 트리

```
1. 주제의 "분기 N≥3건 → 1점 수렴 가능성" 확인 — N개 분기 변수가 한 점 결정체로 압축되는가?
   YES → 다층분기수렴 엔진 채택 (이미지 강화형).
   NO → 분기 N=2는 영역확장 / 단일 원인 다층 수렴은 다층원인수렴 / 시간 축 단일 인과는 인과추적 검토.

2. 분기 단위 결정 — 사례·국가·환경·시기·기술 중 1종

3. 수렴 1점 명확화 — 모든 분기 변수가 압축되는 한 점 결정체 정체 확정
   수렴 1점이 모호하면 본 엔진 채택 금지 — 다른 엔진 검토.

4. [승] 분기 N개 설계 (3~5건) — 각 분기 1블록씩 (또는 정점 분기는 2블록)
   분기마다 사례·일화 중심 + 인과·메커니즘 보조
   [승] 마지막 = 가장 강한 분기 정점

5. 수렴 진입 1블록 — [전] 첫 블록에 보상+오픈루프
   "이처럼 [모든 분기 변수]가 한 점에서 X로 수렴" 어법
   반전형 + 경고형 결합어

6. [전] 수렴 사다리 설계 — 수렴 정체(1차 아하) + 보너스 강점 + 양면성/그림자(2차 아하)
   이중 Aha 구조 권장

7. 결론부 — 수렴 결정체의 시간축 마무리 + 현대 회귀 + 메시지
```

### 8.8 아하 완결 조건

**이중 아하 본질:**

- **1차 아하** = 수렴 결정체 정체 공개 (분기 → 1점 압축 완성).
- **2차 아하** = 양면성·그림자 재해석 (수렴 결정체의 보너스 또는 그림자).

**완결 4조건:**

1. [승] 분기 N≥3건이 누적되었는가?
2. 수렴 진입에서 분기 변수의 한 점 압축이 작동했는가?
3. [전]에서 수렴 결정체 정체가 공개되었는가? (1차 아하)
4. (이중 아하 권장 시) [전] 후반부에 양면성·그림자 재해석이 작동했는가? (2차 아하)

**해소 위치:** 수렴 진입 + [전] 결정체 정체 + [전] 양면성.

### 8.9 엔딩 궁합

- **주력:** 통찰형(⑥) — 수렴 결정체 압축에 정합.
- **허용:** 미래희망(③) — 결정체 시간축 마무리 후 현대 회귀.
- **회피:** 위트반전(⑤) — 수렴 본질과 정합 약함.

### 8.10 다른 엔진과의 차이

- vs **다층원인수렴 엔진** — 다층분기수렴 분기 단위 = 사례·국가·환경 / 다층원인수렴 분기 단위 = 원인 차원.
- vs **영역확장 엔진** — 다층분기수렴은 N분기(3+) → 1수렴 / 영역확장은 2영역(A·B) 균형 확장.
- vs **사례나열 엔진** — 다층분기수렴은 분기 → 1점 결정체 수렴 / 사례나열은 N건 균등 나열 (수렴 1점 없음).

## §9. [전] 진입 용어 8엔진 정합표

본 채널 8엔진의 [전] 진입 용어를 표준화하여 엔진 식별 + 본문 표기 정합을 유지한다.

| 엔진 | [전] 진입 용어 | 본질 |
| --- | --- | --- |
| §1 역설 | **피벗** | 속성·의미 반전 (단조 빌드업 → 반전 폭발) |
| §2 영역확장 | **영역 전환점** | 영역 A → 영역 B 확장 (분류축 이동) |
| §3 인과추적 | **[전] 진입** | 시기 단절 (옛 시기 → 현 시기 결과) |
| §4 결합형 | **피벗** | 속성 반전 + 인과 단절 결합 (역설 본질을 포함하므로 피벗 용어 사용) |
| §5 다층원인수렴 | **[전] 진입** | 원인 차원 추가 (인과추적 엔진과 동일 용어) |
| §6 사례비교 | **사례 전환점** | 사례 A → 사례 B 분기 |
| §7 사례나열 (보류) | [전] 진입 — 사례 나열 진입 | 배경·원리 → 사례 N건 시작 |
| §8 다층분기수렴 (보류) | 수렴 진입 | N개 분기 → 1점 수렴 |

**용어 정합 원칙:**

- 본문 헤더 표기 시 본 표 용어 그대로 사용 (`### 피벗` / `### 영역 전환점` / `### [전] 진입` 등).
- 동의어 표기 금지 — 엔진 식별을 위해 표준 용어 유지.

## §10. 가변성 인정 원칙

### 10.1 가변 원칙 3단계

본 모듈의 분포 권장 범위·이문호공 조합 권장·영역별 정보 배치는 **권장 범위**이며 강제 비율이 아니다. 새 주제마다 정보 유형·후킹 정보가 다양하므로 양식 본질만 유지하고 세부 비중은 주제에 따라 가변 적용한다.

| 단계 | 범위 | 적용 대상 |
| --- | --- | --- |
| **핵심 패턴 (강제)** | 양식 본질 유지 | [승]·[전] 비율 핵심 범위·[전] 진입 위치·정점 형성·이중 Aha 등 양식 식별 영역 |
| **변주 ±5%p (주 엔진)** | 9유형 비율·이문호공 조합 | §1 역설·§2 영역확장·§3 인과추적 |
| **변주 ±10%p (잠정 엔진)** | 9유형 비율·이문호공 조합 | §4 결합형·§5 다층원인수렴·§6 사례비교 |

### 10.2 잠정 엔진 한계 인지

§4 결합형·§5 다층원인수렴·§6 사례비교는 잠정 엔진이다 (§7 사례나열·§8 다층분기수렴은 사용 보류 — §0.2). 운영 시 다음을 인지하고 진입한다:

- 핵심 패턴은 강제 유지하되 세부 비중 변주 폭 ±10%p 확대 허용.
- 후속 분석 누적 시 주 엔진으로 격상 가능.
- 잠정 엔진 채택 시 P3 조감도에 잠정 표기 명시.

**운영 영역 위임:** 광고·협찬 통합 운영(광고 회수 단락 배치·광고 CTA 처리 등)은 본 모듈 영역이 아니며 P4 집필 본체 영역에서 다룬다.
