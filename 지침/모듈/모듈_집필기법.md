> **역할:** 본 채널 집필 기법 카탈로그. 해소 은유 · 시청자 반박 흡수 · 단서 배치 · 간접 확인 · 이탈 방지 · 비유 5역할 · 인터넷 밈 · 브릿지 18종.

## §1. 해소 은유 공식 — SL 해소 시점 사용 시 적용 양식

> **사용은 자율.** 본 공식은 해소 은유를 **사용할 때 적용하는 양식**이지, 편당 정량 의무가 아니다. SL 본문의 정보·임팩트가 자력 각인 가능하면 은유 없이 진행한다.

본문 SL 해소 시점에서 한 줄 각인이 필요할 때 아래 공식을 적용한다. 해소 은유는 **"열림"이 아니라 "해소"에서만 작동**한다. 본 공식의 "은유"는 좁은 의미 은유(A=B)와 직유(A는 B처럼) 둘 다 포함한다.

### 1.1 구성 안내

해소 은유는 핵심 대상에 은유·직유를 결합하여 한 줄로 각인한다. 상식파괴 중심으로 유형별 보조 요인(모순형=경고 / 이미지강화형=비교 / 과정추적형=공감)을 자연 결합한다.

- **은유 ("A는 B다"):** 본질·정체가 같을 때
- **직유 ("처럼·같이·같은·마치·마찬가지로·비슷한·비유하면"):** 속성·행동 방식이 비슷할 때

과정추적형은 인과 완성·감정 압축으로 상식파괴를 대체 가능.

### 1.2 본체 형식 — 한 문장 인정

해소 은유는 **한 문장 안에 본체를 담은 형태**도 본체로 인정한다 (별도 단락 강제 아님). 패턴 골격: **[대비 구도] + [한 문장 결합]**.

P4 집필 시 한 문장 은유·직유가 자연 발생하면 별도 단락을 무리하게 추가하지 않는다. P5 검증 시 한 문장 은유·직유는 본체 부재로 판정하지 않는다.

### 1.3 작동 지점

해소 은유는 서사의 **전환·정점·완성 지점**에서 자연 발생한다.

| 지점 | 발생 조건 |
| --- | --- |
| 피벗·반전 (모순형) | A극→B극 전환 시 역설을 한 줄로 압축 |
| Scale Jump 정점 (이미지강화형) | 대상의 파급력이 최고조에 달하는 순간 |
| 인과 완성 (과정추적형) | "이래서 이렇게 됐구나"의 핵심 고리 |
| 누적 충격 정점 | 문제·가치가 쌓인 후 한 줄로 각인 |
| 메커니즘 완성 | How의 원리가 결과로 증명되는 순간 |

### 1.4 적용 범위 + 사용 여부

- SL 해소 시점에서만 사용. SL 열림에는 팩트·질문으로 공백을 생성하고, 해소에서 비유로 각인
- 편당 정량 의무 없음. 자력 각인 가능하면 미사용

### 1.5 대체 각인 도구

해소 은유 미사용 시 다음 6종 중 하나로 보상 단위 충족. 적용 원칙은 §7 비유 적용 원칙 정합 — 이해 격차가 있을 때 사용, 무리한 빈칸 채우기 금지.

| 도구 | 의미 |
| --- | --- |
| 임팩트 팩트 | 정점 직전 단일 팩트로 충격 각인 |
| 단호 선언 | 화자가 핵심 판단을 단정문으로 압축 |
| 풍자 | 모순·아이러니를 유머로 압축 |
| 수치 압축 비교 | 두 수치를 한 줄에 병치하여 격차 각인 |
| 시간 격차 위트 | 과거·현재 격차를 위트 한 줄로 압축 |
| 결과 공유 셋업 | 결과를 먼저 던지고 시청자와 함께 해석 |

### 1.6 면제 사례

다음 해소 패턴은 해소 은유 공식 미적용: 사건·사례 해소 / 감정 해소 / 인과 해소 / 수미상관 회수. 광고 연착·신뢰 상환 분산 같은 특수 운영도 면제. 면제 사례에서도 SL 해소 자체는 필수.

### 1.7 ML 비유(열림) vs 해소 은유(해소) 차이

ML 비유는 호기심(3) 중심으로 "궁금하게 만들기"(루프 열기), 해소 은유는 상식파괴(5) 중심으로 "각인시키기"(루프 닫기). 해소 은유 적용 시 **"해소"가 "열림"보다 우선**.

## §2. 시청자 반박 선제 흡수 + 자기언급형 서사

본문 중 SL 1~2개마다 시청자가 제기할 수 있는 반박을 선제 흡수한다.

### 2.1 반박 흡수 패턴

- **"~라고 하실 수도 있는데"** — 시청자 반박을 미리 제시
- **"엥? ~ 아니냐"** — 자조적 반박 시뮬레이션
- **"이 정도면 ~ 아니야?"** — 의심 선점
- **"물론 그렇다고 ~ 건 아니고요"** — 평서문 부정형 오해 차단. 앞 문장에서 촉발될 수 있는 오해를 "물론"으로 완화한 뒤 즉시 부정

### 2.2 유형별 반박 흡수 초점

- 모순형: "이게 진짜로 모순되냐?" 식 의심 차단
- 이미지강화형: "과장 아니냐?" 식 의심 차단
- 과정추적형: "논리가 왜 이렇게 연결되냐?" 식 의심 차단

## §3. 단서 배치 3기법

본문 단서들이 최종 통찰을 향해 수렴하도록 배치하는 공통 기법.

| 기법 | 내용 | 사용 시점 |
| --- | --- | --- |
| **질문 연쇄** | 질문 → 부분 해소 → 새 질문으로 계단식 해소 | 본문 전체 관통 |
| **부분 공개** | 정보의 일부만 먼저 공개하고 나머지는 뒤로 보류 | 본문 중반 |
| **대조 쌍** | 대조되는 2개 정보를 짝으로 제시하여 시청자가 스스로 연결 | 피벗·Scale Jump·전환점 전후 |

**원리:** 단서가 독립적 조각이 아니라 서로 연쇄되어 최종 통찰을 향해 수렴하도록 배치한다 (모듈_레고이론 §0.3 단서 연쇄 수렴 정합).

## §4. 간접 확인 원리

썸네일 문구를 본문에서 **직접 인용하지 않고** 우회적으로 확인시키는 우회 기법 (모듈_도입부 §2 자발적 발견 단일 정의에 종속).

### 4.1 간접 확인의 핵심 원리

- **(가) 동의어 치환:** 썸네일 문구를 다른 표현으로 재진술 (예: "지키고 있는" → "살리는")
- **(나) 속성 대체:** 썸네일이 선언한 지위를 다른 속성으로 증명
- **(다) 반대 방향 진입:** 썸네일과 반대 방향에서 시작하여 썸네일 약속 결론을 유도

### 4.2 본 채널 3유형별 주력 기법

| 유형 | 주력 |
| --- | --- |
| 모순형 | (다) 반대 방향 진입 |
| 이미지강화형 | (나) 속성 대체 + (가) 동의어 |
| 과정추적형 | (다) 반대 방향 진입 |

집필 시 유형에 맞는 주력 기법 우선 검토. 단 모든 기법이 모든 유형에서 사용 가능 (강제 매칭 아님).

### 4.3 도입부 노출 강도 3구간

도입부에 썸네일 핵심 어휘 노출 강도를 3구간으로 구분.

| 강도 | 정의 | 처리 |
| --- | --- | --- |
| **강 (전면)** | 핵심 어휘 절반 이상이 어미·조사만 변환되어 노출 | **회피 필수** |
| **중 (부분)** | 명사구 1개가 동일 등장, 술어만 변환 | **허용** (술어 변환 또는 추가 우회로 자연 정합) |
| **약 (단어)** | 단일 단어 노출 | **허용** (핵심 결론어가 우회되면 통과) |

**판정 기준:** "핵심 결론어"(약속의 결론을 만드는 단어)가 우회됐으면 (가)·(나)·(다) 중 하나로 분류, 결론어 자체가 노출됐으면 위반.

### 4.4 분류 차원 안내

본 §4의 (가)·(나)·(다)는 **"썸네일 우회 방식"** 기준이고, 모듈_엔진의 도입부 진입 기법은 **"도입부 진입 방식"** 기준이다. 두 차원은 다른 평면이므로 동일 명칭이 등장해도 정의가 다를 수 있다.

## §5. 이탈 방지 패턴 (변주 + 감정 고조)

텐션이 하락하기 쉬운 구간에 **변주**(주의 재점화) 또는 **감정 고조**(관성 유지)를 삽입하여 이탈을 방지한다. 두 기법은 작동 원리가 다르므로 상황에 맞춰 선택·병용한다.

### 5.1 변주 — 주의 재점화

**삽입 지점:**

- 본문 장점만 길게 쌓일 때 (이미지강화형 — 긍정 피로)
- 시간 흐름만 이어질 때 (과정추적형 — 시간 관성 유지)
- 속성 누적이 길어질 때 (모순형 — A극 누적)

**기법:**

- **문제 제기 삽입:** "근데 이게 문제가 있다" 식 삽입
- **반대 시각 도입:** "~반대로 볼 수도 있다"
- **예상 역전:** "끝이냐? 아니다" 패턴

**본 채널 3유형별 사용 기준:**

- 이미지강화형 = 필수 (긍정 피로 구간에 변주 삽입)
- 모순형 = 권장 (A극 누적 길어질 때 "종결 부정형" 등)
- 과정추적형 = 부분 (감정 고조 주력 보완)

### 5.2 감정 고조 — 관성 유지

**기법:**

- **분노 유발:** 부당함·불의를 드러내어 분노 자극
- **감정 이입 장치:** 시청자가 인물·사건에 감정 이입하도록 서술
- **감탄 유발:** 놀라움·경이로움으로 감정 고조

**본 채널 3유형별 주력 감정:**

- 모순형: 경악·분노 (A극·B극 전환점, 부분)
- 이미지강화형: 경이·감탄 (Scale Jump, 부분)
- 과정추적형: 공감·분노·감사 (역사적 사건, 필수 — 감정 이입 장치 최소 1회)

## §6. 비유 5역할 분류

비유는 시청자가 즉각 이해하지 못하는 내용을 이해시키기 위한 도구다. 본문에서 아래 5가지 역할로 사용된다. 은유는 §1 해소 은유 공식으로 별도 운용한다.

| 역할 | 설명 | 사용 시점 |
| --- | --- | --- |
| **① 대상에 지위·인격 부여** | 대상에 별명·지위를 부여해 중요성 각인 | ML 선언부 / 본문 정점 |
| **② 스케일·속성 체감** | 추상 규모·추상 속성(정교함·견고함·복잡함·내구성·정밀함 등)을 직관 체감 표현으로 전환 | Scale Jump · 수치 직후 + 본문 자유 위치 (메커니즘·작동 원리·추상 속성 등장 시) |
| **③ 역설 압축** | A극·B극 모순을 한 표현에 충돌시킴 | 피벗·반전 지점 |
| **④ 인과 압축** | 복잡한 인과 체인을 한 문장으로 압축 | 절정·결말 |
| **⑤ 정성 강도 비유** | 추상 속성(정교함·견고함·복잡함·강력함·정확함·민첩함·거대함 등)의 강도를 일상 사물·생활 도구·게임 용어·역사 인물·사건·전쟁 영역 비유로 직관 체감 전환. 형식: 직유("처럼·같이·같은·마냥·수준·급" 어휘 변주) 또는 은유("A는 B다" 형태). 남발 금지 | 본문 SL 진입 / SL 정점 / 도입부 오프닝 훅 |

**적용 원칙:** 시청자가 즉각 이해할 수 있는 내용에는 비유를 추가하지 않는다. 이해 격차가 있을 때만 사용한다. §1 해소 은유 공식과 별개로, 본문 중 비유 사용 시 5역할 중 하나를 명확히 선택한다. **삽입형(문장 안 삽입) 우선. 비교 대상이 길어 삽입이 어려울 때만 별도 문장 허용.**

본문 자유 위치 비유(② 역할 신규 시점)는 자연 흐름 우선. 무거운 구간(추모·재난) 회피·억지 끼워맞춤 금지.

## §7. 인터넷 밈 은유·환산

인터넷 밈을 **은유**(호명·동사·형용 등치) 또는 **환산**(추상 개념을 인터넷 밈 척도로 변환)으로 활용하여 시청자 몰입도를 높인다. 3유형 모두 자유 사용.

### 7.1 영역 정의

- **은유:** 인터넷 밈을 직접 등치 형태로 사용 (예: "빤스런 했다" / "원툴이다")
- **환산:** 추상 개념·시스템·정도를 인터넷 밈 척도로 변환 (예: "~급" / "~수준" 접미사로 정도 압축)

### 7.2 사용 원칙

- 주제에 자연스럽게 녹아드는 밈만 사용 — 억지 끼워 맞춤 금지
- 과도한 사용은 집중도 저하 유발 — 한 편 내 같은 밈 반복 금지
- 타겟 시청층(주로 20~40대)이 이해 가능한 밈으로 한정
- 비유 직후 풀이 동반 권장 (밈 비익숙 시청자 고려)

## §8. 브릿지 Type A·B 본체 18종

> 공식 용어는 "브릿지"로 통일.

본문 레이어(N)가 바뀔 때마다 **Type A (서브 오픈루프 15종)**를 배치하여 몰입을 유지한다. Type B 논리적 브릿지는 서사의 선형적·인과적 연결이 필수적인 구간에서만 예외적으로 사용한다.

### 8.1 본문 레이어 연결 전략 개요

- **서브 오픈루프 (몰입 유지형):** 각 레이어·SL이 끝날 때마다 다음에 대한 '작은 갈증'을 유발하는 전환구. **질문·선언·예고 어느 형태든 새 SL을 여는 역할**. 15종 하위 유형 (질문형 6종 + 선언·예고형 9종)
- **논리적 브릿지 (체계 전달형):** SL 없이 정보 정리·매듭. 선형·인과 연결. 3종

**레이어 전환 원칙:** Type A를 주력 기준선으로 유지. 중첩 1개인 SL이 닫힐 때 새 SL을 열어 결핍 상태 유지. 복수 대상 병렬·지역별 수평 확대 등 선형 연결 본질 구간에서 Type B 비중 증가 허용. 주제 성격에 따라 SL 형태·BR 비중 자연 조정 허용.

> **어법 카탈로그(채널 시그니처 어법·도입부 예시·말맛 변주·SL·BR 어법 도구 어휘 영역)는 모듈_어법.md 참조.** 본 §8은 브릿지의 기능 분류·운영 원칙만 정의한다.
