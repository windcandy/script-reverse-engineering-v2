## 1. 서사 유형 검증 + 도입부 구조 확정

P1에서 판단된 서사 유형 방향을 딥리서치 팩트 리포트로 재확인하고, **도입부 구조 변주**도 이 단계에서 확정한다. P1은 퀵서치 팩트 기준으로 방향만 잡았고, P3는 전체 팩트 리포트를 보고 유효성 검증 + 유형 내 세부 구조 선택.

**검증 절차:**

1. P1에서 판단된 서사 유형 방향을 확인한다
2. 딥리서치 팩트 리포트 전체를 검토하여 해당 유형을 뒷받침하는 팩트가 충분한지 확인한다
3. **방향 유지:** 팩트가 충분하면 P1 방향 그대로 확정한다
4. **방향 변경 필요 시:** 더 강한 팩트 유형이 발견되면 유저에게 보고 후 변경 여부를 판단한다
5. **도입부 구조 확정:** 유형 내 구조 변주를 선택한다. 세부 판단 플로우는 유형 파일 P3 §1 참조.

| 서사 유형 | 검증 기준 | 도입부 구조 선택지 (세부는 유형 파일 P3 §1) |
|---|---|---|
| **모순형** | 역설의 양극(A극/B극)을 뒷받침하는 팩트가 충분하고, 낙차가 크다 | 기본형·대안형·압축형·예외형 중 선택 (+ D 특수 예외) |
| **이미지강화형** | Scale Jump를 뒷받침하는 팩트가 있고, 숨겨진 메커니즘이 존재한다 | 표준 3단·권위 인용형·미래 제시형 중 선택 |
| **과정추적형** | 핵심 전환점 팩트가 있고, 과정의 의미가 결말보다 강하다 | 표준 3단·맥락 선행형·상식 전복 확장 중 선택 |

과정추적형은 **본문 하위 유형**(사건/역사/비교)까지 이 단계에서 확정한다.

> **레이어 구분 주의:** 과정추적형의 **본문 하위 유형**(사건/역사/비교)과 **도입부 구조**(표준 3단/맥락 선행형/상식 전복 확장)는 **서로 다른 층위**다. 본문 구조는 주제의 시간 전개 성격에 따라, 도입부 구조는 썸네일 약속 성격에 따라 **독립적으로 결정**한다.

### 도입부 전략 선택 절차 (의무 4단계)

도입부 A~H 전략(공통 §A.3) 선택 시 아래 4단계를 의무로 수행한다.

1. 해당 유형의 주력 전략 우선 검토 — 모순형: B·F / 이미지강화형: B·A / 과정추적형: B·A (공통 §A.3 유형별 주력 전략 표)
2. 주력으로 충분한 후킹이 가능하면 주력 채택
3. 보조 전략(C·D·E·G·H) 채택 시 → **"왜 주력으로 부족한가" 사유 한 줄 명시 의무**
4. GATE 3 체크리스트에서 사유 누락 시 통과 불가

> **근거:** 과정추적형 25편 실측에서 보조 전략 사용은 모두 주제 특수성 때문 — 21 유엔군 H(추모 정서) / 13 호르무즈 G(현재 위기) (`03_유형_과정추적형.md` §540~548 도입부 주력 전략 표). 사유 명시는 자연스러운 절차이며, 절차 부재 시 주력 검토 누락 위험.

### 모순 공존형 채택 시 사전 의무 (썸네일 약속 보호 — 공통 §0.3 #2 정합)

P1에서 모순 공존형(공통 §A.1) 썸네일을 채택한 경우, 썸네일 핵심 단어(2~4개)의 본문 동의어 후보를 P3 단계에서 미리 정의한다. 이를 통해 P5 §C10(썸네일 약속 단어 본문 인용 검사) 위반을 사전 차단한다.

| 항목 | 내용 |
|---|---|
| 썸네일 핵심 단어 추출 | 예: "영원의 거짓말" → "영원" / "거짓말" |
| 본문 동의어 후보 (단어당 2~3개) | 예: "영원" → "한 세대" / "백 년 신화" / "끝없을 줄 알았던" |
| 적용 위치 | A극·B극·엔딩 전반에서 직접 단어 대신 동의어 후보 사용 |

> **근거:** 모순 공존형은 25편 실측 중 04 석면 · 14 까트 2편(8%, `00_공통_원칙.md` §A.1). 빈도는 낮으나 채택 시 두 단어 모두 본문 역설의 핵심이라 무의식적 직접 인용 위험 100% 노출. 7회 시뮬레이션 검증에서 1건 위반 발견 (다이아몬드 "영원의 거짓말" → 본문 "영원" 등장).

## 2. 본문 첫 레이어 — 기원·배경 도입

본문이 시작되는 첫 레이어를 대상의 **기원과 배경**으로 구성하여 몰입을 형성하는 전략이다.

| 서사 유형 | 적용 여부 | 이유 |
|---|---|---|
| **과정추적형** | **필수** | 발단(배경+촉발)이 곧 본문 첫 레이어 — 기원 없이 과정을 추적할 수 없다 |
| **이미지강화형** | **대상 성격 분기** | 사물·기술·문화관습(시간축형 What) → 기원·역사로 진입 / 시스템·현상·구조(정의형 What) → 정체·시스템 분석으로 진입. 10편 중 8편 적용 (명확 6편: 05·07·09·10·11·24 / 부분 2편: 08·20). 정의·시스템만 진입 2편 (02·17) — (세부는 `02_유형_이미지강화형.md` P3 §4 참조) |
| **모순형** | **권장** | 기초SL이 기원·배경 도입을 자연 수행 — 7편 중 6편 적용 (명확 4편: 03·04·19·25 / 부분 2편: 01·23). 단 14 까트 1편은 본문 첫 레이어가 **정의·각성제 진입(기원·배경 부재)**이라 예외. |

**적용 시 주의:** 도입부 임팩트(A 규모폭로·D 속성폭격) 직후 본문 첫 레이어를 **추상적 기원 설명**으로 전환하면 흐름이 꺾일 수 있다. 단 임팩트와 기원이 **인과·시간 연결**로 매끄럽게 이어지면 정상 적용 가능하다 (예: 01 감자 "3억8천만 톤" → "7000년 안데스 작물화", 04 석면 속성 5개 → "광물 형성 기원"). 연결이 어색할 때만 기원·배경 도입을 압축하거나 SL 안에 흡수시킨다.

## 3. 서사 유형별 구조 템플릿

서사 유형에 따라 해당 섹션만 사용한다.

> **층위 독립 원칙 (중요):** 도입부 전략(A~H), 도입부 구조(유형별 3~4가지), 본문 구조 변주는 **서로 독립 층위**다. 전략이 구조를 강제하지 않으며, 구조가 본문 변주를 강제하지 않는다. 각 층위는 주제 성격에 따라 **독립 선택**한다.
>
> **실측 근거:** 같은 B 전략이어도 19 대통령제(양면 동시 건설형)·23 아보카도(시간축 전개)·25 소주(시간축 전개)처럼 본문 변주가 다르며, 같은 A 전략도 07 옥수수(파동형)·08 서울(누적 스케일업 4회)·09 여권(지침 충실형 1회)처럼 각기 다름.

> **신뢰 상환·ML 배치 원칙:**
>
> "신뢰 상환" = 썸네일 약속에 대한 **초기 증거 제공**. "왜 그런지"의 완전한 답이 아니라 "미끼가 실제로 존재함"을 증명하는 기능.
>
> **유형별 기본 원칙:**
>
> - **모순형**: 신뢰 상환 **본문 이연**. 도입부에서는 "썸네일 약속 재확인"(동의어 치환)만 수행. 역설 증명은 본문 A극·B극에서.
> - **이미지강화형**: 신뢰 상환 **도입부 수행**. 1단계(스케일 충격/공감 진입)가 약속 증명 기능.
>   - 단 공감 진입형(17·24)은 본문 10% 지점까지 이연 허용.
> - **과정추적형**: 신뢰 상환 **도입부 수행**. 1단계(현재 상태/결과 제시)가 약속 증명 기능.
>   - 단 일부 주제(22 온돌·13 호르무즈)는 본문 이연.
>
> **본문 이연 시 상환 위치:** 구조 변주에 따라 다르다(본문 10%~68%). 세부 가이드는 `[추출] 대본 - 분석/09_신뢰_상환_위치_분석.md` 참조.
>
> **세부 기준은 유형별 지침 P3 §1 참조** (01 모순형·02 이미지강화형·03 과정추적형)

### 모순형

```text
인트로 → [신뢰 상환 + ML] → A극(기초SL→심화SL→정점SL) → 피벗BR → B극(분할: 문제·원인·메커니즘 3요소) → 엔딩
```

> **B극 분할 단일 지침:** 3요소(문제·원인·메커니즘)는 `01_유형_모순형.md` P3 §4 정합 — 직전 묶음 C-5 정비에서 P4 §C-4 정정 완료.

### 이미지강화형

```text
인트로 → [신뢰 상환 + ML] → 기초레이어(What) → How레이어 → Scale Jump → Why레이어 → 통찰 → 엔딩
```

### 과정추적형 — 사건 서사형

```text
인트로 → [신뢰 상환 + ML]("이 사건은 왜/어떻게 이 결말에 이르렀나")
→ 발단(배경+촉발) → 전개(확대+가속) → 위기(정점) → 결말 → 의미부여 → 엔딩
```

### 과정추적형 — 역사 서사형

```text
인트로 → [신뢰 상환 + ML]("이 흐름이 지금 우리에게 왜 중요한가")
→ 기원 → 전환점1 → 전환점2 → 전환점N → 현재 도달 → 미래 함의 → 엔딩
```

### 과정추적형 — 비교 서사형

```text
인트로 → [신뢰 상환 + ML]("이 둘/셋의 차이가 만드는 결과는?")
→ 공통 전제 → 사례A → 사례B → (사례C) → 교차 대비 → 귀납적 통찰 → 엔딩
```

> **과정추적 변주 차원 안내 (Phase 9 단계 9-C 신설):**
>
> P3에서 정의하는 과정추적 3가지 변주 (사건 서사형·역사 서사형·비교 서사형)는 **도입부·본문 구조 차원**.
>
> `지침/04_언어_장치.md` §4-3 부록에서 정의하는 4가지 변주 (사건 서사형 정통·시즌 2막형·하이브리드형·현재 이슈 액자형)는 **결론부·도구 활용 차원**. 두 분류는 서로 다른 차원이므로 단일 영상에 동시 적용 가능. 예: 06편 = P3 사건 서사형 + 04 사전 사건 서사형 정통 / 12편 = P3 사건 서사형 + 04 사전 시즌 2막형.

> **레이어 구분 재확인:** 위의 과정추적형 3개 서브 템플릿(사건/역사/비교)은 **본문 구조** 층위다. **도입부 구조**(표준 3단/맥락 선행형/상식 전복 확장)는 독립 층위이며, 도입부 구조 선택은 `지침/03_유형_과정추적형.md` P3 §1 참조.

## 5. 모순형 설계

### A극 설계

- **A극 정점 문장을 먼저 정한다** (해소 은유)
- SL의 인지적 기능 구분:
  - **기초SL** (A극 초반) — 기본 전제 건설
  - **심화SL** (A극 중반) — 전제 위에 깊이 추가
  - **정점SL** (A극 끝) — 확신 최고점 도달
- 확신 스케일 확대 순서: 인지 → 놀람 → 확신 → 확신 최고점 → 피벗 추락
- A극의 모든 SL은 A극 정점 방향에 기여해야 한다
- B극 정보가 A극에 미리 노출되면 안 된다 — 피벗의 낙차가 사라진다. 시청자가 이미 알고 있으면 반전이 아니라 확인이 된다

### B극 설계

- **B극 임팩트를 먼저 정한다**
- B극 분할: **문제 · 원인 · 메커니즘 3요소** (`01_유형_모순형.md` P3 §4 단일 지침 정합)
- Aha-moment를 한 문장으로 정의한다 — Aha는 "A극 속성 X가 B극에서 역전됨"의 발견 (`01_유형_모순형.md` P3 §6 참조)
- 메커니즘은 중반에 해소한다 — 먼저 해소해야 "이 사람 아는 게 맞네" 신뢰가 쌓인다
- 원인은 최대한 늦게 배치한다 — "왜?"라는 긴장이 끝까지 유지되어야 이탈을 막는다

### 모순형 7가지 변주

## 6. 이미지강화형 설계

### 레이어별 역할

| 레이어 | 설계 목표 | 시청자 상태 |
|---|---|---|
| 기초레이어(What) | 시청자가 "안다고 생각하는 이미지" 확립 | "아 그거 알지" |
| How레이어 | 표면 아래 작동 원리 해부. 신뢰 확보 구간 | "오, 이렇게 돌아가는 거였어?" |
| Scale Jump | 스케일/맥락을 비약적으로 확장. 대상의 진짜 위상 폭로 | "이 정도였어?!" |
| Why레이어 | Scale Jump가 가능한 근본 원인/구조적 이유 해부 | "그래서 이게 이렇게 된 거구나" |
| 통찰 | 새 이미지를 한 문장으로 각인 (지적 전리품 전달) | "이제 다르게 보인다" |

### 핵심 전환점

**How → Scale Jump BR:**
- How레이어에서 "원리를 이해했다"는 안정감을 준 뒤
- "그런데 이 원리가 작동하는 스케일을 보면" → Scale Jump 진입

**Scale Jump → Why BR:**
- Scale Jump의 충격 직후 "왜 이게 가능한 건데?" 질문을 자연스럽게 유발
- Why레이어에서 근본 원인/구조적 배경을 밝히며 Aha-moment 도달

## 7. 과정추적형 설계

### 하위 유형별 텐션 메커니즘

| 하위 유형 | 텐션 엔진 | 설명 |
|---|---|---|
| 사건 서사형 | 불가피성의 축적 | "이러면 어떻게 되는지 뻔한데..." → 그래도 벌어짐 |
| 역사 서사형 | 현재와의 거리 좁히기 | 과거에서 출발해 "이게 지금 우리 얘기"로 수렴 |
| 비교 서사형 | 패턴 발견의 쾌감 | 사례가 쌓일수록 시청자가 스스로 공통점을 눈치챔 |

### SL 보충 전략

전환점만으로 SL이 부족할 경우 (2분당 1개 미달):
- **사건 서사형:** 각 단계에 "만약 다른 선택이었다면?" 반사실 SL 추가
- **역사 서사형:** 전환점 사이에 "이때 왜 다른 경로를 안 택했는가?" SL 삽입
- **비교 서사형:** 사례 내부에서 "이 조건이 왜 결정적인가?" SL 분기

## 8. 공통 SL 배열 규칙

- **전제 건설 체인:** 앞 SL의 해소가 뒤 SL의 전제가 된다
- **편당 SL 수:** 2분당 1개 기준
  - 10분 = 5개 / 13분 = 6~7개 / 16분 = 8개
- **2분 이상 SL 없는 구간 금지** — 시청자는 궁금증(SL)이 없는 구간에서 이탈한다. 2분은 유튜브 시청 유지율이 급락하는 임계점이다
- SL로 포장할 정보와 SL 없이 서술할 정보를 구분한다

## 9. 본문 BR 운영 원칙

> **BR 분류 단일 지침 문서:** BR(Type A 서브 오픈루프 15종 + Type B 논리적 브릿지 3종) · SL·BR 어법 도구 4종 · SL 열림 표현 형태 5유형 분류는 `지침/00_공통_원칙.md` §C.9 본체 + 부속 ①·②·③ 참조. 본 §9는 P3 구조설계 시 BR 운영 원칙만 다룬다.

- **BR-SL 동시 작동 원칙:** 절반 가까이를 동시 작동으로 설계한다 — 모든 BR이 SL을 동반하면 과부하, 전혀 없으면 전환 구간에서 이탈. 절반이 정보 전달과 긴장 유지의 균형점이다
- **핵심 피벗 BR:** 편당 1회, 모순형은 A극→B극 전환점에 배치 — 2회 이상이면 "또 뒤집네" 피로감이 생겨 신뢰가 떨어진다

### 논리적 브릿지 (비 오픈루프) 사용 가이드

오픈루프(SL)를 열지 않고 정보를 정리하는 구간에서 사용한다. 전체 BR 중 20% 이하로 제한한다. — 논리적 브릿지는 SL 없이 진행되므로 시청자 긴장이 풀린다. 20% 이상이면 "강의"처럼 느껴진다

| 유형 | 기능 | 패턴 |
|---|---|---|
| 선형적 흐름 연결 | 시간/공간의 계보 나열 | "유대교가 → 기독교를 → 이슬람교를" |
| 인과적 흐름 연결 | 사건의 결과가 다음의 원인이 되는 꼬리물기 | "이러한 이유로", "때문에" |

### 루프 중첩 제약

- 서브 오픈루프 중첩은 1개 유지를 원칙으로 하되, 서사 전개상 필요한 경우에만 최대 2개까지 허용
- 3개 이상의 서브 오픈루프가 동시에 열려 있는 상태는 절대 불가
- **세부 원칙·"순차 교체" 실전 메커니즘·교체 조건은 `00_공통_원칙.md` §B.8 참조** (부분 실측·부분 규범 주석 포함)
- 2개가 열려 있다면 새 루프를 열기 전에 반드시 기존 루프 중 하나를 해소한다

## 11. 루프 해소와 보상 원칙

> **단일 지침 정합:** 본 섹션 보상 원칙은 공통 §B.1 (오픈 루프 설계 8원칙 — ① 보상의 비례·② 지적 보상 품질·④ 빌드업 농도) + §B.2 (보상의 비례 원칙) + §C.1 (해소 은유 공식) 정합. 절충 금지 원칙으로 단일 지침 정의 그대로 인용.

- **ML 보상 (공통 §B.1 ①·§B.2 정합):** 영상 전체 관통 루프이므로 가장 오래 유지되며 가장 큰 보상이 필요
- **A극 SL 보상 (공통 §B.1 ④ 빌드업 농도 조절):** A극 전체를 지탱한 SL은 피벗의 낙차가 보상 역할
- **개별 SL 보상 (공통 §C.1 + §B.1):** SL 보상 단위 = **해소 은유 또는 각인 도구(임팩트 팩트·단호 선언·풍자·수치 압축 비교·시간 격차 위트·결과 공유 셋업)** + 뒷받침. 2분 단위 정량 강제 없음. 안전망 — 은유·각인 도구 모두 부재한 SL이 2분 이상 지속되지 않도록 자가점검

> **보상 품질 테스트 (공통 §B.1 ② 지적 보상의 품질 정합):** "이 해소를 들은 시청자가 다른 사람에게 설명하고 싶어지는가?" — 아니면 보상이 약한 것.

## 12-1. 감정 태그 — 사후 진단 도구

조감도의 SL 배열을 완성한 뒤, 각 SL에 아래 6개 태그 중 가장 가까운 것을 부여한다. 태그는 설계를 제약하는 규칙이 아니라, 완성된 설계의 감정 흐름을 진단하는 도구다.

### 태그 정의

| 태그 | 정의 | 판별 기준 | 이것이 아니다 |
|---|---|---|---|
| **전제** | 다음 SL의 기초를 놓는 배경 정보 | "이것 없이 다음 SL이 이해되는가?" → 아니면 전제 | 그 자체로 충격적인 정보 (→ 충격) |
| **충격** | 시청자의 기존 인식을 깨는 팩트. 분노·슬픔 등 감정적 충격도 포함 | "시청자가 '에?' 반응을 하는가?" → 예이면 충격 | 규모/스케일의 비약적 확장 (→ 경탄) |
| **경탄** | 규모/깊이/맥락이 비약적으로 확장되는 지점 | "시청자가 '이 정도였어?!' 반응을 하는가?" → 예이면 경탄 | 단일 팩트의 놀라움 (→ 충격) |
| **추락** | 직전 흐름과 **반대 방향**으로 전환 | "직전 SL의 방향과 반대인가?" → 예이면 추락 | 같은 부정적 방향의 심화 (→ 충격) |
| **공감** | 시청자 경험/감정 소환. 화자 감정 전달, 감사, 기념 감정 포함 | "시청자나 화자의 감정이 직접 드러나는가?" → 예이면 공감 | 정보를 통한 간접적 감정 유발 (→ 충격 또는 추락) |
| **이완** | 긴장 해소. 유머, 안도, 희망적 전환 포함 | "긴장 해소 또는 긍정적 전환이 목적인가?" → 예이면 이완 | 정보를 전달하면서 긴장도 해소 (→ 해당 태그+이완 병기) |

**충격 vs 경탄 경계선:** 충격은 단일 팩트의 임팩트("소금이 화폐였다"), 경탄은 스케일/맥락의 비약적 확장("소금 때문에 제국이 운영됐다")이다.

### 부여 규칙

- SL 배열 설계를 자유롭게 완료한 뒤 **사후에** 부여한다. 태그에 맞춰 SL을 설계하지 않는다
- 복합적인 구간은 **1~2개를 자유 병기**한다 (예: 충격+공감, 경탄+이완)
- 6개 태그 중 어디에도 해당하지 않으면 가장 가까운 태그를 부여하고 괄호로 보충한다 (예: 전제(서사 배경))
