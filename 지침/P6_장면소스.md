# Phase 6: 장면 소스 제작

> **로드 시점:** P5 퇴고 완료 후
> **입력:** 최종 대본 + P3 조감도 + 해소 은유 위치 목록
> **함께 로드:** ❌ (조감도가 유형별 구조를 이미 반영, 추가 지침 로드 불필요)
> **완료 조건:** 전체 장면에 연출 가이드 + 소스 배정 완료

---

## 진행 순서 (필수)

```
[1단계] 대본을 7~10초 단위로 장면 분할
  ↓
[2단계] 각 장면 연출 가이드 작성 (4가지 기준 적용)
  ↓
[3단계] 연출 가이드 기반으로 소스 배정
  (실사 링크 / Freepik / Auto Whisk 프롬프트)
```

**연출 가이드를 먼저 확정한 뒤 소스를 찾는다. 순서를 바꾸지 않는다.**

---

## 연출 가이드 작성 기준 (4가지)

### 기준 1. 서사 구조 위치 (P3 조감도 참조)

| 장면 유형 | 연출 방향 |
|---|---|
| SL 열림 | 긴장감 조성 — 질문을 시각적으로 던지는 구도 |
| SL 정보 전달 | 정보 보조 — 대본 청취에 집중시키는 단순한 시각 |
| ML 해소 / Aha-moment | 충격 또는 해소 — 정점 컷, 임팩트 최대화 |
| 엔딩 | 여운 — 넓은 시야, 느린 무빙 |

### 기준 2. 해소 은유(강) vs 뒷받침(약)

| 구분 | 연출 |
|---|---|
| 강 (해소 은유) | 단독 컷, 클로즈업, 화면 전체 점유 — 임팩트 우선 |
| 약 (뒷받침) | 와이드 or 텍스트 오버레이 — 정보 보조 역할 |

### 기준 3. 시청자 감정 상태

각 장면에서 시청자가 느껴야 할 감정을 먼저 명시한다.
감정이 확정되면 색감·카메라 무빙·속도가 자동으로 결정된다.

| 감정 | 연출 특성 |
|---|---|
| 호기심 | 부분 노출, 줌인, 어두운 톤 |
| 놀라움 | 급격한 전환, 고대비 조명 |
| 긴장 | 정적인 구도, 클로즈업, 침묵 |
| 해소/납득 | 와이드샷, 밝은 톤, 느린 무빙 |
| 공감 | 일상적 장면, 따뜻한 색온도 |

### 기준 4. 정보 밀도

| 밀도 | 연출 원칙 |
|---|---|
| 높음 (팩트·수치 집중 구간) | 시각 단순하게 — 청각(대본)에 집중시킴 |
| 낮음 (호흡 구간, 전환부) | 시각 풍부하게 — 영상미로 몰입 유지 |

---

## 연출 가이드 출력 형식

각 장면마다 아래 형식으로 작성한다.

```
[장면 NN] "대본 구간 텍스트"

● 서사 위치: SL열림 / SL정보 / ML해소 / Aha-moment / 엔딩 등
● 강/약: 강(해소 은유) / 약(뒷받침)
● 감정: [목표 감정]
● 밀도: 높음 / 낮음
● 연출: [카메라 무빙, 구도, 색감 등 구체적 묘사]
```

---

## 방식 선택

대본 완성 후 아래 두 방식 중 하나를 선택한다.

| | 방식 A — 애니메이션 | 방식 B — 실사 혼합 |
|---|---|---|
| 스타일 | 일관된 애니메이션 캐릭터 | 실사 + 부분 생성 이미지 |
| 소스 | Auto Whisk 프롬프트 전체 | 실사 링크 + Freepik + Auto Whisk |
| 크레딧 소모 | 많음 | 적음 |
| 신뢰도 | 보통 | 높음 (실제 사진) |
| 적합한 주제 | 개념/경제/과학 등 추상적 주제 | 사건/인물/장소가 명확한 주제 |

---

## 방식 A — 애니메이션 전체 구성

### 진행 순서

```
[유저] 참고 이미지 1장 제공 (스타일 기준)
  ↓
[나] 대본을 7~10초 단위로 장면 분할
[나] 4가지 기준으로 각 장면 연출 가이드 작성
[나] 연출 가이드 기반으로 Auto Whisk 프롬프트 생성 (영어)
[나] 프롬프트 목록 텍스트 파일로 저장
  ↓
[유저] Auto Whisk에 참고 이미지 + 프롬프트 목록 입력 → 일괄 생성
[유저] 01_영상, 썸네일/ 폴더에 저장
```

### 프롬프트 작성 규칙

- 영어로 작성
- 장면 사이 **한 줄 공백** 필수 (Auto Whisk 구분자)
- 참고 이미지 스타일과 일관성 유지하는 묘사 포함
- 텍스트/숫자가 필요한 장면은 별도 표기

### 출력 형식 예시

```
[장면 01] "한국에만 존재하는 독특한 주거 제도"

이미지 프롬프트: A small cartoon figure standing in front of a giant apartment building, black and white sketch style, dark dramatic atmosphere

연출: 건물 전체 부감 → 인물 클로즈업으로 줌인
AI 영상화: Camera slowly zooms in on a tiny figure standing before a massive apartment complex, dark and oppressive atmosphere, cinematic


[장면 02] ★ "보증금 3억이 하루아침에 사라졌습니다"

이미지 프롬프트: A cartoon figure staring at an empty open vault with shocked expression, money scattered on the floor, high contrast lighting

연출: 금고 문이 열리며 텅 빈 내부 드러남 — 충격 정점
AI 영상화: Vault door slowly swings open to reveal emptiness, shocked cartoon character, dramatic lighting shift from warm to cold
```

## 방식 B — 실사 혼합 구성

### 소스 우선순위

| 순위 | 소스 | 적합한 장면 | 저작권 |
|---|---|---|---|
| 1순위 | **Wikimedia Commons** | 실제 사건·인물·역사·보도 장면 | CC 라이선스 — 사용 전 확인 필수 |
| 2순위 | **Pexels / Pixabay** | 영상 클립 필요 장면, 감성·배경·오브젝트 | 완전 무료 (Pixabay 유료 배지 제외) |
| 3순위 | **Unsplash** | 고화질 분위기 사진 (도입부·엔딩) | 완전 무료 |
| 4순위 | **Auto Whisk 프롬프트** | 실사 없는 개념·데이터·추상 장면 | — |

> Wikimedia Commons 라이선스 확인 기준: CC0·CC BY는 사용 가능(CC BY는 출처 표기), CC BY-NC·CC BY-ND는 사용 불가

### 진행 순서

```
[나] 대본을 7~10초 단위로 장면 분할
[나] 4가지 기준으로 각 장면 연출 가이드 작성
[나] 연출 가이드 기반으로 소스 배정:
  - 1순위: 실사 검색 → 공개 출처 링크 + 이미지 위치 설명
  - 2순위: Freepik 검색 → 웹 링크
  - 3순위: Auto Whisk 프롬프트 생성
[나] 장면 소스 목록 파일로 저장
  ↓
[유저] 실사 다운로드 + Auto Whisk 실행 + Freepik 저장
[유저] 연출 가이드 참고하여 편집 / AI 영상화 필요 시 G-Labs Veo 사용
[유저] 01_영상, 썸네일/ 폴더에 저장
```

### 실사 출처 기준

| 출처 유형 | 예시 |
|---|---|
| 국내 공공기관 | 통계청, 국토교통부, 한국은행, LH공사 |
| 국제기구 | UN, IMF, World Bank, WHO |
| 보도사진 (CC라이선스) | AP, Reuters, Wikimedia Commons |
| 정부 공식 자료 | 대한민국 정책브리핑, 각 부처 보도자료 |

## 공통 규칙

## 썸네일 이미지 생성 (P6 부속 단계)

> **진입 조건:** 장면 소스 목록 완성 후 진행  
> **목적:** P1에서 확정한 썸네일 문구를 배경 이미지 + 텍스트로 최종 완성

### 진행 순서

```
[1단계] 감정 확인 — 이 썸네일이 시청자에게 유발해야 할 감정을 명시
[2단계] 문구-이미지 관계 유형 선택
[3단계] 배경 이미지 소스 결정 (실사 / 합성 / AI 생성)
[4단계] make_thumbnail.py 설정값 출력
```

---

### 1단계: 감정 확인

P1에서 확정한 썸네일 문구를 보고, 이 썸네일이 클릭 전 시청자에게 줘야 할 핵심 감정 하나를 명시한다.

| 감정 | 적합한 이미지 특성 |
|---|---|
| 호기심 | 부분 노출, 어두운 톤, 미완성 느낌 |
| 놀라움 | 고대비, 충격적 장면, 극단적 구도 |
| 긴장 | 클로즈업, 정적인 구도, 차가운 색온도 |
| 공감 | 일상적 장면, 따뜻한 색온도, 친숙한 배경 |

---

### 2단계: 문구-이미지 관계 유형 선택

| 유형 | 문구-이미지 관계 | 클릭 메커니즘 |
|---|---|---|
| 역설적 | 문구와 이미지가 모순/충돌 | "이게 왜 같이 있지?" |
| 강화 | 문구와 이미지가 같은 감정 증폭 | "이게 진짜야?" |
| 공감 | 이미지(경험 소환) + 문구(이유 제시) | "나도 궁금했어" |
| 미스터리 | 이미지(단서) + 문구(질문) | "비밀이 뭐지?" |

**선택 기준:**
- 이미지만 봐도 주제 즉시 파악 가능한가?
- 문구와 이미지가 서로 다른 정보를 전달하는가? (중복 금지)
- 제목·문구·이미지 세 가지가 각각 다른 역할을 하는가?

---

### 3단계: 배경 이미지 소스 결정

아래 세 방식 중 하나를 선택한다.

#### 방식 A — 실사 이미지

무료 이미지 사이트에서 선정. 우선순위:

| 순위 | 사이트 | 특징 |
|---|---|---|
| 1 | Wikimedia Commons | 역사·보도 사진 강점, CC 라이선스 확인 필수 |
| 2 | Pexels / Pixabay | 사진+영상, 완전 무료 |
| 3 | Unsplash | 고화질 분위기 사진 |

선정 기준: 감정 유발 가능 여부 → 문구-이미지 관계 유형 충족 여부 순으로 판단

#### 방식 B — AI 생성 이미지 (Auto Whisk)

실사가 없는 추상·개념 장면에 사용. 아래 형식으로 프롬프트 생성:

```
배경 이미지 프롬프트: [영어로 작성]
  - 감정: [목표 감정을 영어로 명시]
  - 구도: [원하는 카메라 앵글/구도]
  - 색감: [톤 및 색온도]
  - 포함 요소: [핵심 오브젝트/배경]
  - 제외 요소: [텍스트, 인물 얼굴 등 필요 시]
```

#### 방식 C — 실사 합성

실제 인물·오브젝트 컷아웃 PNG를 배경 위에 합성.  
`make_thumbnail.py`의 `composite_subject()` 함수 사용.

**준비물:**
- 배경 이미지 (방식 A 또는 B)
- 인물/오브젝트 PNG — 배경 제거 필수 (remove.bg 또는 포토샵)

**합성 파라미터:**

| 파라미터 | 설명 | 기본값 |
|---|---|---|
| `subject_scale` | 캔버스 높이 대비 인물 크기 비율 | 0.7 |
| `subject_position` | 수평 위치 — "left" / "center" / "right" | "right" |
| `subject_bottom_offset` | 하단에서 올리는 픽셀 | 0 |

> 텍스트가 왼쪽에 배치될 경우 인물은 `"right"`, 반대면 `"left"` 선택

---

### 4단계: make_thumbnail.py 설정값 출력

아래 형식으로 실행 코드를 출력한다. 유저가 복사하여 `__main__` 블록에 붙여넣으면 즉시 실행 가능하도록 완성된 형태로 제공한다.

**방식 A / B (배경만):**

```python
# ── [주제명] 썸네일 ───────────────────────────────────
make_thumbnail(
    bg_path=os.path.join(MEDIA_DIR, "배경파일명.jpg"),
    lines=[
        [("문구1", "red")],
        [("문구2", "white"), ("문구3", "yellow")],
    ],
    output_path=os.path.join(MEDIA_DIR, "주제명_썸네일.jpg"),
)
```

**방식 C (합성):**

```python
# ── [주제명] 썸네일 ───────────────────────────────────
make_thumbnail(
    bg_path=os.path.join(MEDIA_DIR, "배경파일명.jpg"),
    subject_path=os.path.join(MEDIA_DIR, "인물컷아웃.png"),
    subject_scale=0.75,
    subject_position="right",
    subject_bottom_offset=0,
    lines=[
        [("문구1", "red")],
        [("문구2", "white"), ("문구3", "yellow")],
    ],
    output_path=os.path.join(MEDIA_DIR, "주제명_썸네일.jpg"),
)
```

**색상 기준:**

| 색상 | 사용 대상 |
|---|---|
| `"red"` | 역설의 대상 — 시청자가 "설마?" 하는 핵심 명사 |
| `"yellow"` | 충격적 결과/행동 — 동사·결과 키워드 |
| `"white"` | 조사, 어미, 연결어 |
