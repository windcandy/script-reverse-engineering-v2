## PART A: 도입부 집필

> 전략별 실행 패턴 참고: `00_지식파일/인트로_예시_전략별.md`

### 도입부 집필 전 스캐폴딩 선언 (필수)

도입부를 집필하기 **전에** 아래 선언을 먼저 작성한다. 선언 없이 도입부 산문부터 쓰지 않는다.

```text
[도입부 — 전략: {A~H} / 도입부 이문호공: {요인+요인} / 충격 주체: {누가 무엇을 했는지 한 줄} / 신뢰 상환 팩트: {팩트리포트 원본 수치 나열} / 소화 문장 방향: {어떤 의미를 짚을 것인지 / 근거 팩트: 나열된 팩트 중 이 결론을 지지하는 팩트 명시} / ML 이문호공: {요인+요인+요인} / ML 형태: 질문형]
```

- 전략 코드를 선언하면 → 전략 이탈 방지
- 도입부 이문호공 요인을 미리 지정하면 → 요인 수 미달 방지
- 충격 주체를 명시하면 → 주체/인과 방향 누락 시 선언과 불일치가 드러남
- 팩트리포트 원본 수치를 그대로 복사하면 → 팩트 열화 방지
- 소화 문장 방향을 미리 정하면 → 소화 문장 누락 방지. **근거 팩트를 명시하면 → 팩트 나열 후 마지막 팩트를 자동으로 받는 관성을 차단. 결론을 지지하지 않는 팩트가 소화 문장의 근거로 사용되는 논리 비약 방지**
- ML 형태를 "질문형"으로 고정하면 → 선언형 오류 방지

### 도입부 구성 요소

P3에서 선택한 도입부 전략(A~H)을 실행한 뒤, 아래 두 가지 요소를 배치한다. **순서는 고정하지 않는다.** 충격의 성격에 따라 신뢰 상환과 ML의 순서를 결정한다.

> **흐름 원리:** 충격이 감정이면 ML로 먼저 넘어가고 신뢰 상환은 본문 초반으로 이연한다. 충격이 수치면 신뢰 상환을 도입부에서 바로 수행한 뒤 ML로 넘어간다. 판단 기준은 전략 코드가 아니라 해당 도입부의 충격 소재다.

**1) 신뢰 상환:** 썸네일이 약속한 내용을 실제 팩트/현실로 증명한다. 팩트 나열이 아니라 팩트의 체감 시간을 늘린다. 썸네일 문구를 대본에 그대로 인용하지 않는다.
   - **신뢰 상환 ≠ ML:** 신뢰 상환은 "이미 일어난 일/현재 상태"를 팩트로 보여주는 것이고, ML은 "왜/어떻게"를 묻는 질문이다. 같은 문장에 섞지 않는다.
   - **필수:** 팩트 후 소화 문장(한 박자 쉬며 의미를 짚는 문장)을 삽입한다. 단, 팩트 자체가 충분히 직관적이면 삽입하지 않는다. 앞 문장의 단어를 반복하는 것은 소화 문장이 아니다
   - **필수:** 소화 문장은 직전 팩트가 아니라 결론을 가장 잘 지지하는 팩트를 근거로 삼는다. 팩트가 여러 개 나열된 경우, 마지막 팩트를 자동으로 받지 않는다 — 나열된 팩트 중 어느 것이 결론의 논리적 근거인지 먼저 확인한 뒤 소화 문장을 작성한다
   - **필수:** 숫자/전문 개념은 일상 언어로 풀어서 서술한다
   - **선택:** 시청자 인식과 팩트의 괴리를 명시한다 (모순형, 이질감/맥락재설정 전략일 때)
   - **선택:** 시청자 본인의 상황으로 대입한다 (생활 밀착형, 불안소환/자조적체감 전략일 때)
**2) ML 생성:** 영상 전체를 관통하는 핵심 궁금증으로 연결. **2문장 이내, 이문호공 3요인 이상** 적용한다. 한 문장에 3요인을 모두 담거나, 2중첩 + 1복합으로 구성한다. **호기심(3) 또는 상식파괴(5) 중 하나를 반드시 포함**한다. 나머지 요인은 주제와 팩트 흐름에 맞게 선택한다.

### 도입부 길이

- 도입부가 길어지는 것 자체가 문제가 아니라, 시청자 이탈을 유발하는 흐름 단절이 문제다

## PART B: 본문 집필 — 서브루프 + 박스 이미지

### 섹션 집필 전 스캐폴딩 선언 (필수)

각 SL을 집필하기 **전에** 아래 한 줄을 먼저 적는다. 집필 후 제거해도 되지만, 선언 없이 본문부터 쓰지 않는다.

```text
[SL번호 — 선행브릿지: {이전 섹션 상태태그의 브릿지 문구 복사 → 실제 집필할 브릿지 문장} / SL열림유형 / 박스이미지: 역할(여는|닫는) — 핵심 대상 한 줄 — 이문호공 2요인({요인+요인}) / 핵심 팩트: [{마킹}] {팩트리포트 원본 복사} / 의인화 대상: {대상+역할 또는 "없음"} / 이완 장치: {유형+위치} / 속어감탄 누적: {N}/6~10]
예) [SL2 — 선행브릿지: "그럼 이 공포가 통했냐? 아닙니다" → "학살 이듬해인 1297년, 하급 귀족 출신의 윌리엄 월리스가 반란을 일으킵니다" / 부정선행형 / 박스이미지: 닫는 — "죽일수록 적이 늘어났다" — 이문호공 2요인(상식파괴(5)+반전(6)) / 핵심 팩트: [사실] 로마 병사 급여=소금(salarium), [사실] 에티오피아 암염 화폐 19세기까지 / 의인화 대상: 없음 / 이완 장치: 확인사격, 팩트 나열 후 / 속어감탄 누적: 3/6~10]
```

- **선행브릿지를 선언 시점에 이전 섹션 상태태그에서 복사하면 → 브릿지 누락 방지. 계획으로만 존재하고 실제 집필이 누락되는 구조적 공백을 차단한다.** 선언 후 반드시 실제 브릿지 문장을 산문으로 먼저 쓴 뒤 SL 본문에 진입한다
- SL 열림 유형을 먼저 정하면 첫 문장 형태가 자동으로 결정된다
- 박스 이미지 핵심 대상을 미리 선언하면 집필 중 비유/은유가 흐릿해지는 것을 막는다
- **핵심 팩트는 반드시 팩트리포트 파일을 열어 해당 구간을 복사한다. 기억으로 작성하지 않는다.** 기억으로 쓰면 스캐폴딩 자체가 오염되어 보호 효과가 무력화된다
- **핵심 팩트 복사 시 마킹([사실]/[추정]/[해석]/[논쟁])을 반드시 함께 복사한다.** 마킹이 있어야 집필 시 서술 방식(단정형/추정형)이 자동으로 결정된다
- **의인화 대상을 미리 선언하면 B2 의인화 장치 누락을 방지한다.** 주제와 무관한 억지 의인화는 "없음"으로 표기
- **이완 장치 유형과 위치를 미리 정하면 누락을 방지한다**
- **속어감탄 누적 카운터를 매 SL마다 갱신하면 6~10회 범위 이탈을 방지한다**
- 선언 없이 쓴 섹션은 완성 후 아래 **섹션 완성 후 미니 체크**를 반드시 통과해야 한다

### 섹션 완성 후 필수 이행 (섹션마다 즉시)

각 SL 집필이 끝나면 아래 두 가지를 즉시 수행한다. 미수행 시 해당 섹션은 미완성으로 간주한다.

**1) 기존 검증 (변경 없음)**
- 박스 이미지가 없으면 스캐폴딩 선언으로 돌아가 재작성
- SL 열림 유형이 첫 문장과 일치하지 않으면 재작성

**2) 상태 태그 작성 (신규)**
섹션 끝에 아래 형식의 HTML 주석을 삽입한다:

```html
<!-- [상태] SL{N} 완료
  - 루프: 열림(유형) / 해소 여부
  - 열린 루프 현황: {현재 미해소 루프 전체 나열}
  - 브릿지→SL{N+1}: {전환 전략 한 줄}
  - 아하모먼트: {박스이미지 문구} ({여는|닫는}, 이문호공 요인)
  - 마킹↔서술 일치: {[추정]/[해석] 팩트를 단정형으로 쓰지 않았는가? 위반 시 해당 문장 명시. 위반 없으면 "OK"}
  - 추가 팩트: {팩트리포트에 없는 정보를 추가한 경우, 내용과 출처를 기록. 없으면 "없음"} ※ 추가 시 반드시 웹 검색 후 출처 기재. 미확인 시 [추정] 서술로 전환
  - 의인화 적용: {적용 시 대상+표현 기록 / 미적용 시 "없음"}
  - 속어감탄 누적: {N}/6~10
-->
```

> 이완 장치 2분 규칙은 전체 흐름을 봐야 판단 가능하므로 GATE 4에서만 검증한다.

### SL 열림 5유형

| # | 유형 | 형태 | 빈도 |
|---|---|---|---|
| A | 직접 질문형 | "왜 ~냐?" | 최다 |
| B | 기대 위반형 | "~인데, 그런데..." | 중간 |
| C | 부정 선행형 | "~면 되냐? 아니다" | 중간 |
| D | 경험 소환형 | "~해 보신 적?" | 소수 |

### 성공 서사 텐션 규칙

정점 직후 의도적 추락(경고형) 1회 이상

### 시청자 예상 반박 선제 흡수

"~라고 하실 수도 있는데"

### 박스 이미지 공식

```text
박스 이미지 = (핵심 대상 + 비유/은유) + 강화문구(이문호공 2요인 이상)
```

강화문구에는 이문호공 요인을 **2개 이상** 중첩 적용한다. 주제와 내용 흐름에 맞는 요인 조합을 전략적으로 선택한다.

### 비유 vs 은유

- **비유(직유):** "~처럼" → 이해를 돕는다
- **은유:** "A는 B다" → 뇌에 새 개념을 박는다 (더 강력)

### 비유/은유 영역 선정 원칙

비유/은유의 영역은 **주제의 핵심 메커니즘에서 꺼낸다.** 스포츠·게임 등 외부 영역을 억지로 끼워 맞추지 않는다. 억지 비유는 이해를 돕는 게 아니라 흐름을 깬다.

- **영역 선정:** 주제가 자연스럽게 연상시키는 언어 영역을 먼저 찾는다
  - 예) 경쟁·구도 주제 → 게임/경기, 생존·적응 주제 → 자연/진화, 권력·통제 주제 → 물리적 힘/구조물
- **일관성:** 선택한 영역은 **같은 섹션(SL) 내에서 유지한다.** 한 단락에서 두 영역을 섞으면 이물감이 생긴다
- **절제:** 무거운 구간(전쟁 피해·재난·빈곤)에서는 비유를 생략하거나 최소화한다 — 톤 미스매치가 몰입을 깨뜨린다
- **생성 순서:** 비유를 먼저 찾지 않는다. 사건을 비유 없이 한 문장으로 쓴 뒤 → 핵심 메커니즘을 추출하고 → 같은 메커니즘을 가진 일상 이미지를 찾는다

> **마스터 워크플로우 4번("같은 비유 반복 금지")과의 관계:** 동일한 표현을 반복하지 말라는 규정이다. 같은 영역(체계)을 유지하되 표현은 매번 다르게 쓰는 것과 충돌하지 않는다.

### 박스 이미지 vs 설명용 비유

| 구분 | 박스 이미지 | 설명용 비유 |
|---|---|---|
| 목적 | 각인 (정체 정의) | 이해 (내용 해설) |
| 위치 | SL 열림/해소 | 본문 중간 |

모든 비유가 박스 이미지는 아니다. 복잡한 원리를 쉽게 설명하는 비유는 설명용이지 각인용이 아니다.

### 박스 이미지 2가지 역할

- **여는 박스:** 비유 먼저 → 궁금증 (SL 열림)
- **닫는 박스:** 팩트 후 비유 → 지적 보상 (SL 해소)

### 박스 이미지 자격 테스트

> "이 문장을 누군가에게 말한다면 '오 그래?' 반응이 나오는가?" — 나오지 않으면 박스 이미지가 아니라 일반 서술이다.

### 강약 구조

```text
팩트 투입 → 연결 문장 → 박스 이미지(강) → 논리적 뒷받침(약) → 브릿지/다음 전개
```

- 뒷받침 없으면 "뇌피셜" 위험 — 박스 이미지가 강한 만큼 팩트 없이 던지면 "근거는?" 반응 → 채널 신뢰 하락
- 연결 문장 없으면 "비약" 위험 — 팩트에서 박스 이미지로 직행하면 시청자가 논리적 경로를 따라오지 못해 "갑자기 왜?" 반응이 나온다. 연결 문장은 "그러니까 ~인 구조인 거예요", "결국 ~라는 얘기죠" 등
- **예외:** 팩트 임팩트형 (숫자 자체가 충격이면 비유 불필요)

### 배치 규칙: 한 편당 최소 3개

| 위치 | 모순형 | 이미지강화형 | 과정추적형 |
|---|---|---|---|
| 정점 직전 | A극 정점 (필수) | Scale Jump (필수) | 핵심 전환점 (필수) |
| 핵심 해소 | B극 해소 (필수) | 통찰 (필수) | 의미부여 (필수) |
| SL 해소 | 권장 | 권장 | 권장 |
| 엔딩 직전 | 권장 | 권장 | 권장 |

## PART C: 13개 언어 장치

### 1. 의인화

국가/물질/체내 반응을 캐릭터처럼. 직후 객관적 사실 보완.

### 2. 수치 체감 치환

**치환하는 경우:** 면적/거리 큰 수, 전문 단위, 비교 없이 판단 불가한 수치, 높이/길이의 공간적 크기

**치환하지 않는 경우:** 극단적 충격 수치("사망률 90%"), 연도/날짜/시각, 이미 직관적인 비율

**권위 구축 밀집 나열:** 원본 수치·날짜·인명·고유명사를 밀집 나열하여 신뢰감 부여. 이 경우 치환하면 오히려 권위가 약해진다.

### 3. 전문용어 치환

전문용어를 쉬운 말로 풀어 말하듯 서술한다. 용어가 용어 형태로 남지 않고 문장 속에 녹아야 한다. 외국어·학술 용어·업계 용어는 한국어 뜻을 먼저 제시한 뒤 원어를 병기한다.

- **예:** "소금 전매제" → "소금은 나라만 독점으로 팔 수 있었거든요"
- **예외:** 권위 구축 밀집 나열(§2) — 원본 용어를 유지해야 신뢰감이 살아남
- **핵심 메커니즘 설명 시:** 비유를 함께 사용하여 작동 원리까지 전달할 수 있다

### 4. 반전 질문 → 즉시 해소

레이어 전환 구간과 정보 밀도 피크 직후에 자연 배치한다. 유형은 되묻기/확인사격/시청자대입/침묵여백/뒤집기후재강화/유머전환 중 맥락에 맞는 것을 선택한다. SL이 열린 구간에서 즉시 해소형 반전 질문을 남발하지 않는다. 횟수는 강제하지 않는다.

### 5. 구어체 리듬

세 가지 장치를 발동 조건에 맞게 배치한다:

- 확인 사격: 긴 설명 직후, 핵심 한 문장으로 각인 ("뭐다? 감자다")
- 한 줄 요약: 복잡한 설명 후, 체감 문장으로 압축
- 감탄사 리셋: 정보 밀도 높아지기 직전 삽입

30초 이상 긴 문장 연속 시 반드시 짧은 확인 사격을 삽입한다.

### 6. 시청자 대입

편당 최소 1회, 주제와 시청자가 자연스럽게 연결되는 구간에 배치한다. 위치는 강제하지 않는다. 방식은 맥락에 맞게 선택한다 (질문형, 상황 제시형, 경험 소환형, 1인칭 경험 공유 등).

### 7. 속어·구어 감탄/밈

6~10회. 정보 피크 직후. 사물/상황만 대상. 무거운 구간 자제. 도입부에는 거의 사용하지 않는다 — 도입부 초반에 속어가 먼저 나오면 채널 첫인상이 가벼워져 이탈률이 올라간다. 5회 이하면 톤이 딱딱해지고, 11회 이상이면 채널이 가벼워 보인다. 같은 표현을 반복 사용하지 않는다. 해당 팩트에만 쓸 수 있는 맥락 특이적 표현을 사용한다. **욕설(씨발, 지랄 등)은 사용 금지.** "미쳤다", "대박", "걸작", "레전드" 등 구어 감탄 수준만 허용한다.

### 8. 현재형 서술

| 단계 | 시제 | 전환 마커 |
|---|---|---|
| 배경 설명 | 과거형 | — |
| 현재 상황 | 현재형 | "현재는", "지금은", "지금도" |
| 가상 시나리오 | 현재형 가정 | "만약에", "~하면", "~해 버리면" |

### 9. 대조 쌍

"A가 아니라 B다"가 "A다"보다 항상 강하다. 교차 대조(Chiasmus) A-B-B-A.

### 10. 엔딩 감정 전환

P3 엔딩 5유형(일상행위연결/감정적여운형/미래희망형/시의성마무리형/위트반전형) 참조.

### 11. 반사실 가정

"~이 없었다면". 1~3회. 추상적 가치를 구체적 부재로 체감시킬 때. 구문을 매번 변형한다.

### 12. 화자 감정 삽입

1~2회. 1인칭 감정. 시사/시의성 주제에서 빈도 높음. — 3회 이상이면 "이 사람 주관 강하네" 인식 → 정보 채널이 아니라 의견 채널로 분류된다. 진입 구문을 매번 변형한다.

### 13. "이미 이렇게 돼 있다" 현실 폭로

일상 속 대상의 숨겨진 사실. 모든 주제에 적용 불가.

### 이완 장치 강제 규칙

- 팩트 3개+ 연속 후 반드시 이완 1회 — 팩트가 연속되면 인지 부하가 누적되어 시청자가 처리를 포기하고 이탈한다
- 2분 이상 이완 없으면 수정 대상
- 이완 장치: 속어·구어 감탄/밈, 자조적 한 줄, 시청자 공모형 유머, 감탄사+짧은 판단
- **무거운 구간**에서는 자조적 한 줄 / 감탄사+짧은 판단 / 화자 감정 삽입으로 이완한다

### 정보 밀도 파동 사이클

- 60~90초 단위 반복
- 15분 기준 10~12회

### 정보 분류별 서술 규칙

- **사실:** 단정형 사용 가능
- **추정/해석/논쟁:** 단정형 금지. 출처나 불확실성을 드러내는 표현을 자연스럽게 사용

## PART D: 엔딩 집필

### 엔딩 집필 전 스캐폴딩 선언 (필수)

엔딩을 집필하기 **전에** 아래 선언을 먼저 작성한다. 선언 없이 엔딩 산문부터 쓰지 않는다.

```text
[엔딩 — 엔딩유형: {1~5} / 쿨다운 착지: {조감도 착지 문구 복사} / 도입부 소재 회수: {사용|미사용, 회수할 소재}]
```

- 엔딩 유형을 선언하면 → 톤 이탈 방지
- 조감도 착지 문구를 복사하면 → 설계된 요소 누락 방지
- 도입부 소재 회수 여부를 명시하면 → 수미상관 누락 방지

### 결론부 쿨다운 규칙

**3단계 구조:**

1. 정리: 본문 인과 체인을 한 호흡으로 압축 (새 정보 없이 요약만)
2. 착지: 일상 행동 제안 / 수미상관 회수 / 감정 여운 중 택1
3. 시그니처 아웃트로

**규칙:**

- **ML 해소는 마지막 SL(본문)에서 완료한다.** 결론부는 착지만 수행하며, ML 해소 이후 새로운 정보를 추가하지 않는다 — 결론부에 신규 정보가 들어가면 "끝났는데 왜 안 끝나지" 느낌 → 마지막 인상이 나빠져 구독/좋아요 전환에 악영향
- "교훈"이나 "경고"로 끝내지 말 것
- 착지 방식은 맥락에 맞게 선택한다: 일상 행동 제안, 도입부 소재 수미상관 회수, 감정 여운, 희망적/유머러스 전환 등

## GATE 4 직전 팩트 검증 (필수)

GATE 4 진입 전, 대본 전문의 수치·고유명사·날짜를 팩트리포트와 대조한다.

1. 대본에서 모든 수치·고유명사·날짜를 추출한다
2. 팩트리포트 원본과 기계적으로 대조한다
3. 불일치 항목을 아래 형식으로 출력한다:
   - **[열화]** 대본 표현 → 팩트리포트 원본 (예: "115경기 이상" → "116경기")
   - **[미수록]** 대본 표현 → 팩트리포트에 없음. 출처 확인 필요
4. [열화] 항목은 즉시 수정한다
5. [미수록] 항목은 웹 검색으로 출처를 확인하고, 확인 불가 시 삭제하거나 [추정] 서술로 전환한다
6. 대본에서 창작한 장면·상황 묘사(도입부 훅, 시청자 대입, 일상 비유 등)에 상식 오류가 없는지 검증한다

## GATE 4 체크리스트

- [ ] 대본 전문이 완성되었는가?
- [ ] 영상 목표 길이가 10~20분인가?
- [ ] 도입부(2분 30초 이내, 약 750자 공백 포함) 안에 ML이 열려 있는가?
- [ ] SL 열림 유형이 교차 배치되었는가? (5유형 중 최소 3유형 사용, 같은 유형 연속 2회 초과 금지)
- [ ] 2분+ SL 없는 구간이 없는가?
- [ ] 박스 이미지(강) + 뒷받침(약)이 최소 3세트 있는가?
- [ ] 반전 질문이 레이어 전환 구간 또는 정보 밀도 피크 직후에 배치되었는가?
- [ ] "우리/한국/여러분" 배치가 후반부에 있는가?
- [ ] 속어·구어 감탄이 6~10회 있는가? (5회 이하 = 딱딱함, 11회 이상 = 채널 경박)
- [ ] 시청자 대입이 최소 1회 있는가? (질문형/상황제시형/경험소환형 무관)
- [ ] 2분 이상 이완 장치 없는 구간이 없는가? (속어·자조·감탄사+판단 중 하나)
- [ ] [추정]/[해석] 마킹 팩트가 대본에서 단정형으로 서술되지 않았는가? — 마킹이 [추정]이면 "~로 알려져 있다", [해석]이면 "~라는 분석이 있다" 형식 필수
- [ ] 팩트리포트에 없는 정보가 출처 미확인 상태로 삽입되지 않았는가? — 상태 태그의 "추가 팩트"에 기록되지 않은 미수록 정보는 삭제 또는 [추정] 전환
- [ ] 결론부에 ML 해소나 신규 정보가 포함되어 있지 않은가? (정리+착지+아웃트로만 존재하는가?)
- [ ] 결론이 복합 원인 중 하나를 유일 원인으로 단정하지 않았는가? — "~가 결정했다"보다 "~가 핵심 변수였다" 형식 권장
- [ ] 의인화 장치가 최소 1회 적용되었는가? — 주제 특성상 불가능한 경우 사유 명시

## 할루시네이션 방지 규칙

- **지침 예시 문구 복제 금지:** 지침의 모든 예시는 원리 파악용. 실제 집필 시 해당 구간의 맥락에서만 가능한 표현을 새로 생성한다.
- **화자가 자기 정보에 놀라는 척 금지** — 감탄/감상을 제거했을 때 정보나 대비가 남지 않는 문장("말도 안 되죠", "대박이죠")은 놀라는 척이다. 대비를 명시하는 문장("골대를 뽑을 점수가 아닙니다")으로 교체한다
- **비유의 메커니즘 일치 확인:** 표면적 유사성이 아니라 작동 원리가 일치하는지 검증
- **수정 후 정합성 확인 (일반):** 문장 수정 시 같은 단락 전체를 다시 읽는다
- **수정 후 정합성 확인 (구조 요소):** ML, 피벗, 박스 이미지를 수정할 경우 해당 요소를 전제하거나 참조하는 모든 문장을 검색하여 재검증한다
- **수치 최신성 확인:** 현재 시점 기준 최신인지, 주체/범위/조건이 보존되었는지 확인
- **집필 중 추가 팩트 출처 기록:** 팩트리포트에 없는 정보를 집필 중 추가할 경우, 반드시 웹 검색으로 확인한 뒤 상태 태그의 "추가 팩트" 항목에 내용과 출처를 기록한다. 미확인 정보를 단정형으로 서술하지 않는다
