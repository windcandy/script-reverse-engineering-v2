## 집필 전 필수 절차

**이 파일(`P4_집필.md`)을 집필 직전에 반드시 다시 읽는다.** 조감도와 팩트 리포트만 보고 집필하지 않는다. 지침을 설계한 시점과 집필 시점 사이에 맥락이 밀려 지침 적용이 누락되기 때문이다.

**재로드 후 집필 시작 전에 P3 조감도의 SL 배열을 보고, 각 SL의 사전 선언(PART B 참고)을 먼저 작성한다.** 도입부 → SL1 → SL2 … 순서대로 선언을 나열한 뒤 집필에 들어간다. 조감도 없이 산문으로 먼저 쓰는 것은 금지한다.

## 공통 집필 원칙

- **정보의 단순 나열 금지:** 팩트를 나열할 때 문장 간 인과·대비·스케일 확대 등 흐름 연결이 있어야 한다. 연결 없이 독립된 사실을 순서대로 늘어놓지 않는다. 단, 권위 구축 밀집 나열(§PART C-2)은 제외한다.
- **썸네일 문구 직접 인용 금지:** 도입부·본문 어디에서도 썸네일 문구를 그대로 반복하지 않는다. 간접 확인(수치·동의어·암시)으로만 연결한다.
- **시청자 반박 선제 흡수:** SL 1~2개마다 1회 "~라고 하실 수도 있는데요" 패턴으로 의심을 차단한다. 모든 서사 유형에 공통 적용.
- **ML 해소 완료 위치:** ML은 반드시 본문 마지막 SL에서 완료 후 결말 진입한다. 결말에서 ML을 해소하지 않는다.

## PART A: 도입부 집필

> 전략별 실행 패턴 참고: `00_지식파일/인트로_예시_전략별.md`

### 도입부 집필 전 사전 선언 (필수)

도입부를 집필하기 **전에** 아래 선언을 먼저 작성한다. 선언 없이 도입부 산문부터 쓰지 않는다.

```text
[도입부 — 서사유형: {모순형|이미지강화형|과정추적형} / 전략: {A~H} / 도입부 이문호공: {요인+요인} / 충격 주체: {누가 무엇을 했는지 한 줄} / 신뢰 상환: {도입부 수행|이연} / 신뢰 상환 팩트: {팩트리포트 원본 수치 나열} / 소화 문장 방향: {어떤 의미를 짚을 것인지} / 근거 팩트: {나열된 팩트 중 이 최종 통찰(L3)을 지지하는 팩트 명시} / ML 이문호공: {요인+요인} / ML 형태: {서사유형별 허용 형태}]
```

- **서사유형을 선언**하면 → 유형별 규칙 적용 이탈 방지
- 전략 코드를 선언하면 → 전략 이탈 방지
- 도입부 이문호공 요인을 미리 지정하면 → 요인 수 미달 방지
- **신뢰 상환 위치를 선언**하면 → 유형 불일치 방지
- 팩트리포트 원본 수치를 그대로 복사하면 → 팩트 열화 방지
- **근거 팩트를 명시**하면 → 팩트 나열 후 마지막 팩트를 자동으로 받는 관성 차단, 논리 비약 방지
- ML 형태를 미리 정하면 → 유형별 허용 형태 이탈 방지

### A-1. 도입부 공통 구조

P3에서 선택한 도입부 전략(A~H)을 실행한 뒤, 아래 두 요소를 배치한다.

> **흐름 원리:**
>
> - **신뢰 상환 위치는 서사 유형 기본 원칙(P3 §3) + 구조 변주(09_신뢰_상환_위치_분석.md)에 따라 판단한다.**
> - 도입부 전략(A~H)과 신뢰 상환 위치는 강제 매핑이 아니다. P3 조감도에서 확정한 위치·배치 순서를 그대로 실행한다.
> - **서사 유형별 기본 원칙:** 모순형 = 본문 이연 / 이미지 강화형·과정 추적형 = 도입부 수행 (예외는 P3 §3 참조).

**1) 신뢰 상환:** 썸네일이 약속한 내용을 실제 팩트로 증명한다. 팩트 나열이 아니라 팩트의 체감 시간을 늘린다.
   - **신뢰 상환 ≠ ML:** 신뢰 상환은 "이미 일어난 일/현재 상태"를 팩트로 보여주는 것, ML은 "왜/어떻게"를 묻는 질문. 같은 문장에 섞지 않는다.
   - **필수:** 팩트 후 소화 문장 삽입. 단, 팩트 자체가 충분히 직관적이면 생략 가능.
   - **필수:** 소화 문장은 직전 팩트가 아니라 최종 통찰(L3)을 가장 잘 지지하는 팩트를 근거로 삼는다.
   - **필수:** 숫자/전문 개념은 일상 언어로 풀어서 서술한다. **이미지강화형은 수치 단독 제시가 아니라 비교 대상을 반드시 동반한다** ("한반도보다 넓은", "밀+쌀 합쳐야 비빌 수준").

**2) ML 생성:** 영상 전체를 관통하는 핵심 궁금증. **호기심(3) 또는 상식파괴(5) 중 하나를 반드시 포함한다.**

## PART B: 본문 집필

### B-0. 오픈루프 4단계 + 브릿지 (공통)

> 모든 서사 유형에 적용. 출처: `[추출] 대본/07_오픈루프_4단계_접착제_분석.md`

| 단계 | 위치 | 작동 방식 |
|---|---|---|
| **열기 (Opening)** | 도입부 ML | ML 질문으로 인지적 공백 생성 |
| **유지 (Holding)** | 본문 전반 | 핵심 답변 보류, 서브루프로 분산 해소 |
| **강화 (Reinforcing)** | 피벗 직전·Scale Jump 직전 | 확신 극대화 또는 방향 재상기 |
| **닫기 (Closing)** | 본문 후반 | 분할 제공·역전 해소·Aha-moment 순 |

**브릿지 유형:**

| 유형 | 정의 | 비율 |
|---|---|---|
| **[A-질문] 서브 오픈루프 — 질문형** | "다 알았다"는 순간 차단 — 종결 부정·인과 추적·완벽함 뒤의 함정·사회 통념 반전·스케일업·서사적 초필살기 (6종) | 40~65% |
| **[A-선언] 서브 오픈루프 — 선언·예고형** (구 심화 브릿지) | 확언으로 새 SL 진입 — "진짜는 지금부터"·"결정적 사건"·"예상치 못한 또 다른"·"마침내"·"사실은 이미"·"양반이었다"·"시즌 전환"·"시간 역행"·"아 맞다" (9종) | 20~35% |
| **[B] 논리적 브릿지** | 정보 정리 + 감정 매듭 — 선형적·인과적·철학적 (3종) | 10~25% |

> **SL 선언·예고형 [A-선언]은 필수 적용 대상이다.** 실측상 단독 사용이 아니라 **새 SL을 여는 순간 동반 사용되는 선언·예고 문법**이므로 SL과 함께 설계한다. 9종 패턴(진짜는지금부터·결정적사건·양반이었다·시즌전환 등)은 `지침/00_공통_원칙.md` C.9 [A-선언] 참조.

### B-1. 섹션 집필 전 사전 선언 (필수)

각 SL을 집필하기 **전에** 아래 한 줄을 먼저 적는다. 집필 후 제거해도 되지만, 선언 없이 본문부터 쓰지 않는다.

```text
[SL번호 — 선행브릿지: {이전 섹션 상태태그의 브릿지 문구 복사 → 실제 집필할 브릿지 문장} / SL열림유형 / 해소비유: 있음|없음(팩트임팩트형) — 핵심 대상 한 줄 — 이문호공 2요인({요인+요인}) / 핵심 팩트: [{마킹}] {팩트리포트 원본 복사} / 의인화 대상: {대상+역할 또는 "없음"} / 이완 장치: {유형+위치} / 속어감탄 누적: {N}/6~10]
```

- **선행브릿지를 이전 상태태그에서 복사**하면 → 브릿지 누락·집필 공백 차단
- **핵심 팩트는 팩트리포트 파일을 열어 복사**한다. 기억으로 작성 금지
- **마킹([사실]/[추정]/[해석]/[논쟁])을 함께 복사**하면 서술 방식이 자동 결정됨
- 의인화·이완 장치·속어감탄 누적을 미리 선언하면 누락 방지

### B-2. 섹션 완성 후 필수 이행

각 SL 집필이 끝나면 아래 두 가지를 즉시 수행한다.

**1) 기존 검증:** 해소 비유가 없으면 사전 선언으로 돌아가 재작성 / SL 열림 유형이 첫 문장과 불일치하면 재작성

**2) 상태 태그 작성:**
```html
<!-- [상태] SL{N} 완료
  - 루프: 열림(유형) / 해소 여부

```

### B-4. 서사 유형별 본문 구조

> **근거:** `[추출] 대본 - 분석/04·05·06_본문_분석_*.md`

#### 모순형

**핵심 구조:**
```
A극(기초SL → 심화SL → 정점SL) → 피벗BR → B극(What → How → Why → 현재의미)
```

**A극 규칙:**
- 기초SL·심화SL 경계가 불분명해도 된다. 2단(기초+정점) 처리도 허용
- 신뢰 상환은 도입부가 아닌 **A극 첫 SL에서 흡수**하는 것이 자연스러운 방식
- A극/B극 비율: 기본 1:1. A극이 역사 서사 포함 시 최대 3:1까지 불균형 허용 (A극 자체가 충분히 흥미로운 경우)
- A극 정점에서 확신을 최고조로 끌어올린 직후 피벗 진입 — "절정 후 역전" 패턴

**피벗 브릿지 5유형:**

| 유형 | 예시 | 특징 |
|---|---|---|
| 직접 선언형 | "여기서 문제는 ~" | 반전 질문 없이 선언만으로 전환 |
| 반어 질문형 | "그런데 다들 아시죠?" | 시청자가 이미 아는 사실을 반어적으로 확인 |
| 단호 선언형 | "누가 봐도 마약 맞죠" | A극 인식을 단호하게 부정 |
| 대비 선언형 | "하지만 우리는 다 알고 있죠" | 시청자 공감을 활용한 전환 |
| 유보형 | "있긴 있죠. 잠시만요" | 장점 소개를 극적으로 지연 |

**변주 6유형:**

| # | 명칭 | 정의 |
|---|---|---|
| 1 | 정적 역설 | A극 속성이 그대로 B극의 독이 됨. A·B 원인 동일 |
| 2 | 시간축 전개 | A극을 역사·성공 스토리로 전개 후, B극에서 대가를 보여줌 |
| 4 | 누적 반복형 | B극에서 문제를 하나씩 누적 투하. 다발성 역전 |
| 5 | 양면 동시 건설형 | A극에서 장점 설명과 동시에 단점의 씨앗 노출 |
| 6 | 감정 피벗형 | 피벗이 논리적 반전 아닌 감정적 유보로 전환 |

**Aha-moment:** B극 70~85% 지점 / "A극 핵심 속성이 B극에서 역전" = 역설 발견

#### 이미지강화형

**핵심 구조:**
```
What → How → Scale Jump(복수) → Why/통찰
```

**Scale Jump 규칙:**
- **Scale Jump는 1회 전제가 아니다. 2~5회 발생이 표준이다.** (10편 중 7편이 2회 이상)
- Scale Jump 간 전환 브릿지: "그런데 예상치 못한 또 다른 효과가", "그럼 끝날까요? 아니죠"

**Scale Jump 7변주:**

| # | 명칭 | 정의 |
|---|---|---|
| 1 | 지침 충실형 | What→How→Scale Jump(1회)→Why 단선 |
| 2 | 단계적 스케일업 | Scale Jump 2~5회 단계적 확대 |
| 3 | 파동형 | 장점→문제 제기→해결→더 큰 장점의 파동 |
| 4 | 누적 스케일업 | 같은 방향을 겹겹이 쌓음. "끝이냐? 아니다" 반복 |
| 5 | 수평 확장형 | 같은 원리가 다른 문화권/지역에서도 작동함을 보여줌 |
| 6 | 복수 대상 병렬형 | 복수 대상을 병렬로 다루되 공통 메커니즘으로 묶음 |
| 7 | 시간 역행형 | "사실 더 먼저 발명됐다"로 시간을 거슬러 현재 복귀 |

**긍정 피로 방지 (필수):** 5분 이상 장점(스케일)만 쌓이면 텐션 하락. 다음 중 하나로 재점화:
- **문제 제기 삽입:** "근데 이게 또 문제가 있다" → 해결 → 다음 Scale로
- **"끝이냐? 아니다" 패턴:** "그럼 이제 끝일까요? 어림도 없지, 이제부터가 본 게임"
- **시청자 반박 선제 흡수:** "이 정도면 과장 아니냐?" → 차단 후 증거 투하

**How가 범용 열쇠:** How에서 공개한 메커니즘이 Scale Jump마다 반복 적용됨 → 시청자가 "이것도 같은 원리?"를 능동 유추하게 된다.

**Aha-moment:** 마지막 Scale Jump 또는 Why 레이어 / "이것까지 영향이 미쳤다" = 스케일 발견 (누적 과정에서 임계점 도달)

#### 과정추적형

**핵심 구조:**
```
발단(배경+촉발) → 전개(전환점1→2→N) → 절정(결정적 전환) → 결말(현재의미)
```

**전환점 규칙:**
- **전환점 수는 평균 4.5회가 표준.** 2~3회 전제는 시간 범위가 짧은 사건 서사형에만 해당
- 시간 범위가 넓을수록 전환점이 많고 각 전개가 압축됨 (범위 1000년 → 6회 전환점)
- **하이브리드 구조 허용:** 과정 추적 중 이미지강화형적 메커니즘 해설(How) 삽입 가능

**과정추적형 고유 장치:**

| 장치 | 내용 |
|---|---|
| **시간 관성** | "다음엔 어떻게 됐어?"라는 시간적 궁금증이 핵심 동력. 모순형의 피벗·이미지강화형의 스케일에 해당 |
| **감정이입 장치** | 감정을 통해 "다음은?"의 동기를 자발적으로 생성하게 만든다. 모순형·이미지강화형에는 없는 고유 장치 |
| **ML 계단식 해소** | "어쩌다?" ML이 한 번에 해소되지 않고 전환점마다 부분 해소 → 시청자가 계속 따라오게 됨 |
| **복선 배치** | 초반 정보가 후반에 다른 의미로 재등장. "아 그때 그게 이래서 이렇게 된 거구나" — 같은 요소의 의미가 시간 경과에 따라 변화하는 구조 |
| **시간 전환 브릿지** | "그렇게 N세기에는…", "그러던 N년…" — 시대 간 전환이 핵심 브릿지 |

**텐션 유지 장치:**
- 전환점마다 상황 역전: 우세 → 위기 → 역전의 반복으로 시청자를 묶음
- 감정 고조 문장: "경찰이 후퇴했다", "기록 말살형에 처해졌다" — 분노·억울함 유발
- "여기서 끝이 아니다" 패턴: 새 전환점 진입 직전에 반드시 긴장 재점화

**Aha-moment:** 위치 % 강제 아님. **완결 조건 체크리스트** 충족 시 배치 (유형 파일 §Aha 참조). 과정추적형은 "이래서 이렇게 됐구나" 인과 발견 또는 "이런 사람들이 있었구나" 감정 결집 지점. 실측 분포 35~90%는 참고.

### B-5. 공통 이탈 방지 장치

**보상의 비례 원칙:** 빌드업이 길수록 보상(해소)도 구체적·풍부하게. 빌드업과 보상의 비율이 맞지 않으면 시청자가 배신감을 느낀다.

**보상의 분할 제공:** 가장 큰 보상을 한 번에 투하하지 않고 분할한다. 3~4단계 점진적 공개.

**단서 배치 원칙 (정답형 서술 회피):**
```
단서 배치 → 시청자 유추 → 확인
```
핵심 Why를 최대한 늦게 공개. 그 전에 단서를 흩뿌려 시청자가 스스로 조립하게 만든다.

**성공 서사 텐션 규칙:** 정점 직후 의도적 추락(경고형) 1회 이상.

### B-6. 해소 비유 공식

```text
해소 비유 = (핵심 대상 + 비유/은유) + 강화문구(이문호공 2요인 이상)
```

**해소 비유 작동 지점** (서사 유형별):
- **모순형:** 피벗/반전, B극 해소 시점
- **이미지강화형:** Scale Jump 정점, 통찰 시점
- **과정추적형:** 핵심 전환점, 인과 완성 시점

> **실전 근거:** 지식해적단 25편 분석 결과 — SL 열림에서의 비유 0건 / SL 해소에서의 비유 16건. 해소 비유는 반드시 SL 해소 시점에서만 작동한다.

**강약 구조:**
```
팩트 투입 → 연결 문장 → 해소 비유(강) → 논리적 뒷받침(약) → 브릿지/다음 전개
```

- **연결 문장 없으면 "비약" 위험** — "그러니까 ~인 구조인 거예요", "결국 ~라는 얘기죠" 등
- **뒷받침 없으면 "뇌피셜" 위험** — 해소 비유가 강할수록 팩트 없이 던지면 신뢰 하락
- **예외:** 팩트 임팩트형 (숫자 자체가 충격이면 비유 불필요)

**배치 규칙: 한 편당 최소 3개**

| 위치 | 모순형 | 이미지강화형 | 과정추적형 |
|---|---|---|---|
| 정점 직전 | A극 정점 (필수) | Scale Jump (필수) | 핵심 전환점 (필수) |
| 핵심 해소 | B극 해소 (필수) | 통찰 (필수) | 의미부여 (필수) |
| SL 해소 | 권장 | 권장 | 권장 |
| 엔딩 직전 | 권장 | 권장 | 권장 |

**해소 비유 자격 테스트:** "이 문장을 누군가에게 말한다면 '오 그래?' 반응이 나오는가?" — 나오지 않으면 해소 비유가 아니라 일반 서술이다.

**비유 vs 은유:**
- 비유(직유): "~처럼" → 이해를 돕는다
- 은유: "A는 B다" → 뇌에 새 개념을 박는다 (더 강력)

**비유/은유 영역 선정 원칙:**
- 주제의 핵심 메커니즘에서 꺼낸다. 외부 영역을 억지로 끼워 맞추지 않는다
- 같은 섹션(SL) 내에서 영역을 일관 유지한다
- 무거운 구간(전쟁·재난·빈곤)에서는 최소화한다 — 톤 미스매치가 몰입을 깨뜨림
- **생성 순서:** 사건을 비유 없이 한 문장으로 쓴 뒤 → 핵심 메커니즘 추출 → 같은 메커니즘을 가진 일상 이미지 탐색

**해소 비유 vs 설명용 비유:**

| 구분 | 해소 비유 | 설명용 비유 |
|---|---|---|
| 목적 | 각인 (정체 정의) | 이해 (내용 해설) |
| 위치 | SL 해소 | 본문 중간 |

## PART C: 13개 언어 장치

### 1. 의인화
국가/물질/체내 반응을 캐릭터처럼. 직후 객관적 사실 보완. 최소 1회 적용 (주제상 불가능한 경우 사유 명시).

### 2. 수치 체감 치환
**치환하는 경우:** 면적/거리 큰 수, 전문 단위, 비교 없이 판단 불가한 수치, 높이/길이의 공간적 크기

**치환하지 않는 경우:** 극단적 충격 수치("사망률 90%"), 연도/날짜/시각, 이미 직관적인 비율

**권위 구축 밀집 나열:** 원본 수치·날짜·인명·고유명사를 밀집 나열하여 신뢰감 부여. 이 경우 치환하면 오히려 권위가 약해진다.

### 3. 전문용어 치환
전문용어를 쉬운 말로 풀어 말하듯 서술한다. 용어가 용어 형태로 남지 않고 문장 속에 녹아야 한다. 외국어·학술 용어·업계 용어는 한국어 뜻을 먼저 제시한 뒤 원어를 병기한다.

- **예:** "소금 전매제" → "소금은 나라만 독점으로 팔 수 있었거든요"
- **예외:** 권위 구축 밀집 나열 — 원본 용어 유지

### 4. 반전 질문 → 즉시 해소
레이어 전환 구간과 정보 밀도 피크 직후에 자연 배치한다. 유형은 되묻기/확인사격/시청자대입/침묵여백/뒤집기후재강화/유머전환 중 맥락에 맞게 선택한다.

### 5. 구어체 리듬
- **확인 사격:** 긴 설명 직후, 핵심 한 문장으로 각인 ("뭐다? 감자다")
- **한 줄 요약:** 복잡한 설명 후, 체감 문장으로 압축
- **감탄사 리셋:** 정보 밀도 높아지기 직전 삽입
- 30초 이상 긴 문장 연속 시 반드시 짧은 확인 사격 삽입

### 6. 시청자 대입
편당 최소 1회. 주제와 시청자가 자연스럽게 연결되는 구간에 배치. 방식: 질문형·상황 제시형·경험 소환형·1인칭 경험 공유.

### 7. 속어·구어 감탄/밈
**6~10회.** 정보 피크 직후. 사물/상황만 대상. 무거운 구간 자제. 도입부 초반 사용 금지(첫인상이 가벼워져 이탈률 상승). 같은 표현 반복 금지. 해당 팩트에만 쓸 수 있는 맥락 특이적 표현 사용. **욕설 금지.** "미쳤다", "대박", "걸작", "레전드" 등 구어 감탄 수준만 허용.

### 8. 현재형 서술

| 단계 | 시제 | 전환 마커 |
|---|---|---|
| 배경 설명 | 과거형 | — |
| 현재 상황 | 현재형 | "현재는", "지금은", "지금도" |
| 가상 시나리오 | 현재형 가정 | "만약에", "~하면", "~해 버리면" |

### 9. 대조 쌍
"A가 아니라 B다"가 "A다"보다 항상 강하다. 교차 대조(Chiasmus) A-B-B-A.

### 10. 엔딩 감정 전환
PART D 엔딩 유형 참조.

### 11. 반사실 가정
"~이 없었다면". 1~3회. 추상적 가치를 구체적 부재로 체감시킬 때. 구문을 매번 변형한다.

### 12. 화자 감정 삽입
1~2회. 1인칭 감정. 시사/시의성 주제에서 빈도 높음. 3회 이상이면 "의견 채널"로 분류됨. 진입 구문 매번 변형.

### 13. "이미 이렇게 돼 있다" 현실 폭로
일상 속 대상의 숨겨진 사실. 모든 주제에 적용 불가.

### 이완 장치 강제 규칙
- **팩트 3개+ 연속 후 반드시 이완 1회** — 인지 부하 누적 방지
- **2분 이상 이완 없으면 수정 대상**
- 이완 장치: 속어·구어 감탄/밈, 자조적 한 줄, 시청자 공모형 유머, 감탄사+짧은 판단
- **무거운 구간:** 자조적 한 줄 / 감탄사+짧은 판단 / 화자 감정 삽입으로 대체

### 정보 밀도 파동 사이클
- 60~90초 단위 반복 / 15분 기준 10~12회

### 정보 분류별 서술 규칙
- **사실:** 단정형 사용 가능
- **추정/해석/논쟁:** 단정형 금지. 불확실성을 드러내는 표현 필수

### D-1. 결론부 쿨다운 규칙 (공통)

**3단계 구조:**
1. **정리:** 본문 인과 체인을 한 호흡으로 압축 (새 정보 없이 요약만)
2. **착지:** 아래 엔딩 유형 중 선택
3. **시그니처 아웃트로:** "이상 이거봐봐 지식해적단이었습니다"

**규칙:**
- **ML 해소는 마지막 SL(본문)에서 완료한다.** 결론부는 착지만 수행
- 결론부에 신규 정보 추가 금지 — "끝났는데 왜 안 끝나지" 느낌 → 마지막 인상 저하
- "교훈"이나 "경고"로 끝내지 말 것. 경고가 필요하면 위트로 감싸서 완화
- **도입부 소재 회수(수미상관):** 도입부 드립·소재를 결말에서 재활용. 가능하면 반드시 사용

---

### D-2. 엔딩 8유형 + 유형별 자연 궁합

> **본격 정의는 `00_공통_원칙.md` §B.6 참조.** P4는 집필 실행 시 사용 안내만 표기.

| 유형 | 설명 | 유형별 자연 궁합 |
|---|---|---|
| ① 일상행위연결 | "오늘 해 먹어 보세요", "다음에 가보세요" | 이미지강화형·과정추적형 |
| ② 감정적여운형 | "억울했던 마음도 같이 생각해 보면" | **과정추적형 최다(25%)** |
| ③ 미래희망형 | "통일이 돼서 서울이 후방이 되는 날을" | **이미지강화형 최다(60%)** |
| ④ 시의성마무리형 | 현재 이슈와 연결하여 착지 | 과정추적형·이미지강화형 |
| ⑤ 위트반전형 | 도입부 드립 회수 + 유머로 착지 | **모순형 선호** |
| ⑥ 통찰형 | 본문 전체를 관통하는 한 문장 통찰 (예: 10 종교와 고기) | 모순 2·이미지 3·과정 1편 |
| ⑦ 경고형 | 대상의 한계·문제·위험 경고로 착지 (예: 13 호르무즈) | 모순 2·이미지 1·과정 1편 |
| ⑧ 광고·협찬형 | 협찬·제품 홍보와 결합한 착지 (예: 24 전기차) | 모순 1·이미지 1·과정 1편 |

> **혼합형이 표준이다:** 단일 유형이 아닌 "주1+보조1" 조합 사용. 예) 미래희망(주)+위트(보조).

### D-3. 서사 유형별 엔딩 패턴

#### 모순형
- **선호:** 위트반전형(⑤) + 도입부 소재 회수. 경고 톤이 필요할 때 위트로 감싸서 완화
- **결말 길이:** 공백포함 평균 ~243자

#### 이미지강화형
- **선호:** 미래희망형(③) — 6/10(60%). 대상의 파급력이 "앞으로도 계속된다"로 착지하는 것이 가장 자연스러움
- **결론부 문제 제기 허용:** 스케일 극대화 후 마지막에 한계·부작용을 짧게 제시 → 미래희망 착지. 이미지강화형 고유 패턴
- **결말 길이:** 공백포함 평균 ~236자

#### 과정추적형
- **선호:** 감정적여운형(②). 역사적 사건의 감정적 무게가 자연스러운 여운을 만든다
- **고유 패턴:**
  - **후일담:** "사건 이후 어떻게 됐나" — 신규 정보가 아닌 ML 해소의 자연스러운 연장으로 허용
- **결말 길이:** 공백포함 평균 ~152자 (3유형 중 가장 짧음)

## GATE 4 직전 팩트 검증 (필수)

GATE 4 진입 전, 대본 전문의 수치·고유명사·날짜를 팩트리포트와 대조한다.

1. 대본에서 모든 수치·고유명사·날짜를 추출한다
2. 팩트리포트 원본과 기계적으로 대조한다
3. 불일치 항목 출력:
   - **[열화]** 대본 표현 → 팩트리포트 원본
   - **[미수록]** 대본 표현 → 팩트리포트에 없음. 출처 확인 필요
4. [열화] 즉시 수정 / [미수록] 웹 검색 확인, 불가 시 삭제 또는 [추정] 전환
5. 창작 장면·상황 묘사에 상식 오류가 없는지 검증

- [ ] 2분+ SL 없는 구간이 없는가?
- [ ] 해소 비유(강) + 뒷받침(약)이 최소 3세트 있는가?
- [ ] [A-선언] 선언·예고형 9종 ("진짜는 지금부터"·"결정적 사건"·"양반이었다" 등) 중 최소 1회 이상 사용되었는가? (구 심화 브릿지)
- [ ] **[이미지강화형]** Scale Jump가 2회 이상 설계되었는가? 긍정 피로 방지 장치가 있는가?
- [ ] **[과정추적형]** 전환점이 3회 이상인가? 감정이입 장치가 있는가? 복선이 최소 1개 설계되었는가?
- [ ] **[모순형]** 피벗 브릿지 유형이 명확하게 작동하는가?
- [ ] 반전 질문이 레이어 전환 구간 또는 정보 밀도 피크 직후에 배치되었는가?
- [ ] 속어·구어 감탄이 6~10회 있는가?
- [ ] 시청자 대입이 최소 1회 있는가?
- [ ] 2분 이상 이완 장치 없는 구간이 없는가?
- [ ] [추정]/[해석] 마킹 팩트가 단정형으로 서술되지 않았는가?
- [ ] 팩트리포트에 없는 정보가 출처 미확인 상태로 삽입되지 않았는가?
- [ ] 결론부에 ML 해소나 신규 정보가 없는가? (과정추적형의 후일담·시의성 맥락은 허용)
- [ ] 최종 통찰(L3)이 복합 원인 중 하나를 유일 원인으로 단정하지 않았는가?

- [ ] 엔딩 유형이 서사 유형과 자연 궁합을 이루는가? (D-3 기준)

## 할루시네이션 방지 규칙

- **지침 예시 문구 복제 금지:** 지침의 모든 예시는 원리 파악용. 실제 집필 시 해당 구간의 맥락에서만 가능한 표현을 새로 생성한다.
- **화자가 자기 정보에 놀라는 척 금지** — 감탄/감상을 제거했을 때 정보나 대비가 남지 않는 문장은 제거. 대비를 명시하는 문장으로 교체한다.
- **비유의 메커니즘 일치 확인:** 표면적 유사성이 아니라 작동 원리가 일치하는지 검증
- **수정 후 정합성 확인 (일반):** 문장 수정 시 같은 단락 전체를 다시 읽는다
- **수정 후 정합성 확인 (구조 요소):** ML·피벗·해소 비유를 수정할 경우 해당 요소를 참조하는 모든 문장을 검색하여 재검증한다
- **수치 최신성 확인:** 현재 시점 기준 최신인지, 주체/범위/조건이 보존되었는지 확인
- **집필 중 추가 팩트 출처 기록:** 팩트리포트에 없는 정보는 반드시 웹 검색 확인 후 상태 태그 "추가 팩트"에 내용+출처 기록. 미확인 정보 단정형 서술 금지
