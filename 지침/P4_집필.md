## 공통 집필 원칙

- **정보의 단순 나열 금지:** 팩트를 나열할 때 문장 간 인과·대비·스케일 확대 등 흐름 연결이 있어야 한다. 연결 없이 독립된 사실을 순서대로 늘어놓지 않는다. 단, 권위 구축 밀집 나열(§PART A-2)은 제외한다.
- **썸네일 문구 직접 인용 금지:** 썸네일 약속(L1)은 직접 언급·반복하지 않고 간접 확인(수치·동의어·암시)으로만 연결한다 (공통 §0.4 "자발적 발견" + §C.4 "간접 확인" 참조).
- **시청자 반박 선제 흡수:** SL 1~2개마다 1회 시청자 반박을 선제 흡수한다 (공통 §C.2 패턴·예시 참조).
- **ML 해소 완료 위치:** ML은 반드시 본문 마지막 SL에서 완료 후 결말 진입한다. 결말에서 ML을 해소하지 않는다.

## 할루시네이션 방지 규칙

- **지침 예시 문구 복제 금지:** 지침의 모든 예시는 원리 파악용. 실제 집필 시 해당 구간의 맥락에서만 가능한 표현을 새로 생성한다.
- **화자가 자기 정보에 놀라는 척 금지** — 감탄/감상을 제거했을 때 정보나 대비가 남지 않는 문장은 제거. 대비를 명시하는 문장으로 교체한다.
- **비유의 메커니즘 일치 확인:** 표면적 유사성이 아니라 작동 원리가 일치하는지 검증
- **수정 후 정합성 확인 (일반):** 문장 수정 시 같은 단락 전체를 다시 읽는다
- **수정 후 정합성 확인 (구조 요소):** ML·피벗·해소 비유를 수정할 경우 해당 요소를 참조하는 모든 문장을 검색하여 재검증한다
- **수치 최신성 확인:** 현재 시점 기준 최신인지, 주체/범위/조건이 보존되었는지 확인
- **집필 중 추가 팩트 출처 기록:** 팩트리포트에 없는 정보는 반드시 웹 검색 확인 후 상태 태그 "추가 팩트"에 내용+출처 기록. 미확인 정보 단정형 서술 금지

## PART A: 13개 언어 장치

### 1. 의인화
국가/물질/체내 반응을 캐릭터처럼. 직후 객관적 사실 보완. 최소 1회 적용 (주제상 불가능한 경우 사유 명시).

### 2. 수치 체감 치환
**치환하는 경우:** 면적/거리 큰 수, 전문 단위, 비교 없이 판단 불가한 수치, 높이/길이의 공간적 크기

**치환하지 않는 경우:** 극단적 충격 수치("사망률 90%"), 연도/날짜/시각, 이미 직관적인 비율

**권위 구축 밀집 나열:** 원본 수치·날짜·인명·고유명사를 밀집 나열하여 신뢰감 부여. 이 경우 치환하면 오히려 권위가 약해진다.

### 3. 전문용어 치환
전문용어를 쉬운 말로 풀어 말하듯 서술한다. 용어가 용어 형태로 남지 않고 문장 속에 녹아야 한다. 외국어·학술 용어·업계 용어는 한국어 뜻을 먼저 제시한 뒤 원어를 병기한다.

- **예:** "소금 전매제" → "소금은 나라만 독점으로 팔 수 있었거든요"
- **예외:** 권위 구축 밀집 나열 — 원본 용어 유지

### 4. 반전 질문 → 즉시 해소
레이어 전환 구간과 정보 밀도 피크 직후에 자연 배치한다. 유형은 되묻기/확인사격/시청자대입/침묵여백/뒤집기후재강화/유머전환 중 맥락에 맞게 선택한다.

### 5. 구어체 리듬
- **확인 사격:** 긴 설명 직후, 핵심 한 문장으로 각인 ("뭐다? 감자다")
- **한 줄 요약:** 복잡한 설명 후, 체감 문장으로 압축
- **감탄사 리셋:** 정보 밀도 높아지기 직전 삽입
- 30초 이상 긴 문장 연속 시 반드시 짧은 확인 사격 삽입

### 6. 시청자 대입
편당 최소 1회. 주제와 시청자가 자연스럽게 연결되는 구간에 배치. 방식: 질문형·상황 제시형·경험 소환형·1인칭 경험 공유.

### 7. 속어·구어 감탄/밈
**6~10회.** 정보 피크 직후. 사물/상황만 대상. 무거운 구간 자제. 도입부 초반 사용 금지(첫인상이 가벼워져 이탈률 상승). 같은 표현 반복 금지. 해당 팩트에만 쓸 수 있는 맥락 특이적 표현 사용. **욕설 금지.** "미쳤다", "대박", "걸작", "레전드" 등 구어 감탄 수준만 허용.

### 8. 현재형 서술

| 단계 | 시제 | 전환 마커 |
|---|---|---|
| 배경 설명 | 과거형 | — |
| 현재 상황 | 현재형 | "현재는", "지금은", "지금도" |
| 가상 시나리오 | 현재형 가정 | "만약에", "~하면", "~해 버리면" |

### 9. 대조 쌍
"A가 아니라 B다"가 "A다"보다 항상 강하다. 교차 대조(Chiasmus) A-B-B-A.

### 10. 엔딩 감정 전환
PART D 엔딩 유형 참조.

### 11. 반사실 가정
"~이 없었다면". 1~3회. 추상적 가치를 구체적 부재로 체감시킬 때. 구문을 매번 변형한다.

### 12. 화자 감정 삽입
1~2회. 1인칭 감정. 시사/시의성 주제에서 빈도 높음. 3회 이상이면 "의견 채널"로 분류됨. 진입 구문 매번 변형.

### 13. "이미 이렇게 돼 있다" 현실 폭로
일상 속 대상의 숨겨진 사실. 모든 주제에 적용 불가.

### 이완 장치 강제 규칙
- **팩트 3개+ 연속 후 반드시 이완 1회** — 인지 부하 누적 방지
- **2분 이상 이완 없으면 수정 대상**
- 이완 장치: 속어·구어 감탄/밈, 자조적 한 줄, 시청자 공모형 유머, 감탄사+짧은 판단
- **무거운 구간:** 자조적 한 줄 / 감탄사+짧은 판단 / 화자 감정 삽입으로 대체

### 정보 밀도 파동 사이클
- 60~90초 단위 반복 / 15분 기준 10~12회

### 정보 분류별 서술 규칙
- **사실:** 단정형 사용 가능
- **추정/해석/논쟁:** 단정형 금지. 불확실성을 드러내는 표현 필수

### B-0. 도입부 집필 전 사전 선언 (필수)

도입부를 집필하기 **전에** 아래 선언을 먼저 작성한다. 선언 없이 도입부 산문부터 쓰지 않는다.

- 서사유형 = 공통원칙 §0.5 + 유형 파일

**선언 효과 (각 항목별):**

- **서사유형을 선언**하면 → 유형별 규칙 적용 이탈 방지
- 전략 코드를 선언하면 → 전략 이탈 방지
- 도입부 이문호공 요인을 미리 지정하면 → 요인 수 미달 방지

- 팩트리포트 원본 수치를 그대로 복사하면 → 팩트 열화 방지
- **근거 팩트를 명시**하면 → 팩트 나열 후 마지막 팩트를 자동으로 받는 관성 차단, 논리 비약 방지
- ML 형태를 미리 정하면 → 유형별 허용 형태 이탈 방지

### B-1. 도입부 공통 구조

> **분류:** 단일 지침 정의 있음 (공통 §0.6 + §B.4 + §B.5 + P3 §3 + 02 유형 파일 P4 §2) — 절충 금지 원칙으로 단일 지침 정의 그대로 인용. 신뢰 상환 운영 안내는 P4 자체 운영 절차로 보존.

**도입부 역할 (공통 §0.6 인용):**

도입부는 썸네일이 예고한 최종 통찰(L3)을 암시·직접 인용하지 않고, 최종 통찰(L3)이 왜 의외인지의 전제 조건을 세팅한다. 신뢰 상환 전략에 따라 도입부의 이원 기능이 달라진다 (공통 §0.6 표 참조).

**도입부 3구성 (공통 §B.4 인용):**

도입부는 인트로(블록) + ML(블록) + 신뢰 상환(기능)으로 구성된다. 각 구성의 구체 구현(진입 단계 수·내용·신뢰 상환 위치)은 유형 파일 P3 §1에서 정의된다.

| 구성 | 분류 | 공통 원칙 |
|---|---|---|
| 인트로 | 블록 | 썸네일 약속 재확인 + 진입 전략 A~H 선택 (도입부 첫 단계, 유형별 1~3개 단계로 분기 — P3 §1) |
| ML (메인 루프) | 블록 | 제목·썸네일의 약속을 질문·선언으로 문장화 (도입부 말미) |
| 신뢰 상환 | 기능 | 시청자에게 "영상이 약속을 지킬 것"이라는 신호 제공 |

**신뢰 상환 위치 (P3 §3 그대로 인용):**

> - 모순형: 신뢰 상환 본문 이연. 도입부에서는 "썸네일 약속 재확인"(동의어 치환)만 수행. 역설 증명은 본문 A극·B극에서.
> - 이미지 강화형·과정 추적형: 신뢰 상환 도입부 수행. 단 일부 주제(22 온돌·13 호르무즈)는 본문 이연.
> 본문 이연 시 상환 위치: 구조 변주에 따라 다르다(본문 10%~68%). `09_신뢰_상환_위치_분석.md` 참조.

25편 실측 분포·예외 편은 §B-2 표 + 09 분석 파일 참조.

**ML 정의 (공통 §B.5 그대로 인용):**

ML은 도입부 말미에서 본문이 답할 핵심 질문을 공식화하는 단위이다. 서사 유형에 따라 변주하며, 유형별 구체 구조는 유형 파일 P3 ML 섹션 참조. 이문호공 3요인 이상 권장. 자연스러운 궁금증 유발이 요인 카운팅보다 우선 (공통 §B.5). 9요인 정의는 `00_지식파일/03 이문호공.md` 참조.

#### 신뢰 상환 운영 안내 — 도입부 수행 시 (P4 자체 운영 절차)

> **분류:** P4 자체 운영 절차 (단일 지침에 운영 안내 없음 — 가치 있는 절차로 P4 보존). 신뢰 상환을 도입부에서 수행하는 경우에 적용. 모순형 본문 이연 시에는 본문 영역 (PART C — 특히 §C-4 서사 유형별 본문 구조) 참조.

- 신뢰 상환은 썸네일이 약속한 내용을 실제 팩트로 증명한다. 팩트 나열이 아니라 팩트의 체감 시간을 늘린다.
- **신뢰 상환 ≠ ML:** 신뢰 상환은 "이미 일어난 일/현재 상태"를 팩트로 보여주는 것, ML은 "왜/어떻게"를 묻는 질문. 같은 문장에 섞지 않는다.
- **필수:** 팩트 후 소화 문장 삽입 (단, 팩트 자체가 충분히 직관적이면 생략 가능). "소화 문장" 정의 = `00_용어정리.md` 참조.
- **필수:** 소화 문장은 직전 팩트가 아니라 최종 통찰(L3)을 가장 잘 지지하는 팩트를 근거로 삼는다 (관성 차단 + 논리 비약 방지).
- **필수:** 숫자/전문 개념은 일상 언어로 풀어서 서술한다. 이미지강화형 수치 선택 기준은 02 유형 파일 P4 §2 참조 (압도적·체감 가능·1개로 충분).

**비유 vs 은유:**

- 비유(직유): "~처럼" → 이해를 돕는다
- 은유: "A는 B다" → 뇌에 새 개념을 박는다 (더 강력)

**비유/은유 영역 선정 원칙:**

- 주제의 핵심 메커니즘에서 꺼낸다. 외부 영역을 억지로 끼워 맞추지 않는다
- 같은 섹션(SL) 내에서 영역을 일관 유지한다
- 무거운 구간(전쟁·재난·빈곤)에서는 최소화한다 — 톤 미스매치가 몰입을 깨뜨림
- **생성 순서:** 사건을 비유 없이 한 문장으로 쓴 뒤 → 핵심 메커니즘 추출 → 같은 메커니즘을 가진 일상 이미지 탐색

**해소 비유 vs 설명용 비유:**

| 구분 | 해소 비유 | 설명용 비유 |
|---|---|---|
| 목적 | 각인 (정체 정의) | 이해 (내용 해설) |
| 위치 | SL 해소 | 본문 중간 |

## PART D: 엔딩 집필

### D-0. 엔딩 집필 전 사전 선언 (필수)

> **분류:** P4 자체 운영 절차 (단일 지침에 운영 안내 없음 — 가치 있는 절차로 P4 보존). 양식 안 각 항목은 단일 지침 정의 참조.

```text

```

**참조 — 양식 안 각 항목 정의 위치:**

- 서사유형 = 공통 §0.5 + 유형 파일
- 엔딩유형 (8유형 + 유형별 궁합) = 공통 §B.6 + 유형 파일 P3 엔딩 섹션
- 쿨다운 착지 = P3 조감도 (산출물 — 착지 문구)
- 도입부 소재 회수 (수미상관) = §D-1 결론부 쿨다운 규칙

**선언 효과:**

- **서사유형을 선언**하면 → 유형별 엔딩 궁합 자동 매칭 (잘못된 유형 선택 방지)
- **엔딩유형을 선언**하면 → 공통 §B.6 8유형 중 선택 명확화 (혼합형도 주1+보조1 명시)

- **도입부 소재 회수 사전 선언**하면 → 수미상관 효과 누락 방지

### D-1. 결론부 쿨다운 규칙 (공통)

**3단계 구조:**
1. **정리:** 본문 인과 체인을 한 호흡으로 압축 (새 정보 없이 요약만)
2. **착지:** 아래 엔딩 유형 중 선택
3. **시그니처 아웃트로:** "이상 이거봐봐 지식해적단이었습니다"

**규칙:**
- **ML 해소는 마지막 SL(본문)에서 완료한다.** 결론부는 착지만 수행
- 결론부에 신규 정보 추가 금지 — "끝났는데 왜 안 끝나지" 느낌 → 마지막 인상 저하
- "교훈"이나 "경고"로 끝내지 말 것. 경고가 필요하면 위트로 감싸서 완화
- **도입부 소재 회수(수미상관):** 도입부 드립·소재를 결말에서 재활용. 가능하면 반드시 사용

---

### D-2. 엔딩 8유형 + 유형별 자연 궁합

> **본격 정의는 `00_공통_원칙.md` §B.6 참조.** P4는 집필 실행 시 사용 안내만 표기.

| 유형 | 설명 | 유형별 자연 궁합 |
|---|---|---|
| ① 일상행위연결 | "오늘 해 먹어 보세요", "다음에 가보세요" | 이미지강화형·과정추적형 |
| ② 감정적여운형 | "억울했던 마음도 같이 생각해 보면" | **과정추적형 최다(25%)** |
| ③ 미래희망형 | "통일이 돼서 서울이 후방이 되는 날을" | **이미지강화형 최다(60%)** |
| ④ 시의성마무리형 | 현재 이슈와 연결하여 착지 | 과정추적형·이미지강화형 |
| ⑤ 위트반전형 | 도입부 드립 회수 + 유머로 착지 | **모순형 선호** |
| ⑥ 통찰형 | 본문 전체를 관통하는 한 문장 통찰 (예: 10 종교와 고기) | 모순 2·이미지 3·과정 1편 |
| ⑦ 경고형 | 대상의 한계·문제·위험 경고로 착지 (예: 13 호르무즈) | 모순 2·이미지 1·과정 1편 |
| ⑧ 광고·협찬형 | 협찬·제품 홍보와 결합한 착지 (예: 24 전기차) | 모순 1·이미지 1·과정 1편 |

> **혼합형이 표준이다:** 단일 유형이 아닌 "주1+보조1" 조합 사용. 예) 미래희망(주)+위트(보조).

### D-3. 서사 유형별 엔딩 패턴

#### 모순형
- **선호:** 위트반전형(⑤) + 도입부 소재 회수. 경고 톤이 필요할 때 위트로 감싸서 완화
- **결말 길이:** 공백포함 평균 ~243자

#### 이미지강화형
- **선호:** 미래희망형(③) — 6/10(60%). 대상의 파급력이 "앞으로도 계속된다"로 착지하는 것이 가장 자연스러움
- **결론부 문제 제기 허용:** 스케일 극대화 후 마지막에 한계·부작용을 짧게 제시 → 미래희망 착지. 이미지강화형 고유 패턴
- **결말 길이:** 공백포함 평균 ~236자

#### 과정추적형
- **선호:** 감정적여운형(②). 역사적 사건의 감정적 무게가 자연스러운 여운을 만든다
- **고유 패턴:**
  - **후일담:** "사건 이후 어떻게 됐나" — 신규 정보가 아닌 ML 해소의 자연스러운 연장으로 허용
- **결말 길이:** 공백포함 평균 ~152자 (3유형 중 가장 짧음)

## GATE 4 직전 팩트 검증 (필수)

GATE 4 진입 전, 대본 전문의 수치·고유명사·날짜를 팩트리포트와 대조한다.

1. 대본에서 모든 수치·고유명사·날짜를 추출한다
2. 팩트리포트 원본과 기계적으로 대조한다
3. 불일치 항목 출력:
   - **[열화]** 대본 표현 → 팩트리포트 원본
   - **[미수록]** 대본 표현 → 팩트리포트에 없음. 출처 확인 필요
4. [열화] 즉시 수정 / [미수록] 웹 검색 확인, 불가 시 삭제 또는 [추정] 전환
5. 창작 장면·상황 묘사에 상식 오류가 없는지 검증

## GATE 4 체크리스트

- [ ] 대본 전문이 완성되었는가?
- [ ] 영상 목표 길이가 10~20분인가?
- [ ] 도입부(1분~2분 30초, 약 300~750자 공백 포함) 안에 ML이 열려 있는가? (과정추적형은 750자 초과 허용)
- [ ] 서사 유형에 맞는 도입부 구조가 적용되었는가? (B-2 기준)
- [ ] SL 열림 표현 형태(공통 §C.9 부속 ①, 형A~형E)가 교차 배치되었는가? (형0 자유 단정 진입 제외 5유형 중 최소 3유형 사용, 같은 유형 연속 2회 초과 금지)
- [ ] 2분+ SL 없는 구간이 없는가?
- [ ] 해소 비유(강) + 뒷받침(약)이 최소 3세트 있는가?
- [ ] [A-선언] 선언·예고형 9종 ("진짜는 지금부터"·"결정적 사건"·"양반이었다" 등) 중 최소 1회 이상 사용되었는가? (구 심화 브릿지)
- [ ] **[이미지강화형]** Scale Jump가 2회 이상 설계되었는가? 긍정 피로 방지 장치가 있는가?
- [ ] **[과정추적형]** 전환점이 3회 이상인가? 감정이입 장치가 있는가? 복선이 최소 1개 설계되었는가?
- [ ] **[모순형]** 피벗 브릿지 유형이 명확하게 작동하는가?
- [ ] 반전 질문이 레이어 전환 구간 또는 정보 밀도 피크 직후에 배치되었는가?
- [ ] 속어·구어 감탄이 6~10회 있는가?
- [ ] 시청자 대입이 최소 1회 있는가?
- [ ] 2분 이상 이완 장치 없는 구간이 없는가?
- [ ] [추정]/[해석] 마킹 팩트가 단정형으로 서술되지 않았는가?
- [ ] 팩트리포트에 없는 정보가 출처 미확인 상태로 삽입되지 않았는가?
- [ ] 결론부에 ML 해소나 신규 정보가 없는가? (과정추적형의 후일담·시의성 맥락은 허용)
- [ ] 최종 통찰(L3)이 복합 원인 중 하나를 유일 원인으로 단정하지 않았는가?
- [ ] 의인화 장치가 최소 1회 적용되었는가?
- [ ] 엔딩 유형이 서사 유형과 자연 궁합을 이루는가? (D-3 기준)
- [ ] 공유하고 싶은 한 줄이 엔딩에 있는가? (모순형 필수 확인)
