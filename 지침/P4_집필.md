## 공통 집필 원칙

- **정보의 단순 나열 금지:** 팩트 나열 시 문장 간 흐름 연결(인과·대비·스케일)이 있어야 한다. 자가점검 — "직전 문장과 본 문장 사이에 흐름 연결이 있는가? 같은 행위의 분할 묘사인가?" Yes면 연결어로 통합 또는 흐름 문장 삽입. 단, 권위 구축 밀집 나열(§PART A-2)이 **효과 기준 두 조건(맥락 인지 + 임팩트 발생)** 충족 시 예외.
- **인과 마커 오용 금지:** A→B 순서 나열 시 자가점검 — "A가 실제로 B의 원인인가, A 이후에 일어난 것뿐인가?" 원인 아니면 인과 마커 금지. 단순 시간 순서는 "이후"·"그 뒤에"로, 복잡한 인과는 "우여곡절 끝에"·"여러 협상 끝에" 같은 압축 브릿지로 대체.
- **새 정보 등장 시 셋업 선행:** 새 정보(수치·고유명사·국가·기관·상태 표현(부족·심각·만성 등))가 처음 등장할 때, 직전 1~2문장에 그 정보의 등장 배경을 깔아둔다. 자가점검 — "처음 읽는 사람이 '갑자기 X가 왜 Y하지?'라 의문을 가질 수 있는가? Yes면 셋업 누락" (P5 §D D12 사후 검출 정합).
- **낭독 적합 표기 원칙:** 본문은 입으로 읽히는 글. 시각 전용 부호는 사용 금지 — `+` (플러스 나열) → 조사 누적 ("A에 B에 C에"), `—` 줄표 삽입구 → 별도 문장으로 풀어쓰기, `→` `※` `/` (슬래시 나열) → 한국어 풀어쓰기. 단, 마침표·쉼표·물음표·따옴표·가운뎃점(·)은 허용 (낭독 시 자연스러움). 자가점검 — "이 문장을 입으로 읽을 때 부호를 어떻게 발음할지 모호한가? Yes면 위반"
- **썸네일 문구 직접 인용 금지:** 썸네일 약속(L1)은 직접 언급·반복하지 않고 간접 확인(수치·동의어·암시)으로만 연결한다.
- **시청자 반박 선제 흡수:** SL 1~2개마다 1회 시청자 반박을 선제 흡수한다 (공통 §C.2 패턴·예시 참조).
- **ML 해소 완료 위치:** ML은 본문 마지막 SL에서 완료 후 결말 진입한다. ML 위치 본체 규칙과 과정추적형 "결말 40% 역전" 예외는 **§D-1 결론부 쿨다운 규칙** 참조.

### 1. 의인화

국가/물질/체내 반응을 캐릭터처럼. 직후 객관적 사실 보완. 최소 1회 적용 (주제상 불가능한 경우 사유 명시).

### 2. 수치 체감 치환

**치환하는 경우:** 면적/거리 큰 수, 전문 단위, 비교 없이 판단 불가한 수치, 높이/길이의 공간적 크기

**치환하지 않는 경우:** 극단적 충격 수치("사망률 90%"), 이미 직관적인 비율. 단발 사건의 월·일은 사건 명칭으로 압축 (예: "1940년 6월 25일, 프랑스가 항복한 그날" → "프랑스가 항복한 바로 그날"). **연도·시각은 익숙한 표현 가능 시 익숙한 표현 우선** (예: "1940년 6월" → "2차 대전 때 프랑스 항복 직후"). 익숙한 표현 부재 시 원본 유지. 권위 구축 밀집 나열은 **효과 기준 두 조건(맥락 인지 + 임팩트 발생) 충족 시** 예외.

**권위 구축 밀집 나열 (효과 기준):** 원본 수치·날짜·인명·고유명사·연표·기관명을 밀집 나열하여 신뢰감 부여. **적용 조건 (둘 다 충족):** ① 맥락 인지 — 직전 단락에서 본 나열의 맥락(주제·영역·관계)이 시청자에게 셋업 / ② 효과 발생 — 처음 듣는 시청자 시각에서 권위·신뢰 임팩트 발생, "왜 이게 나오지?" 혼란 미발생. **한 조건이라도 미충족 시 적용 외 → §공통 셋업 의무 적용.** **자가점검:** "직전 단락 정보만 가지고 시청자가 이 나열을 받았을 때 임팩트가 발생하는가, 혼란이 발생하는가?" (영역 — 학술/일반 — 무관, 효과 기준). **마지막 항목은 흐름·톤에 맞게 자연스럽게 마무리한다.**

#### 가격·요금·비율 수치 한정 동반 의무

다음 3종 수치는 평소 시세·변화율·익숙한 비교 대상 중 1개 동반 의무. 시청자가 시세 정보 없이는 가치 판단 불가하기 때문이다.

| 강제 대상 (3종) | 예시 |
| --- | --- |
| 가격·요금 | 전용기 차터 70만 달러 / 아파트 가격 / 항공권 가격 |
| 비율·변동률 | GDP 1% 미만 / 부동산 5.9% 하락 / 점유율 |
| 외국 통화 단독 | $·€·¥ 단독 표시 |

**위반 → 수정 예시:** "전용기 차터 70만 달러" 단독 → "평소 25만 달러였던 차터가 2주 만에 70만 달러까지 폭등" (변화율 동반)

**강제 제외 (단위 자체 직관):**

- 인구·인원 (백만장자 81,200명·관중 N명)
- 물리량 (미사일 438발·드론 2,012대·기업 8,700개)
- 시간 (40년·2주)
- 백분율 충격 수치 (사망률 90%)

### 3. 전문용어 치환

전문용어를 쉬운 말로 풀어 말하듯 서술한다. 풀이 없이 용어 단독 사용 금지. 풀이 방식·자가 진단은 `04_언어_장치.md` §어려운 용어 풀이 참조.

- **예:** "소금 전매제" → "소금은 나라만 독점으로 팔 수 있었거든요"
- **예외:** 권위 구축 밀집 나열 — **효과 기준 두 조건 충족 시** 원본 용어 유지

### 4. 반전 질문 → 즉시 해소

레이어 전환 구간과 정보 밀도 피크 직후에 자연 배치한다.

**공통 4종** (되묻기·확인사격·시청자대입·뒤집기후재강화) — 3유형 모두 사용 / **자문자답 압축형** — 비-모순형(이미지강화·과정추적) 한정, 모순형 차용 금지. 04 §4 참조.

반전 질문 빈도: 모순형 편당 10~18회 (A극 SL당 0~1회 / 단순 사실 나열 SL은 0회 정상 / B극 SL당 1~2회) / 이미지강화 5회+ / 과정추적 1회 이상 (전환점 하나당 최대 1회).

**같은 SL 안에서 같은 형태 연속 사용 금지** — 다른 유형 중 선택하여 변주.

### 5. 구어체 리듬

- **확인 사격:** 긴 설명 직후, 핵심 한 문장으로 각인 ("뭐다? 감자다")
- **한 줄 요약:** 복잡한 설명 후, 체감 문장으로 압축
- **감탄사 리셋:** 정보 밀도 높아지기 직전 삽입

### 6. 시청자 대입

편당 최소 1회. 주제와 시청자가 자연스럽게 연결되는 구간에 배치. 방식: 질문형·상황 제시형·경험 소환형·1인칭 경험 공유.

### 7. 엔딩 감정 전환 **(P5 §A A17·A18 정합)**
**혼합형 권장** — 8유형 중 1~2개 조합 (공통 §B.6 정합 — 단일 매핑보다 혼합 권장). 예: 깨달음+여운·기대+유머·경고+안도. PART D 엔딩 유형 참조.

### 8. 반사실 가정
"~이 없었다면" 변형 — 추상적 가치를 구체적 부재로 체감. 구문을 매번 변형한다.

**유형별 분기:**

- **모순형: 0~1회** (사실상 부재 — 모순형은 "현재 폭로" SL 골격으로 가정 표현 회피)
- 이미지강화: 1~3회
- 과정추적: 0~2회

**가정법 시뮬레이션 ("만약 ~한다면")**: 이미지+과정 공유 도구 — 04 사전 §5 참조. 추상 제도/정책·미래 시뮬레이션 주제 강함.

### 9. 화자 감정 삽입
1~2회. 1인칭 감정. **사용 위치:** 정보 피크 직후·놀랄만한 정보 직후 (정보 피크 직전·SL 진입 위치 사용 금지). 시사/시의성 주제에서 빈도 높음. 3회 이상이면 "의견 채널"로 분류됨. 진입 구문 매번 변형.

### 10. "이미 이렇게 돼 있다" 현실 폭로
일상 속 대상의 숨겨진 사실. 모든 주제에 적용 불가.

**마커 우선순위:**

- **1순위:** "사실(상)"·"실제로" (편당 8~10회 합산)
- 2순위: "이미" (변화 강조 시 보조)
- **❌ "알고 보면"·"실은"** (사용 금지)

과정추적 한정 추가 마커: **상태 변화 폭로 마커** ("이제서야 깨닫게 됐죠"·"마침내 멸망"·"변모해 지금까지") — 04 사전 §6-3 참조.

### 11. 지시어 명확성

지시 대명사(이것·저것·그것·여기·거기·저기·이때·그때) 사용 시 지시 대상이 직전 1~2 문장 안에 명확히 등장해야 한다. 음성 매체(낭독 영상) 특성상 시청자가 앞 문장 다시 못 봄 → 모호한 지시어는 즉시 이탈 유발.

**위반 사례 (두바이 대본):**

- "두바이의 약점은 사실 1985년부터 거기 있었어요" — '거기'가 가리키는 대상 모호 (외국 자본 의존 구조? 사막? 자유무역지구?)

**수정 사례:**

- "두바이의 약점은 그 구조 안에 있었어요. 외국 자본 100% 의존이라는 구조 말이죠."

**점검 방법:**

1. 지시어 사용 시 가리키는 명사가 직전 1~2 문장 안에 있는지 확인
2. 없으면 명사로 풀어쓰기
3. P5 §C C8 자체 퇴고 + P5_퇴고검증 에이전트로 사후 검증

### B-1. 도입부 공통 구조

**도입부 역할 (공통 §0.6 인용):**

도입부는 썸네일이 예고한 최종 통찰(L3)을 암시·직접 인용하지 않고, 최종 통찰(L3)이 왜 의외인지의 전제 조건을 세팅한다. 신뢰 상환 전략에 따라 도입부의 이원 기능이 달라진다 (공통 §0.6 표 참조).

**도입부 3구성 (공통 §B.4 인용):**

도입부는 인트로(블록) + ML(블록) + 신뢰 상환(기능)으로 구성된다. 각 구성의 구체 구현(진입 단계 수·내용·신뢰 상환 위치)은 유형 파일 P3 §1에서 정의된다.

| 구성 | 분류 | 공통 원칙 |
|---|---|---|
| 인트로 | 블록 | 썸네일 약속 재확인 + 진입 전략 A~H 선택 (도입부 첫 단계, 유형별 1~3개 단계로 분기 — P3 §1) |
| ML (메인 루프) | 블록 | 제목·썸네일의 약속을 질문·선언으로 문장화 (도입부 말미) |
| 신뢰 상환 | 기능 | 시청자에게 "영상이 약속을 지킬 것"이라는 신호 제공 |

**신뢰 상환 위치 (P3 §3 그대로 인용):**

> - 모순형: 신뢰 상환 본문 이연. 도입부에서는 "썸네일 약속 재확인"(동의어 치환)만 수행. 역설 증명은 본문 A극·B극에서.
> - 이미지강화형·과정추적형: 신뢰 상환 도입부 수행. 단 일부 주제(22 온돌·13 호르무즈)는 본문 이연.
> 본문 이연 시 상환 위치는 주제·서사 구조·정보 배치 전략에 따라 결정 (강제 기준 없음 — 유형 파일 P3 §1 참조).

**ML 정의 (공통 §B.5 그대로 인용):**

ML은 도입부 말미에서 본문이 답할 핵심 질문을 공식화하는 단위이다. 서사 유형에 따라 변주하며, 유형별 구체 구조는 유형 파일 P3 ML 섹션 참조. 이문호공 3요인 이상 권장. 자연스러운 궁금증 유발이 요인 카운팅보다 우선 (공통 §B.5). 13요인 (9 기본 + 4 확장) 정의는 `00_지식파일/03 이문호공.md` 참조.

#### 신뢰 상환 운영 안내 — 도입부 수행 시

> 신뢰 상환을 도입부에서 수행하는 경우에 적용. 모순형 본문 이연 시에는 본문 영역 (PART C — 특히 §C-4 서사 유형별 본문 구조) 참조.

- 신뢰 상환은 썸네일이 약속한 내용을 실제 팩트로 증명한다. 팩트 나열이 아니라 팩트의 체감 시간을 늘린다.
- **신뢰 상환 ≠ ML:** 신뢰 상환은 "이미 일어난 일/현재 상태"를 팩트로 보여주는 것, ML은 "왜/어떻게"를 묻는 질문. 같은 문장에 섞지 않는다.
- **필수:** 팩트 후 소화 문장 삽입 (단, 팩트 자체가 충분히 직관적이면 생략 가능). "소화 문장" 정의 = `00_용어정리.md` 참조.
- **필수:** 소화 문장은 직전 팩트가 아니라 최종 통찰(L3)을 가장 잘 지지하는 팩트를 근거로 삼는다 (관성 차단 + 논리 비약 방지).
- **필수:** 숫자/전문 개념은 일상 언어로 풀어서 서술한다. 이미지강화형 수치 선택 기준은 02 유형 파일 P4 §2 참조 (압도적·체감 가능·1개로 충분).

> - **생애주기 4단계** (열기·유지·강화·닫기) + **서사 유형별 ML 4단계 구현 표** (유지·강화·닫기 단계별 모순형·이미지강화형·과정추적형 차이) = 공통 §B.7
> - **브릿지 본체 18종** ([A-질문] 6종·[A-선언] 9종·[B] 3종) + **부속 ①·②·③** (SL 열림 표현 5유형·SL/BR 어법 4종·유형별 본문 활성화 지침) = 공통 §C.9
> - **루프 중첩 원칙** (1개 유지·순차 교체·3개 이상 절대 불가) = 공통 §B.8
> - **3분할 비율** (Type A 75~90% / 질문형 45~60% / 선언·예고형 20~35% / Type B 10~25%) = 공통 §B.9

#### 핵심 식별

**오픈루프 4단계 (공통 §B.7 인용):**

| 단계 | 위치 | 작동 방식 |
|---|---|---|
| 열기 (Opening) | 도입부 ML | 질문/예기치 못한 상황 → 인지적 공백 생성 |
| 유지 (Holding) | 본문 전반 | 답변 보류 + 기초 맥락 + 서브 오픈루프 6~10개 연쇄 배치 |
| 강화 (Reinforcing) | 본론 중간 | 결정적 단서 / 정보 필요성 재상기 |
| 닫기 (Closing) | 본문 후반 | 궁금증 해소 → 지적 카타르시스 |

**브릿지 유형 (공통 §C.9 인용 + §B.9 비율):**

| 유형 | 정의 | 비율 (공통 §B.9) |
|---|---|---|
| [A-질문] 서브 오픈루프 — 질문형 (6종) | "다 알았다"는 순간 차단 | 45~60% |
| [A-선언] 서브 오픈루프 — 선언·예고형 (9종) | 확언으로 새 SL 진입 | 20~35% |
| [B] 논리적 브릿지 (3종) | 정보 정리 + 감정 매듭 (선형·인과·철학) | 10~25% (Type B 전체) |

> **[A-선언] 필수 적용:** 새 SL을 여는 순간 동반 사용되는 선언·예고 문법 (단독 사용 0건). 9종 패턴 + 부속 코드 매핑(S1~S9) = 공통 §C.9 [A-선언] 참조.

### C-1. 섹션 집필 전 사전 선언 (필수)

각 SL을 집필하기 **전에** 아래 한 줄을 먼저 적는다. 집필 후 제거해도 되지만, 선언 없이 본문부터 쓰지 않는다.

```text
[SL번호
- 선행브릿지: {이전 섹션 상태태그의 브릿지 문구 복사 → 실제 집필할 브릿지 문장}
- SL열림유형: {형A~형E}
- 해소비유: 있음|없음(팩트임팩트형) — 핵심 대상 한 줄 — 이문호공 2요인({요인+요인})
- 핵심 팩트: [{마킹}] {팩트리포트 원본 복사}
]
```

**참조 — 양식 안 각 항목 정의 위치:**

- SL번호 = P3 조감도 (확정된 SL 배열)
- 선행브릿지 (브릿지 18종 + 부속 ①·②·③) = 공통 §C.9
- SL열림유형 (형A~형E 5유형) = 공통 §C.9 부속 ①
- 해소비유 (있음/없음 + 공식) = 공통 §C.1
- 이문호공 2요인 = 공통 §B.5 + `00_지식파일/03 이문호공.md` + P1 §8
- 핵심 팩트 + 마킹 ([사실]/[추정]/[해석]/[논쟁]) = P2 팩트 리포트 (`P2_딥리서치.md`)
- 상태태그 = §C-2 섹션 완성 후 필수 이행 (의인화 적용·속어감탄 누적 등 사후 관리)

### C-2. 섹션 완성 후 필수 이행

각 SL 집필이 끝나면 아래 두 가지를 즉시 수행한다.

**1) 기존 검증:** 해소 은유가 없으면 §C-1 사전 선언으로 돌아가 재작성 / SL 열림 유형이 첫 문장과 불일치하면 재작성 / **단락 간 사실 중복 점검:** 직전 단락에서 다룬 사실(인물·기관·동작·정제 내용)이 본 단락에 단어만 다르게 재인용되지 않았는가? 재인용 시 직전 단락과 통합 또는 본 단락에서 결과·재해석으로 변경. (P5 §0 D7 / §C C11 정합)

**2) 상태 태그 작성:**

```text
[상태] SL{N} 완료
- 루프: 열림(유형) / 해소 여부
- 열린 루프 현황: {현재 미해소 루프 전체 나열}
- 브릿지→SL{N+1}: {전환 전략 한 줄}
- 아하 모먼트: {해소 은유 문구} (이문호공 요인)
- 마킹↔서술 일치: {위반 없으면 "OK" / 위반 시 해당 문장 명시}
- 추가 팩트: {팩트리포트에 없는 정보 내용+출처 / 없으면 "없음"}
- 의인화 적용: {대상+표현 / 미적용 시 "없음"}
- 속어감탄 누적: {N}/6~10
```

**참조 — 상태 태그 안 각 항목 정의 위치:**

- 루프 열림 유형 / 해소 여부 = 공통 §B.7 (생애주기 4단계) + §C.9 (브릿지 18종)
- 열린 루프 현황 = 공통 §B.8 (루프 중첩 원칙 — 1개 유지·최대 2개·3개 이상 절대 불가)
- 브릿지→SL{N+1} = 공통 §C.9 (브릿지 본체 18종 + 부속 ①·②·③)
- 아하 모먼트 (해소 은유 + 이문호공 요인) = 공통 §0.5 + §C.1 (해소 은유 공식) + §B.5 (이문호공)
- 마킹↔서술 일치 ([사실]/[추정]/[해석]/[논쟁]) = P2 팩트 리포트 + §A 정보 분류별 서술 규칙
- 추가 팩트 = 공통 §할루시네이션 방지 — P4 집필 단계 "집필 중 추가 팩트 출처 기록"
- 의인화 적용 = §A 12개 언어 장치 1번 의인화
- 속어/밈 누적 = 04 §2 속어/밈 본체 (계층 1 시그니처)

| 엔딩 직전 | 선택 | 선택 | 선택 |

→ 자연 발생 위치에서 이해 격차가 있으면 해소 은유 또는 각인 도구(§C.1 ①~⑥) 중 자연스러운 것 선택. 격차 없으면 사용하지 않는다.

**해소 은유 자격 테스트:** "이 문장을 누군가에게 말한다면 '오 그래?' 반응이 나오는가?" — 나오지 않으면 해소 은유가 아니라 일반 서술이다.

**은유 형태 선택:** → 공통 §C.1 "공식 — 은유/직유 선택 기준" 참조 (은유 A=B / 직유 A는 B처럼). **사용 자율 + 미사용 시 각인 도구 6종 대체** 안내는 §558 "정량 강제 없음 — §C.1 자율" 인용 정합 (중복 제거).

**비유·은유 영역 선정 원칙:**

→ 주제 핵심에서 꺼내기·무거운 구간 최소화 2원칙은 공통 §C.8 §비유·은유 영역 선정 원칙 위임. P4 고유:

- 같은 섹션(SL) 내에서 영역을 일관 유지한다
- **생성 순서:** 사건을 비유 없이 한 문장으로 쓴 뒤 → 핵심 메커니즘 추출 → 같은 메커니즘을 가진 일상 이미지 탐색

**해소 은유 vs 설명용 비유:**

| 구분 | 해소 은유 (§C.1) | 설명용 비유 (§C.7) |
|---|---|---|
| 목적 | 각인 (정체 정의) | 이해 (내용 해설) |
| 위치 | SL 해소 | 본문 중간 |
| 사용 강제 | 자율 (정량 없음). 미사용 시 각인 도구 6종으로 대체 가능 | 이해 격차 있을 때만 사용 |

## PART D: 엔딩 집필

### D-0. 엔딩 집필 전 사전 선언 (필수)

엔딩을 집필하기 **전에** 아래 선언을 먼저 작성한다. 선언 없이 엔딩 산문부터 쓰지 않는다.

```text
[엔딩
- 서사유형: {모순형|이미지강화형|과정추적형}
- 본문 결산 양식: {유형별 4종 중 1종 — 19번 §1-2}
- 결론 브릿지 어법: {5종 중 1~3종 결합 — 19번 §1-1}
- 엔딩유형: {유형명}
- 쿨다운 착지: {조감도 착지 문구 복사}
- 회수 대상: {도입부 소재 1개 또는 본문 비유 1개 (단일 선택, 동시 회수 금지)}
- 톤: {가벼운 연착 / 음모론·경고 강요 금지}
- 시그니처: {본 채널 이름 정확 표기 — CLAUDE.md "채널 이름 명확 분리" 정합}
- 맥락 일관성: {엔딩 각 문장이 직전 문장과 논리 연결되는가}
]
```

**참조 — 양식 안 각 항목 정의 위치:**

- 서사유형 = 공통 §0.5 + 유형 파일
- 본문 결산 양식 = 19번 §1-2 (모순형·이미지강화·과정추적 각 4종 분류)
- 결론 브릿지 어법 = 19번 §1-1 (5종 — ① 결과 호명·② 이완 호명·③ 시간 전환·④ 회수·⑤ 메타)
- 엔딩유형 (8유형 + 유형별 궁합) = 공통 §B.6 + 유형 파일 P3 엔딩 섹션
- 쿨다운 착지 = P3 조감도 (산출물 — 착지 문구)
- 회수 대상 = §D-1 결론부 쿨다운 규칙 "회수 대상 단일화"
- 톤 = 가벼운 연착 (음모론·경고 톤 금지)
- 시그니처 = `CLAUDE.md` "채널 이름 명확 분리" (본 채널 ↔ 참고 채널 혼동 차단)
- 맥락 일관성 = 엔딩 단편 나열 차단

> - 결론부 위치 정의 → `P3_구조설계.md` §12 구조 보고 SL 배열 표 (결론부 — ML 해소)

#### 결론부 3단계 구성 (P4 자체 운영 절차)

| 단계 | 기능 | 단일 지침 위임 |
|---|---|---|
| 1. 정리 | 본문 인과 체인을 한 호흡으로 압축 (요약만, 새 정보 없음) | 공통 §B.7 "루프 닫기" 후반부 위치와 정합 |
| 2. 착지 | 엔딩 8유형 중 1~2개 조합으로 마무리 | 공통 §B.6 + P4 §D-2 (유형별 자연 궁합) |

#### P4 자체 운영 규칙 (단일 지침 외 영역)

- **ML 해소 위치 고정:** ML은 본문 마지막 SL에서 완료하고, 결론부는 착지만 수행 (공통 §B.7 "후반부 닫기" + P3 §12 구조 보고 SL 배열 표 정합)
  - **예외 (과정추적형 고유):** ML Q2가 결말 파트에서 해소되는 "결말 40% 역전" 패턴은 허용 (03 P3 §10 / 12 가야 사례 / 결론부 40% 이내). 본 규칙은 모순형·이미지강화형의 기본 위치를 의미 — 과정추적형은 공통 §B.7 ML 4단계 닫기 표 "결말에서 현재 의미 부여"와 정합
- **신규 정보 추가 금지:** 결론부에 ML 해소에 영향 주는 **새 팩트·새 인과·새 SL** 도입 금지 → "끝났는데 왜 안 끝나지" 인상 차단 → 마지막 인상 저하 방지
  - **예외 (이미지강화형 고유):** 02 P3 §8 "부가 정보 삽입" 패턴은 허용 — ML 해소 무관 "관련 상식"(보너스 정보), 결론부 20% 이내, ML이 본문에서 완전히 해소된 후 (결론부 본체 단락 영역 — §D-1-2-B 규칙 3은 결론부 진입 단락 어법 영역으로 별개)
  - **예외 (⑥ 통찰형):** 본문 관통 통찰 한 문장은 허용 — 새 정보가 아닌 메타 압축 (공통 §B.6 ⑥ 정합)
- **자연 마무리 호흡 (자가점검):** 결론부가 통찰 단정으로 압축되어 갑자기 끝나는 느낌이 들지 않는가? 자가점검 — "시청자가 결론부 마지막 문장 후 자연스러운 호흡 단락에 도착하는가? 단정 직후 영상이 끊기는 느낌인가?" 갑자기 끝나면 호흡 보강. **권위 톤 회피 — 1인칭 동조 톤 유지 (04 §결과 공유 셋업 도구의 주의 사항 정합).** 표현 형태는 자율 (미래 관찰·시청자 동참·시간 여운 등 다양한 형태 가능 — 본 항목은 형태 강제 아님).
- **도입부 소재 회수(수미상관):** 가능 시 도입부 드립·소재를 결말에서 재활용 (공통 §C.1 E형 + §B.6 ⑤ 위트반전과 정합)
- **회수 대상 단일화:** 결론부 회수는 다음 중 **한 가지**만 선택.
  - (a) **도입부 소재 회수 (수미상관)** — 도입부 키워드·인물·드립 1개 변형 회수
  - (b) **본문 비유 회수** — 본문 핵심 비유 1개 변형 회수
  - ❌ **동시 회수 금지** — (a) + (b) 동시 시도 시 통찰 단편 나열 발생
  - **자가 점검:** 엔딩에 회수된 요소가 **2개 이상**이면 통찰 단절 위험 → 1개로 압축
- **본문 결산 양식 마무리:** 본문 마지막 단락(= 결론부 직전 단락)에서 ML Q1·Q2 답을 정리·통찰·부가 정보·시의성 등으로 마무리. 유형별 4종 중 1종 선택 (§D-1-2-B 참조 — 19번 §1-2 정의 위임).
- **결론 브릿지 어법 1~3종 결합:** 본문 → 결론부 진입은 결론 브릿지 어법 5종 (① 결과 호명·② 이완 호명·③ 시간 전환·④ 회수·⑤ 메타) 중 1~3종 결합 사용. 본문 결산 양식과의 자연 연동은 §D-1-2-B 강제 규칙 4건 참조 (19번 §1-1 정의 위임).
- **결론부 본문 헤더·상태 태그 보존:** 헤더는 `## 결론부 ({결론 브릿지 어법} / {엔딩 유형})` 표기 (CLAUDE.md "대본 본문 섹션 헤더 표기 규칙" 정합). 결론부 본문 후 §C-2 SL 상태 태그와 동일 패턴으로 **코드 블록 보존** — 본문 결산 양식·결론 브릿지·엔딩 유형·회수 대상·톤·시그니처 명시 (P5 추출 시 자동 제거 / 시청자 미노출).

#### D-1-2-B. 본문 결산 양식 → 진입 어법 자연 연동 (강제 규칙 4건 + 권장 1건)

본문 결산 양식이 결정되면 다음 강제 규칙에 따라 결론 브릿지 어법을 사용한다.

**[규칙 1] 정리·결산 양식 → ① 결과 호명**

본문 결산 양식이 다음 중 하나일 때 결론 브릿지 어법은 ① 결과 호명을 사용한다.

- 모순형 A. 정리·복귀형
- 과정추적형 B. 현재 의미 부여형

**[규칙 2] 모순형 통찰 압축형 → ② 이완 호명 필수**

모순형 본문 결산 양식이 통찰 압축형(B)일 때, ② 이완 호명을 필수로 사용한다. 직접 ① 결과 호명 점프는 금지 ("성급한 마무리" 차단).

**[규칙 3] 이미지강화 부가 정보형 → ② 이완 호명**

이미지강화형 본문 결산 양식이 부가 정보형(B)일 때, ② 이완 호명을 사용한다 ("여담으로"·"뭐 어찌됐든" 등).

**[규칙 4] 시청자 시점 결합 결산 양식 → ④ 회수**

본문 결산 양식이 다음 중 하나일 때 ④ 회수를 사용한다.

- 모순형 D. 자기언급+갈등형
- 이미지강화형 D. 시의성형
- 과정추적형 A. 후일담형

**[권장 1건]** 이미지강화형에서 시청자 경험 진입형 도입부(B 공감·E 밈진입·F 맥락재설정 ML 외부화)와 본문 통찰형 아하 모먼트를 결정한 경우, 본문 결산 양식은 A 통찰 압축형을 권장한다.

#### D-1-2-C. 결론 브릿지 어법 5종 (옵션 풀)

결론 브릿지 어법 5종은 자유 조합으로 사용한다 (1~3종 결합).

| 어법 | 사용 예시 |
|---|---|
| ① 결과 호명 | "이게 ~한 일", "상황이 좀 달라졌습니다" |
| ② 이완 호명 | "물론 뭐", "아 근데 뭐", "무튼" |
| ③ 시간 전환 | "최근 몇 년간", "그러나 이후" |
| ④ 회수 | "앞서 말한", "영상 도입부에도 말씀드렸듯" |
| ⑤ 메타 | "이 영상 ~"·"선원 여러분 ~" / "등잔이 어둡다"·"공중 보건은 조별 과제" (메타 격언) |

§D-1-2-B 강제 규칙 4건이 적용되지 않는 영역에서는 자유 조합으로 사용한다.

**유형별 결말 어법 참고 가이드 (결론부 본체 단락 어법 — 결론 브릿지 어법과 별도 영역):**

각 유형의 결론부 본체 단락 어법은 다음 빈출 경향을 보인다 (강제 아님 — 변형 영상 자유).

| 유형 | 결말 톤 | 빈출 어법 |
|---|---|---|
| 이미지강화형 | 희망적·미래 지향 | ③ 미래희망 |
| 모순형 | 통찰적·격언 호소 | ⑥ 통찰형 |
| 과정추적형 | 실용적·관여 호소 | ② 행동 권유 |

> **영역 차이 명시:** 본 표의 ① ② ③ ⑥ ⑦ 번호는 결론부 본체 단락 어법 5종 (② 행동 권유·③ 미래희망·④ 의미 부여·⑥ 통찰형·⑦ 경고). §D-1-2-B·C 결론 브릿지 어법 5종(① 결과 호명·② 이완 호명·③ 시간 전환·④ 회수·⑤ 메타)과 다른 영역. 같은 번호도 다른 어법 — 혼동 금지.
>
> **영역 차이 명시:** 결론부 본체 단락 진입 방향(본 표) vs 결론부 마지막 영역 착지 방식(엔딩 8유형, 공통 §B.6) — 다른 영역이므로 혼동 금지.
