## 공통 집필 원칙

- **정보의 단순 나열 금지:** 팩트 나열 시 문장 간 흐름 연결(인과·대비·스케일)이 있어야 한다. 자가점검 — "직전 문장과 본 문장 사이에 흐름 연결이 있는가? 같은 행위의 분할 묘사인가?" Yes면 연결어로 통합 또는 흐름 문장 삽입. 단, 권위 구축 밀집 나열(§PART A-2)이 **효과 기준 두 조건(맥락 인지 + 임팩트 발생)** 충족 시 예외.
- **인과 마커 오용 금지:** A→B 순서 나열 시 자가점검 — "A가 실제로 B의 원인인가, A 이후에 일어난 것뿐인가?" 원인 아니면 인과 마커 금지. 단순 시간 순서는 "이후"·"그 뒤에"로, 복잡한 인과는 "우여곡절 끝에"·"여러 협상 끝에" 같은 압축 브릿지로 대체.
- **새 정보 등장 시 셋업 선행:** 새 정보(수치·고유명사·국가·기관·상태 표현(부족·심각·만성 등))가 처음 등장할 때, 직전 1~2문장에 그 정보의 등장 배경을 깔아둔다. 자가점검 — "처음 읽는 사람이 '갑자기 X가 왜 Y하지?'라 의문을 가질 수 있는가? Yes면 셋업 누락" (P5 §2.3 (새 정보 등장 시 셋업 선행) 사후 검출 정합).
- **낭독 적합 표기 원칙:** 본문은 입으로 읽히는 글. 시각 전용 부호는 사용 금지 — `+` (플러스 나열) → 조사 누적 ("A에 B에 C에"), `—` 줄표 삽입구 → 별도 문장으로 풀어쓰기, `→` `※` `/` (슬래시 나열) → 한국어 풀어쓰기. 단, 마침표·쉼표·물음표·따옴표·가운뎃점(·)은 허용 (낭독 시 자연스러움). 자가점검 — "이 문장을 입으로 읽을 때 부호를 어떻게 발음할지 모호한가? Yes면 위반"
- **썸네일 문구 직접 인용 금지:** 썸네일 약속(L1)은 직접 언급·반복하지 않고 간접 확인(수치·동의어·암시)으로만 연결한다.
- **시청자 반박 선제 흡수:** SL 1~2개마다 1회 시청자 반박을 선제 흡수한다 (모듈_집필기법 §2 (시청자 반박 흡수) 패턴·예시 참조).
- **ML 해소 완료 위치:** ML은 본문 마지막 SL에서 완료 후 결말 진입한다. ML 위치 본체 규칙과 과정추적형 "결말 40% 역전" 예외는 **§D-1 결론부 쿨다운 규칙** 참조.
- **시의성 이슈 활용 시 본문 방향:** 시의성 이슈를 제목·썸네일 후킹에 활용한 경우, 본문은 해당 이슈가 생겨난 배경·원인·작동 원리·구조적 역학 중심으로 구성한다. 현재 진행 중인 사안의 옳고 그름을 판단하거나 특정 입장을 직접 지지하는 방향으로 서술하지 않는다 (P2 §2 수집 방향 연계).

## 본 채널 서사 구조 프레임

> 본 §은 본 채널 3유형(모순형·이미지강화형·과정추적형) 전체에 적용되는 서사 구조 영역. 능동 추리자·L1·L2·L3·아하·단서 수렴 본체는 모듈_레고이론 §0 참조.

### 결론 보류 + 간접 회수

#### ① 결론 보류

최종 통찰은 본문 후반부까지 보류한다. 아하 모먼트 자체는 완결 조건 충족 시 본문 어느 지점에서도 발생 가능하다.

#### ② 간접 회수 (수미상관 구현)

도입부에서 던진 소재·에피소드·비유·진입 장치를 결론부에서 돌려받을 때 명시적 문구 반복이 아닌 동의어 치환·맥락 이동·간접 연결로 처리한다.

- **적용 대상:** 도입부 장치 (썸네일 약속이 아닌 도입부 소재)
- **목적:** 완결감·위트 부여 ("아, 이게 돌아왔네!" 효과)
- **주의:** 본 회수는 도입부 소재 재등장 허용 (방식 제한). 모듈_레고이론 §0.3 자발적 발견과 적용 대상이 다르다 — §0.3은 썸네일 메인 자체 금지, 본 ②는 도입부 소재 재등장 허용
- **회수 처리 방식:** 동의어 치환·맥락 이동·간접 연결로 회수한다. 도입부 소재 그대로 반복은 회피

### 서사 4단계 프레임

| 단계 | 정의 | 집필 지시 |
| --- | --- | --- |
| 1단계 | 약속의 선언과 공백 형성 | 제목+썸네일+ML을 결합하여 시청자에게 의외성을 약속한다 |
| 2단계 | 단서의 연쇄적 수렴 | 서사 유형별 구조에 따라 단서를 연쇄 배치한다 |
| 3단계 | 아하 모먼트와 발견 | 시청자가 스스로 최종 통찰을 발견하는 순간(아하 모먼트)을 설계한다 |
| 4단계 | 가치 확장 (결론부 착지) | 도달한 최종 통찰을 현대 사회·일상·미래·정서·실천 등으로 확장하여 여운을 남긴다 (모듈_본문구조 §2 결론부 8유형 참조) |

**결론부 5단락 영역 (4단계 세부 분할):** 본문 정점 SL → 결론부 직전 단락 → 결론부 진입 단락 → 결론부 본체 단락 → 시그니처. 세부 운영은 §D-1 결론부 쿨다운 규칙에서 정의한다.

### 약속의 완결성 + 신뢰 상환

#### ① 약속의 완결성

썸네일·제목이 제시한 "의외성"은 **2단계로 완결**된다.

1. **신뢰 상환 (1차 단계):** 신뢰 상환 전략(도입부 / 본문 이연)에 따라 도입부 또는 본문 내에서 초기 증거 제시. 시청자 이탈 방지 기능
2. **최종 통찰 도달 (2차 단계):** 본문 후반부에서 완성된 결론. 영상 전체의 지적 전리품

**결론부에서는 신뢰 상환·최종 통찰을 수행하지 않는다.** 결론부는 모듈_본문구조 §2 엔딩 8유형으로 여운 착지한다.

#### ② 신뢰 상환

서사 유형별로 신뢰 상환 위치가 다르다. 세부 위치·이유는 모듈_엔진 해당 엔진 절 참조, 유형별 궁합 표는 모듈_본문구조 §2 참조. 신뢰 상환 전략 결정은 P2 본체에서 정의한다.

### 인과관계의 접착

단서와 단서 사이에 논리적 유격이 없어야 한다. 연결 문장으로 인과를 명시한다.

(인과 마커 사용 시 "공통 집필 원칙 — 인과 마커 오용 금지" 자가점검 동반.)

### 집필 흐름 순서

집필 흐름: **서사 유형 확정 → 썸네일 → 도입부 → 본문 → 간접 확인 → 아하 → 해소 은유(사용 시) → 원칙 검증**. 세부 운영은 각 PART 본문 참조.

## 할루시네이션 방지 규칙

### 원칙 — 전 Phase 공통

#### ① 지침 예시 활용 방식은 종류에 따라 다르다

- **패턴형 예시** (`~`·`[N]`·`OOO` 등 자리채움 마커 포함): 마커 자리를 주제에 맞게 채워 그대로 사용 가능. 다만 한 대본 내 동일 패턴 2회 이상 사용 시 다른 패턴 또는 새 표현으로 교체한다
- **완성 문장형 예시** (자리채움 마커 없는 완성 문장·문단): 구조·어조·분량 감각만 참고하고, 실제 단어·문장은 해당 주제와 맥락에 맞게 새로 작성한다

#### ② 지침 내부 용어 노출 금지

SL·ML·BR·A극·B극·역설엔진·이문호공·해소 비유·피벗 등 지침 전용 용어를 대본에 노출하지 않는다.

#### ③ 같은 비유·밈·감탄사 한 편 내 반복 금지

같은 표현을 한 영상에서 두 번 이상 사용하지 않는다.

#### ④ 확인되지 않은 수치·사실 단정 금지

웹 검색으로 팩트 확인 후 사용한다.

### P4 집필 단계 적용

- **화자가 자기 정보에 놀라는 척 금지** — 감탄·감상을 제거했을 때 정보나 대비가 남지 않는 문장은 제거. 대비를 명시하는 문장으로 교체한다
- **비유의 메커니즘 일치 확인** — 표면적 유사성이 아니라 작동 원리가 일치하는지 검증
- **수정 후 정합성 확인 (일반):** 문장 수정 시 같은 단락 전체를 다시 읽는다
- **수정 후 정합성 확인 (구조 요소):** ML·피벗·해소 은유를 수정할 경우 해당 요소를 참조하는 모든 문장을 검색하여 재검증한다
- **수치 최신성 확인** — 현재 시점 기준 최신인지, 주체·범위·조건이 보존되었는지 확인
- **집필 중 추가 팩트 출처 기록** — 팩트리포트에 없는 정보는 반드시 웹 검색 확인 후 상태 태그 "추가 팩트"에 내용·출처 기록. 미확인 정보 단정형 서술 금지

## PART A: 12개 언어 장치 + 04 사전 위임 도구

> **04 언어 장치 사전 위임:** 일부 항목은 모듈_집필기법 §6 (비유 영역 선정) + §7 (인터넷 밈) + `지침/지침/모듈/모듈_어법.md` §4·§5·§6-1·§6-2·§6-3.
>
> **위임 도구:** 비-모순형 공통 (자문자답 압축형) → §4 / 가정 (반사실 가정·가정법 시뮬레이션) → §5 / 유형별 분기 (모순 6·이미지 5·과정 5) → §6-1·§6-2·§6-3.
>
> **시점 분류 위임:** 본 §A의 12개 언어 장치 + 위임 도구는 모두 04 §1.1 시점 분류 체계(P4 강제 / P4 권장 / P5 전담)에 따라 분배된다. 집필 중 적용 대상은 **P4 강제 + P4 권장 항목**이며, **P5 전담 항목은 P5 검수 시 P5_퇴고.md §2.1·§2.2 (도구 적용·위치·빈도 / 차용·톤 정합) 체크리스트로 일괄 검증**한다. 도구별 시점 분류는 04 §2~§6 정의 블록 + 04 §7 부속 표 참조.

### 1. 의인화

국가/물질/체내 반응을 캐릭터처럼. 직후 객관적 사실 보완. 최소 1회 이상 적용 권장. 적용 결과·미적용 사유는 P5 검수 시 메인에게 보고 (메인 1차 검수 영역).

### 2. 수치 체감 치환

**치환하는 경우:**

- 면적/거리 큰 수
- 전문 단위
- 비교 없이 판단 불가한 수치
- 높이/길이의 공간적 크기
- **시간 표기**
  - 단발 사건의 월·일 → 사건 명칭으로 압축
    - 예: "1940년 6월 25일, 프랑스가 항복한 그날" → "프랑스가 항복한 바로 그날"
  - 연도·시각 → 익숙한 표현 가능 시 익숙한 표현 우선
    - 예: "1940년 6월" → "2차 대전 때 프랑스 항복 직후"
    - 익숙한 표현 부재 시 원본 유지
- **추상 속성·메커니즘** (정교함·견고함·복잡함·내구성·정밀함·보존력·작동 원리 등)
  - 익숙한 사물·캐릭터·영역 비유로 변환
    - 예: 정교함 → "아이폰 설계도 수준" / 보존력 → "캡틴 아메리카 박제" / 견고함 → "강철"

**환산 단위 풀:**

> · 가구 월 가계비 환산 (예: 5배 비싸다 → "매달 24만원 더 빠짐")
> · 일상 소비재 환산 (예: "라면 한 그릇 값", "치킨 한 마리", "아이폰 N대")
> · 자산 환산 (예: "단돈 720만 달러", "강남 아파트 2채", "중형차 한 대")
> · 시간 환산 (예: "월급 6,000년치", "주 5일 야근 1년치")
> · 등급 비유 (예: 원유 등급 → "막걸리·청주·소주", 풍속 → "선풍기·태풍")

**치환하지 않는 경우:**

- 극단적 충격 수치 (예: "사망률 90%")
- 이미 직관적인 비율
- **권위 구축 밀집 나열** (효과 기준 두 조건 충족 시 예외 — 아래 참조)
- 본 예외 항목은 `지침/모듈/모듈_어법.md` §0 즉시 변환 원칙에 그대로 적용 (§0 발동 시에도 본 예외 우선)

**권위 구축 밀집 나열 (효과 기준):**

원본 수치·날짜·인명·고유명사·연표·기관명을 밀집 나열하여 신뢰감 부여.

- **적용 조건 (둘 다 충족):**
  - ① 맥락 인지 — 직전 단락에서 본 나열의 맥락(주제·영역·관계)이 시청자에게 셋업
  - ② 효과 발생 — 처음 듣는 시청자 시각에서 권위·신뢰 임팩트 발생, "왜 이게 나오지?" 혼란 미발생
- **한 조건이라도 미충족 시:** 적용 외 → §공통 셋업 의무 적용
- **자가점검:** "직전 단락 정보만 가지고 시청자가 이 나열을 받았을 때 임팩트가 발생하는가, 혼란이 발생하는가?" (영역 분류 무관 — 위 두 조건만 판단)
- **마지막 항목은 흐름·톤에 맞게 자연스럽게 마무리한다.**

### 5. 구어체 리듬·어법

#### 리듬

- **확인 사격:** 긴 설명 직후, 핵심 한 문장으로 각인 ("뭐다? 감자다")
- **한 줄 요약:** 복잡한 설명 후, 체감 문장으로 압축
- **감탄사 리셋:** 정보 밀도 높아지기 직전 삽입

#### 어법 (학습 필요 채널 톤 — 누적 카탈로그)

향후 발견되는 학습 필요 구어체 어법을 본 카탈로그에 누적한다. 각 어법은 **원리(WHY) + 변주 예시**로 등재.

**공통 운영 원칙:**

- 예시 활용 원칙은 `지침/모듈/` "원칙 — 전 Phase 공통" §1 참조 (어법 카탈로그 변주 예시는 패턴형)
- **자연 흐름 우선** — 어법은 적용 가치 있는 위치에서만 선택 사용 (강제 아님). 자연 흐름에 마커 인위 삽입 금지

**등재 어법:**

- **자기 진술 후 반전 어법:**
  - **원리:** 화자가 직전 흐름에 대한 자기 진술(상태 인정·평가·가정)을 한 뒤 곧바로 반전으로 진입. 시청자와 같은 자리에서 한 발 떨어져 보는 친근 톤
  - **사용 조건:** 직전 흐름·전제에 대한 자기 진술이 시청자 추론과 같은 자리에 있을 때, 그 자세를 명시할 가치가 있는 위치에서만 적용
  - **변주 예시:**
    - **상태 인정형:**
      - "사실 이때까지만 보면 ~. 그러나 ~"
      - "여기까지는 ~. 진짜 ~"
      - "다만 이때까지는 그래도 ~"
      - "그러나 그럼에도 불구하고 ~까지만 해도 ~"
    - **화자 평가형:**
      - "~ 아닌가 싶기도 한데 이게 진짜 ~"
      - "~ 아닐까 싶긴 합니다. 물론 ~"
      - "~ 아닌가 싶을 정도예요"
    - **화자 가정형:**
      - "~ 했으면 또 몰라. 문제는 ~"

- **일상 폭로 어법:**
  - **원리:** 시청자가 일상에서 접하는 대상의 숨겨진 사실을 마커로 폭로하여 인지 전환 ("어? 그게 그런 거였어?") 유도
  - **사용 조건:** 시청자가 이미 안다고 생각하는 일상·상식에 실제로는 숨겨진 사실이 있을 때만 적용. 단순 사실 진술에 마커 끼워 넣기 금지
  - **변주 예시:**
    - "사실 우리가 매일 X하는 Y는 ~"
    - "사실상 ~ 급이라고 볼 수 있지요"
    - "실제로 이 동네는 ~"
    - "이미 ~ 뒤였습니다"
    - "알고 보면 ~"
    - "실은 ~"

편당 최소 1회. 주제와 시청자가 자연스럽게 연결되는 구간에 배치. 방식: 질문형·상황 제시형·경험 소환형·1인칭 경험 공유.

1~2회. 1인칭 감정. **사용 위치:** 정보 피크 직후·놀랄만한 정보 직후 (정보 피크 직전·SL 진입 위치 사용 금지). 시사/시의성 주제에서 빈도 높음. 3회 이상이면 "의견 채널"로 분류됨. 진입 구문 매번 변형.

### 9. 지시어 명확성

지시 대명사(이것·저것·그것·여기·거기·저기·이때·그때) 사용 시 지시 대상이 직전 1~2 문장 안에 명확히 등장해야 한다. 음성 매체(낭독 영상) 특성상 시청자가 앞 문장 다시 못 봄 → 모호한 지시어는 즉시 이탈 유발.

**예시:** 위반 — "사실 1985년부터 거기 있었어요" (`거기` 모호) → 수정 — "그 구조 안에 있었어요. ○○○라는 구조 말이죠"

**점검 방법:**

1. 지시어 사용 시 가리키는 명사가 직전 1~2 문장 안에 있는지 확인
2. 없으면 명사로 풀어쓰기
3. P5 §2.3 (지시어 명확성) 메인 1차 검수 + (선택) P5_퇴고검증 에이전트 수동 호출 시 추가 검증

### 10. 이완 장치 강제 규칙

- 이완 장치: 속어/밈, 자조적 한 줄, 시청자 공모형 유머, 감탄사+짧은 판단

- **무거운 구간:** 자조적 한 줄 / 감탄사+짧은 판단 / 화자 감정 삽입으로 대체

### B-1. 도입부 공통 구조

**도입부 역할 (모듈_도입부 §2 (역할 분담) + 모듈_본문구조 §1 (ML) 인용):**

도입부는 썸네일이 예고한 최종 통찰(L3)을 암시·직접 인용하지 않고, 최종 통찰(L3)이 왜 의외인지의 전제 조건을 세팅한다. 신뢰 상환 전략에 따라 도입부의 이원 기능이 달라진다 (모듈_도입부 §2 (역할 분담) + 모듈_본문구조 §1 (ML) 표 참조).

**도입부 3구성 (모듈_도입부 §2 (인트로+ML+신뢰 상환) 인용):**

도입부는 인트로(블록) + ML(블록) + 신뢰 상환(기능)으로 구성된다. 각 구성의 구체 구현(진입 단계 수·내용·신뢰 상환 위치)은 유형 파일 P3 §1에서 정의된다.

| 구성 | 분류 | 공통 원칙 |
|---|---|---|
| 인트로 | 블록 | ML 이전 진입 영역 — 유형별 차등 (모순형 ①②③ 3요소·이미지/과정 ①② 2요소). 세부는 P3 §1 |
| ML (메인 루프) | 블록 | 제목·썸네일의 약속을 질문·선언으로 문장화 (도입부 말미) |
| 신뢰 상환 | 기능 | 시청자에게 "영상이 약속을 지킬 것"이라는 신호 제공 |

**신뢰 상환 위치 (P3 §3 그대로 인용):**

> - 모순형: 신뢰 상환 본문 이연. 도입부에서는 "썸네일 약속 재확인"(동의어 치환)만 수행. 역설 증명은 본문 A극·B극에서.
> - 이미지강화형·과정추적형: 신뢰 상환 도입부 수행. 단 일부 주제(22 온돌·13 호르무즈)는 본문 이연.
> 본문 이연 시 상환 위치는 주제·서사 구조·정보 배치 전략에 따라 결정 (강제 기준 없음 — 유형 파일 P3 §1 참조).

**ML 정의 (모듈_본문구조 §1 (ML) / §2 (결론부 8유형) 그대로 인용):**

ML은 도입부 말미에서 본문이 답할 핵심 질문을 공식화하는 단위이다. 서사 유형에 따라 변주하며, 유형별 구체 구조는 유형 파일 P3 ML 섹션 참조. 이문호공 3요인 이상 권장. 자연스러운 궁금증 유발이 요인 카운팅보다 우선 (모듈_본문구조 §1 (ML) / §2 (결론부 8유형)). 10요인 (9 기본 + 1 신규) 정의는 `지침/모듈/모듈_이문호공.md` 참조.

#### 신뢰 상환 운영 안내 — 도입부 수행 시

> 신뢰 상환을 도입부에서 수행하는 경우에 적용. 모순형 본문 이연 시에는 본문 영역 (PART C — 특히 §C-4 서사 유형별 본문 구조) 참조.

- 신뢰 상환은 썸네일이 약속한 내용을 실제 팩트로 증명한다. 팩트 나열이 아니라 팩트의 체감 시간을 늘린다.
- **신뢰 상환 ≠ ML:** 신뢰 상환은 "이미 일어난 일/현재 상태"를 팩트로 보여주는 것, ML은 "왜/어떻게"를 묻는 질문. 같은 문장에 섞지 않는다.
- **필수:** 팩트 후 소화 문장 삽입 (단, 팩트 자체가 충분히 직관적이면 생략 가능). "소화 문장" 정의 = `00_용어정리.md` 참조.
- **필수:** 소화 문장은 직전 팩트가 아니라 최종 통찰(L3)을 가장 잘 지지하는 팩트를 근거로 삼는다 (관성 차단 + 논리 비약 방지).
- **필수:** 숫자/전문 개념은 일상 언어로 풀어서 서술한다. 이미지강화형 수치 선택 기준은 02 유형 파일 P4 §2 참조 (압도적·체감 가능·1개로 충분).

### C-1. 섹션 집필 전 사전 선언 (필수)

각 SL을 집필하기 **전에** 아래 한 줄을 먼저 적는다. 집필 후 제거해도 되지만, 선언 없이 본문부터 쓰지 않는다.

```text
[SL번호
- 해소비유: 있음|없음(팩트임팩트형) — 핵심 대상 한 줄 — 이문호공 2요인({요인+요인})
- 핵심 팩트: [{마킹}] {팩트리포트 원본 복사}
]
```

> **선행브릿지·SL열림유형은 사전 선언 불필요:** 선행브릿지는 직전 SL의 §C-2 상태 태그 "브릿지→SL{N+1}"을 그대로 활용 / SL열림유형은 P3 조감도 SL 배열표 컬럼에서 확인. 두 항목 모두 사전 양식 작성 없이 집필 직전 조감도·직전 상태 태그에서 직접 도출.

**참조 — 양식 안 각 항목 정의 위치:**

- SL번호 = P3 조감도 (확정된 SL 배열)
- 해소비유 (있음/없음 + 공식) = 모듈_집필기법 §1 (해소 은유 공식)
- 이문호공 2요인 = 모듈_본문구조 §1 (ML) / §2 (결론부 8유형) + `지침/모듈/모듈_이문호공.md` + P1 §7
- 핵심 팩트 + 마킹 ([사실]/[추정]/[해석]/[논쟁]) = P2 팩트 리포트 (`P2_딥리서치.md`)
- 상태태그 = §C-2 섹션 완성 후 필수 이행 (의인화 적용·속어감탄 누적 등 사후 관리)

### C-2. 영역 완성 후 필수 이행 (도입부·SL·결론부 공통)

각 영역(도입부·SL·결론부) 집필이 끝나면 아래 두 가지를 즉시 수행한다. **사후 점검 도구로서 채널 강제 시그니처·유형별 강제 도구 망각 여부를 즉시 발견하여 해당 영역만 재작성하는 안전망 역할을 한다.**

**1) 기존 검증:** 해소 은유가 없으면 §C-1 사전 선언으로 돌아가 재작성 / P3 조감도 SL 배열표의 SL 열림 유형과 첫 문장이 불일치하면 재작성 / **단락 간 사실 중복 점검:** 직전 단락에서 다룬 사실(인물·기관·동작·정제 내용)이 본 단락에 단어만 다르게 재인용되지 않았는가? 재인용 시 직전 단락과 통합 또는 본 단락에서 결과·재해석으로 변경. (P5 §2.3 단락 간 사실 중복 점검 정합)

**2) 상태 태그 작성:**

```text
[상태] {도입부|SL{N}|결론부} 완료
- 루프: 열림(유형) / 해소 여부
- 열린 루프 현황: {현재 미해소 루프 전체 나열}
- 브릿지→다음: {전환 전략 한 줄}
- 아하 모먼트: {해소 은유 문구} (이문호공 요인)
- 마킹↔서술 일치: {위반 없으면 "OK" / 위반 시 해당 문장 명시}
- 추가 팩트: {팩트리포트에 없는 정보 내용+출처 / 없으면 "없음"}
```

#### P4 자체 운영 안내 (이탈 방지)

**보상의 비례 원칙:** 공백의 크기(궁금증의 강도)에 비례해 해소 시 보상(지적 쾌감·스케일 확인·인과 완성)의 강도가 커야 한다. 시청자가 "기다린 만큼 값어치가 있었다"고 느끼게 한다.

**유형별 보상의 성격:**

- 모순형: 역설 해소 (A극·B극이 동일 원천에서 나온 양면성이었음의 발견 — "어? 같은 거였네?")
- 이미지강화형: 스케일 확인 ("이것까지 영향이 미쳤다고?")
- 과정추적형: 인과 체인 완성 ("이래서 이렇게 됐구나!")

**모순형 비율 편차:** 모순형에서 A극이 역사적 서사나 공감 소재로 "읽는 재미"를 제공하는 경우, A극/B극 비율이 2:1~3:1까지 불균형해도 허용된다.

**기타 운영 규칙:**

- **보상의 분할 제공:** 가장 큰 보상을 한 번에 투하하지 않고 분할한다. 3~4단계 점진적 공개.
- **성공 서사 텐션 규칙:** 정점 직후 의도적 추락(경고형) 1회 이상.

- 같은 섹션(SL) 내에서 영역을 일관 유지한다
- **생성 순서:** 사건을 비유 없이 한 문장으로 쓴 뒤 → 핵심 메커니즘 추출 → 같은 메커니즘을 가진 일상 이미지 탐색

**해소 은유 vs 설명용 비유:**

## PART D: 엔딩 집필

### D-1. 결론부 쿨다운 규칙 (공통)

> **본격 정의는 다음 단일 지침 참조.** P4는 집필 실행 시 핵심 식별과 P4 자체 운영 절차만 표기.
>
> - 루프 닫기 위치 → `지침/모듈/` §B.7 (생애주기 4단계 中 ④ 닫기)
> - 착지 엔딩 유형 → `지침/모듈/` §B.6 + P4 §D-2 (유형별 자연 궁합)
> - 수미상관(도입부 소재 회수) → `지침/모듈/` §C.1 + §B.6 ⑤ 위트반전
> - 결론부 위치 정의 → `P3_구조설계.md` §12 구조 보고 SL 배열 표 (결론부 — ML 해소)

#### 결론부 3단계 구성 (P4 자체 운영 절차)

| 단계 | 기능 | 단일 지침 위임 |
|---|---|---|
| 1. 정리 | 본문 인과 체인을 한 호흡으로 압축 (요약만, 새 정보 없음) | 모듈_집필기법 §8 (브릿지 18종) + 모듈_본문구조 §1.3 (3유형 ML 단계) "루프 닫기" 후반부 위치와 정합 |
| 2. 착지 | 엔딩 8유형 중 1~2개 조합으로 마무리 | 모듈_본문구조 §2 (엔딩 8유형) + 모듈_본문구조 §1.2 (SL 중첩) + P4 §D-2 (유형별 자연 궁합) |
| 3. 시그니처 아웃트로 | "이상 이거봐봐였습니다" 고정 문구 (CLAUDE.md "채널 이름 명확 분리" 정합) | (P4 자체 고정 — 단일 지침 외) |

#### P4 자체 운영 규칙 (단일 지침 외 영역)

- **ML 해소 위치 고정:** ML은 본문 마지막 SL에서 완료하고, 결론부는 착지만 수행 (모듈_집필기법 §8 (브릿지 18종) + 모듈_본문구조 §1.3 (3유형 ML 단계) "후반부 닫기" + P3 §12 구조 보고 SL 배열 표 정합)
  - **예외 (과정추적형 고유):** ML Q2가 결말 파트에서 해소되는 "결말 40% 역전" 패턴은 허용 (모듈_엔진 §3 (인과추적 엔진) §X / 12 가야 사례 / 결론부 40% 이내). 본 규칙은 모순형·이미지강화형의 기본 위치를 의미 — 과정추적형은 모듈_집필기법 §8 (브릿지 18종) + 모듈_본문구조 §1.3 (3유형 ML 단계) ML 4단계 닫기 표 "결말에서 현재 의미 부여"와 정합
- **신규 정보 추가 금지:** 결론부에 ML 해소에 영향 주는 **새 팩트·새 인과·새 SL** 도입 금지 → "끝났는데 왜 안 끝나지" 인상 차단 → 마지막 인상 저하 방지
  - **예외 (이미지강화형 고유):** 모듈_엔진 §2 (영역확장 엔진) §X "부가 정보 삽입" 패턴은 허용 — ML 해소 무관 "관련 상식"(보너스 정보), 결론부 20% 이내, ML이 본문에서 완전히 해소된 후 (결론부 본체 단락 영역 — §D-1-2-B 규칙 3은 결론부 진입 단락 어법 영역으로 별개)
  - **예외 (⑥ 통찰형):** 본문 관통 통찰 한 문장은 허용 — 새 정보가 아닌 메타 압축 (모듈_본문구조 §2 (엔딩 8유형) + 모듈_본문구조 §1.2 (SL 중첩) ⑥ 정합)
- **자연 마무리 호흡 (자가점검):** 결론부가 통찰 단정으로 압축되어 갑자기 끝나는 느낌이 들지 않는가? 자가점검 — "시청자가 결론부 마지막 문장 후 자연스러운 호흡 단락에 도착하는가? 단정 직후 영상이 끊기는 느낌인가?" 갑자기 끝나면 호흡 보강. **권위 톤 회피 — 1인칭 동조 톤 유지.** 표현 형태는 자율 (미래 관찰·시청자 동참·시간 여운 등 다양한 형태 가능 — 본 항목은 형태 강제 아님).
- **도입부 소재 회수(수미상관):** 가능 시 도입부 드립·소재를 결말에서 재활용 (모듈_집필기법 §1 (해소 은유 공식) E형 + §B.6 ⑤ 위트반전과 정합)
- **회수 대상 단일화:** 결론부 회수는 다음 중 **한 가지**만 선택.
  - (a) **도입부 소재 회수 (수미상관)** — 도입부 키워드·인물·드립 1개 변형 회수
  - (b) **본문 비유 회수** — 본문 핵심 비유 1개 변형 회수
  - ❌ **동시 회수 금지** — (a) + (b) 동시 시도 시 통찰 단편 나열 발생
  - **자가 점검:** 엔딩에 회수된 요소가 **2개 이상**이면 통찰 단절 위험 → 1개로 압축
- **본문 결산 양식 마무리:** 본문 마지막 단락(= 결론부 직전 단락)에서 ML Q1·Q2 답을 정리·통찰·부가 정보·시의성 등으로 마무리. 유형별 4종 중 1종 선택 (§D-1-2-B 참조).
- **결론 브릿지 어법 1~3종 결합:** 본문 → 결론부 진입은 결론 브릿지 어법 5종 (① 결과 호명·② 이완 호명·③ 시간 전환·④ 회수·⑤ 메타) 중 1~3종 결합 사용. 본문 결산 양식과의 자연 연동은 §D-1-2-B 강제 규칙 4건 참조.
- **결론부 본문 헤더·상태 태그 보존:** 헤더는 `## 결론부 ({결론 브릿지 어법} / {엔딩 유형})` 표기 (CLAUDE.md "대본 본문 섹션 헤더 표기 규칙" 정합). 결론부 본문 후 §C-2 SL 상태 태그와 동일 패턴으로 **코드 블록 보존** — 본문 결산 양식·결론 브릿지·엔딩 유형·회수 대상·톤·시그니처 명시 (P5 추출 시 자동 제거 / 시청자 미노출).

#### D-1-2-A. 5단락 영역 + 결론부 직전 단락 안전장치 (권장)

본문 → 결론부는 다음 5단락 영역으로 분할된다 (용어정리.md §4 정합).

본문 정점 SL → 결론부 직전 단락 → 결론부 진입 단락 → 결론부 본체 단락 → 시그니처

**결론부 직전 단락 안전장치 (권장):**

본문 마지막 단락(= 결론부 직전 단락)에 결론 진입 신호를 사전 배치한다. 안전장치 형태는 본문 정점 위상에 따라 다음 중 하나를 선택한다.

| 본문 정점 위상 | 안전장치 형태 |
|---|---|
| 위상 격상 정점 | 강화 신호 또는 균형 회복용 단점 호명 |
| 부정 정점·위기 정점 | 강화 신호 또는 호소 |
| 의외성 정점 | 모순 사례·논쟁거리 사전 배치 |
| 해법 호소 정점 | 해법 사례 사전 배치 |

> **본문 정점 위상 분류 4종:** 위상 격상 / 부정·위기 / 의외성 / 해법 호소 (본 §D-1-2-A 표 정합).
>
> **권장 영역**: 본 가이드는 강제 규칙 아님. 자연 발생 사례 분석 결과로 권장 등재.

#### D-1-2-B. 본문 결산 양식 → 진입 어법 자연 연동 (강제 규칙 4건 + 권장 1건)

본문 결산 양식이 결정되면 다음 강제 규칙에 따라 결론 브릿지 어법을 사용한다.

**[규칙 1] 정리·결산 양식 → ① 결과 호명**

본문 결산 양식이 다음 중 하나일 때 결론 브릿지 어법은 ① 결과 호명을 사용한다.

- 모순형 A. 정리·복귀형
- 과정추적형 B. 현재 의미 부여형

**[규칙 2] 모순형 통찰 압축형 → ② 이완 호명 필수**

모순형 본문 결산 양식이 통찰 압축형(B)일 때, ② 이완 호명을 필수로 사용한다. 직접 ① 결과 호명 점프는 금지 ("성급한 마무리" 차단).

**[규칙 3] 이미지강화 부가 정보형 → ② 이완 호명**

이미지강화형 본문 결산 양식이 부가 정보형(B)일 때, ② 이완 호명을 사용한다 ("여담으로"·"뭐 어찌됐든" 등).

**[규칙 4] 시청자 시점 결합 결산 양식 → ④ 회수**

본문 결산 양식이 다음 중 하나일 때 ④ 회수를 사용한다.

- 모순형 D. 자기언급+갈등형
- 이미지강화형 D. 시의성형
- 과정추적형 A. 후일담형

**[권장 1건]** 이미지강화형에서 시청자 경험 진입형 도입부(B 공감·E 밈진입·F 맥락재설정 ML 외부화)와 본문 통찰형 아하 모먼트를 결정한 경우, 본문 결산 양식은 A 통찰 압축형을 권장한다.

> **유형별 가이드 정의는 단일 지침 위임:**
>
> - 모순형 → 모듈_엔진 §1 (역설 엔진)
> - 이미지강화형 → 모듈_엔진 §2 (영역확장 엔진)
> - 과정추적형 → 모듈_엔진 §3 (인과추적 엔진)

#### D-1-2-C. 결론 브릿지 어법 5종 (옵션 풀)

결론 브릿지 어법 5종은 자유 조합으로 사용한다 (1~3종 결합).

| 어법 | 사용 예시 |
|---|---|
| ① 결과 호명 | "이게 ~한 일", "상황이 좀 달라졌습니다" |
| ② 이완 호명 | "물론 뭐", "아 근데 뭐", "무튼" |
| ③ 시간 전환 | "최근 몇 년간", "그러나 이후" |
| ④ 회수 | "앞서 말한", "영상 도입부에도 말씀드렸듯" |
| ⑤ 메타 | "이 영상 ~"·"선원 여러분 ~" / "등잔이 어둡다"·"공중 보건은 조별 과제" (메타 격언) |

§D-1-2-B 강제 규칙 4건이 적용되지 않는 영역에서는 자유 조합으로 사용한다.

**유형별 결말 어법 참고 가이드 (결론부 본체 단락 어법 — 결론 브릿지 어법과 별도 영역):**

각 유형의 결론부 본체 단락 어법은 다음 빈출 경향을 보인다 (강제 아님 — 변형 영상 자유).

| 유형 | 결말 톤 | 빈출 어법 |
|---|---|---|
| 이미지강화형 | 희망적·미래 지향 | ③ 미래희망 |
| 모순형 | 통찰적·격언 호소 | ⑥ 통찰형 |
| 과정추적형 | 실용적·관여 호소 | ② 행동 권유 |

> **영역 차이 명시:** 본 표의 ① ② ③ ⑥ ⑦ 번호는 결론부 본체 단락 어법 5종 (② 행동 권유·③ 미래희망·④ 의미 부여·⑥ 통찰형·⑦ 경고). §D-1-2-B·C 결론 브릿지 어법 5종(① 결과 호명·② 이완 호명·③ 시간 전환·④ 회수·⑤ 메타)과 다른 영역. 같은 번호도 다른 어법 — 혼동 금지.
>
> **영역 차이 명시:** 결론부 본체 단락 진입 방향(본 표) vs 결론부 마지막 영역 착지 방식(엔딩 8유형, 모듈_본문구조 §2 (엔딩 8유형) + 모듈_본문구조 §1.2 (SL 중첩)) — 다른 영역이므로 혼동 금지.

### D-2. 엔딩 8유형 + 유형별 자연 궁합 (단일 지침 위임)

> **본격 정의는 다음 단일 지침 참조.** P4는 집필 실행 시 핵심 식별과 P4 자체 운영 안내만 표기.
>
