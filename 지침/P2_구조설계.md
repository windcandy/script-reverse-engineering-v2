## 1. 서사 구조 유형 선택

Phase 1의 핵심 질문 유형에 따라 결정한다.

### 모순형 (핵심 질문 A)
```
인트로 → 신뢰 상환 → ML → A극(기초SL→심화SL→정점SL) → 피벗BR → B극(What→How→Why→현재의미) → 엔딩
```

### 이미지 강화형 (핵심 질문 B)
```
인트로 → 신뢰 상환 → ML → 기초레이어 → How레이어 → Scale Jump → 통찰 → 엔딩
```

### 과정 추적형 (핵심 질문 C)
모순형 변주(시간축 전개 / 사례 순회 / 누적 반복형 등)에서 선택한다.

## 2. 모순형 7가지 변주

| # | 변주 | 설명 |
|---|------|------|
| 1 | 정적 역설 | 하나의 대상이 가진 두 얼굴을 정지 상태에서 대비한다 |
| 2 | 시간축 전개 | 시간 순서에 따라 A극→B극이 자연스럽게 드러나는 구조 |
| 3 | 사례 순회 | 복수 사례를 돌며 공통 패턴(B극)을 귀납적으로 도출한다 |
| 4 | 누적 반복형 | 같은 논리를 층층이 쌓아 정점에서 한꺼번에 뒤집는다 |
| 5 | 양면 동시 건설형 | A극과 B극 정보를 교차 배치하여 동시에 쌓아 올린다 |
| 6 | 감정 피벗형 | 감정의 방향 전환 자체가 서사의 핵심 동력이 된다 |
| 7 | 원인 역추적형 | 결과(현상)에서 출발하여 원인을 역으로 추적한다 |

---

## 3. A극 설계

- **A극 정점 문장을 먼저 정한다** (박스 이미지)
- SL의 인지적 기능 구분:
  - **기초SL** (A극 초반) — 기본 전제 건설
  - **심화SL** (A극 중반) — 전제 위에 깊이 추가
  - **정점SL** (A극 끝) — 확신 최고점 도달
- 확신 스케일 확대 순서: 인지 → 놀람 → 확신 → 확신 최고점 → 피벗 추락
- A극의 모든 SL은 A극 정점 방향에 기여해야 한다
- B극 정보가 A극에 미리 노출되면 안 된다

---

## 4. B극 설계

- **B극 임팩트를 먼저 정한다**
- B극 순서: **현상(What) → 메커니즘(How) → 근본원인(Why) → 현재의미**
- Why SL의 Aha-moment를 한 문장으로 정의한다
- How는 중반에 해소한다 (신뢰 확보 목적)
- Why는 최대한 늦게 배치한다 (텐션 유지 목적)

---

## 5. SL 배열 규칙

- **전제 건설 체인:** 앞 SL의 해소가 뒤 SL의 전제가 된다
- **편당 SL 수:** 2분당 1개 기준
  - 10분 = 5개
  - 13분 = 6~7개
  - 16분 = 8개
- **2분 이상 SL 없는 구간 금지**
- SL로 포장할 정보와 SL 없이 서술할 정보를 구분한다

---

## 6. BR(브릿지) 5유형

| # | 유형 | 패턴 | 빈도 |
|---|------|------|------|
| 1 | 역접 전환형 | "하지만/그런데" | 최다 |
| 2 | 질문 전환형 | "~냐?/~까요?" | 중간 |
| 3 | 정산 전환형 | "자/이쯤 되면" | 중간 |
| 4 | 확장 전환형 | "~뿐만이 아니라" | 소수 |
| 5 | 감정 피벗형 | 감정 전환 | 편당 최대 1회 |

- **BR-SL 동시 작동 원칙:** 절반 가까이를 동시 작동으로 설계한다
- **핵심 피벗 BR:** 편당 1회, A극 → B극 전환점에 배치한다

### 논리적 브릿지 (비 오픈루프) 사용 가이드

오픈루프(SL)를 열지 않고 정보를 정리하는 구간에서 사용한다. 전체 BR 중 20% 이하로 제한한다.

| 유형 | 기능 | 패턴 |
|---|---|---|
| 선형적 흐름 연결 | 시간/공간의 계보 나열 (A 다음 B) | "유대교가 → 기독교를 → 이슬람교를" |
| 인과적 흐름 연결 | 사건의 결과가 다음의 원인이 되는 꼬리물기 (A이기에 B) | "이러한 이유로", "때문에", "덕분에" |

- 논리적 브릿지 사용 시 해당 구간 뒤에 왜 오픈루프 대신 이것을 썼는지 스스로 확인한다
- 서브 오픈루프가 닫힐 때 새로운 서브 오픈루프를 열어 정보의 결핍 상태를 유지하는 것이 기본이다

### 루프 중첩 제약

- 서브 오픈루프 중첩은 1개 유지를 원칙으로 하되, 서사 전개상 필요한 경우에만 최대 2개까지 허용
- 3개 이상의 서브 오픈루프가 동시에 열려 있는 상태는 절대 불가
- 2개가 열려 있다면 새 루프를 열기 전에 반드시 기존 루프 중 하나를 해소한다

## 8. 정보 배치 원리

- 익숙한 것의 숨겨진 의미를 폭로한다
- 없는 세계의 고통으로 가치를 증명한다
- 시청자가 스스로 패턴을 발견하게 한다
- 아는 것을 게이트웨이로 활용한다

## 9. 루프 해소와 보상 원칙

### 보상의 비례 원칙

루프를 유지(지연)시킨 시간만큼 보상의 가치(정보의 밀도)도 커야 한다. 기다림에 비해 결론이 평범하면 이탈률이 급증한다.

- ML(영상 전체 관통 루프): 가장 오래 유지되므로 가장 큰 보상이 필요
- A극 전체를 지탱한 SL: 피벗의 낙차가 보상 역할
- 개별 SL(2분 단위): 박스 이미지(강) + 뒷받침(약)이 최소 보상 단위

### 지적 보상의 품질

루프를 닫을 때 시청자가 충분히 납득할 만한 수준의 통찰을 제공해야 '낚시'가 아닌 '학습'으로 인지된다.

**품질 테스트:** "이 해소를 들은 시청자가 다른 사람에게 설명하고 싶어지는가?" — 아니면 보상이 약한 것.

### 빌드업의 농도 조절

보상 직전까지의 과정이 너무 길면(Dragging) 보상이 주어졌을 때 시청자는 이미 지친 상태가 된다. 보상 직전의 텐션을 최고조로 올리되 호흡을 너무 끌지 않는다.

### 보상의 분할 제공

가장 큰 보상을 한 번에 던지지 않는다. 정보를 쪼개어 배치한다.

```text
결과 공개 → 비하인드/의미 부여 → 추가 팁(또는 현재 연결)
```

### 지적 유능감 자극

시청자가 정보를 얻었을 때 "나만 이걸 알게 됐다", "이제 이해가 간다"라는 느낌을 받도록 서술한다. 보상 해소 후 이 내용을 타인에게 공유하고 싶게 만드는 한 줄 요약이나 사회적 가치를 덧붙이면 시청자는 더 큰 보상을 받았다고 느낀다.

## 10. 도입부 전략 8가지

| 코드 | 전략 | 핵심 |
|------|------|------|
| A | 규모폭로 | 숫자나 스케일로 즉각 충격 |
| B | 공감적이질감 | 익숙한 것에서 낯선 요소 발견 |
| C | 자조적체감 | 자기비하적 유머로 거리감 제거 |
| D | 속성폭격 | 짧은 팩트를 연속 투하 |
| E | 밈진입 | 밈/트렌드를 진입점으로 활용 |
| F | 맥락재설정 | 알려진 사실의 맥락을 바꿔 제시 |
| G | 현재불안소환 | 지금 시청자의 불안을 건드림 |
| H | 감정직접소환 | 강한 감정으로 즉시 몰입 유도 |

### 도입부 2단계 연결
1. **신뢰 상환** — 도입부 직후, 시청자의 시간 투자가 가치 있음을 증명
2. **메인루프(ML)** — 영상 전체를 관통하는 핵심 궁금증으로 연결

## 12. 할루시네이션 방지 규칙

- SL 개수를 억지로 맞추기 위해 불필요한 정보를 넣지 않는다
- 감정 곡선을 도식적으로 따르느라 부자연스러운 전환을 만들지 않는다
- 구조설계 시 리서치 전 단계의 정보로 확정 짓지 않는다 — **"리서치 후 조정 가능"을 명시한다**
