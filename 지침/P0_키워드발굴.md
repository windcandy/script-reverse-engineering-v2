## 3유형 적합도 판단 기준 (방식 A·B 공통)

주제 후보를 발굴할 때 각 주제가 **어느 서사 유형에 적합한지** 평가한다. 이는 가중치 점수(방식 A) + 주제 태그(방식 A·B 출력)로 반영된다.

### 3유형 적합도 지표

| 서사 유형 | 적합도 판단 지표 | 강한 신호 |
|---|---|---|
| **모순형 적합도** | A극·B극 낙차 + 역설 강도 | "금보다 비쌌던 소금이 지금 가장 싸다" 같은 대비 팩트 |
| **이미지강화형 적합도** | 스케일 수치 + 숨겨진 메커니즘 존재 | "연 3억 8천만 톤", "193개국 무비자" 같은 수치 + 왜·어떻게 해명 가능한 원리 |
| **과정추적형 적합도** | 명확한 전환점 + 시간 흐름 팩트 | "어쩌다 이렇게 됐나" 추적 가능한 시점·사건 존재 |

### 판단 방식

- 각 유형 적합도를 **1(약함)~5(매우 강함)** 으로 평가
- 주제별로 **최고 적합도 유형**을 예상 서사 유형으로 태그 부여
- 2개 유형이 비슷하게 강한 경우 복수 태그 허용 (P1 §2에서 최종 결정)

### P1 연계

P0에서 부여한 서사 유형 태그는 **P1 §2 "서사 유형 방향 판단"의 초기 힌트**로 사용된다. P1 단계에서 퀵서치 팩트를 수집한 뒤 최종 확정한다.

## 방식 A — API 검색

### 실행 절차

`scripts/keyword_search.py` 실행.

기간 기본값: **최근 7일** (`TIMEFRAME = "now 7-d"`)
유저가 기간을 언급하면 해당 기간으로 변경.

### 수행 내용

1. **Google Trends** — 키워드 검색량 비교 + 연관 키워드 추출
2. **YouTube API** — 경쟁 영상 조회수 + 오션 판별 (레드 / 퍼플 / 블루)
3. **네이버 API** — 최신 뉴스/블로그 반응 수집
4. **가중치 채점** — 한국 관련도×3 + 일상 연결성×2 + **3유형 적합도 최고값×2** + 보편 궁금증×1 + 30~50대 공감도×2 → 최종 주제 추천
   - 3유형 적합도 최고값: 모순형·이미지강화형·과정추적형 적합도 중 가장 높은 점수 (상단 "3유형 적합도 판단 기준" 참조)
   - 최종 주제 추천 시 **예상 서사 유형 태그** 함께 부여 (예: `[서사 유형: 모순형 — 역설 낙차 강]`)

## 방식 B — 참고 채널 분석

### 실행 절차

#### 1단계 — 채널 목록 로드

`차기주제_채널목록.md`를 읽어 등록된 채널 목록을 가져온다.
채널 추가·변경은 유저가 해당 파일을 직접 수정한다.

#### 2단계 — 채널 최근 영상 수집

등록된 채널별로 웹 검색하여 최근 영상 3개 제목을 수집한다.

```
검색어 예시:
  "[채널명] 유튜브 최근 영상"
  site:youtube.com "[채널명]"
```

- 수집한 제목에서 이미 다룬 주제 목록 작성 → 중복 방지 기준으로 사용

#### 3단계 — 주제 추천

기존 주제와 겹치지 않는 신규 주제를 2가지 유형으로 각 3개씩 추천한다.

**시의성 주제 (최근 이슈 기반)**
- 웹 검색으로 최신 뉴스·이슈 확인
- 채널 스타일(경제·국제·사회구조)에 맞는 이슈 선별
- 유효기간 최소 2~4주 이상
- 단순 사건·사고·연예·스포츠 제외

**에버그린 주제 (구조적 깊이)**
- 시간이 지나도 조회수가 유지되는 구조적 주제
- "왜?"가 있고 표층·심층 정보를 4층 이상 쌓을 수 있는 주제
- 채널이 아직 다루지 않은 분야 우선

### 출력 형식

```
[분석 채널: OOO 외 N개]
최근 커버 주제: [제목 요약]

──────────────────────────────
시의성 주제 추천
──────────────────────────────
① [주제명] [서사 유형: 모순형 / 이미지강화형 / 과정추적형 — 판단 근거]
   핵심 질문: ~
   유효기간: 약 N주
   추천 이유: ~

② [주제명] [서사 유형: ...]
③ [주제명] [서사 유형: ...]

──────────────────────────────
에버그린 주제 추천
──────────────────────────────
④ [주제명] [서사 유형: ...]
   핵심 질문: ~
   추천 이유: ~

⑤ [주제명] [서사 유형: ...]
⑥ [주제명] [서사 유형: ...]

──────────────────────────────
원하시는 번호를 선택해주세요.
```

**서사 유형 태그 부여 방식:**

- 주제별로 **3유형 적합도 평가** (상단 "3유형 적합도 판단 기준" 참조)
- 최고 적합도 유형을 태그로 표기 (예: `[서사 유형: 모순형 — 역설 낙차 강]`)
- 2개 유형이 비슷하게 강하면 복수 태그 허용 (예: `[서사 유형: 모순형 or 이미지강화형]`)
- 태그는 P1 §2 "서사 유형 방향 판단"의 초기 힌트로 사용됨 (P1에서 퀵서치 팩트로 최종 확정)

## 방식 A+B — 통합

1. **방식 B** 실행 → 주제 후보 6개 추출
2. 유저가 관심 있는 후보 선택 또는 전체 진행
3. 선택된 후보 키워드를 **방식 A** (`keyword_search.py`)에 입력하여 검증
4. 가중치 채점 결과 + B 추천 근거를 합산하여 최종 주제 추천

## 방식 C — 떡상 영상 발굴 (신규/소형 채널 우선)

### 핵심 전략

**대형 채널이 아닌 신규/소형 채널(구독자 1만~30만)의 떡상 영상**을 우선 발굴한다. 대형 채널은 충성 구독자와 알고리즘 가산점으로 조회수가 높지만, 소형 채널의 떡상 영상은 **순수하게 주제 자체가 알고리즘을 뚫었다는 직접 증거**다. 시작 단계의 본 채널이 같은 알고리즘 트리거를 노릴 수 있다.

**작동 원리 — 썸네일이 주제를 끌어낸다 (역추출 방식)**

방식 A·B는 "주제 → 썸네일" 순서지만, 방식 C는 **"떡상 썸네일 → 변형 썸네일 → 함의 주제 역추출"** 순서다. 썸네일은 주제를 발굴하는 진입점이며, 역추출된 주제는 **기존 P0 지침(3유형 적합도·가중치 채점)으로 검증**한다. 따라서 방식 C 결과도 방식 A·B와 동일한 평가 기준을 통과해야 한다.

> **원칙:** 대본 내용은 절대 카피하지 않는다. 떡상 썸네일은 주제 발굴 단서로만 쓰고, 제목·썸네일은 **각도 전환 3가지 방식**으로 다르게 만든다.

### 떡상 신호 기준

| 신호 | 기준값 |
|---|---|
| 구독자 규모 | 1만~30만 (소형 우선) |
| 비율 점수 | 조회수 ÷ 구독자 ≥ 5배 |
| 상대 점수 | 해당 영상 ÷ 채널 평균 조회수 ≥ 5배 |
| 절대 하한 | 조회수 ≥ 5만 |
| 영상 길이 | 4분 이상 (쇼츠 제외) |
| 업로드 | 최근 90일 이내 |

### 떡상 유형 자동 분류

스캐너가 영상 업로드 경과일 + 일평균 조회수 기준으로 자동 분류한다. 보고서는 유형별로 섹션을 분리해 출력한다.

| 유형 | 분류 기준 | 시그널 의미 | 활용 전략 |
|---|---|---|---|
| 🔥 신규 떡상 | 업로드 ≤30일 + 비율 ≥5배 | 알고리즘이 지금 밀어주는 중 + 시의성 트렌드 | 빠르게 진입 (라이프사이클 짧음) |
| 📈 중기 안정 | 업로드 31~60일 + 비율 ≥5배 | 안정 트렌드 | 일반 진행 |
| 🌱 후행 에버그린 | 업로드 >60일 + 일평균 조회수 ≥5천 | 장기 검색·추천 노출 가능성 | 천천히 만들어도 됨 (장기 가치) |

### 우상향 추세 검증 (VidIQ + Claude in Chrome)

스캐너는 누적 조회수만 측정 가능하므로 "최근 일자별 조회수 증가 추세"는 자체 측정하지 않는다. 대신 **VidIQ 크롬 확장의 'Views over time' 그래프**로 검증한다.

**검증 방법 2가지 중 선택:**

- **방법 1 — 수동 검증:** 영상 URL 클릭 → VidIQ 패널 'Views' 탭 → 'Views over time' 그래프 확인 → 우상향(꺾이지 않고 계속 증가) 신호 강할 때만 P1 진행
- **방법 2 — Claude in Chrome 자동 검증:** 보고서 하단에 자동 생성된 'Claude in Chrome 검증 프롬프트' 블록을 복사하여 Claude in Chrome 세션에 붙여넣기 → Claude가 상위 10개 영상을 순차 방문하여 추세 일괄 보고

전제 조건: 브라우저에 VidIQ 무료 플랜 설치 + 로그인 + (방법 2 선택 시) Claude in Chrome 활성 세션.

### 실행 절차 (7단계)

1. **스캐너 실행**

   ```bash
   python scripts/knowledge_video_scanner.py
   ```

   4경로 병행 검색을 자동 수행한다.
   - Path 1 — YouTube 카테고리 인기 영상 (mostPopular)
   - Path 2 — 키워드 폭격 검색 (지식 영상 패턴 키워드)
   - Path 3 — 시드 채널 댓글 채널 마이닝
   - Path 4 — 외부 분석 사이트 (수동 큐레이션 보조)

2. **보고서 검토 + 1차 선별**

   `대본/00_떡상 썸네일/[떡상 썸네일]_YYYYMMDD_HHMM.md` 생성됨. 보고서에는 영상별 썸네일 이미지가 임베드되어 시각 비교 가능. 본 채널 톤(지식·경제·국제·역사·심리·사회구조) 부합 영상 5~10개를 1차 선별.

3. **변형 썸네일 3안 생성**

   1차 선별된 원본 떡상 썸네일마다 아래 3가지 방식으로 변형. 단순 동의어 교체 금지.

   | 전환 방식 | 설명 | 예시 |
   |---|---|---|
   | 욕구 전환 | 같은 주제를 다른 감정 동기로 | "배터리 절약" → "배터리 하루 버티는 설정" |
   | 주체 전환 | 대상·행위자를 다르게 | "대기업 절세법" → "개인이 따라 할 수 있는 절세법" |
   | 시점 전환 | 과거·현재·미래 중 다른 시점 강조 | "왜 망했나" → "이렇게 되면 망한다" |

4. **변형 썸네일 → 함의 주제 역추출**

   변형 썸네일 각각이 다루게 될 주제를 한 줄로 명시. 변형마다 주제가 달라야 함(같으면 변형 실패).

   - 예: "한국 정유사가 더러운 원유를 더 비싸게 사는 이유" → 주제: 한국 정유사의 중동 원유 의존 구조

5. **역추출 주제에 기존 지침 평가 적용**

   - **3유형 적합도** 판정 (모순형·이미지강화형·과정추적형 중 어느 유형에 적합한지, 1~5점)
   - **가중치 채점:** 한국 관련도×3 + 일상 연결성×2 + 3유형 적합도 최고값×2 + 보편 궁금증×1 + 30~50대 공감도×2
   - 본 채널 톤 부합도 보조 점검

6. **점수순 정렬 + 유저 보고**

   각 항목에 다음을 모두 표시한다.
   - **제목** (영상 제목 — 검색용 긴 텍스트)
   - **썸네일 문구** (썸네일 이미지의 텍스트 오버레이 — 후킹 카피, OCR 자동 추출)
   - **썸네일 이미지** (자동 다운로드 — `대본/00_떡상 썸네일/thumbnails/{제목}_{video_id}.jpg`, 보고서에 마크다운 임베드)
   - **링크** (`https://youtu.be/...`)
   - **업로드일·경과일** (표기 형식: `2026-04-21 (19일 전)`)
   - **채널명·구독자·조회수·비율 점수·상대 점수** (검증용)
     - 표기 형식: `구독자 N만 / 조회수 N만 / 비율 점수 N배 / 상대 점수 N배`
     - 비율 점수: 구독자 대비 영상 조회수 (조회수 ÷ 구독자)
     - 상대 점수: 평균 대비 영상 조회수 (영상 조회수 ÷ 채널 평균 조회수)

   **명칭 통일:** 보고서·지침·대화에서 영상 텍스트 정보는 "제목"(긴 검색용)·"썸네일 문구"(짧은 후킹 카피)로 통일한다. "원본" 표현 사용 금지.
   - **변형 썸네일** (가제 후보)
   - **역추출 주제**
   - **서사 유형 태그** + **가중치 점수**

   유저가 변형 썸네일 후보를 평가할 때 원본을 즉시 확인할 수 있도록 한다.

7. **유저 선택 → GATE 0 진행**

   선택된 변형 썸네일은 P1 가제로, 역추출 주제는 P1 주제로 인계.

### 시드 풀 관리

발굴된 채널은 `지식채널_시드풀.md`에 자동 등록되어 다음 스캔의 댓글 마이닝(Path 3) 기준이 된다. 유저가 직접 등록·제외 가능.

### 자동화 수준

수동 실행 권장(P0 진입 시점). API 쿼터를 신선한 데이터에 집중 사용한다.
