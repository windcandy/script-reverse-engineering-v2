## 3유형 적합도 판단 기준 (방식 A·B 공통)

> **3유형(모순형·이미지강화형·과정추적형) 정의·8엔진 매칭 본체 → `모듈_엔진.md` §0 참조.** 본 §은 발굴 단계의 적합도 판단 기준(주제 발굴 신호)만 다룬다.

주제 후보를 발굴할 때 각 주제가 **어느 서사 유형에 적합한지** 평가한다. 이는 가중치 점수(방식 A) + 주제 태그(방식 A·B 출력)로 반영된다.

### 3유형 적합도 지표

| 서사 유형 | 적합도 판단 지표 | 강한 신호 |
|---|---|---|
| **모순형 적합도** | A극·B극 낙차 + 역설 강도 | "금보다 비쌌던 소금이 지금 가장 싸다" 같은 대비 팩트 |
| **이미지강화형 적합도** | 영역 확장 가능성(친숙 대상 → 이질·거시 영역 연결) + 숨겨진 메커니즘 존재 | "193개국 무비자"·"연 3억 8천만 톤"처럼 친숙 대상이 예상 밖 영역·규모로 확장되는 팩트 + 왜·어떻게 해명 가능한 원리 |
| **과정추적형 적합도** | 명확한 전환점 + 시간 흐름 팩트 | "어쩌다 이렇게 됐나" 추적 가능한 시점·사건 존재 |

### 판단 방식

- 각 유형 적합도를 **1(약함)~5(매우 강함)** 으로 평가
- 주제별로 **최고 적합도 유형**을 예상 서사 유형으로 태그 부여
- 2개 유형이 비슷하게 강한 경우 복수 태그 허용 (P1 §2에서 최종 결정)

### P1 연계

P0에서 부여한 서사 유형 태그는 **P1 §2 "서사 유형 방향 판단"의 초기 힌트**로 사용된다. P1 단계에서 각 후보의 퀵서치 팩트를 수집한 뒤 최종 확정한다.

### 시의성 이슈 활용 원칙 (방식 무관 공통)

현재 이슈를 제목·썸네일 후킹으로 활용하는 주제는 본문을 해당 이슈가 생겨난 배경·원인·작동 원리·구조적 역학으로 전환 가능한지 함께 확인한다. 본문에서 현재 진행 중인 사안의 옳고 그름을 판단하거나 특정 입장을 직접 지지하는 방향으로 전개되는 주제는 선별에서 제외한다.

## 방식 A — API 검색

### 실행 절차

`scripts/keyword_search.py` 실행.

기간 기본값: **최근 7일** (`TIMEFRAME = "now 7-d"`)
유저가 기간을 언급하면 해당 기간으로 변경.

### 수행 내용

1. **Google Trends** — 키워드 검색량 비교 + 연관 키워드 추출
2. **YouTube API** — 경쟁 영상 조회수 + 오션 판별 (레드 / 퍼플 / 블루)
3. **네이버 API** — 최신 뉴스/블로그 반응 수집
4. **가중치 채점** — 한국 관련도×3 + 일상 연결성×2 + **3유형 적합도 최고값×2** + 보편 궁금증×1 + 30~50대 공감도×2 → 상위 6개 주제 후보 추출
   - 3유형 적합도 최고값: 모순형·이미지강화형·과정추적형 적합도 중 가장 높은 점수 (상단 "3유형 적합도 판단 기준" 참조)
   - 각 후보에 **예상 서사 유형 태그** 함께 부여 (예: `[서사 유형: 모순형 — 역설 낙차 강]`)
5. **시의성·에버그린 분류 + 제시** — 4에서 추출한 6개를 분류하여 방식 B 형식으로 제시 → 유저가 3개 선택
   - 시의성: 최근 이슈·트렌드 기반 검색 급등 주제 (방식 B 시의성 기준 동일 적용)
   - 에버그린: 구조적·역사적·반복 검색 주제 (방식 B 에버그린 기준 동일 적용)
   - 한쪽 유형 0개 시: 전체 제시 후 "(에버그린 0개)" 또는 "(시의성 0개)" 명시

## 방식 B — 참고 채널 분석

### 실행 절차

#### 1단계 — 채널 목록 로드

`차기주제_채널목록.md`를 읽어 등록된 채널 목록을 가져온다.
채널 추가·변경은 유저가 해당 파일을 직접 수정한다.

#### 2단계 — 채널 최근 영상 수집

등록된 채널별로 웹 검색하여 최근 영상 3개 제목을 수집한다.

```
검색어 예시:
  "[채널명] 유튜브 최근 영상"
  site:youtube.com "[채널명]"
```

- 수집한 제목에서 이미 다룬 주제 목록 작성 → 중복 방지 기준으로 사용

#### 3단계 — 주제 추천

기존 주제와 겹치지 않는 신규 주제를 2가지 유형으로 각 3개씩 추천한다.

**시의성 주제 (최근 이슈 기반)**
- 웹 검색으로 최신 뉴스·이슈 확인
- 본 채널 톤(지식·경제·국제·역사·사회구조)에 맞는 이슈 선별
- 유효기간 최소 2~4주 이상
- 단순 사건·사고·연예·스포츠 제외

**에버그린 주제 (구조적 깊이)**
- 시간이 지나도 조회수가 유지되는 구조적 주제
- "왜?"가 있고 표층·심층 정보를 4층 이상 쌓을 수 있는 주제
- 채널이 아직 다루지 않은 분야 우선

### 출력 형식

```
[분석 채널: OOO 외 N개]
최근 커버 주제: [제목 요약]

──────────────────────────────
시의성 주제 추천
──────────────────────────────
① [주제명] [서사 유형: 모순형 / 이미지강화형 / 과정추적형 — 판단 근거]
   핵심 질문: ~
   유효기간: 약 N주
   추천 이유: ~

② [주제명] [서사 유형: ...]
③ [주제명] [서사 유형: ...]

──────────────────────────────
에버그린 주제 추천
──────────────────────────────
④ [주제명] [서사 유형: ...]
   핵심 질문: ~
   추천 이유: ~

⑤ [주제명] [서사 유형: ...]
⑥ [주제명] [서사 유형: ...]

──────────────────────────────
P1에서 탐색할 3개를 선택해주세요.
```

## 방식 A+B — 통합

1. **방식 B** 실행 → 주제 후보 6개 추출
2. 유저가 관심 있는 후보 선택 또는 전체 진행
3. 선택된 후보 키워드를 **방식 A** (`keyword_search.py`)에 입력하여 검증
4. 가중치 채점 결과 + B 추천 근거를 합산하여 6개 후보 제시 → 유저가 3개 선택

## 방식 C — 떡상 영상 발굴 (신규/소형 채널 우선)

### 핵심 전략

**우선 발굴 대상은 신규/소형 채널(구독자 1만~30만)의 떡상 영상**이다. 대형 채널은 충성 구독자와 알고리즘 가산점으로 조회수가 높지만, 소형 채널의 떡상 영상은 **순수하게 주제 자체가 알고리즘을 뚫었다는 직접 증거**다. 시작 단계의 본 채널이 같은 알고리즘 트리거를 노릴 수 있다.

**대형 채널(구독자 30만 초과)도 함께 발굴하되 별도 섹션으로 분리** — 변형 분석 대상에서 제외하고 트렌드 참고용으로만 노출 (본 채널 재현성 낮음).

**작동 원리 — 썸네일이 주제를 끌어낸다 (역추출 방식)**

방식 A·B는 "주제 → 제목·썸네일" 순서지만, 방식 C는 **"떡상 영상 발굴 → 변형(가제+썸네일 문구 짝) 생성 → 함의 주제 역추출"** 순서다. 떡상 영상은 주제를 발굴하는 진입점이며, 역추출된 주제는 **기존 P0 지침(3유형 적합도·가중치 채점)으로 검증**한다. 따라서 방식 C 결과도 방식 A·B와 동일한 평가 기준을 통과해야 한다.

> **원칙:** 대본 내용은 절대 카피하지 않는다. 떡상 썸네일은 주제 발굴 단서로만 쓰고, 제목·썸네일은 **5가지 감정 방향**(충격/궁금증/비교/다큐/사회문화)으로 다르게 만든다.

### 용어 정의 (모호성 제거 — 새 세션 AI도 즉시 이해 가능)

본 방식 C에서 사용하는 모든 용어를 명확히 정의한다. 모호한 단축형은 사용 금지.

| 용어 | 정의 | 계산식 | 단위 |
|---|---|---|---|
| **구독자 대비** | 영상 조회수가 채널 구독자 수의 몇 배인지 (외부 시청자 유입 강도) | 영상 조회수 ÷ 채널 구독자 수 | 배 |
| **평균 대비** | 영상 조회수가 채널 평균 영상 조회수의 몇 배인지 (채널 내 이변성) | 영상 조회수 ÷ 채널 평균 영상 조회수 | 배 |
| **영상당 구독자** | 채널 영상 1개당 평균 구독자 수 (채널 효율성) | 채널 구독자 수 ÷ 채널 총 영상 수 | 명 |
| **떡상** | 조회수가 이례적으로 많이 나온 상태. 본 시스템에서는 정량화함 | 구독자 대비 ≥ 5배 OR 평균 대비 ≥ 5배 | 정성 |
| **신호 강도** | 떡상의 강도를 나타내는 단일 지표 | max(구독자 대비, 평균 대비) | 배 |
| **영상당 구독자 보너스** | 영상당 구독자 값에 따른 가중치 가산점 (채널 자체 품질 신호) | 1,000 이상 +1 / 5,000 이상 +2 / 10,000 이상 +3 | 점 |
| **소형 채널 강신호 보너스** | 작은 채널 + 강한 신호 조합 가산점 (본 채널 재현 가능성 신호) | 신호 강도 × 구독자 크기 매트릭스 (하단 표 참조) | 점 |
| **최신성 보너스** | 영상 업로드 경과일에 따른 가산점 (알고리즘 푸시 윈도우 반영) | ≤7일 +3 / 8~14일 +2 / 15~30일 +1 / 31일~ 0 | 점 |
| **썸네일 문구 (메인)** | 썸네일 이미지 큰 글씨 텍스트 — 영상의 후킹 카피 | OCR 결과 중 글자 박스 height ≥ 최대 height × 60% | 텍스트 |
| **썸네일 문구 (서브)** | 썸네일 이미지 작은 글씨 (말풍선·부가 설명) — 참고용 | OCR 결과 중 글자 박스 height < 최대 height × 60% | 텍스트 |
| **감정 방향** | 변형의 후킹 톤 5가지 분류 | 충격형 / 궁금증형 / 비교형 / 다큐형 / 사회문화형 | 카테고리 |
| **서사 유형** | 본 채널 콘텐츠 전개 구조 3가지 분류 | 모순형 / 이미지강화형 / 과정추적형 | 카테고리 |
| **추천 조합** | 변형 풀(5~15개) 중 점수·전략별로 선정한 최종 후보 3개 | A 무난형 / B CTR 강함 / C 다큐형 | 조합 |
| **감정 포인트** | 영상이 시청자에게 자극하는 감정·욕구 4~5개 | (Step 1 분석 결과) | 키워드 묶음 |
| **알고리즘 키워드 조합** | 알고리즘이 연관 영상으로 노출시킬 핵심 키워드 조합 | (Step 2 분석 결과) | 키워드 묶음 |
| **보존 핵심 키워드** | 변형 시 절대 빼면 안 되는 명사·고유명사 5~7개 (이 풀 중 1~3개를 가제·썸네일 양쪽에 반드시 포함 — §원칙 1) | (Step 3 분석 결과) | 키워드 묶음 |

**금지 표현:** 단순 "비율" / "상대" / "효율" 단독 사용 금지 (반드시 위 용어 사용).
**변형 입력 원칙:** 변형은 반드시 **썸네일 문구 (메인)** 만 기준으로 한다. 서브는 참고용으로만 표시.
**변형 우선순위 원칙:** **썸네일 문구가 1순위(시청자 1차 인지·CTR 결정), 가제(제목)가 2순위(검색·SEO + 보조)**. 사고 순서는 썸네일 문구 먼저 → 가제는 그것을 SEO 형식으로 확장.
**감정 방향 vs 서사 유형:** 둘은 다른 차원의 분류. 감정 방향은 후킹 톤(방식 C 내부 변형 분류 차원), 서사 유형은 콘텐츠 구조(P1·P3·P4용).

### 떡상 신호 기준

**필수 조건 (AND 모두 충족):**

| 신호 | 기준값 |
|---|---|
| 구독자 규모 | 1만 이상 (1만 미만 제외 / 30만 초과는 별도 섹션 분류) |
| 절대 하한 | 조회수 ≥ 5만 |
| 영상 길이 | 4분 이상 (쇼츠 제외) |
| 업로드 | 최근 90일 이내 |

**떡상 신호 조건 (OR — 둘 중 하나 충족):**

| 신호 | 기준값 |
|---|---|
| 구독자 대비 | 조회수 ÷ 구독자 ≥ 5배 |
| 평균 대비 | 해당 영상 ÷ 채널 평균 조회수 ≥ 5배 |

### 비지식 영상 3중 제외 필터 (스캐너 자동)

본 채널은 지식 집중 채널이므로 뉴스·정치·연예·게임·드라마·브랜드 콘텐츠를 제외한다. 스캐너에서 다음 3단계로 자동 필터링.

| 필터 | 검사 대상 | 예시 키워드 |
|---|---|---|
| EXCLUDE_CHANNEL_KEYWORDS | 채널명 | MBC·KBS·뉴스·은행·카드·보험·증권·통신·BMW·로블록스·**심리** 등 |
| EXCLUDE_TITLE_KEYWORDS | 영상 제목 | 윤석열·이재명·먹방·참교육·대리뷰·탐방·진엔딩·공략법·≪≫·**뇌과학·심리학** 등 |
| EXCLUDE_THUMBNAIL_KEYWORDS | OCR 추출 썸네일 문구 | 참교육·먹방·광팬·엔딩·공략법·ㅋㅋ·ㅎㅎ 등 |

이 3중 필터에도 누락된 비지식 영상은 메인 클로드의 2단계 "1차 선별"에서 추가 제거한다.

### 떡상 유형 자동 분류

스캐너가 **채널 구독자 수 + 영상 업로드 경과일 + 일평균 조회수** 기준으로 자동 분류한다. 보고서는 유형별로 섹션을 분리해 출력한다. **30만 초과 채널은 다른 분류보다 우선 적용**되어 🏢 대형 채널 그룹으로 분류된다.

| 유형 | 분류 기준 | 시그널 의미 | 활용 전략 |
|---|---|---|---|
| 🔥 신규 떡상 | 업로드 ≤30일 + (구독자 대비 ≥5배 OR 평균 대비 ≥5배) | 알고리즘이 지금 밀어주는 중 + 시의성 트렌드 | 빠르게 진입 (라이프사이클 짧음) |
| 📈 중기 안정 | 업로드 31~60일 + (구독자 대비 ≥5배 OR 평균 대비 ≥5배) | 안정 트렌드 | 일반 진행 |
| 🌱 후행 에버그린 | 업로드 >60일 + 일평균 조회수 ≥5천 | 장기 검색·추천 노출 가능성 | 천천히 만들어도 됨 (장기 가치) |
| 📊 일반 | 위 3유형 미해당 | 보통 신호 | 보조 검토 |
| 🏢 대형 채널 | 구독자 30만 초과 (다른 분류 우선 적용) | 본 채널 재현성 낮음 (대형 알고리즘 가산점) | **변형 분석 대상에서 제외 — 참고용 별도 섹션** |

**대형 채널 처리 원칙:** 30만 초과 채널 영상은 다른 유형보다 우선해서 "🏢 대형 채널" 섹션으로 분류된다. 본 채널 재현성이 낮아 메인 클로드의 1차 선별·변형 분석에서 제외하고 참고용으로만 노출한다.

**시의성·에버그린 레이블:** 역추출 주제에 아래 대응으로 레이블을 병기한다.
- 🔥 신규 떡상·📈 중기 안정 → [시의성]
- 🌱 후행 에버그린 → [에버그린]
- 📊 일반 → 역추출 주제 성격으로 판단 (방식 B 시의성·에버그린 기준 동일 적용)

### 우상향 추세 검증 (VidIQ + Claude in Chrome)

스캐너는 누적 조회수만 측정 가능하므로 "최근 일자별 조회수 증가 추세"는 자체 측정하지 않는다. 대신 **VidIQ 크롬 확장의 'Views over time' 그래프**로 검증한다.

**검증 방법 2가지 중 선택:**

- **방법 1 — 수동 검증:** 영상 URL 클릭 → VidIQ 패널 'Views' 탭 → 'Views over time' 그래프 확인 → 우상향(꺾이지 않고 계속 증가) 신호 강할 때만 P1 진행
- **방법 2 — Claude in Chrome 자동 검증:** 보고서 하단에 자동 생성된 'Claude in Chrome 검증 프롬프트' 블록을 복사하여 Claude in Chrome 세션에 붙여넣기 → Claude가 상위 10개 영상을 순차 방문하여 추세 일괄 보고

전제 조건: 브라우저에 VidIQ 무료 플랜 설치 + 로그인 + (방법 2 선택 시) Claude in Chrome 활성 세션.

### 실행 절차 (6단계)

1. **스캐너 실행**

   ```bash
   python scripts/knowledge_video_scanner.py
   ```

   4경로 병행 검색을 자동 수행한다.
   - Path 1 — YouTube 카테고리 인기 영상 (mostPopular)
   - Path 2 — 키워드 폭격 검색 (지식 영상 패턴 키워드)
   - Path 3 — 시드 채널 댓글 채널 마이닝
   - Path 4 — 외부 분석 사이트 (수동 큐레이션 보조)

2. **보고서 검토 + 본 채널 톤 부합 영상 1차 선별 (영상 단위)**

   `대본/00_떡상 썸네일/[떡상 썸네일]_YYYYMMDD_HHMM.md` 생성됨. 보고서에는 영상별 썸네일 이미지가 임베드되어 시각 비교 가능. 본 채널 톤(지식·경제·국제·역사·사회구조) 부합 영상 5~10개 선별.

   **선별 대상:** 🔥 신규 떡상 / 📈 중기 안정 / 🌱 후행 에버그린 / 📊 일반 4그룹.
   **선별 제외:** 🏢 대형 채널 그룹 (재현성 낮음 — 참고용으로만 활용).

3. **변형 5~15안 생성 (감정 방향당 1~3개 — 썸네일 문구 우선 + 가제 + 역추출 주제)**

   1차 선별 영상마다 아래 분석 흐름을 거쳐 5가지 감정 방향별로 1~3개씩 = **총 5~15개 변형**을 생성한다. 개수는 자율 정지 규칙(자체 검증 체크리스트 참조)에 따라 가변.

   **참고 입력:** 제목(완전 맥락) + **썸네일 문구 (메인)**(실제 후킹 카피)
   (썸네일 문구 (서브)는 변형 기준에서 제외 — 말풍선·부가 설명은 후킹 카피가 아님)

   **변형 1개 = 썸네일 문구 + 가제 + 역추출 주제 (시의성·에버그린 레이블 포함, 3개 항목 한 묶음)**

   #### 우선순위 — 썸네일 문구가 1순위 (시청자 인지 순서)

   시청자는 알고리즘 추천 화면에서 **썸네일 이미지 + 큰 글씨를 먼저 인지하고 클릭을 결정**한다. 제목은 썸네일 본 후 또는 검색 결과에서 인지하는 2차 정보.

   | 항목 | 우선순위 | 길이 | 역할 |
   |---|---|---|---|
   | **썸네일 문구** | **1순위** (CTR 결정) | 원본 썸네일 문구 글자수 ±20% 폭 (예: 원본 20자 → 변형 16~24자) | 1차 인지·클릭 후킹 |
   | 가제 (제목) | 2순위 (검색·SEO) | 15~30자 | 검색 노출 + 클릭 후 정보 보강 |
   | 역추출 주제 | (분석용) | 한 줄 명시 | 변형이 다루게 될 주제 (변형마다 달라야 함) |

   **사고 순서:** ① 썸네일 문구 먼저 생성 (후킹 + 키워드) → ② 가제는 썸네일 문구를 SEO 형식으로 확장 → ③ 역추출 주제 도출

   #### 분석 흐름 1 — 영상의 감정 포인트 4~5개 추출

   원본 영상이 시청자에게 어떤 감정·욕구를 자극하는지 4~5개 키워드로 정리.

   > 예 (칠면조 영상):
   > - 미국 문화의 숨겨진 이유
   > - 한국 vs 미국의 식문화 차이
   > - 우리가 몰랐던 의외성
   > - "왜 굳이 그걸?" 호기심

   #### 분석 흐름 2 — 알고리즘 반응 키워드 조합 정의

   알고리즘이 연관 영상으로 노출시킬 핵심 키워드 조합을 추출.

   > 예: **미국 + 칠면조 + 의외성 + 문화충돌**

   #### 분석 흐름 3 — 보존 핵심 키워드 식별

   변형 시 절대 빼면 안 되는 명사·고유명사·식별자 5~7개.

   > 예: 미국 / 칠면조 / 추수감사절 / 미국인 / 한국인

   #### 분석 흐름 4 — 5가지 감정 방향별 변형 1~3개씩 생성 (총 5~15개)

   **각 카테고리당 변형 1~3개 = 총 5~15개 생성.** 자율 정지 규칙은 자체 검증 체크리스트 참조. 아래 표는 카테고리당 예시 1개만 표시.

   | 감정 방향 | 톤 | 예시 (가제 / 썸네일 문구) |
   |---|---|---|
   | **충격형** | 의외성·반전·놀라움 | "한국에선 망했는데 미국은 환장하는 음식" / "한국:안 먹음 / 미국:없으면 난리" |
   | **궁금증형** | 호기심·미스터리 | "미국인들이 왜 이 큰 새를 먹을까" / "왜 굳이 이걸?" |
   | **비교형** | 한국 vs 미국, A vs B | "미국은 환장 한국은 외면" / "미국 vs 한국 식탁" |
   | **다큐형** | 역사·교양·구조 | "미국은 어떻게 칠면조의 나라가 되었나" / "미국의 국민 음식" |
   | **사회문화형** | 문화충돌·세대·전통 | "추수감사절의 진짜 주인공은 왜 칠면조일까" / "미국의 이상한 전통" |

   **변형 원칙 — 두 가지 동시 만족 (필수)**

   **[원칙 1] 연관 영상 노출 가능성 유지**
   알고리즘이 유사 주제로 인식하여 추천 영상 슬롯에 노출되도록 함.
   - **보존 핵심 키워드 1~3개를 가제와 썸네일 문구 양쪽에 반드시 포함**
   - 추상적 슬로건 금지 (예: "끊는 날, 며칠?" 같이 무엇인지 모호한 표현 — 키워드 빠지면 알고리즘 매칭 실패)

   **[원칙 2] 복제 느낌 줄이기**
   원본 영상의 표현을 그대로 따라가지 않음 (저작권·중복 회피).
   - **문장 구조 변형:** 의문문 ↔ 평서문, 능동 ↔ 수동, 서술 시점 변경
   - **표현·어휘 변경:** 원본 단어 묶음 그대로 옮기기 금지 (의미는 유지하되 다른 어휘)
   - **각도·시점 전환:** 다른 주체·시점·욕구로 5가지 감정 방향 다양화

   **[자체 검증 체크리스트 — 변형 1개 추가 시마다 통과 필수]**
   새 변형을 풀에 추가하기 전 아래 4개 항목 점검. 하나라도 NO면 변형 폐기 + 해당 카테고리 작성 종료 (자율 정지).
   - [ ] 보존 핵심 키워드가 가제·썸네일 양쪽에 1~3개 포함되었는가? (원칙 1)
   - [ ] 원본·기존 변형과 보존 키워드 외 연속 단어 4개 이상 겹치지 않는가? (원칙 2 — 표현 복제 차단. 보존 키워드는 의도적 포함이라 카운트 제외)
   - [ ] 같은 카테고리 내 기존 변형과 역추출 주제가 본질적으로 다른가? (원칙 2 — 구조 중복 차단. 단어만 다르고 통찰이 같으면 NO)
   - [ ] 원본을 모르고 봐도 매력적인 후킹인가?

   - 변형마다 역추출 주제 함께 부여

   #### 모범 패턴 — 챗지피티 분석 사고 적용 사례 (칠면조 영상)

   **원본:**
   - 제목: "미국인들도 잘 모르는 칠면조의 진실 (한국에서 안 먹는 이유)"
   - 썸네일: "미국은 왜 / 칠면조를 먹나"
   - 보존 핵심 키워드: 미국, 칠면조, 추수감사절, 미국인, 한국인

   **변형 예시 (모범):**

   | 감정 방향 | 가제 | 썸네일 문구 | 키워드 보존 | 구조 변형 |
   |---|---|---|---|---|
   | 충격형 | "한국에선 망했는데 미국은 환장하는 음식" | "한국: 안 먹음 / 미국: 없으면 난리" | 한국·미국 ✅ | 의문 → 대비 구조 |
   | 궁금증형 | "미국인들이 왜 이 큰 새를 먹을까" | "미국은 왜 / 이 큰 새를 먹나?" | 미국·미국인 ✅ | 표현 변경 (칠면조 → 큰 새) |
   | 비교형 | "한국인은 안 먹는데 미국은 환장하는 음식" | "한국: 안 먹음 / 미국: 없으면 난리" | 한국·미국·한국인 ✅ | 대비 구조 신설 |
   | 다큐형 | "미국은 어떻게 칠면조의 나라가 되었나" | "미국의 국민 음식" | 미국·칠면조 ✅ | 평서문 변환 |
   | 사회문화형 | "추수감사절의 진짜 주인공은 왜 칠면조일까" | "추수감사절 필수템" | 추수감사절·칠면조 ✅ | 명사구화 |

   → 모든 변형이 **키워드 보존(원칙 1) + 다른 표현·구조·각도(원칙 2)** 동시 만족.

4. **추천 조합 3가지 선정 + 점수 적용**

   변형 풀(5~15개) 모두 점수를 부여하지 않는다(보고서 비대화 방지). 대신 3개 조합을 다른 기준으로 선정 후 그 3개에만 가중치 점수 부여.

   #### 추천 조합 3가지 선정 기준 (감정 방향 분배)

   변형 풀(5~15개, 5가지 감정 방향 × 1~3개)을 3개 조합에 분배 선정.

   | 조합 | 대상 감정 방향 | 선정 기준 |
   |---|---|---|

### 발굴 절차

1. **커뮤니티별 인기글 수집** — 5곳 × 5개 내외 = 총 25개 후보
2. **주제 필터링** — 수집 조건 적용하여 채널 주제 부합 게시물 선별
3. **중복 제거** — 같은 주제가 복수 커뮤니티에서 반복 시 반응 가장 높은 커뮤니티 기준 1개로 통합
4. **핵심 키워드 추출 + 주제 설계** — 선별된 게시물별로 핵심 키워드(인물·사건·개념·구조)를 추출하고, 해당 키워드를 바탕으로 채널 3유형(모순형·이미지강화형·과정추적형)에 맞는 주제를 새로 설계한다. 게시물 제목을 그대로 주제로 전환하지 않는다.
5. **시의성·에버그린 분류 + 서사 유형 확정** — 방식 B 기준 동일 적용 / 4에서 설계한 주제에 서사 유형 태그 부여
6. **시의성 3개 + 에버그린 3개 제시** → 유저가 3개 선택

## 방식 E — 유저 제공 영상 정보 기반 발굴

### 핵심 전략

유저가 직접 찾은 영상의 정보(제목·썸네일 문구 중 1개 이상)를 입력하면, 핵심 키워드를 추출하여 채널 3유형에 맞는 시의성·에버그린 주제를 설계한다.

### 입력 형식

유저가 아래 중 1개 이상 제공:
- 영상 제목
- 썸네일 문구 (이미지에서 읽은 텍스트)

복수 영상 동시 입력 가능. 영상당 독립적으로 분석한다.

### 발굴 절차

1. **핵심 키워드 추출** — 제목·썸네일 문구에서 인물·사건·개념·구조 키워드 추출
2. **주제 설계** — 추출한 키워드를 바탕으로 채널 3유형(모순형·이미지강화형·과정추적형)에 맞는 주제를 새로 설계한다. 입력 제목·문구를 그대로 주제로 전환하지 않는다.
3. **시의성·에버그린 분류 + 서사 유형 확정** — 방식 B 기준 동일 적용
4. **시의성 3개 + 에버그린 3개 제시** → 유저가 3개 선택

### 출력 형식

방식 B·D 출력 형식과 동일. [출처] 항목에 유저 제공 영상 제목·키워드를 표기한다.
