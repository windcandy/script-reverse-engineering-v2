## 3유형 적합도 판단 기준 (방식 A·B 공통)

주제 후보를 발굴할 때 각 주제가 **어느 서사 유형에 적합한지** 평가한다. 이는 가중치 점수(방식 A) + 주제 태그(방식 A·B 출력)로 반영된다.

### 3유형 적합도 지표

| 서사 유형 | 적합도 판단 지표 | 강한 신호 |
|---|---|---|
| **모순형 적합도** | A극·B극 낙차 + 역설 강도 | "금보다 비쌌던 소금이 지금 가장 싸다" 같은 대비 팩트 |
| **이미지강화형 적합도** | 스케일 수치 + 숨겨진 메커니즘 존재 | "연 3억 8천만 톤", "193개국 무비자" 같은 수치 + 왜·어떻게 해명 가능한 원리 |
| **과정추적형 적합도** | 명확한 전환점 + 시간 흐름 팩트 | "어쩌다 이렇게 됐나" 추적 가능한 시점·사건 존재 |

### 판단 방식

- 각 유형 적합도를 **1(약함)~5(매우 강함)** 으로 평가
- 주제별로 **최고 적합도 유형**을 예상 서사 유형으로 태그 부여
- 2개 유형이 비슷하게 강한 경우 복수 태그 허용 (P1 §2에서 최종 결정)

### P1 연계

P0에서 부여한 서사 유형 태그는 **P1 §2 "서사 유형 방향 판단"의 초기 힌트**로 사용된다. P1 단계에서 퀵서치 팩트를 수집한 뒤 최종 확정한다.

## 방식 A — API 검색

### 실행 절차

`scripts/keyword_search.py` 실행.

기간 기본값: **최근 7일** (`TIMEFRAME = "now 7-d"`)
유저가 기간을 언급하면 해당 기간으로 변경.

### 수행 내용

1. **Google Trends** — 키워드 검색량 비교 + 연관 키워드 추출
2. **YouTube API** — 경쟁 영상 조회수 + 오션 판별 (레드 / 퍼플 / 블루)
3. **네이버 API** — 최신 뉴스/블로그 반응 수집
4. **가중치 채점** — 한국 관련도×3 + 일상 연결성×2 + **3유형 적합도 최고값×2** + 보편 궁금증×1 + 30~50대 공감도×2 → 최종 주제 추천
   - 3유형 적합도 최고값: 모순형·이미지강화형·과정추적형 적합도 중 가장 높은 점수 (상단 "3유형 적합도 판단 기준" 참조)
   - 최종 주제 추천 시 **예상 서사 유형 태그** 함께 부여 (예: `[서사 유형: 모순형 — 역설 낙차 강]`)

## 방식 B — 참고 채널 분석

### 실행 절차

#### 1단계 — 채널 목록 로드

`차기주제_채널목록.md`를 읽어 등록된 채널 목록을 가져온다.
채널 추가·변경은 유저가 해당 파일을 직접 수정한다.

#### 2단계 — 채널 최근 영상 수집

등록된 채널별로 웹 검색하여 최근 영상 3개 제목을 수집한다.

```
검색어 예시:
  "[채널명] 유튜브 최근 영상"
  site:youtube.com "[채널명]"
```

- 수집한 제목에서 이미 다룬 주제 목록 작성 → 중복 방지 기준으로 사용

#### 3단계 — 주제 추천

기존 주제와 겹치지 않는 신규 주제를 2가지 유형으로 각 3개씩 추천한다.

**시의성 주제 (최근 이슈 기반)**
- 웹 검색으로 최신 뉴스·이슈 확인
- 채널 스타일(경제·국제·사회구조)에 맞는 이슈 선별
- 유효기간 최소 2~4주 이상
- 단순 사건·사고·연예·스포츠 제외

**에버그린 주제 (구조적 깊이)**
- 시간이 지나도 조회수가 유지되는 구조적 주제
- "왜?"가 있고 표층·심층 정보를 4층 이상 쌓을 수 있는 주제
- 채널이 아직 다루지 않은 분야 우선

### 출력 형식

```
[분석 채널: OOO 외 N개]
최근 커버 주제: [제목 요약]

──────────────────────────────
시의성 주제 추천
──────────────────────────────
① [주제명] [서사 유형: 모순형 / 이미지강화형 / 과정추적형 — 판단 근거]
   핵심 질문: ~
   유효기간: 약 N주
   추천 이유: ~

② [주제명] [서사 유형: ...]
③ [주제명] [서사 유형: ...]

──────────────────────────────
에버그린 주제 추천
──────────────────────────────
④ [주제명] [서사 유형: ...]
   핵심 질문: ~
   추천 이유: ~

⑤ [주제명] [서사 유형: ...]
⑥ [주제명] [서사 유형: ...]

──────────────────────────────
원하시는 번호를 선택해주세요.
```

**서사 유형 태그 부여 방식:**

- 주제별로 **3유형 적합도 평가** (상단 "3유형 적합도 판단 기준" 참조)
- 최고 적합도 유형을 태그로 표기 (예: `[서사 유형: 모순형 — 역설 낙차 강]`)
- 2개 유형이 비슷하게 강하면 복수 태그 허용 (예: `[서사 유형: 모순형 or 이미지강화형]`)
- 태그는 P1 §2 "서사 유형 방향 판단"의 초기 힌트로 사용됨 (P1에서 퀵서치 팩트로 최종 확정)

## 방식 A+B — 통합

1. **방식 B** 실행 → 주제 후보 6개 추출
2. 유저가 관심 있는 후보 선택 또는 전체 진행
3. 선택된 후보 키워드를 **방식 A** (`keyword_search.py`)에 입력하여 검증
4. 가중치 채점 결과 + B 추천 근거를 합산하여 최종 주제 추천

## 방식 C — 떡상 영상 발굴 (신규/소형 채널 우선)

### 핵심 전략

**대형 채널이 아닌 신규/소형 채널(구독자 1만~30만)의 떡상 영상**을 우선 발굴한다. 대형 채널은 충성 구독자와 알고리즘 가산점으로 조회수가 높지만, 소형 채널의 떡상 영상은 **순수하게 주제 자체가 알고리즘을 뚫었다는 직접 증거**다. 시작 단계의 본 채널이 같은 알고리즘 트리거를 노릴 수 있다.

**대형 채널(구독자 30만 초과)도 발굴은 하지만 별도 섹션 분리** — 변형 분석 대상에서 제외하고 참고용으로만 노출 (재현성 낮음).

**작동 원리 — 썸네일이 주제를 끌어낸다 (역추출 방식)**

방식 A·B는 "주제 → 제목·썸네일" 순서지만, 방식 C는 **"떡상 영상 발굴 → 변형(가제+썸네일 문구 짝) 생성 → 함의 주제 역추출"** 순서다. 떡상 영상은 주제를 발굴하는 진입점이며, 역추출된 주제는 **기존 P0 지침(3유형 적합도·가중치 채점)으로 검증**한다. 따라서 방식 C 결과도 방식 A·B와 동일한 평가 기준을 통과해야 한다.

> **원칙:** 대본 내용은 절대 카피하지 않는다. 떡상 썸네일은 주제 발굴 단서로만 쓰고, 제목·썸네일은 **5가지 감정 방향**(충격/궁금증/비교/다큐/사회문화)으로 다르게 만든다.

### 용어 정의 (모호성 제거 — 새 세션 AI도 즉시 이해 가능)

본 방식 C에서 사용하는 모든 용어를 명확히 정의한다. 모호한 단축형은 사용 금지.

| 용어 | 정의 | 계산식 | 단위 |
|---|---|---|---|
| **구독자 대비** | 영상 조회수가 채널 구독자 수의 몇 배인지 (외부 시청자 유입 강도) | 영상 조회수 ÷ 채널 구독자 수 | 배 |
| **평균 대비** | 영상 조회수가 채널 평균 영상 조회수의 몇 배인지 (채널 내 이변성) | 영상 조회수 ÷ 채널 평균 영상 조회수 | 배 |
| **영상당 구독자** | 채널 영상 1개당 평균 구독자 수 (채널 효율성) | 채널 구독자 수 ÷ 채널 총 영상 수 | 명 |
| **떡상** | 조회수가 이례적으로 많이 나온 상태. 본 시스템에서는 정량화함 | 구독자 대비 ≥ 5배 OR 평균 대비 ≥ 5배 | 정성 |
| **신호 강도** | 떡상의 강도를 나타내는 단일 지표 | max(구독자 대비, 평균 대비) | 배 |
| **영상당 구독자 보너스** | 영상당 구독자 값에 따른 가중치 가산점 (채널 자체 품질 신호) | 1,000 이상 +1 / 5,000 이상 +2 / 10,000 이상 +3 | 점 |
| **소형 채널 강신호 보너스** | 작은 채널 + 강한 신호 조합 가산점 (본 채널 재현 가능성 신호) | 신호 강도 × 구독자 크기 매트릭스 (하단 표 참조) | 점 |
| **최신성 보너스** | 영상 업로드 경과일에 따른 가산점 (알고리즘 푸시 윈도우 반영) | ≤7일 +3 / 8~14일 +2 / 15~30일 +1 / 31일~ 0 | 점 |
| **썸네일 문구 (메인)** | 썸네일 이미지 큰 글씨 텍스트 — 영상의 후킹 카피 | OCR 결과 중 글자 박스 height ≥ 최대 height × 60% | 텍스트 |
| **썸네일 문구 (서브)** | 썸네일 이미지 작은 글씨 (말풍선·부가 설명) — 참고용 | OCR 결과 중 글자 박스 height < 최대 height × 60% | 텍스트 |
| **감정 방향** | 변형의 후킹 톤 5가지 분류 | 충격형 / 궁금증형 / 비교형 / 다큐형 / 사회문화형 | 카테고리 |
| **서사 유형** | 본 채널 콘텐츠 전개 구조 3가지 분류. **P0 방식 C에서는 결정하지 않음** — P1 §2에서 퀵서치 팩트로 확정 (방식 A·B와 동일) | 모순형 / 이미지강화형 / 과정추적형 | 카테고리 |
| **추천 조합** | 25개 변형 중 점수·전략별로 선정한 최종 후보 3개 | A 무난형 / B CTR 강함 / C 다큐형 | 조합 |
| **감정 포인트** | 영상이 시청자에게 자극하는 감정·욕구 4~5개 | (Step 1 분석 결과) | 키워드 묶음 |
| **알고리즘 키워드 조합** | 알고리즘이 연관 영상으로 노출시킬 핵심 키워드 조합 | (Step 2 분석 결과) | 키워드 묶음 |
| **보존 핵심 키워드** | 변형 시 절대 빼면 안 되는 명사·고유명사 5~7개 | (Step 3 분석 결과) | 키워드 묶음 |

**금지 표현:** 단순 "비율" / "상대" / "효율" 단독 사용 금지 (반드시 위 용어 사용).
**변형 입력 원칙:** 변형은 반드시 **썸네일 문구 (메인)** 만 기준으로 한다. 서브는 참고용으로만 표시.
**감정 방향 vs 서사 유형:** 둘은 다른 차원의 분류. 감정 방향은 후킹 톤(P0 변형용), 서사 유형은 콘텐츠 구조(P1·P3·P4용). **P0 방식 C에서는 감정 방향 태그만 부여하며, 서사 유형은 P1 §2 퀵서치 후 결정**한다.

### 떡상 신호 기준

| 신호 | 기준값 |
|---|---|
| 구독자 규모 | 1만 이상 (1만 미만 제외 / 30만 초과는 별도 섹션 분류) |
| 구독자 대비 | 조회수 ÷ 구독자 ≥ 5배 |
| 평균 대비 | 해당 영상 ÷ 채널 평균 조회수 ≥ 5배 |
| 절대 하한 | 조회수 ≥ 5만 |
| 영상 길이 | 4분 이상 (쇼츠 제외) |
| 업로드 | 최근 90일 이내 |

### 비지식 영상 3중 제외 필터 (스캐너 자동)

본 채널은 지식 집중 채널이므로 뉴스·정치·연예·게임·드라마·브랜드 콘텐츠를 제외한다. 스캐너에서 다음 3단계로 자동 필터링.

| 필터 | 검사 대상 | 예시 키워드 |
|---|---|---|
| EXCLUDE_CHANNEL_KEYWORDS | 채널명 | MBC·KBS·뉴스·은행·카드·보험·증권·통신·BMW·로블록스 등 |
| EXCLUDE_TITLE_KEYWORDS | 영상 제목 | 윤석열·이재명·먹방·참교육·대리뷰·탐방·진엔딩·공략법·≪≫ 등 |
| EXCLUDE_THUMBNAIL_KEYWORDS | OCR 추출 썸네일 문구 | 참교육·먹방·광팬·엔딩·공략법·ㅋㅋ·ㅎㅎ 등 |

이 3중 필터에도 누락된 비지식 영상은 메인 클로드의 2단계 "1차 선별"에서 추가 제거한다.

### 떡상 유형 자동 분류

스캐너가 **채널 구독자 수 + 영상 업로드 경과일 + 일평균 조회수** 기준으로 자동 분류한다. 보고서는 유형별로 섹션을 분리해 출력한다. **30만 초과 채널은 다른 분류보다 우선 적용**되어 🏢 대형 채널 그룹으로 분류된다.

| 유형 | 분류 기준 | 시그널 의미 | 활용 전략 |
|---|---|---|---|
| 🔥 신규 떡상 | 업로드 ≤30일 + (구독자 대비 ≥5배 OR 평균 대비 ≥5배) | 알고리즘이 지금 밀어주는 중 + 시의성 트렌드 | 빠르게 진입 (라이프사이클 짧음) |
| 📈 중기 안정 | 업로드 31~60일 + (구독자 대비 ≥5배 OR 평균 대비 ≥5배) | 안정 트렌드 | 일반 진행 |
| 🌱 후행 에버그린 | 업로드 >60일 + 일평균 조회수 ≥5천 | 장기 검색·추천 노출 가능성 | 천천히 만들어도 됨 (장기 가치) |
| 📊 일반 | 위 3유형 미해당 | 보통 신호 | 보조 검토 |
| 🏢 대형 채널 | 구독자 30만 초과 (다른 분류 우선 적용) | 본 채널 재현성 낮음 (대형 알고리즘 가산점) | **변형 분석 대상에서 제외 — 참고용 별도 섹션** |

**대형 채널 처리 원칙:** 30만 초과 채널 영상은 다른 유형보다 우선해서 "🏢 대형 채널" 섹션으로 분류된다. 본 채널 재현성이 낮아 메인 클로드의 1차 선별·변형 분석에서 제외하고 참고용으로만 노출한다.

### 우상향 추세 검증 (VidIQ + Claude in Chrome)

스캐너는 누적 조회수만 측정 가능하므로 "최근 일자별 조회수 증가 추세"는 자체 측정하지 않는다. 대신 **VidIQ 크롬 확장의 'Views over time' 그래프**로 검증한다.

**검증 방법 2가지 중 선택:**

- **방법 1 — 수동 검증:** 영상 URL 클릭 → VidIQ 패널 'Views' 탭 → 'Views over time' 그래프 확인 → 우상향(꺾이지 않고 계속 증가) 신호 강할 때만 P1 진행
- **방법 2 — Claude in Chrome 자동 검증:** 보고서 하단에 자동 생성된 'Claude in Chrome 검증 프롬프트' 블록을 복사하여 Claude in Chrome 세션에 붙여넣기 → Claude가 상위 10개 영상을 순차 방문하여 추세 일괄 보고

전제 조건: 브라우저에 VidIQ 무료 플랜 설치 + 로그인 + (방법 2 선택 시) Claude in Chrome 활성 세션.

### 실행 절차 (6단계)

1. **스캐너 실행**

   ```bash
   python scripts/knowledge_video_scanner.py
   ```

   4경로 병행 검색을 자동 수행한다.
   - Path 1 — YouTube 카테고리 인기 영상 (mostPopular)
   - Path 2 — 키워드 폭격 검색 (지식 영상 패턴 키워드)
   - Path 3 — 시드 채널 댓글 채널 마이닝
   - Path 4 — 외부 분석 사이트 (수동 큐레이션 보조)

2. **보고서 검토 + 본 채널 톤 부합 영상 1차 선별 (영상 단위)**

   `대본/00_떡상 썸네일/[떡상 썸네일]_YYYYMMDD_HHMM.md` 생성됨. 보고서에는 영상별 썸네일 이미지가 임베드되어 시각 비교 가능. 본 채널 톤(지식·경제·국제·역사·사회구조) 부합 영상 5~10개 선별.

   **선별 대상:** 🔥 신규 떡상 / 📈 중기 안정 / 🌱 후행 에버그린 / 📊 일반 4그룹.
   **선별 제외:** 🏢 대형 채널 그룹 (재현성 낮음 — 참고용으로만 활용).

3. **변형 25안 생성 (가제 + 썸네일 문구 + 역추출 주제)**

   1차 선별 영상마다 아래 분석 흐름을 거쳐 5가지 감정 방향별로 5개씩 = **총 25개 변형**을 생성한다.

   **참고 입력:** 제목(완전 맥락) + **썸네일 문구 (메인)**(실제 후킹 카피)
   (썸네일 문구 (서브)는 변형 기준에서 제외 — 말풍선·부가 설명은 후킹 카피가 아님)

   **변형 1개 = 가제 + 썸네일 문구 + 역추출 주제 (3개 항목 한 묶음)**
   - 가제: 15~30자 제목 형식 (검색·SEO용)
   - 썸네일 문구: 5~15자 짧은 후킹 카피 (썸네일 이미지 오버레이용)
   - 역추출 주제: 변형이 다루게 될 주제를 한 줄로 명시 (변형마다 달라야 함 — 같으면 변형 실패)

   **서사 유형은 P0에서 결정하지 않는다.** P1 §2 "서사 유형 방향 판단"에서 퀵서치 팩트 수집 후 확정한다 (방식 A·B와 동일한 일관성).

   #### 분석 흐름 1 — 영상의 감정 포인트 4~5개 추출

   원본 영상이 시청자에게 어떤 감정·욕구를 자극하는지 4~5개 키워드로 정리.

   > 예 (칠면조 영상):
   > - 미국 문화의 숨겨진 이유
   > - 한국 vs 미국의 식문화 차이
   > - 우리가 몰랐던 의외성
   > - "왜 굳이 그걸?" 호기심

   #### 분석 흐름 2 — 알고리즘 반응 키워드 조합 정의

   알고리즘이 연관 영상으로 노출시킬 핵심 키워드 조합을 추출.

   > 예: **미국 + 칠면조 + 의외성 + 문화충돌**

   #### 분석 흐름 3 — 보존 핵심 키워드 식별

   변형 시 절대 빼면 안 되는 명사·고유명사·식별자 5~7개.

   > 예: 미국 / 칠면조 / 추수감사절 / 미국인 / 한국인

   #### 분석 흐름 4 — 5가지 감정 방향별 변형 5개씩 생성 (총 25개)

   | 감정 방향 | 톤 | 예시 (가제 / 썸네일 문구) |
   |---|---|---|
   | **충격형** | 의외성·반전·놀라움 | "한국에선 망했는데 미국은 환장하는 음식" / "한국:안 먹음 / 미국:없으면 난리" |
   | **궁금증형** | 호기심·미스터리 | "미국인들이 왜 이 큰 새를 먹을까" / "왜 굳이 이걸?" |
   | **비교형** | 한국 vs 미국, A vs B | "미국은 환장 한국은 외면" / "미국 vs 한국 식탁" |
   | **다큐형** | 역사·교양·구조 | "미국은 어떻게 칠면조의 나라가 되었나" / "미국의 국민 음식" |
   | **사회문화형** | 문화충돌·세대·전통 | "추수감사절의 진짜 주인공은 왜 칠면조일까" / "미국의 이상한 전통" |

   **변형 원칙:**
   - 보존 핵심 키워드는 반드시 유지
   - 문장 구조만 변형 (단순 동의어 교체 금지)
   - 핵심 키워드 ≠ 단어 그대로의 카피, "조합"을 변주
   - 변형마다 역추출 주제 함께 부여 (서사 유형은 P0에서 결정하지 않음)

4. **추천 조합 3가지 선정 + 점수 적용**

   25개 변형 모두 점수를 부여하지 않는다(보고서 비대화 방지). 대신 3개 조합을 다른 기준으로 선정 후 그 3개에만 가중치 점수 부여.

   #### 추천 조합 3가지 선정 기준 (감정 방향 분배)

   25개 변형(5가지 감정 방향 × 5개)을 3개 조합에 분배 선정.

   | 조합 | 대상 감정 방향 | 선정 기준 |
   |---|---|---|
   | **조합 A (무난형)** | 비교형 5개 | 본 채널 톤 부합도 1위 (균형·무난) |
   | **조합 B (CTR 강함)** | 충격형 5개 + 궁금증형 5개 = 10개 | 후킹 강도 1위 (강한 단어·반전·놀라움) |
   | **조합 C (다큐형)** | 다큐형 5개 + 사회문화형 5개 = 10개 | 본 채널 톤 부합도 1위 (교양 깊이) |

   - **본 채널 톤 부합도** = 지식·경제·국제·역사·사회구조 부합 정도
   - **후킹 강도** = 의외성·반전·강한 단어·호기심 자극 강도

   #### 가중치 채점 (만점 59) — 추천 조합 3개에만 적용

   **① 기본 점수 (만점 50)**

   ```
   기본 점수 = 한국 관련도×3
            + 일상 연결성×2
            + 3유형 적합도 최고값×2
            + 보편 궁금증×1
            + 30~50대 공감도×2
   ```

   **② 영상당 구독자 보너스 (최대 +3)** — 채널 자체 품질 신호

   | 영상당 구독자 (구독자 ÷ 영상수) | 보너스 |
   |---|---|
   | 10,000 이상 | +3 |
   | 5,000 ~ 9,999 | +2 |
   | 1,000 ~ 4,999 | +1 |
   | 1,000 미만 | 0 |

   **③ 소형 채널 강신호 보너스 (최대 +3)** — 본 채널 재현 가능성 신호

   기준값 = **신호 강도** = max(구독자 대비, 평균 대비)

   |  | 신호 강도 5~9배 | 신호 강도 10~19배 | 신호 강도 20배 이상 |
   |---|---|---|---|
   | 구독자 1만~3만 | +1 | +2 | **+3** |
   | 구독자 3만~10만 | 0 | +1 | +2 |
   | 구독자 10만~30만 | 0 | 0 | +1 |

   **④ 최신성 보너스 (최대 +3)** — 알고리즘 푸시 윈도우 반영

   영상 업로드 경과일 기준. 신선할수록 알고리즘이 한창 밀어주는 중이라 클릭 유효성 더 높음.

   | 업로드 경과일 | 보너스 |
   |---|---|
   | ≤7일 | +3 |
   | 8~14일 | +2 |
   | 15~30일 | +1 |
   | 31일~ | 0 |

   **최종 점수 (만점 59) = 기본 + 영상당 구독자 보너스 + 소형 채널 강신호 보너스 + 최신성 보너스**

   **추천 조합에는 서사 유형 태그를 부여하지 않는다** — P1 §2 "서사 유형 방향 판단"에서 퀵서치 후 결정 (방식 A·B와 동일).

5. **유저 보고 + 보고서 파일 추가 기록 (필수)**

   메인 클로드는 3·4단계 결과를 **반드시 스캐너 보고서 파일 하단 "## 변형 분석" 섹션에 추가 기록**한다. 대화창에만 출력하지 말 것 — 다음 세션에서 재참조 불가능하기 때문.

   - 작성 위치: `대본/00_떡상 썸네일/[떡상 썸네일]_YYYYMMDD_HHMM.md` 파일의 placeholder 영역
   - 작성 방법: Edit 도구로 `_(메인 클로드가 분석 후 본 영역을 채웁니다)_` 라인을 실제 분석 표로 교체

   각 항목에 다음을 모두 표시한다.
   변형 분석 섹션의 후보별 표기는 떡상 영상 리스트와 동일한 메타정보를 모두 포함한다(분석만 보고도 후보 평가 가능하도록).

   - **제목** (영상 제목 — 검색용 긴 텍스트)
   - **썸네일 문구 (메인)** (큰 글씨 — 영상의 후킹 카피, OCR 자동 추출, 변형 기준)
   - **썸네일 문구 (서브)** (작은 글씨·말풍선 — 부가 설명, 참고용)
   - **썸네일 이미지** (자동 다운로드 — `대본/00_떡상 썸네일/thumbnails/{제목}_{video_id}.jpg`, 보고서에 마크다운 임베드)
   - **링크** (`https://youtu.be/...`)
   - **업로드일·경과일·일평균 조회수·길이** (표기 형식: `2026-04-21 (19일 전) / 일평균 조회수 N / 길이 N분 N초`)
   - **채널명·구독자·영상수·영상당 구독자·조회수·구독자 대비·평균 대비** (검증용)
     - **떡상 영상 리스트 표기 형식 (영상당 구독자 채널 라인에 표시):**
       - `채널: {채널명} (구독자 N만 / 영상수 N / 영상당 구독자 N)`
       - `영상 지표: 조회수 N만 / 구독자 대비 N배 / 평균 대비 N배`
     - **변형 분석 표기 형식 (영상당 구독자는 보너스 계산 라인에만 — 중복 방지):**
       - `채널: {채널명} (구독자 N만 / 영상수 N / 조회수 N / 구독자 대비 N배 / 평균 대비 N배)`
       - 별도 라인: `영상당 구독자 N (N구간) → 영상당 구독자 보너스 +N`
     - 영상당 구독자: 구독자 ÷ 영상수 (영상 1개당 평균 구독자 수). 1,000 이상이면 가중치 보너스 가산 대상.
     - 구독자 대비: 조회수 ÷ 구독자 (구독자 1명당 조회수 배수)
     - 평균 대비: 영상 조회수 ÷ 채널 평균 조회수 (채널 평균 영상 대비 배수)

   **명칭 통일:** 보고서·지침·대화에서 영상 텍스트 정보는 "제목"(긴 검색용)·"썸네일 문구"(짧은 후킹 카피)로 통일한다. "원본" 표현 사용 금지.

   (변형 영역 항목은 양식 예시 참조 — 가제 / 썸네일 문구 / 역추출 주제. 서사 유형은 P0에서 결정하지 않음)

   #### 보고서 전체 구조 (스캐너 자동 + 메인 클로드 추가)

   ```
   # 떡상 영상 스캔 보고서 — YYYYMMDD_HHMM
   > [메타정보 3줄: 발굴 결과 / 임계값 / 워크플로우 참조]
   ```
