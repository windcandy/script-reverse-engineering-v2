## 1. 주제 유형 판별 (Q1~Q8)

아래 8개 질문에 순서대로 답변하여 주제의 성격을 확정한다.

| # | 질문 | 판별 기준 |
|---|------|----------|
| Q1 | "물질/현상"인가, "사건/인간"인가? | 물질/현상 → **인지적 채무** (몰랐던 사실을 알게 됨) / 사건/인간 → **감정적 채무** (감정이 움직임) |
| Q2 | 시청자의 긴장(텐션)은 어디서 오는가? | 모순/역설(아는 것 ≠ 실제) → 모순형 / 숨겨진 규모·가치(아는 것보다 훨씬 큼) → 이미지강화형 / 과정·전개의 궁금증(어떻게 이 결과에?) → 과정추적형. 복수 선택 가능, 서사 유형 확정은 P3 |
| Q3 | 핵심 정보의 깊이는 단층인가 다층인가? | 모순형: 반전이 하나 vs 여러 겹 / 이미지강화형: How만으로 충분 vs Why까지 필요 / 과정추적형: 전환점이 하나 vs 여러 개 |
| Q4 | 표면 정보와 심층 정보를 시간순으로 분리할 수 있는가? | 분리 가능 → 순차 배열 (모순형: A극→B극, 이미지강화형: What→How→Why) / 분리 불가 → 교차 배열 (모순형: 양면동시건설, 과정추적형: 사례 교차) |
| Q5 | 가치는 "있어서 좋은 것"인가, "없으면 고통인 것"인가? | 긍정적 가치 강조 vs 결핍/고통 프레임 |
| Q6 | 시의성이 있는가? | 뉴스(즉시) / 기념일(주기) / 에버그린(상시) |
| Q7 | 시간축이 긴 주제인가? | 단기 사건 vs 장기 역사·추이 |
| Q8 | 여러 사례를 다뤄야 하는가? | 단일 사례 집중 vs 복수 사례 비교 |

## 2. 핵심 질문 발굴 (3가지 유형)

### 유형 A: 역설 (모순형)

**공식:** [시청자가 아는 것 A] + [모순되는 사실 B]

| 하위 유형 | 설명 |
|-----------|------|
| 장점-단점 역전형 | 좋다고 알려진 것이 실은 해롭다 (또는 그 반대) |
| 상식 파괴형 | 누구나 믿는 상식이 사실이 아니다 |
| 인과 단절형 | 원인과 결과가 직관과 반대로 작동한다 |
| 미스터리형 | 설명 불가능한 현상이 존재한다 |

### 유형 B: 숨겨진 가치 (이미지 강화형)

| 하위 유형 | 설명 |
|-----------|------|
| 이면 폭로형 | 겉으로 보이지 않는 이면을 드러낸다 |
| 가치 증명형 | 과소평가된 대상의 진정한 가치를 입증한다 |
| 전략 해부형 | 성공/실패 뒤에 숨겨진 전략을 분석한다 |

### 유형 C: 과정 추적 (서사형)

| 하위 유형 | 설명 |
|-----------|------|
| 사건 서사형 | 하나의 사건을 시간 순서로 추적한다 |
| 역사 서사형 | 긴 역사적 흐름을 압축하여 전달한다 |
| 비교 서사형 | 둘 이상의 대상을 비교하며 서사를 전개한다 |

### 핵심 질문 테스트

> 발굴한 핵심 질문에 대해 시청자가 **"에? 왜?"** 반응을 보이는가?
> 이 반응이 나오지 않으면 핵심 질문을 재설정한다.

### 주제 발굴 원칙

트렌드 키워드에서 **"사건 자체"**가 아니라 **"핵심 대상"**을 추출한다.
사건은 시의성과 함께 소멸하지만, 핵심 대상에서 출발하면 **에버그린 지식**이 된다.

### Pivot + Cross-Over: 기존 주제에서 파생 주제 발굴

성공한 영상이나 검토 중인 주제에서 새 주제를 체계적으로 도출하는 두 가지 방법이다.

#### Pivot (관점 전환)

같은 대상을 다른 각도로 바라본다. 기존 영상이 다룬 "What/How" 관점을 "Why/Risk"로 전환하거나, 표면 현상을 메커니즘으로, 결과를 원인으로 뒤집는다.

| 전환 방향 | 예시 |
| --------- | ---- |
| How-to → Why/Risk | "전세 계약 방법" → "전세가 사기에 취약한 구조적 이유" |
| 현상 → 메커니즘 | "집값이 오른다" → "집값을 올리는 실제 작동 원리" |
| 결과 → 원인 | "한국 저출산 심각" → "출산율을 낮추는 경제 메커니즘" |
| 긍정 → 이면 | "삼성의 성공" → "삼성 성공이 은폐한 비용" |

#### Cross-Over (키워드 이식)

한 영역에서 성공한 핵심 키워드를 다른 영역에 이식하여 새로운 역설을 만든다.

| 이식 방향 | 예시 |
| --------- | ---- |
| 경제 → 생물 | "독점"이라는 키워드 → "자연계의 독점 생물" |
| 역사 → 현재 | "제국주의 수탈 구조" → "현대 플랫폼 기업의 수탈 구조" |
| 국내 → 해외 | "한국 전세 사기" → "미국/일본에도 유사 구조가 있는가?" |

**적용 시점:** 차기 주제 발굴 시, 또는 기존 성공 영상에서 파생 주제를 찾을 때. 발굴한 주제는 위 §2의 핵심 질문 유형(A/B/C)으로 분류하고 GATE 1을 통과해야 다음 Phase로 진행한다.

## 3. 프레임 선택

| 프레임 | 본론 중심 | ML 방향 | 톤 |
|--------|----------|---------|-----|
| 기술적 | 메커니즘, 공정 | "이 기술이 어떻게 가능한가?" | 분석적 |
| 문화적 | 역사, 정체성 | "왜 이것이 마음을 잡았는가?" | 서사적 |
| 경제적 | 시장, 산업 구조 | "왜 이런 경제 현상이?" | 시사적 |
| 지정학적 | 국가 관계, 자원 | "왜 이곳이 운명을 좌우?" | 긴박한 |
| 과학적 | 생물, 화학, 물리 | "원리는 무엇인가?" | 지적 |

## 4. 최종 결론 + 지적 전리품 정의

| 항목 | 정의 |
|------|------|
| **최종 결론** | 시청자가 영상을 다 본 뒤 소유하게 되는 결론 |
| **지적 전리품** | 시청자가 세상을 다르게 보게 되는 방식 |
| **논리 엔진** | 최종 결론에 도달하기 위한 핵심 추론 경로 |

세 항목 모두 Phase 1 단계에서 명확히 정의해야 이후 구조 설계(Phase 2)로 진행할 수 있다.

## 5. 경쟁 환경 점검

주제 후보 확정 전, GATE 1 통과 전에 아래 세 가지를 점검한다.

### 레드오션 판별

YouTube에서 동일 주제를 이미 다룬 영상을 검색한다.

| 경쟁 상황 | 판단 기준 | 대응 |
| --------- | --------- | ---- |
| 경쟁 영상 없음 | 블루오션 | 그대로 진행 |
| 경쟁 영상 있음, 조회수 10만 미만 | 아직 여지 있음 | 차별화 포인트 확인 후 진행 |
| 경쟁 영상 있음, 조회수 100만+ | 레드오션 가능성 | 반드시 차별화 각도 확보 후 진행 |

### 차별화 포인트 분석

경쟁 영상이 존재하는 경우, 해당 영상의 내용을 확인하여 "이미 다뤄진 각도"를 파악한다. §2의 Pivot 방법을 적용해 다른 관점을 선택한다.

- 경쟁 영상이 How-to로 다뤘다 → Pivot: Why/Risk 각도로 전환
- 경쟁 영상이 현상 설명에 그쳤다 → Pivot: 메커니즘 해부로 전환
- 경쟁 영상이 동일 각도다 → Cross-Over: 다른 영역에 이식

### 차별화 포인트 정의

점검 후, 아래 한 문장으로 정의한다.

> "기존 영상은 [A]를 다뤘다. 이 영상은 [B]를 다룬다."

## 6. 주제 선정 가중치 공식

```
기대 조회수 = (한국 관련도 × 3) + (일상 연결성 × 2) + (역설 강도 × 2) + (보편 궁금증 × 1)
```

각 항목을 1~5점으로 채점한 뒤 가중 합산하여 주제 간 우선순위를 판단한다.

## 8. 할루시네이션 방지 규칙 (Phase 1 전용)

- 존재하지 않는 트렌드나 통계를 만들어내지 않는다.
- 주제의 역설 강도를 과장하지 않는다 -- 실제 사실로 뒷받침 가능한 역설만 채택한다.
- "모든 주제에 역설이 있어야 한다"는 강박을 갖지 않는다 -- 과정 추적형(유형 C)도 유효한 핵심 질문 유형이다.
