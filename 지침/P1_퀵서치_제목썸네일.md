## 1. 퀵서치

3개 P0 후보 각각의 키워드로 즉시 웹 검색하여 핵심 팩트를 수집한다 (유저 직접 입력 시 입력된 키워드 수만큼 처리).

### 수집 목표

- 제목·썸네일 후보 생성에 충분한 임팩트 있는 수치·사실 (공통 §0.5 서사 유형 정의 정합)
- 시청자가 "에? 왜?" 반응을 보일 만한 인지부조화 팩트 우선
- **출처 등급:** P2 §3 출처 신뢰도 등급 참조 (3순위 권위 있는 언론사 1차 보도 이상 권장)
- **본문 구성용 팩트 완전 세트는 P2 딥리서치에서 보강** (P2 §3 참조)

### 핵심 팩트 리포트 출력 형식

> **저장 위치:** 3개 후보 분석은 임시 보고서로 화면 출력. 유저가 GATE 1에서 1개 선택 시 선택된 후보의 분석만 `대본/[NNN_주제명]/[퀵서치] 주제명.md`로 저장 (폴더 생성도 GATE 1 통과 시점 — `CLAUDE.md` "대본 파일 저장 규칙" 정합)

```text
## 핵심 팩트 리포트 (퀵서치)

### P0 후보 1: [주제명]
1. [팩트] (출처: )
2. [팩트] (출처: )
3. [팩트] (출처: )
4. [팩트] (출처: )
5. [팩트] (출처: )

### P0 후보 2: [주제명]
(동일 형식, 팩트 5개)

### P0 후보 3: [주제명]
(동일 형식, 팩트 5개)
```

## 2. 서사 유형 방향 판단

3개 P0 후보 각각의 퀵서치 팩트를 수집한 뒤, 제목/썸네일 후보를 만들기 **전에** 각 후보의 서사 유형 방향을 판단한다.

> **P0 연계:** P0(키워드 발굴)에서 각 후보에 서사 유형 태그가 부여된 경우, 해당 태그를 초기 힌트로 확인한 뒤 각 후보의 퀵서치 팩트로 재검증한다. P0 태그와 P1 판단이 다르면 P1 판단을 우선한다.

### 판단 기준: 이 주제에서 가장 강한 팩트가 무엇인가?

| 가장 강한 팩트 유형 | 서사 유형 방향 | 제목/썸네일 구조 |
| --- | --- | --- |
| **역설(A vs B)** — "~인데 ~하다"는 모순 팩트 | **모순형** | 역설을 암시하는 구조 ("~못하는 이유", "~인데 ~하다") |
| **스케일/메커니즘** — 숨겨진 구조나 규모 팩트 | **이미지강화형** | 스케일·위상을 드러내는 구조 ("~의 위엄", "~의 진짜 이유") |
| **시간순 전환점** — 과정·변천사 팩트 | **과정추적형** | 과정·변화를 암시하는 구조 ("~의 역사", "어쩌다 ~", "~의 과정") |

> **순위 없음** — 가장 강한 팩트 1개를 기준으로 선택한다.

### 서사 유형 방향 출력 형식

```text
## 서사 유형 방향

### P0 후보 1: [주제명]
- 가장 강한 팩트: [팩트 요약]
- 서사 유형 방향: 모순형 / 이미지강화형 / 과정추적형
- 판단 근거: [왜 이 유형인지 한 줄]
- 제목/썸네일 구조 방향: [역설 암시 / 스케일 제시 / 과정 암시]

### P0 후보 2: [주제명]
(동일 형식)

### P0 후보 3: [주제명]
(동일 형식)
```

> **주의:** 같은 주제도 어떤 팩트를 중심으로 잡느냐에 따라 서사 유형이 달라진다. 가장 강한 팩트가 방향을 결정한다.

## 3. 레고이론 참조 (요약)

> 제목·썸네일·지적 전리품 개념, 기본 메커니즘(역할 분담), 3대 핵심 원칙은 `지침/00_공통_원칙.md` 0.2·0.3·A.2 참조.

### 3.1 제목 이원 구조 ("/" 구분자)

제목은 반드시 "/" 구분자로 이원화한다. 단일 문장형 제목은 사용하지 않는다.

**구조식:**

```text
[전단부 — 후킹·임팩트 문구] / [후단부 — 주제명·SEO 키워드]
```

| 부분 | 역할 | 평균 길이 (공백 제외) | 기능 |
| --- | --- | --- | --- |
| 전단부 | 후킹·인지부조화·질문·반전 | 32.6자 | 클릭 유도 |
| 후단부 | 주제명·영상 내용 예고 | 28.0자 | YouTube SEO·검색 의도 매칭 |

### 3.2 SEO 최적화 4가지 원칙

제목·썸네일은 후킹뿐 아니라 검색 노출도 함께 설계한다.

| 원칙 | 효과 |
| --- | --- |
| **① 후단부 주제 키워드 100% 배치** | 검색 의도(원인·역사·메커니즘) 매칭 |
| **② YouTube 60자 기준 의도적 초과 (전체 60~110자)** | 전단부(60자 내) 모바일 미리보기 + 후단부 검색용 동시 확보 |
| **③ 질문형 제목 비중** | 구글 SERP "People also ask" 진입 유도 |
| **④ 키워드 분산 배치** | 주 키워드 후단부 + 보조 키워드 전단부 후반 |

**적용 원칙:**

- 전단부 길이는 60자 이내로 유지 (모바일 미리보기 노출 확보)
- 후단부에는 반드시 주제명 키워드를 명시 (검색 의도 매칭)
- 질문형 제목은 검색량이 많은 "이유·역사·과정" 키워드와 결합 우선

## 5. 썸네일 메인 문구 생성 5단계

본 §5는 썸네일 메인 문구(이미지 상단 핵심 문장)만 다룬다. 메인 문구와 함께 표시되는 서브 문구(말풍선·박스 내 보조 문구) 작성 기준은 §5.B 참조.

1. **퀵서치 팩트에서 후킹력이 가장 강한 것을 고른다**

2. **§2에서 판단된 서사 유형 확인**

3. **§4.1 썸네일 3분류 중 선택** (공통 §A.1 정합)

   유형별 선택 기준 → §4.2 참조

4. **동사형과 명사형을 균형 있게 섞는다** — 후킹 방식에 다양성을 부여하여 패턴이 굳어지는 것을 막는다

   문형 적용 원칙(단정형·질문형·혼합형)은 **§5.A 부속 참조**.

5. **각 P0 후보당 1-2개씩, 총 3-6개의 제목·썸네일 후보 생성** — 각 후보에 이문호공 결합 근거·핵심 내용 한 줄을 표기 (도입부 전략 진단은 §6 — GATE 1 통과 후 최종 1개 후보에 대해서만 수행)

### 5.A [부속] 썸네일 메인 문구 길이·문형 권장 기준

썸네일 메인 문구는 단일 문장 + 정량 기준을 따른다. 다중행 구성은 사용하지 않는다. (서브 문구 기준은 §5.B 참조)

**길이 권장:** 공백 제외 **11~30자** (평균 21.3자, 31자 이상은 예외 허용이나 비권장).

**문형 적용 원칙:**

- 이미지강화형·과정추적형은 단정형 우위 (지위·결과 선언 자연)
- 모순형은 질문형·단정형 동등 (역설 구조상 양극 표현)
- 혼합형(단정+질문 동시)은 과정추적형 변형 사용

**메인 변주 예외(드물게 사용):** 메인 내 화자 부연(예: 04 `(진짜임)`)·구어 기호(예: 18 `...ㄷㄷ`)·상단 보조 수치(예: 20 `100m`)·재사용 템플릿(01·07) 사용 가능.

### 5.B [부속] 썸네일 서브 문구 작성 기준

**적용률:** 24편 100% 서브 문구 적용 (23편 둥근 말풍선 + 1편 각진 박스). 서브 문구는 메인 문구와 별도로 작성한다.

**정량 임계값:**

| 항목 | 권장 기준 |
| --- | --- |
| 길이 (공백 제외) | **3~16자, 평균 9.1자** |
| 행 수 | **단일행** |
| 어조 | **반말 구어체** |

**발화 주체 6분류 — 1종 선택:**

| 분류 | 정의 |
| --- | --- |
| ① 화면 속 인물 대사 | 썸네일 이미지 인물·캐릭터의 입에서 나오는 말 (꼬리가 인물 입으로 향함) |
| ② 화자 부연·메타 | 채널 화자가 메인을 보충·메타 정보 노출 |
| ③ 주제·사물 의인화 | 영상 주제(국가·물건·역사적 주체)에 입을 달아 1인칭 발화 |
| ④ 시청자 혼잣말 | 시청자가 메인을 본 순간 떠올릴 반응 선반영 |
| ⑤ 답·결론 노출 | 영상의 답·결론을 한 단어·한 줄로 슬쩍 노출 (§9 답 유추 금지 예외 — 메인은 답 유추 불가, 서브만 살짝 노출 허용) |
| ⑥ 제3자 시선 | 영상 주제 외부의 제3자(외국인·관찰자) 시점 발언 |

**서사 유형별 발화 주체 우위 (선택 권장):**

| 유형 | 우위 분류 | 작동 방식 |
| --- | --- | --- |
| 모순형 | ② 부연 + ④·⑥ 공감 **양극 분담** | 이성 부연 ↔ 감정 공감 동시 분담 |
| 이미지강화형 | **①+③ 장면 몰입** | 인물 대사·주제 의인화로 장면 즉시 호출 |
| 과정추적형 | **⑤ 답 노출** | 답·결론 살짝 노출로 즉시 보상·신뢰 |

## 7. 이문호공 10요인 체계 (9 기본 + 1 신규)

### 7.1 ~ 7.3 정의·적용 범위 (단일 본체 위임)

→ **`00_지식파일/03 이문호공.md` 단일 본체 참조.** 9 기본 요인 정의(응용 변주 포함)·10번 도덕공분 정의·요인별 적용 방법·도덕공분 적용 범위 모두 03 이문호공.md에 정의됨.

**P1 적용 요약 (작업 시점 안내):**

- 썸네일·제목 후킹 = 10요인 모두 + 9·4번 부정·감정 변주 활용
- ML 도입부 = 3요인 이상 권장 (공통 §B.5)
- 해소 은유 (사용 시) = 2요인 중첩 고정 (공통 §C.1 — 사용 자체는 자율)
- 본문 문장 중첩 = 9 기본 요인 중심
- 10번은 "도덕적 공분" 표기 고정 ("자조/유머" 등 변형 금지)

### 7.4 제목·썸네일 메인 이문호공 요인 분담 원칙

제목과 썸네일 메인은 서로 다른 이문호공 요인을 사용하여 이중 후킹 구조를 형성한다. (썸네일 서브의 발화 주체 차원 분담은 §5.B 참조)

**분담률:**

| 패턴 | 비율 | 효과 |
| --- | --- | --- |
| 제목 ≠ 썸네일 메인 (요인 분담) | **71% (17/24편)** | 이성 후킹 ↔ 감정 후킹 이중 효과 |
| 제목 = 썸네일 메인 (요인 일관) | 29% (7/24편) | 단일 메시지 강화형 |

**역할 분담 패턴:**

- **제목 전단부:** 호기심·반전·상식파괴 (이성적 후킹)
- **썸네일 메인:** 도덕공분·이익제시·최신정보(과거 부정 변주) (감정적 후킹)

**적용 원칙:**

- 후보 생성 시 제목·썸네일 메인 각각의 이문호공 요인을 별도로 명시한다 (§8 후보 제시 형식 참조)
- 분담 시 §7.5 유형별 요인 우위 매핑 정합 확인

### 7.5 유형별 이문호공 요인 우위 매핑

각 서사 유형은 제목·썸네일 메인에서 우세한 이문호공 요인이 다르다.

| 유형 | 제목 1순위 | 제목 2순위 | 썸네일 메인 1순위 | 썸네일 메인 2순위 |
| --- | --- | --- | --- | --- |
| **모순형** | 상식파괴(5) | 반전(6) | 상식파괴(5) | 도덕공분(10) |
| **이미지강화형** | 상식파괴(5)·이익제시(1) | 호기심(3) | **이익제시(1)** | 비교(8) |
| **과정추적형** | 상식파괴(5) | 반전(6) | 상식파괴(5)·반전(6) | 도덕공분(10) |

**3유형 공통 우세:**

- **상식파괴(5)** — 모든 유형 제목·썸네일 메인에서 최다 사용. 범용 1순위 후보

**유형별 고유 요인:**

| 유형 | 고유 요인 | 작동 방식 |
| --- | --- | --- |
| 모순형 | **도덕공분(10)** | 모순 공존형 썸네일에서 우세. A극(긍정)+B극(부정) 충돌 시 분노 유발 |
| 이미지강화형 | **이익제시(1)** | 통찰적 우월감 직접 제공 ("이걸 알면 이득" 신호) |
| 과정추적형 | **반전(6)** | 서사 전환점의 극적 효과 ("어쩌다 이렇게?" 진입) |

**적용 원칙:**

- 후보 생성 시 유형 1순위·2순위 요인 중 선택 권장
- 다른 요인 사용 가능하나 §10 GATE 1 체크리스트에서 정합 확인

## 9. 테스트

- 메인 문구만 보고 답을 유추할 수 있는가? → **유추 가능하면 탈락** (공통 §0.3 자발적 발견 — 서브 문구는 §5.B ⑤ 답·결론 노출 분류에 따라 답 살짝 노출 허용)
- 이문호공이 팩트를 포장하는 프레이밍으로 작동하는가, 아니면 억지로 끼워 맞춘 것인가?
- **선택된 A~H 전략이 서사 유형의 주력 범위 내에 있는가?** (공통 §A.3)
- **이 제목은 팩트로 증명 가능한가?** — 도입부 신뢰 상환 또는 본문 이연 신뢰 상환 중 하나로 증명 가능해야 통과 (P2 §1 신뢰 상환 전략 2종 참조). 어느 쪽으로도 불가능한 해석적 주장형 제목은 P2 재리서치 위험으로 P1 단계에서 제목 후보 교체 검토
