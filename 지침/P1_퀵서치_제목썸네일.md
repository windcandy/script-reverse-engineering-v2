## 1. 퀵서치

키워드를 받으면 즉시 웹 검색으로 핵심 팩트를 수집한다.

### 수집 목표

- 제목·썸네일 후보 생성에 충분한 임팩트 있는 수치·사실 (공통 §0.5 서사 유형 정의 정합 — 모순형 = 역설 / 이미지강화형 = 스케일·메커니즘 / 과정추적형 = 시간순 전환점)
- 시청자가 "에? 왜?" 반응을 보일 만한 인지부조화 팩트 우선
- **출처 등급:** P2 §3 출처 신뢰도 등급 참조 (3순위 권위 있는 언론사 1차 보도 이상 권장)
- **본문 구성용 팩트 완전 세트는 P2 딥리서치에서 보강** (P2 §3 참조)

### 핵심 팩트 리포트 출력 형식

> **저장 위치:** `대본/[NNN_주제명]/[퀵서치] 주제명.md` (`CLAUDE.md` "대본 파일 저장 규칙" 정합 — P1 완료 시 저장)

```text
## 핵심 팩트 리포트 (퀵서치)

1. [팩트] (출처: )
2. [팩트] (출처: )
3. [팩트] (출처: )
4. [팩트] (출처: )
5. [팩트] (출처: )
```

## 2. 서사 유형 방향 판단

퀵서치 팩트를 수집한 뒤, 제목/썸네일 후보를 만들기 **전에** 서사 유형 방향을 먼저 판단한다. 제목/썸네일의 구조가 서사 유형을 암묵적으로 결정하기 때문이다.

> **P0 연계:** P0(키워드 발굴)에서 이미 서사 유형 태그가 부여된 경우, 해당 태그를 초기 힌트로 확인한 뒤 퀵서치 팩트로 재검증한다. P0 태그와 P1 판단이 다르면 P1 판단을 우선한다(팩트 기반 검증이 더 정확).

### 판단 기준: 이 주제에서 가장 강한 팩트가 무엇인가?

| 가장 강한 팩트 유형 | 서사 유형 방향 | 제목/썸네일 구조 | 도입부 대표 구조 (유형 P3 §1 실측) |
|---|---|---|---|
| **역설(A vs B)** — "~인데 ~하다"는 모순 팩트 | **모순형** | 역설을 암시하는 구조 ("~못하는 이유", "~인데 ~하다") | **기본형 50% (3편, 최다)** — 4가지 구조 (01 P3 §1) |
| **스케일/메커니즘** — 숨겨진 구조나 규모 팩트 | **이미지강화형** | 스케일·위상을 드러내는 구조 ("~의 위엄", "~의 진짜 이유") | **표준 3단 75% (6/8편, 최다)** — 3가지 구조 (02 P3 §1) |
| **시간순 전환점** — 과정·변천사 팩트 | **과정추적형** | 과정·변화를 암시하는 구조 ("~의 역사", "어쩌다 ~", "~의 과정") | **표준 3단 71.4% (5/7편, 최다)** — 3가지 구조 (03 P3 §1) |

> **P1 역할 한계:** 이 표는 서사 유형 "방향" 판단용이다. 도입부 구조의 세부 변형(모순형 4가지 / 이미지강화형 3가지 / 과정추적형 3가지) 선택은 **P3 §1에서 확정**한다. 유형별 세부 구조·선택 조건·판단 플로우는 `지침/01·02·03_유형_*.md` P3 §1 참조.

### 판단 우선순위

1. **모순형 우선 검토** — 역설의 양극(A극/B극)을 뒷받침하는 팩트가 충분하고 낙차가 크면 모순형
2. **이미지강화형** — 모순 구조는 없지만 스케일 수치 + 숨겨진 메커니즘 팩트가 있을 때
3. **과정추적형** — 핵심 전환점 + 시간 흐름 팩트가 있을 때 (1·2를 먼저 검토 후 선택)

### 서사 유형 방향 출력 형식

```text
## 서사 유형 방향

가장 강한 팩트: [팩트 요약]
서사 유형 방향: 모순형 / 이미지강화형 / 과정추적형
판단 근거: [왜 이 유형인지 한 줄]
제목/썸네일 구조 방향: [역설 암시 / 스케일 제시 / 과정 암시]
```

> **주의:** 같은 주제도 어떤 팩트를 중심으로 잡느냐에 따라 서사 유형이 달라진다. 예: "소금" → 모순형("금보다 비쌌던 소금이 지금은 가장 싼 조미료") / 이미지강화형("소금이 인체에서 하는 일") / 과정추적형("소금 무역이 문명을 바꾼 과정"). 가장 강한 팩트가 방향을 결정한다.

## 5. 썸네일 문구 생성 6단계

> **분석 기준 외, 실무 프로세스:** 6단계 절차 자체는 25편 실측 도출 아닌 집필 에이전트 실무 프로세스. 단계 내 판단 기준(§4.1·§6.1·§7)은 단일 지침 정의 적용.

1. **퀵서치 팩트에서 후킹력이 가장 강한 것을 고른다**

2. **§2에서 판단된 서사 유형 확인**

3. **§4.1 썸네일 3분류 중 선택** (공통 §A.1 정합)

   유형별 권장(§4.2):
   - 모순형 → 조각 제시형 우선, 모순 공존형 고려 (주제에 A극·B극 속성 공존 가능 시)
   - 이미지강화형 → 조각 제시형 우선, 결과 제시형은 "지위가 압도적으로 증명된 주제"일 때
   - 과정추적형 → 결과·조각 동등 사용

4. **정체/주어 제거 → 결과/행위만 남기기**

   주어는 썸네일 이미지(국기, 인물, 장면)가 담당하므로 문구는 후킹에만 집중한다. 주어가 문구에 다시 등장하면 이미지와 중복되어 후킹 밀도가 떨어진다.

   문구의 역할은 두 가지 중 선택한다:
   - **후킹형:** 빈칸을 만들어 클릭 유도 ("뭐가? 누가?" 반응)
   - **정보 추가형:** 이미지가 전달할 수 없는 정보를 얹어 기대감 구체화

   | 정보 유형 | 예시 | 시청자 반응 |
   | --- | --- | --- |
   | 규모 | "1,000번 넘게 붙은 라이벌" | "이 정도였어?" |
   | 시간 | "700년째 끝나지 않는다" | "왜 이렇게 오래?" |
   | 감정 | "피의 복수극" | "그냥 경기가 아니었구나" |
   | 결과 | "골대를 뽑고 끝난 경기" | "어떻게 그게 가능해?" |

   단, 주어 자체가 후킹 요소일 때는 유지한다:
   - 주어가 반전인 경우 (예: "에드워드 1세가 스스로 독립전쟁을 만들었다" — 지우면 반전이 사라짐)
   - 주어가 낯선 고유명사인 경우 (예: "베릭이 무너졌다" — 모르면 빈칸이 아니라 무관심)

5. **동사형과 명사형을 균형 있게 섞는다** — 후킹 방식에 다양성을 주어 패턴이 굳어지는 것을 막는다

6. **최소 5개 후보 생성** — 각 후보에 전략·비중·이문호공 결합 근거·핵심 내용 한 줄을 표기

## 6. 도입부 전략 A~H 진단 연결

썸네일 후보가 결정되면, 해당 주제를 **도입부에서 어떻게 풀어낼지** A~H 전략 중 선택한다. 도입부 전략은 집필 단계(P4)에서 실제 구현되지만, P1에서 미리 판단해둬야 썸네일·제목이 도입부 전개와 일관성을 가진다.

### 6.1 진단 체크리스트 (공통 §A.3 8문항 적용)

> **본격 정의는 `00_공통_원칙.md` A.3 참조.** 8문항 진단 + 유형별 주력 전략 + 후킹 강도 위계(B·F > A)는 모두 공통 §A.3 정의 그대로 적용.

### 6.2 예시 참조

전략 A~H별 실제 인트로 예시: `00_지식파일/인트로_예시_전략별.md` (지식해적단 25편 사례 기반)

```text
이문호공: 요인1 + 요인2 (9 기본 또는 4 확장 중 선택)
핵심 내용: 이 제목으로 가면 어떤 내용의 영상이 되는가 (한 줄)
메커니즘: 시청자가 왜 클릭하는지의 심리 경로
```

## 9. 테스트

- 이 문구만 보고 답을 유추할 수 있는가? → **유추 가능하면 탈락** (공통 §0.3 자발적 발견)
- 이문호공이 팩트를 포장하는 프레이밍으로 작동하는가, 아니면 억지로 끼워 맞춘 것인가?
- **썸네일 3분류 판정이 서사 유형의 권장 전략과 일치하는가?** (§4.2)
- **선택된 A~H 전략이 서사 유형의 주력 범위 내에 있는가?** (공통 §A.3)
- **이 제목은 팩트로 증명 가능한가?** — 도입부 신뢰 상환 또는 본문 이연 신뢰 상환 중 하나로 증명 가능해야 통과 (P2 §1 신뢰 상환 전략 2종 참조). 어느 쪽으로도 불가능한 해석적 주장형 제목(예: 22 온돌 "귀신이 없는 이유")은 P2 재리서치 위험으로 P1 단계에서 제목 후보 교체 검토

## 10. 조합 제안 (선택적 — P1 자체 운영)

> **분류:** P1 자체 운영 절차 (단일 지침 부재 — 후보 다양화 보조 도구).

후보 5개를 제시한 뒤, 서로 인과·구조적으로 연결되는 방향이 2개 이상 존재할 경우, 아래 형식으로 조합 옵션을 추가 제안한다.

### 조합 출력 형식

```text
조합 A — 후보 N + 후보 N
통합 제목 방향: (한 줄)
썸네일 문구: (조합에 맞게 새로 생성 — 후킹형 또는 정보 추가형 명시)
썸네일 3분류: 결과 제시형 / 조각 제시형 / 모순 공존형 (공통 §A.1)
도입부 전략 (A~H): 진단 결과 (공통 §A.3)
합치는 이유: 왜 이 두 방향이 하나의 서사로 작동하는가
비중: 상 / 중 / 하
이문호공: 요인1 + 요인2 (9 기본 또는 4 확장)
핵심 내용: 이 조합으로 가면 어떤 내용의 영상이 되는가 (한 줄)
메커니즘: 시청자가 왜 클릭하는지의 심리 경로
```

### 조합 규칙

- 조합이 불가능하거나 억지스러운 경우 생략한다
- 최대 3개까지만 제안한다
- 유저가 조합을 선택하면, 해당 조합 기반으로 제목/썸네일을 재생성 후 GATE 1을 진행한다
