# Phase 1: 퀵서치 + 제목/썸네일

> **로드 시점:** 키워드 입력 시
> **입력:** 유저가 제시한 키워드
> **다음 Phase:** P2_딥리서치.md
> **GATE 통과 조건:** 유저가 제목/썸네일 후보 중 하나를 선택

---

## 1. 퀵서치

키워드를 받으면 즉시 웹 검색으로 핵심 팩트를 수집한다.

### 수집 목표

- 가장 임팩트 있는 수치 또는 사실 5개
- 시청자가 "에? 왜?" 반응을 보일 만한 역설/반전/규모 팩트 우선 — 인지부조화가 클릭을 유발한다. 단순 정보는 검색으로 해결되지만 역설/반전은 "영상을 봐야 풀리는 궁금증"을 만든다
- 출처 등급 3순위 이상 출처만 사용

### 출력 형식

```text
## 핵심 팩트 리포트 (퀵서치)

1. [팩트] (출처: )
2. [팩트] (출처: )
3. [팩트] (출처: )
4. [팩트] (출처: )
5. [팩트] (출처: )
```

---

## 2. 레고 이론 — 박스 이미지

- **제목** = 시청자가 최종 소유할 "박스 이미지" (지적 전리품)
- **썸네일** = "박스 이미지의 강화" 또는 "조립의 단서(조각)"
- 대본은 흩어진 단서들이 제목(박스 이미지)이라는 완성된 형상으로 수렴되는 과정을 증명한다
- 시청자가 제목이라는 지적 전리품을 직접 조립해낸 듯한 효능감을 느끼게 한다
- **역할 순서:** 썸네일 문구는 1차 시선 후킹(이미지와 함께 0.3초 안에 멈추게 하는 것), 제목은 2차 클릭 확정(호기심 또는 기대감을 완결하는 것)으로 역할을 구분한다

---

## 3. 제목 구조

```text
앞부분: 핵심 질문의 한쪽을 감정적/자극적으로 압축 + 감탄 이모티콘(ㄷㄷ, ;; 등)
뒷부분: 주제명 + 형식 표기 (SEO 키워드 포함)
```

---

## 4. 썸네일 전략 2가지

### A. 이미지 강화형

제목(박스 이미지)이 가진 **가장 강력한 결과/보상**을 문구로 직접 제시하여 압도적 기대감을 형성한다.

- 제목과의 관계: **같은 방향** (강화)
- 시청자 반응: "진짜야? 근거는?"
- 대본의 임무: 그 결과의 **이면에 숨겨진 진실과 가치**를 증명

### B. 조각 추출형

수집한 팩트에서 **특정 정보 조각(부품)**을 문구로 던져 조립 욕구를 자극한다.

- 제목과의 관계: **부분 → 전체** (조각이 박스 이미지로 수렴)
- 시청자 반응: "이게 뭔데? 이게 왜?"
- 대본의 임무: 그 조각이 전체 박스 이미지를 완성하는 **필연적 근거**임을 입증

조각은 팩트 그 자체다 — 역설, 수치, 사건, 메커니즘 등 종류를 가리지 않는다. 이문호공은 그 팩트를 **썸네일 문구로 포장할 때** 입히는 후킹 프레이밍이며, 조각 선별 단계가 아니라 문구 생성 단계(§5)에서 적용한다.

**결말이 알려진 사건의 경우:** 결과가 아니라 "과정의 숨겨진 메커니즘"을 조각으로 추출한다.

### 조각 비중 조절 (상·중·하)

| 비중 | 강화형일 때 | 조각 추출형일 때 | 서사적 역할 |
| --- | --- | --- | --- |
| **상** | 핵심 가치를 100% 관통 | 없으면 조립 불가능한 핵심 엔진 | → ML로 활용 |
| **중** | 결과물의 특정 단면 강조 | 논리 전개상 필수 이정표 | → 시청 지속 유도 |
| **하** | 관련된 흥미로운 지점 | 본질과 무관해 보이는 이질적 단서 | → 초반 후킹 후 본문 전이 |

## 5. 썸네일 문구 생성 7단계

1. 퀵서치 팩트에서 후킹력이 가장 강한 것을 고른다
2. 전략 선택: 강화형(결과 제시) vs 조각 추출형(부품 던지기)
3. 정체/주어 제거 → 결과/행위만 남기기 → "뭐가? 누가?" 반응 유발 — 주어가 없으면 빈칸 자체가 시청자의 궁금증이 된다
4. **[선택적] 서사적 은유:** 소재를 군사/생물/신화/재난/예술 도메인으로 치환하여 인지 부조화를 유발한다. 단, 영상 도입부에서 연결 고리를 즉시 납득시킬 수 있는 개연성이 있을 때만 사용한다.
5. 이문호공 2요인 이상을 후킹 프레이밍으로 입힌다
6. 동사형과 명사형을 균형 있게 섞는다 — 후킹 방식에 다양성을 주어 패턴이 굳어지는 것을 막는다
7. 최소 5개 후보 생성 — 각 후보에 전략·비중·이문호공 결합 근거·핵심 내용 한 줄을 표기

## 6. 후보 제시 형식

```text
후보 1: "문구" (제목) / "문구" (썸네일)
전략: 강화형 / 조각 추출형
비중: 상 / 중 / 하
이문호공: 요인1 + 요인2
서사적 은유: 사용 / 미사용 (사용 시 치환 도메인 명시)
핵심 내용: 이 제목으로 가면 어떤 내용의 영상이 되는가 (한 줄)
메커니즘: 시청자가 왜 클릭하는지의 심리 경로
```

## 10. 썸네일 배경 이미지 관계 4유형

- **역설적**: 제목/문구와 반대되는 이미지로 인지 부조화 유발
- **강화**: 제목/문구가 말하는 결과를 시각적으로 극대화
- **공감**: 시청자가 감정 이입할 수 있는 상황 묘사
- **미스터리**: 정체를 알 수 없는 이미지로 호기심 자극

## 11. 조합 제안 (선택적)

후보 5개를 제시한 뒤, 서로 인과·구조적으로 연결되는 방향이 2개 이상 존재할 경우, 아래 형식으로 조합 옵션을 추가 제안한다.

### 조합 출력 형식

```text
조합 A — 후보 N + 후보 N
통합 제목 방향: (한 줄)
합치는 이유: 왜 이 두 방향이 하나의 서사로 작동하는가
```

### 조합 규칙

- 조합이 불가능하거나 억지스러운 경우 생략한다
- 최대 3개까지만 제안한다
- 유저가 조합을 선택하면, 해당 조합 기반으로 제목/썸네일을 재생성 후 GATE 1을 진행한다

## 13. 할루시네이션 방지 규칙

- 퀵서치 팩트는 출처 등급 3순위 이상만 사용한다
- 존재하지 않는 수치를 만들어내지 않는다
- 은유적 과장이 팩트와 완전히 괴리되는 제목을 만들지 않는다
