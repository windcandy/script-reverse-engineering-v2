## §0. 서사 구조 프레임

> **레고 이론 (능동 추리자 메커니즘·L1·L2·L3·자발적 발견·아하·단서 수렴·지적 유능감)** → `지침/모듈_레고이론.md` §0 위임

### 0.1 기본 메커니즘 프레임

```text
제목 + 썸네일 = 썸네일 약속 (최종 통찰 암시) / 사건 개요
                — 본문에서 직접 인용하지 않는다

도입부 = 썸네일 약속이 왜 의외인지의 "전제 조건" 세팅
        + (선택) 신뢰 상환 초기 증거

본문 = 단서의 연쇄적 수렴 (최종 통찰에 도달하는 과정 증명)
       + 아하 모먼트 발생 지점

결론부 = 본문에서 발견한 최종 통찰의 여운 착지
         (§B.5 엔딩 8유형: 실천 제안·미래 기원·통찰·위트 등)
```

### 0.2 결론 보류 + 간접 회수

#### ① 결론 보류

**최종 통찰**은 본문 후반부까지 보류한다. **아하 모먼트 자체**는 완결 조건 충족 시 본문 어느 지점에서도 발생 가능하다.

#### ② 간접 회수 (수미상관 구현)

**도입부에서 던진 소재·에피소드·비유·진입 장치**를 결론부에서 돌려받을 때 명시적 문구 반복이 아닌 동의어 치환·맥락 이동·간접 연결로 처리한다.

- **적용 대상:** 도입부 장치 (썸네일 약속이 아닌 도입부 소재)
- **목적:** 완결감·위트 부여 ("아, 이게 돌아왔네!" 효과)
- **주의:** 본 회수는 도입부 소재 **재등장 허용** (방식 제한). 모듈_레고이론 §0.3 자발적 발견과 적용 대상이 다르다 — §0.3은 썸네일 메인 자체 금지, 본 ②는 도입부 소재 재등장 허용

> **회수 처리 방식:** 동의어 치환·맥락 이동·간접 연결로 회수한다. 도입부 소재 그대로 반복은 회피.

### 0.3 서사 4단계 프레임

| 단계 | 정의 | 집필 지시 |
| --- | --- | --- |
| 1단계 | 약속의 선언과 공백 형성 | 제목+썸네일+ML을 결합하여 시청자에게 의외성을 약속한다 |
| 2단계 | 단서의 연쇄적 수렴 | 서사 유형별 구조에 따라 단서를 연쇄 배치한다 |
| 3단계 | 아하 모먼트와 발견 | 시청자가 스스로 최종 통찰을 발견하는 순간(아하 모먼트)을 설계한다 |
| 4단계 | 가치 확장 (결론부 착지) | 도달한 최종 통찰을 현대 사회·일상·미래·정서·실천 등으로 확장하여 여운을 남긴다 (§B.5 결론부 8유형 참조) |

**결론부 5단락 영역 (4단계 세부 분할):** 본문 정점 SL → 결론부 직전 단락 → 결론부 진입 단락 → 결론부 본체 단락 → 시그니처. 세부 운영은 P4 본체에서 정의한다.

### 0.4 약속의 완결성 + 신뢰 상환

#### ① 약속의 완결성

썸네일·제목이 제시한 "의외성"은 **2단계로 완결**된다.

1. **신뢰 상환 (1차 단계):** 신뢰 상환 전략(도입부 / 본문 이연)에 따라 도입부 또는 본문 내에서 초기 증거 제시. 시청자 이탈 방지 기능
2. **최종 통찰 도달 (2차 단계):** 본문 후반부에서 완성된 결론. 영상 전체의 지적 전리품

**결론부에서는 신뢰 상환·최종 통찰을 수행하지 않는다.** 결론부는 §B.5 엔딩 8유형으로 **여운 착지**한다.

#### ② 신뢰 상환

서사 유형별로 신뢰 상환 위치가 다르다. 세부 위치·이유는 모듈_엔진 해당 엔진 절 참조, 유형별 궁합 표는 §B.5 참조. 신뢰 상환 전략 결정은 P2 본체에서 정의한다.

### 0.5 인과관계의 접착

단서와 단서 사이에 논리적 유격이 없어야 한다. 연결 문장으로 인과를 명시한다.

### 0.6 집필 흐름 순서

집필 흐름: **서사 유형 확정 → 썸네일 → 도입부 → 본문 → 간접 확인 → 아하 → 해소 은유(사용 시) → 원칙 검증**. 세부 운영은 각 § 본문과 P4 본체 참조.

## §A. 썸네일·도입부 공통

### A.1 썸네일 판정 기준 — 3분류

썸네일 문구의 형태에 따라 3가지 전략으로 구분한다.

| 전략 | 판정 기준 | 문형 신호 | 참고 매핑 |
| --- | --- | --- | --- |
| **결과 제시형** | 결론·지위·단정을 직접 선언 | `~이다`, `~나뉜다`, `~하는 X`, `~할 수 없다` | 과정추적형, 이미지강화형 일부 |
| **조각 제시형** | 질문 형태 또는 특정 부품 던지기 | `~이유`, `~일까?`, `~라고?`, `얼마나 ~?`, `어떤 ~?` | 모순형, 이미지강화형 |
| **모순 공존형** | 썸네일 문구 내에 A극 속성과 B극 속성이 동시 제시 | `[긍정 수식] + [부정 명사]`, `[신적·절대적 표현] + [위험·문제 표현]` 조합 | 모순형 우세 |

**참고 매핑 운영:** 위 매핑은 빈도 경향이며 강제 아님. 본 채널 3유형 모두 3종 분류 자유 활용 가능 (39편 분석 기준 약 36% 자유 매핑 발생).

#### 모순 공존형 예시

- "악마의 풀" — 마성의 이미지 + 식물 속성
- "축복의 저주" 식 대조 — A극·B극 속성 충돌

#### 경계 사례 판정 원칙

문형의 **주도적 구조**를 본다.

- 질문 문형이 포함되면 조각 제시형
- 단정·지위 선언이 포함되면 결과 제시형
- A극·B극 속성 충돌이 명시적이면 모순 공존형

### A.2 썸네일 약속·도입부·본문 역할 분담 원리

용어 구분은 §0.1.5 참조.

- **썸네일·제목 = 썸네일 약속** — 최종 통찰 암시
- **도입부 = 썸네일 약속이 왜 의외인지의 전제 조건 세팅** + (도입부 신뢰 상환 전략일 경우) 신뢰 상환 초기 증거 제공
- **본문 = 단서들이 최종 통찰을 향해 수렴** + 아하 모먼트 발생 지점

썸네일 문구를 반복하면 "이미 알고 있는 걸 또 말하네"가 되어 이탈한다. 전제 조건만 깔면 시청자가 스스로 궁금증을 생성하여 자발적 몰입이 일어난다.

> 썸네일 메인 문구 직접 인용 금지 원칙은 §0.3 ② 단일 정의 참조.

### A.3 도입부 전략 A~H 정의 + 유형별 주력

도입부 첫 블록에서 사용하는 진입 전략. 서사 유형 + 주제 성격에 따라 1개 주력 전략을 선택한다.

#### A~H 각 전략 정의

| 코드 | 명칭 | 정의 |
| --- | --- | --- |
| A | 규모폭로 | 압도적 수치·스케일로 대상의 거대함을 즉시 증명 |
| B | 공감적이질감 | 시청자가 당연히 여기는 것에 이질감을 심어 균열 생성 |
| C | 자조적체감 | 대상의 결함·단점을 자조적으로 먼저 제시 |
| E | 밈진입 | 밈·패러디·농담으로 가볍게 진입 |
| F | 맥락재설정 | 시청자의 상식적 맥락을 뒤집는 새 맥락 제시 |
| G | 현재불안소환 | 현재 진행형 위기·불안을 소환 |
| H | 감정직접소환 | 감정·분노·감사를 직접 소환 |

> D 결번 — 과거 통계 진입형 D 코드가 A 규모폭로와 기능 중복으로 폐기, 번호는 안정성 유지.
>
> 각 전략 사용 예시는 모듈_어법 도입부 예시 카탈로그 참조.

#### 본 채널 3유형별 주력 전략

- **모순형**: B·F 우선 (A·C·E·G·H 회피 권장 — 부수 작동 허용)
- **이미지강화형**: B 우선 고려, 주제 적합 시 A·C·E·G·H 선택 (F 회피 권장)
- **과정추적형**: B 우선 고려, 주제 적합 시 A·F·G·H 선택 (C·E 회피 권장)

**범용 전략 안내:** B(공감적이질감)는 3유형 모두에서 우선 고려. 상식 전복 구조로 서사 유형 불문 범용성이 높다.

**B 전략 공감 진입 출발 단위 (3유형 공통):** 시청자 일상 경험에서 출발. 출발점은 이 주제에만 성립하는 구체적 **공통 인지·상식·경험·일상 장면·일상 행동·감각**이어야 한다. 주제 이름만 바꾸면 다른 대본에도 쓸 수 있는 문장은 금지. — **동료 호명 톤 사용** (예: "우리 다 아는 ~"·"~ 한 번쯤 들어봤죠"·"다들 ~ 경험 있죠").

→ 3유형 도입부 종합 비교(3단 구조·글자수·전략·신뢰 상환) + 주력 전략 매트릭스는 **§D.1** 참조.

## §B. 오픈 루프·구조 공통

### B.1 보상의 비례 원칙

공백의 크기(궁금증의 강도)에 비례해 해소 시 보상(지적 쾌감·스케일 확인·인과 완성)의 강도가 커야 한다. 시청자가 "기다린 만큼 값어치가 있었다"고 느끼게 한다.

**유형별 보상의 성격:**

- 모순형: 역설 해소 (A극·B극이 동일 원천에서 나온 양면성이었음의 발견 — "어? 같은 거였네?")
- 이미지강화형: 스케일 확인 ("이것까지 영향이 미쳤다고?")
- 과정추적형: 인과 체인 완성 ("이래서 이렇게 됐구나!")

**단서 — 모순형 비율 편차:** 모순형에서 A극이 역사적 서사나 공감 소재로 "읽는 재미"를 제공하는 경우, A극/B극 비율이 2:1~3:1까지 불균형해도 허용된다.

> **어조 변주 운영:** 위치별 어조 변주(유지=부드러움 / 전환·강화=유형별 단호함·경외·감정 이입 / 결론부=§B.5 참조)는 §C 집필 기법 영역에서 자연 결합. 별도 원칙 영역으로 분리하지 않는다.

### B.2 단일 대상 조건

ML 선언부에 도입부 비유를 삽입하려면 대상이 **단일 사물·개체**여야 한다. 현상·시스템·비교 구조에서는 비유 삽입을 피한다.

- **사용 가능 예시:** 옥수수·서울·여권·에어컨·고래잡이 등 단일 대상
- **사용 회피 예시:** 입식 vs 좌식 / 락스 원리 등 현상·시스템·비교 구조

### B.3 도입부 공통 골격 — 인트로 + ML + 신뢰 상환

도입부는 **인트로(블록) + ML(블록) + 신뢰 상환(기능)**으로 구성된다.

| 구성 | 분류 | 공통 원칙 |
| --- | --- | --- |
| **인트로** | 블록 | ML 이전 진입 영역 — 유형별 차등 (모순형 ①②③ 3요소 / 이미지강화·과정추적 ①② 2요소) |
| **ML (메인 루프)** | 블록 | 제목·썸네일의 약속을 질문·선언으로 문장화 (도입부 말미) |
| **신뢰 상환** | 기능 | 시청자에게 "영상이 약속을 지킬 것"이라는 신호 제공 |

#### 도입부 — 신뢰 상환 전략별 이원 기능

| 신뢰 상환 전략 | 도입부 기능 |
| --- | --- |
| **도입부 신뢰 상환** | ① 전제 조건 세팅 + ② **신뢰 상환**(초기 증거 제공) 겸함 |
| **본문 이연 신뢰 상환** | ① 전제 조건 세팅만 수행 (증명은 본문에서) |

각 구성의 구체 구현(진입 단계 수·내용·신뢰 상환 위치)은 모듈_엔진 해당 엔진 절 참조.

### B.4 ML 구조 공통 원리

ML(메인 루프)은 **도입부 말미에서 본문이 답할 핵심 질문을 공식화하는 단위**이다. 서사 유형에 따라 변주하며, 유형별 구체 구조와 자체 형태 변주는 모듈_엔진 ML 영역 참조.

#### ML 3대 실용 기능

1. **본문 로드맵 제시** — ML 질문이 본문 파트를 예고 (예: "언제부터? 장단점?" → 역사 + 비교분석 2파트)
2. **본문 진입 선언 (선택)** — ML 질문 직후 본문으로 자연스럽게 넘어가면 생략 가능. 삽입 시 "~알아봅니다" / "한번 보시죠" / "지금부터 보시죠" 등 진입 표현만 허용. 채널명·화자 소개 삽입 금지
3. **서사 방향 설정** — 유형별 고유 기능

**ML 봉인 형태 (질문 개수):** 양면 봉인(2개 질문 — "왜? 어떻게?")이 기본. 단면 봉인(1개 질문)도 본문 단일 방향 추적 영상에서 활용 가능.

#### 이문호공 요인 참고 기준

이문호공 10요인 + 4역할 카탈로그는 모듈_이문호공.md 참조. ML 영역에서 이문호공 요인을 자연 결합한다. **자연스러운 궁금증 유발이 요인 카운팅보다 우선**한다.

- 본문 ML·해소 = **9 기본 요인** 중심
- 썸네일·제목 후킹 = **10요인 모두 + 부정·감정 변주** 활용

#### 본문 운영 원칙

본문은 메인 루프(ML) 1개를 축으로, 서브 오픈 루프 6~10개를 **연쇄적으로 배치**하여 ML 해소 동력을 유지한다. 각 SL은 독립적 '문제-해결'이 아니라 ML의 Q1·Q2 분산 해소, 피벗 직전 확신 극대화 등으로 **ML에 수렴**시킨다.

#### 본 채널 3유형별 ML 유지·강화·닫기 구현

| 단계 | 모순형 | 이미지강화형 | 과정추적형 |
| --- | --- | --- | --- |
| **유지** | Q2를 보류하여 공백 유지 (Q1은 A극에서 분산 해소) | How에서 Q1·Q2 해소 후 Scale Jump가 새 공백 생성 | 시간순 맥락을 축적하며 핵심 사건·인과를 보류 |
| **강화** | 피벗 직전 A극 확신 극대화 | Scale Jump 진입 시 새 루프 열기 | 인과 폭로로 절정 직전 텐션 극대화 |
| **닫기** | B극에서 3~4회 분할 제공. Q1·Q2 해소 위치 5패턴 허용 | Why에서 스케일 각인 — Scale Jump 2~5회 | 절정의 사건 자체가 해소 + 결론부에서 현재 의미 부여 — 전환점 2~6회 |

→ 3유형 ML 구조·이문호공 종합 비교는 **§D.2** 참조.

### B.5 결론부 8유형

결론부 전략은 아래 8유형 중 1~2개를 조합한다.

| # | 유형 | 착지 방식 |
| --- | --- | --- |
| ① | 일상행위연결 | "오늘 ~해 보세요" 식 실천 제안 |
| ② | 감정적여운 | "~한번쯤 생각해 보는 것도" 식 감정 착지 |

| ④ | 시의성마무리 | 현재 시점 이슈로 연결 |

**⑦ 경고형 보편성 원칙 (필수 조건):** 경고 대상자가 시청자 대다수와 일치할 때만 채택한다. 일부 소수에게만 적용되는 경고는 ⑦ 채택 금지.

- **모순형:** 미래희망(③) + 위트(⑤) 혼합 주류. 심각 주제는 경고형(⑦) 단독 허용
- **이미지강화형:** 미래희망(③) 주력. 감정적여운(②) 비사용
- **과정추적형:** 감정적여운(②) + 시의성(④) 주류. 위트(⑤) 비사용

본문 → 결론부 진입 어법(결론 브릿지 어법·5단락 영역·결론부 직전 단락 안전장치·본문 결산 양식 등)은 P4 본체 결론부 영역 참조.

### B.6 루프 중첩 원칙

메인 오픈 루프는 아래 가이드에 포함되지 않는다.

- **루프 중첩 원칙:** 서브 오픈 루프의 중첩은 1개 유지를 원칙으로 하되, 서사 전개상 필요한 경우에만 예외적으로 최대 2개까지 허용한다
- **실전 메커니즘 — "순차 교체":** 1개 유지의 실전 구현은 **"순차 교체"**이다. 새 SL을 열기 전에 기존 SL을 해소하여 동시 중첩보다 교체를 우선한다
- **중첩 한계:** 어떠한 경우에도 3개 이상의 서브 오픈 루프가 동시에 열려 있는 상태(Triple Overlap)는 절대 불가하다
- **교체 및 확장 조건:** 이미 2개의 서브 오픈 루프가 열려 있다면, 새로운 서브 오픈 루프를 생성하기 전 **반드시 기존 서브 오픈 루프 중 최소 하나를 폐쇄(결말·해소)**해야 한다

### B.7 레이어 전환 비율 + 주제 성격별 조정 원칙

#### 운영 원칙

Type A(서브 오픈루프)를 주력으로 사용하고, Type B(논리적 브릿지)는 서사의 선형적·인과적 연결이 필수적인 구간에 한정하여 사용한다 (§C.8 참조).

- 중첩이 하나인 서브 오픈 루프가 닫힐 때 새로운 서브 오픈 루프를 열어 정보의 결핍 상태를 유지한다
- 복수 대상 병렬·지역별 수평 확대 등 선형적 연결이 본질인 구간에서는 논리적 브릿지 비중 증가 허용

#### 주제 성격별 자연 조정

주제 성격에 따라 SL 형태·BR 비중은 자연 조정 허용한다. 전체 채널 톤은 Type A 주력을 기준선으로 유지한다.

## §C. 집필 기법 공통

### C.1 해소 은유 공식 — SL 해소 시점 사용 시 적용 양식

> **사용은 자율.** 본 공식은 해소 은유를 **사용할 때 적용하는 양식**이지, 편당 정량 의무가 아니다. SL 본문의 정보·임팩트가 자력 각인 가능하면 은유 없이 진행한다.

본문 SL 해소 시점에서 한 줄 각인이 필요할 때 아래 공식을 적용한다. 해소 은유는 **"열림"이 아니라 "해소"에서만 작동**한다. 본 공식의 "은유"는 좁은 의미 은유(A=B)와 직유(A는 B처럼) 둘 다 포함한다.

#### 구성 안내

해소 은유는 핵심 대상에 은유·직유를 결합하여 한 줄로 각인한다. 상식파괴 중심으로 유형별 보조 요인(모순형=경고 / 이미지강화형=비교 / 과정추적형=공감)을 자연 결합한다.

- **은유 ("A는 B다"):** 본질·정체가 같을 때
- **직유 ("처럼·같이·같은·마치·마찬가지로·비슷한·비유하면"):** 속성·행동 방식이 비슷할 때

과정추적형은 인과 완성·감정 압축으로 상식파괴를 대체 가능.

#### 본체 형식 — 한 문장 인정

해소 은유는 **한 문장 안에 본체를 담은 형태**도 본체로 인정한다 (별도 단락 강제 아님). 패턴 골격: **[대비 구도] + [한 문장 결합]**.

P4 집필 시 한 문장 은유·직유가 자연 발생하면 별도 단락을 무리하게 추가하지 않는다. P5 검증 시 한 문장 은유·직유는 본체 부재로 판정하지 않는다.

#### 작동 지점

해소 은유는 서사의 **전환·정점·완성 지점**에서 자연 발생한다.

| 지점 | 발생 조건 |
| --- | --- |
| 피벗·반전 (모순형) | A극→B극 전환 시 역설을 한 줄로 압축 |
| Scale Jump 정점 (이미지강화형) | 대상의 파급력이 최고조에 달하는 순간 |
| 인과 완성 (과정추적형) | "이래서 이렇게 됐구나"의 핵심 고리 |
| 누적 충격 정점 | 문제·가치가 쌓인 후 한 줄로 각인 |
| 메커니즘 완성 | How의 원리가 결과로 증명되는 순간 |

#### 적용 범위 + 사용 여부

- SL 해소 시점에서만 사용. SL 열림에는 팩트·질문으로 공백을 생성하고, 해소에서 비유로 각인
- 편당 정량 의무 없음. 자력 각인 가능하면 미사용

#### 대체 각인 도구

해소 은유 미사용 시 다음 6종 중 하나로 보상 단위 충족. 적용 원칙은 §C.7 비유 적용 원칙 정합 — 이해 격차가 있을 때 사용, 무리한 빈칸 채우기 금지.

| 도구 | 의미 |
| --- | --- |
| 임팩트 팩트 | 정점 직전 단일 팩트로 충격 각인 |
| 단호 선언 | 화자가 핵심 판단을 단정문으로 압축 |
| 풍자 | 모순·아이러니를 유머로 압축 |
| 수치 압축 비교 | 두 수치를 한 줄에 병치하여 격차 각인 |
| 시간 격차 위트 | 과거·현재 격차를 위트 한 줄로 압축 |
| 결과 공유 셋업 | 결과를 먼저 던지고 시청자와 함께 해석 |

#### 면제 사례

다음 해소 패턴은 해소 은유 공식 미적용: 사건·사례 해소 / 감정 해소 / 인과 해소 / 수미상관 회수. 광고 연착·신뢰 상환 분산 같은 특수 운영도 면제. 면제 사례에서도 SL 해소 자체는 필수.

#### ML 비유(열림) vs 해소 은유(해소) 차이

ML 비유는 호기심(3) 중심으로 "궁금하게 만들기"(루프 열기), 해소 은유는 상식파괴(5) 중심으로 "각인시키기"(루프 닫기). 해소 은유 적용 시 **"해소"가 "열림"보다 우선**.

### C.2 시청자 반박 선제 흡수 + 자기언급형 서사

본문 중 SL 1~2개마다 시청자가 제기할 수 있는 반박을 선제 흡수한다.

#### 반박 흡수 패턴

- **"~라고 하실 수도 있는데"** — 시청자 반박을 미리 제시
- **"엥? ~ 아니냐"** — 자조적 반박 시뮬레이션
- **"이 정도면 ~ 아니야?"** — 의심 선점
- **"물론 그렇다고 ~ 건 아니고요"** — 평서문 부정형 오해 차단. 앞 문장에서 촉발될 수 있는 오해를 "물론"으로 완화한 뒤 즉시 부정

#### 유형별 반박 흡수 초점

- 모순형: "이게 진짜로 모순되냐?" 식 의심 차단
- 이미지강화형: "과장 아니냐?" 식 의심 차단
- 과정추적형: "논리가 왜 이렇게 연결되냐?" 식 의심 차단

#### 자기언급형 서사

화자가 영상 내에서 자기 입장을 직접 드러내어 시청자와의 거리감을 좁히는 기법.

### C.3 단서 배치 3기법

본문 단서들이 최종 통찰을 향해 수렴하도록 배치하는 공통 기법.

| 기법 | 내용 | 사용 시점 |
| --- | --- | --- |
| **질문 연쇄** | 질문 → 부분 해소 → 새 질문으로 계단식 해소 | 본문 전체 관통 |
| **부분 공개** | 정보의 일부만 먼저 공개하고 나머지는 뒤로 보류 | 본문 중반 |
| **대조 쌍** | 대조되는 2개 정보를 짝으로 제시하여 시청자가 스스로 연결 | 피벗·Scale Jump·전환점 전후 |

**원리:** 단서가 독립적 조각이 아니라 서로 연쇄되어 최종 통찰을 향해 수렴하도록 배치한다 (§0.6 원칙 ④ 정합).

### C.4 간접 확인 원리

썸네일 문구를 본문에서 **직접 인용하지 않고** 우회적으로 확인시키는 우회 기법 (§0.3 ② 단일 정의에 종속).

#### 간접 확인의 핵심 원리

- **(가) 동의어 치환:** 썸네일 문구를 다른 표현으로 재진술 (예: "지키고 있는" → "살리는")
- **(나) 속성 대체:** 썸네일이 선언한 지위를 다른 속성으로 증명
- **(다) 반대 방향 진입:** 썸네일과 반대 방향에서 시작하여 썸네일 약속 결론을 유도

#### 본 채널 3유형별 주력 기법

| 유형 | 주력 |
| --- | --- |
| 모순형 | (다) 반대 방향 진입 |
| 이미지강화형 | (나) 속성 대체 + (가) 동의어 |
| 과정추적형 | (다) 반대 방향 진입 |

집필 시 유형에 맞는 주력 기법 우선 검토. 단 모든 기법이 모든 유형에서 사용 가능 (강제 매칭 아님).

#### 도입부 노출 강도 3구간

도입부에 썸네일 핵심 어휘 노출 강도를 3구간으로 구분.

| 강도 | 정의 | 처리 |
| --- | --- | --- |
| **강 (전면)** | 핵심 어휘 절반 이상이 어미·조사만 변환되어 노출 | **회피 필수** |
| **중 (부분)** | 명사구 1개가 동일 등장, 술어만 변환 | **허용** (술어 변환 또는 추가 우회로 자연 정합) |
| **약 (단어)** | 단일 단어 노출 | **허용** (핵심 결론어가 우회되면 통과) |

**판정 기준:** "핵심 결론어"(약속의 결론을 만드는 단어)가 우회됐으면 (가)·(나)·(다) 중 하나로 분류, 결론어 자체가 노출됐으면 위반.

#### 분류 차원 안내

본 §C.4의 (가)·(나)·(다)는 **"썸네일 우회 방식"** 기준이고, 모듈_엔진의 도입부 진입 기법은 **"도입부 진입 방식"** 기준이다. 두 차원은 다른 평면이므로 동일 명칭이 등장해도 정의가 다를 수 있다.

### C.5 이탈 방지 패턴 (변주 + 감정 고조)

텐션이 하락하기 쉬운 구간에 **변주**(주의 재점화) 또는 **감정 고조**(관성 유지)를 삽입하여 이탈을 방지한다. 두 기법은 작동 원리가 다르므로 상황에 맞춰 선택·병용한다.

#### A. 변주 — 주의 재점화

**삽입 지점:**

- 본문 장점만 길게 쌓일 때 (이미지강화형 — 긍정 피로)
- 시간 흐름만 이어질 때 (과정추적형 — 시간 관성 유지)
- 속성 누적이 길어질 때 (모순형 — A극 누적)

**기법:**

- **문제 제기 삽입:** "근데 이게 문제가 있다" 식 삽입
- **반대 시각 도입:** "~반대로 볼 수도 있다"
- **예상 역전:** "끝이냐? 아니다" 패턴

**본 채널 3유형별 사용 기준:**

- 이미지강화형 = 필수 (긍정 피로 구간에 변주 삽입)
- 모순형 = 권장 (A극 누적 길어질 때 "종결 부정형" 등)
- 과정추적형 = 부분 (감정 고조 주력 보완)

#### B. 감정 고조 — 관성 유지

**기법:**

- **분노 유발:** 부당함·불의를 드러내어 분노 자극
- **감정 이입 장치:** 시청자가 인물·사건에 감정 이입하도록 서술
- **감탄 유발:** 놀라움·경이로움으로 감정 고조

**본 채널 3유형별 주력 감정:**

- 모순형: 경악·분노 (A극·B극 전환점, 부분)
- 이미지강화형: 경이·감탄 (Scale Jump, 부분)
- 과정추적형: 공감·분노·감사 (역사적 사건, 필수 — 감정 이입 장치 최소 1회)

### C.6 비유 5역할 분류

비유는 시청자가 즉각 이해하지 못하는 내용을 이해시키기 위한 도구다. 본문에서 아래 5가지 역할로 사용된다. 은유는 §C.1 해소 은유 공식으로 별도 운용한다.

| 역할 | 설명 | 사용 시점 |
| --- | --- | --- |
| **① 대상에 지위·인격 부여** | 대상에 별명·지위를 부여해 중요성 각인 | ML 선언부 / 본문 정점 |
| **② 스케일·속성 체감** | 추상 규모·추상 속성(정교함·견고함·복잡함·내구성·정밀함 등)을 직관 체감 표현으로 전환 | Scale Jump · 수치 직후 + 본문 자유 위치 (메커니즘·작동 원리·추상 속성 등장 시) |
| **③ 역설 압축** | A극·B극 모순을 한 표현에 충돌시킴 | 피벗·반전 지점 |
| **④ 인과 압축** | 복잡한 인과 체인을 한 문장으로 압축 | 절정·결말 |
| **⑤ 정성 강도 비유** | 추상 속성(정교함·견고함·복잡함·강력함·정확함·민첩함·거대함 등)의 강도를 일상 사물·생활 도구·게임 용어·역사 인물·사건·전쟁 영역 비유로 직관 체감 전환. 형식: 직유("처럼·같이·같은·마냥·수준·급" 어휘 변주) 또는 은유("A는 B다" 형태). 남발 금지 | 본문 SL 진입 / SL 정점 / 도입부 오프닝 훅 |

**적용 원칙:** 시청자가 즉각 이해할 수 있는 내용에는 비유를 추가하지 않는다. 이해 격차가 있을 때만 사용한다. §C.1 해소 은유 공식과 별개로, 본문 중 비유 사용 시 5역할 중 하나를 명확히 선택한다. **삽입형(문장 안 삽입) 우선. 비교 대상이 길어 삽입이 어려울 때만 별도 문장 허용.**

본문 자유 위치 비유(② 역할 신규 시점)는 자연 흐름 우선. 무거운 구간(추모·재난) 회피·억지 끼워맞춤 금지.

### C.7 인터넷 밈 은유·환산

인터넷 밈을 **은유**(호명·동사·형용 등치) 또는 **환산**(추상 개념을 인터넷 밈 척도로 변환)으로 활용하여 시청자 몰입도를 높인다. 3유형 모두 자유 사용.

#### 영역 정의

- **은유:** 인터넷 밈을 직접 등치 형태로 사용 (예: "빤스런 했다" / "원툴이다")
- **환산:** 추상 개념·시스템·정도를 인터넷 밈 척도로 변환 (예: "~급" / "~수준" 접미사로 정도 압축)

#### 사용 원칙

- 주제에 자연스럽게 녹아드는 밈만 사용 — 억지 끼워 맞춤 금지
- 과도한 사용은 집중도 저하 유발 — 한 편 내 같은 밈 반복 금지
- 타겟 시청층(주로 20~40대)이 이해 가능한 밈으로 한정
- 비유 직후 풀이 동반 권장 (밈 비익숙 시청자 고려)

#### 비유·은유 영역 선정 원칙 (모든 비유 공통)

- **주제 핵심 메커니즘에서 꺼낸다** — 외부 영역을 억지로 끼워 맞추지 않는다
- **무거운 구간(전쟁·재난·빈곤) 최소화** — 톤 미스매치가 몰입을 깨뜨림 (§C.1 면제 사례 정합)

### C.8 브릿지 Type A·B 본체 18종

> 공식 용어는 "브릿지"로 통일.

#### 본문 레이어 연결 전략 개요

- **서브 오픈루프 (몰입 유지형):** 각 레이어·SL이 끝날 때마다 다음에 대한 '작은 갈증'을 유발하는 전환구. **질문·선언·예고 어느 형태든 새 SL을 여는 역할**. 15종 하위 유형 (질문형 6종 + 선언·예고형 9종)

> **어법 카탈로그(채널 시그니처 어법·도입부 예시·말맛 변주·SL·BR 어법 도구 어휘 영역)는 모듈_어법.md 참조.** 본 §C.8은 브릿지의 기능 분류·운영 원칙만 정의한다.

### 원칙 — 전 Phase 공통

#### ① 지침 예시 활용 방식은 종류에 따라 다르다

- **패턴형 예시** (`~`·`[N]`·`OOO` 등 자리채움 마커 포함): 마커 자리를 주제에 맞게 채워 그대로 사용 가능. 다만 한 대본 내 동일 패턴 2회 이상 사용 시 다른 패턴 또는 새 표현으로 교체한다
- **완성 문장형 예시** (자리채움 마커 없는 완성 문장·문단): 구조·어조·분량 감각만 참고하고, 실제 단어·문장은 해당 주제와 맥락에 맞게 새로 작성한다

#### ② 지침 내부 용어 노출 금지

SL·ML·BR·A극·B극·역설엔진·이문호공·해소 비유·피벗 등 지침 전용 용어를 대본에 노출하지 않는다.

#### ③ 같은 비유·밈·감탄사 한 편 내 반복 금지

같은 표현을 한 영상에서 두 번 이상 사용하지 않는다.

#### ④ 확인되지 않은 수치·사실 단정 금지

웹 검색으로 팩트 확인 후 사용한다.

### P2 딥리서치 단계 적용

- 존재하지 않는 통계나 연구를 만들어내지 않는다
- 상관관계를 인과관계로 단정하지 않는다
- 확인되지 않은 팩트를 아하 모먼트 후보로 채택하지 않는다
- 팩트가 부족한 경우 추가 검색을 수행한다. "팩트가 없는 구조"를 만들지 않는다

### P3 구조 단계 적용

- SL 개수를 억지로 맞추기 위해 불필요한 정보를 넣지 않는다
- 감정 곡선을 도식적으로 따르느라 부자연스러운 전환을 만들지 않는다
- 팩트 리포트에 없는 내용으로 구조를 채우지 않는다

### P4 집필 단계 적용

- **화자가 자기 정보에 놀라는 척 금지** — 감탄·감상을 제거했을 때 정보나 대비가 남지 않는 문장은 제거. 대비를 명시하는 문장으로 교체한다
- **비유의 메커니즘 일치 확인** — 표면적 유사성이 아니라 작동 원리가 일치하는지 검증
- **수정 후 정합성 확인 (일반):** 문장 수정 시 같은 단락 전체를 다시 읽는다
- **수정 후 정합성 확인 (구조 요소):** ML·피벗·해소 은유를 수정할 경우 해당 요소를 참조하는 모든 문장을 검색하여 재검증한다
- **수치 최신성 확인** — 현재 시점 기준 최신인지, 주체·범위·조건이 보존되었는지 확인
- **집필 중 추가 팩트 출처 기록** — 팩트리포트에 없는 정보는 반드시 웹 검색 확인 후 상태 태그 "추가 팩트"에 내용·출처 기록. 미확인 정보 단정형 서술 금지

### P5 퇴고 단계 적용

- 퇴고 시 "통과"를 위해 실제로 고치지 않고 체크만 하는 행위 금지
- 문제를 발견하면 반드시 수정 후 해당 항목 재검증
- "이 정도면 괜찮다"는 판단 금지 — 기준에 미달하면 수정
