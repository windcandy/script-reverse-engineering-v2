## §0 검수 원칙

### 단계 흐름

- **단계 1 (뼈대) → 단계 2 (디테일) 순서 엄수**
- 각 단계 내 위치 순서: **도입부 → 본문 → 결론부**
- 단계 1 미통과 시 단계 2 진입 차단 (구조 문제 발견 시 디테일 작업 무효화 방지)

### 검수 주체 (메인 1차 + 에이전트 2차 + 메인 판단)

- **메인 직접** (1차 필수) — §1·§2 전 영역 자가 검수·수정 완료
- **`P5_검수검증` 에이전트** (2차 선택) — 유저 요청 시 수동 호출. §1·§2 동일 체크리스트로 독립 검수 → 위반·발견 목록 메인 반환
- **`논리검수` 에이전트** (2차 선택) — 유저 요청 시 수동 호출. 층위 점프·수치·집합·프레임 충돌·인과 공백·어휘 반복 4축 검수
- **`팩트검증` 에이전트** (2차 선택) — 유저 요청 시 수동 호출 (§2.4). 수치·고유명사·날짜를 팩트리포트와 대조
- **세 에이전트 모두 결과는 메인이 자동 수정하지 않고 유저에게 보고 후 유저 판단(수정/유지/재검토)에 따라 반영.** 메인이 집필 의도·전체 정합을 고려해 에이전트 발견의 오검출·과검출을 먼저 거른 뒤 보고한다.

### 핵심 용어 정의 (모호성 방지)

- **결론부:** 본문 종료 ~ 시그니처까지 전체 영역
- **엔딩:** 결론부 후반 (시그니처 직전 영역, 모듈_본문구조 §2 엔딩 8유형의 적용 영역)
- **최종 통찰(L3) = 지적 전리품:** 본문 전체를 통해 도달한 완성된 결론. 위치: 본문 후반부 ~ 엔딩 직전. **모든 영상 강제** (모듈_레고이론 §0.5 정합)
