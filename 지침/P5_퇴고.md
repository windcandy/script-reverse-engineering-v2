P5 단계는 **4개 에이전트 병렬·순차 호출**로 검증한다. 메인이 결과 통합·보강 처리.

| 순서 | 에이전트 | 영역 | 호출 방식 |
| --- | --- | --- | --- |
| 1 | `P5_퇴고검증` | 구조·스타일 체크리스트 A~E + 패턴 16 | 우선 호출 |
| 2 | `팩트검증` | 수치·고유명사·날짜·인과 1:1 대조 | 1번과 병렬 가능 |
| 3 | `중립시청자` | 호의적 시청자 인지 흐름·도입부 후킹·본문 몰입·시청 후 만족도 정성 점검 (11영역 — 생소 용어·셋업·주어·맥락 점프·동어반복·사명 변경·지시어·인과 마커 부재 + 도입부 후킹 강도·본문 점층 이탈 위험·지적 보상 만족도+시간 가치) | 1·2 이후 |
| 4 | `비판시청자` | 회의적 시청자 논리 흐름·도입부 후킹 진정성·본문 신뢰 누적·시청 후 시간 가치 정성 점검 (10영역 — 논리 비약·과장·단일 원인 단정·근거 부족·반박 미흡수·톤 불일치·숨은 가정 + 도입부-본문 갭(낚시)·본문 신뢰 누적 약화·약속 대비 보상 강도+시간 낭비 위험) | 3번과 병렬 호출 |

**결과 통합 책임:** 메인이 4개 결과를 통합하고 양쪽 동의 영역(중립시청자 + 비판시청자 모두 발견)은 강한 신호로 우선 보강한다.

> **수행 주체:** 메인 에이전트 (P4 집필자) 직접

### 자체 퇴고 체크리스트 (4영역)

#### A. 도입부 답 조각 점검 (모순형 한정)

- A1. 도입부에 B극 답의 **구체 행동·반응·표현**(예: "환호했다"·"자기 집에 모셨다"·"벼슬 내렸다")이 노출됐는가?
- A2. 본문 SL4(신뢰 상환) 폭로의 **고유명사·인물·기관**이 도입부에 누설됐는가?
- A3. 시청자가 도입부만 보고 B극 답의 구체 행동을 추측 가능한가?

#### B. 비유 자연성 점검

- B1. 비유가 원본 문장 자체에 압축 형태로 발생하는데 별도 비유 단락을 추가했는가?
- B2. 추가된 비유 단락이 원본과 같은 메시지를 두 번 말하는 형태인가?
- B3. 비유 영역(학교·직장·회사 등)이 본 영상 주제와 자연 연결되는가, 끼워 맞춤인가?
- B4. 무거운 구간(전쟁·재난·신분제 등)에 가벼운 비유를 끼워 넣었는가?

#### C. 시청자 인지 흐름 점검 **(§E 시청자 경험 시뮬레이션과 영역 정합: §0 C = 메인 1차 점검 / §E = 에이전트 2차 점검 이중 점검 영역)**

- C1. 시청자 궁금증 발생 후 즉시 답 조각을 던졌는가?
- C2. 능동 추리 차단 지점이 있는가?
- C3. 본문 후반 폭로의 충격이 도입부 누설로 약화됐는가?
- C4. 같은 자리에서 "또 그 얘기" 인상을 주는 구간이 있는가?
- **C5. 직전 정보와 모순되는 질문/거점을 던졌는가?** — 직전 문장에서 정보 X를 노출한 직후 "X일까요? X 아니에요" 같은 부정 거점은 시청자가 이미 답 아는 질문이 되어 흐름 끊김. SL 열림 형C(부정 선행형)는 시청자가 답 모르는 상태에서만 작동.
- **C6. 본문 → 결론부 전환 매끄러움 평가** — 본문 마지막 SL에서 결론부 진입까지 시청자 인지 흐름이 자연스러운가? 결론부 직전 단락에 안전장치(결론 진입 신호)가 사전 배치되었는가? (P4 §D-1-2-A 권장 / 19번 §1-6 매끄러움 평가 정합)
- **C7. 처음 듣는 시청자 즉시 이해 점검** — 모든 SL을 처음 듣는 시청자 입장에서 시뮬레이션. 직접 점검 4차원:
  1) **주어 생략 점검 — 조건부 명시 의무 (한국어 자연성 정합)**
     - 한국어는 주어 생략이 자연스러우나, 다음 조건 충족 시 주어 명시 의무:
       - **조건 a:** 직전 1~2단락 안에 주체가 명시되지 않은 경우
       - **조건 b:** 직전 단락 주체가 모호하여 시청자가 본 문장 주체를 추측 못 하는 경우
       - **조건 c:** 본 문장이 새 정보를 도입하는 경우 (정의·정책·제도 풀이 등)
     - **자가 시뮬레이션:** "이 문장 직전 1~2단락만 보고 시청자가 주체를 즉시 추측할 수 있는가?" 추측 어려우면 주어 명시.
     - **사례 (두바이 대본):** "영국에 살아도 외국에서 번 돈은 영국 세금 안 내도 되는 제도였어요" — 직전 단락 "non-dom 제도가 폐지된 거예요" 직후. 주체("외국에서 온 부자")가 직전 단락에 명시 안 됨 → 조건 a 충족 → 주어 명시 의무 → "외국에서 온 부자가 영국에 살아도"
  2) **앞뒤 표현 매칭 점검** — 같은 개념을 다른 단어로 표현 시 시청자 매칭 실패 위험. 동일 개념은 동일 표현으로 통일.
     - **사례 (두바이 대본):** 앞 "자기 나라에서 번 돈은 영국 세금 안 내도 되는" ↔ 뒤 "전 세계 소득에 영국 세금 매기겠다" → 같은 개념을 다른 단어로 표현 → 시청자 매칭 실패 → 뒤도 "자기 나라에서 번 돈도 빼놓지 않고 영국 세금 매기겠다"로 통일
     - **결산·엔딩 영역 확장:** 본문 호명을 결산·엔딩에서 회수 가능한 형태로 유지. 본문에 없던 새 비유·새 명사가 단독 등장하여 시청자 매칭 실패 위험 발생 시 풀이 한 줄 결합. **사례 (스위스 대본):** 본문 결산 "양쪽 다 거래 인질로 잡을 수 있었어요" — 본문 SL7 "양다리 거래국"에서 "거래 인질"로 표현 변경 → 시청자 매칭 실패. "양다리 거래국이었던 셈" 형태로 회수 또는 풀이 한 줄 결합 필요.
  3) **영문 직역체·추상 용어 단독 사용 점검** — `04_언어_장치.md` §"권위어 친근 풀이" 정합. 영문 직역체(예: worldwide income → "전 세계 소득") · 세무·경제·금융 추상 용어(예: 자본이득세·시가총액·순자산·공시지가·비거주자 세제) 단독 사용 시 친근 풀이 결합 의무 (P4 강제 도구).
  4) **수치·사실 등장 시 직전 문장과의 인과 연결 점검** — 시청자가 "왜 이 사실이 등장하지?" 의문 발생하면 직전 단락에 인과 한 줄 결합 의무. **사례 (스위스 대본):** "독일이 스위스 전체 에너지의 무려 41%를 공급했고" — 직전 단락이 "스위스 은행 환전" 영역인데 갑자기 "독일이 공급" 등장 → 시청자 "왜?" 의문 발생. 직전에 "두 나라는 거래로 단단히 얽혀 있었어요" 같은 인과 한 줄 결합 필요.
- **C8. 지시어(대명사) 명확성 점검 (P4 §A 14 정합)** — 본문 모든 지시 대명사(이것·저것·그것·여기·거기·저기·이때·그때)에 대해 지시 대상이 직전 1~2 문장 안에 있는지 검사. 없으면 명사로 풀어쓰기. 음성 매체 특성상 시청자가 앞 문장 다시 못 봄 → 모호한 지시어는 이탈 유발. 위반 사례(두바이 대본): "두바이의 약점은 사실 1985년부터 거기 있었어요" — '거기' 지시 대상 모호.

#### D. 원본 충실성 점검

- D1. 원본 문장이 이미 좋은데 정량 기준 충족 위해 무리하게 추가했는가?
- D2. 좋은 원본 비유를 다른 비유로 대체했는가?
- D3. 정량 미달이지만 원본이 동등 효과를 발생시키는가?
- **D4. 화자가 본문보다 강하게 호들갑 떠는 감탄 형용사가 있는가?** — "충격적·진짜·엄청" 같은 감탄을 제거 시 정보 0이면 P4 할루시네이션 방지 규칙 위반. **+ 감정 형용사 정합성 점검:** 감탄 형용사가 본문 정보 성격과 어울리는가? **사례 (스위스 대본):** 본문 SL7이 "스위스의 똑똑한 양다리 전략"(통쾌·감탄 영역)인데 "등골이 서늘했는데요"(공포 영역) 사용 → 정보-감정 부조화. 본문 정보 영역과 어긋난 형용사 사용 시 시청자 공감 실패.
- **D5. 동의어 단답·동의어 행위 동사를 단문으로 연속 분할했는가?** — (1) "X입니다. Y입니다. Z입니다" 형태에서 X·Y·Z가 사실상 같은 의미면 redundancy. 자문자답 압축형 도구의 과사용 위험. (2) **동의어성 행위 동사 분할** — "갔다 → 시찰했다", "왔다 → 도착했다" 같은 한 행위를 다른 동사로 분할. 자가점검 — "두 문장이 결국 한 사실을 말하는가? Yes면 한 문장 통합 권장".
- **D6. 메타 언어 + 변주 명칭 노출 점검** — 화자가 서사 구조 자체를 호명("여기서 진짜 핵심이 보입니다"·"이제 본격적으로 들어갑니다"·"이게 핵심이에요") + **사물 주어 SL 진입 톤("X가 등장합니다·~번째 ~은요·다음 단계는 ~")** 또는 P3 §6 변주 작동 원리 명칭("1차 아하·2차 재역전·정점 비유·Scale Jump·정체 폭로·메커니즘 해설형·수치 대비형" 등)을 본문에 직접 노출했는가? 시청자가 메타 라벨 없이 흐름 따라가며 스스로 구조 인지하도록 자연 표현 사용. SL 진입 시 정보 자체로 진입 (04 §1인칭 화자 §금지 정합).
- **D7. 같은 사실이 두 단락에 걸쳐 재인용됐는가?** — "X였다 → (다른 단락) → 양반이 X를 했다"처럼 동일 사실의 단어만 다르게 두 번 등장. 변주 ⑥ 2단 아이러니 순환형 채택 시 특히 발생 — 1차·2차 단락이 같은 사실을 호명할 위험. **한 번만 호명하고 다음 단락은 결과·재해석으로 연결**한다. 단락 간 비교(직전 단락 다시 읽기) 자가 점검 필수.
- **D-A. 원본/이전 대본 표현 반복 점검 (P5 §D D4 메인 이관)** — 이전 대본 25편·동일 채널 영상에서 사용된 표현·비유와 본 대본 표현 비교. 외부 자료(이전 대본 폴더) 필요 영역으로 메인 1차 책임. 에이전트 검증 영역 외.
- **D-B. 수정 후 이전 전제 잔류 점검 (P5 §D D10 메인 이관)** — 수정 사이클에서 ML·피벗·해소 은유 등 구조 요소 변경 시 이전 전제 잔류 여부 검사. 수정 전 대본(히스토리 파일) 필요 영역으로 메인 1차 책임. 에이전트 검증 영역 외.
- **D8. 결산 양식 압축 인과 정확성 점검** — 결산 양식의 "X해서 Y" 압축 인과가 본문 사실에서 실제로 작동했는지 역추적. **사례 (스위스 대본):** "작아서 외교 보증이 통했고" — 본문 SL4는 강대국 8개국 합의가 직접 원인이지 "작은 크기"가 직접 원인 아님. 단순화 비약 시 인과 한 줄 보완 또는 다른 표현으로 변경.
- **D9. 엔딩 8중 점검** — 엔딩 작성 후 다음 8항목 자가 점검:
  1. **맥락 일관성** — 엔딩 각 문장이 직전 문장과 논리 연결되는가? 통찰 단편 나열 아닌가? **+ 회수 톤 자가점검:** 엔딩에서 본문 내용을 다시 호명할 때 회수 표지(예: "아까 봤던") 결합했는가? 결합 없이 단순 사실 단정으로 도입하면 시청자에게 "처음 알려주는" 인상 발생. **사례 (스위스 대본):** "어두운 그림자도 분명히 있었으니까요" — 본문 SL7에서 다룬 내용인데 회수 표지 부재 → "처음 알려주듯" 톤.
  2. **회수 단일화** — 도입부 소재·본문 비유 동시 회수 안 했는가? 한 가지만 선택했는가? (P4 §D-1 회수 대상 단일화 정합 / P5_퇴고검증.md 패턴 6 정합)
  3. **톤 점검** — 시청자에게 의심·경고·음모론 강요하지 않는가? 가벼운 연착인가?
  4. **시그니처 점검** — 채널 이름 정확 표기? 본 채널(이거봐봐) ↔ 참고 채널(지식해적단) 혼동 없는가? (CLAUDE.md "채널 이름 명확 분리" 정합)
  5. **본문 결산 양식 → 진입 어법 자연 연동 (P4 §D-1-2-B 강제 규칙 4건 정합 / P5_퇴고검증.md 패턴 7 정합)** — 본문 결산 양식이 모순형 통찰 압축형(B)인데 ② 이완 호명 없이 직접 ① 결과 호명 점프했는가? ("성급한 마무리" 차단 핵심) / 정리·결산 양식인데 ① 결과 호명 부재? / 이미지강화 부가 정보형인데 ② 이완 호명 부재? / 시청자 시점 결합 결산 양식인데 ④ 회수 부재?
  6. **결론부 직전 단락 안전장치 (P4 §D-1-2-A 권장)** — 본문 마지막 단락에 결론 진입 신호(강화·호소·모순 사례·해법 사례 등)가 사전 배치되었는가?
  7. **결론 브릿지 어법 1~3종 결합 사용 (P4 §D-1-2-C / P5_퇴고검증.md 패턴 8 정합)** — 결론부 진입 단락에 결론 브릿지 어법 5종(① 결과 호명·② 이완 호명·③ 시간 전환·④ 회수·⑤ 메타) 중 1~3종이 사용되었는가? 한 종도 사용 안 했으면 [브릿지 어법 부재] 위험.
  8. **결론부 ML 답 직접 정리 금지 (P4 §D-1 / P5_퇴고검증.md 패턴 5 정합)** — 결론부에 ML 답(표준 Q1·Q2 또는 ML 압축형 단일 질문 답·암시적 루프 해소 답 포함 — `01_유형_모순형.md` P4 §4 변주 정합)을 직접 정리하지 않았는가? 본문 마지막 SL에서 완료가 정상. 단, 과정추적형 "결말 40% 역전" 패턴(03 P3 §10 / 12 가야 — 결론부 40% 이내 ML 해소)은 허용.
  9. **엔딩 ⑦ 경고형 대상자 보편성 (공통 §B.6 ⑦ + P5_퇴고검증.md 패턴 11 정합)** — ⑦ 경고형 채택 시 경고 대상자가 시청자 다수(50%+)와 일치하는가? 1% 미만 소수 대상 경고는 ⑦ 경고형 위반. 위반 사례(두바이 대본): "두바이 부동산·관광 자산 보유자 주의" — 한국 일반 시청자 1% 미만 적용 → 보편 메시지로 변환 필요.

### 검증 에이전트 결과 처리 차등 원칙

§A~E 검증 에이전트가 반환하는 결과 처리 시:

- **위반(❌):** 즉시 처리. 시청자에게 명백히 인지되는 표면 문제.
- **경고(⚠️):** 자연스러움 vs 효과 트레이드오프 평가. 자가 점검:
  1. 원본이 이미 해당 효과를 발생시키는가?
  2. 추가/변경이 원본 흐름을 끊는가?
- 자가 점검 후 모호하면 **유저 합의 단계** 거친다.

### 에이전트 보고 자가 검증 3단계

검증 에이전트가 위반·경고를 반환하면, 메인은 **즉시 적용하지 않고** 다음 3단계 자가 점검을 거친다. 한 단계라도 막히면 유저 합의 단계 거친다.

1. **원본 효과 점검** — 원본 문장/단락이 이미 해당 효과(비유·시청자 대입·정보 전달)를 발생시키는가? 발생시키고 있으면 추가 불필요.
2. **에이전트 판정 정확성 점검** — 에이전트가 "별도 단락 형태"만 인식하여 한 문장 안 압축 형태(예: 한 문장 비유)를 놓치지 않았는가? 특히 비유·시청자 대입 같은 질적 영역.
3. **추가/변경 흐름 영향 점검** — 추가/변경이 원본 흐름을 끊거나, 같은 메시지를 두 번 말하는 형태가 되거나, 직전 정보와 모순되는 부정 거점이 되는가?

**3단계 모두 통과해야 적용.**

## 검증 에이전트 분리 원칙

**P5는 반드시 별도 서브 에이전트(Agent tool)로 실행한다.** P4를 집필한 에이전트가 직접 퇴고하지 않는다.

**이유:** 자기 작성물을 자기가 검증하면 확증 편향이 발생한다. 서브 에이전트는 대본을 "처음 보는 텍스트"로 읽기 때문에 편향이 구조적으로 제거된다.

**메인 에이전트 → 검증 에이전트 전달 항목:**

```text
- 대본 파일 경로
- 팩트리포트 파일 경로
- 조감도 파일 경로
- 서사 유형
- 확정 제목/썸네일
```

**검증 에이전트 반환:** 위반 항목 목록 + 수정 제안. 메인 에이전트가 수정 후 유저에게 보고한다.

## B-2. 친밀 톤 검증 체크리스트

집필 후 P5 퇴고 단계에서 친밀 톤 적용을 검증한다. 모든 항목 통과해야 친밀 톤 적합 판정.

### B2-1. 채널 공통 5대 시그니처 누락 검증 (강제)

- [ ] **1인칭 화자** 우세 — "저는·제가" 빈도 높음. 부적절 위치 강제 삽입 금지.
- [ ] **한국 정체성 환산** 적용 — 외국 정보 등장 시 100% 환산
- [ ] **인터넷 슬랭** 적용 — 편당 6~10회, 정보 피크 직후 위치
- [ ] **권위어 친근 풀이** 적용 — 권위 학술어 직후 친근 풀이 결합 (편당 5~15회)
- [ ] **직접 비교 정량 나열** 적용 — 같은 단위 직접 병치 (편당 4~10회)

### B2-2. 채널 핵심 보조 3종 적용 검증 (권장)

- [ ] **자기 한계 인정** 편당 1~2회 (어려운·논쟁·추정 영역에서)
- [ ] **자문자답 호명** 시청자 의문 예상 지점(모순 진입·정보 피크 직전)에서 사용. 횟수 강제 없음.
- [ ] **외국어/사료 어원 풀이** 적용 — 외국 명칭/사료 등장 시 100% 풀이 (편당 2~5회)

### B2-3. 유형별 분기 강제 도구 적용 검증

> **검증 범위:** 🔴 강제 등급 도구 (P4 강제 — SL 본질·골격) 위주. 🟠 권장 도구의 P5 검증은 B2-4·B2-5·B2-7 + 도구별 정량 기준 04 §7 부속 표 참조.

- [ ] **(모순형)** 모순 진입 마이크로 빌드업 적용 (단계적 의문 누적 후 반전)
- [ ] **(모순형)** 결과 공유 셋업 적용 (도입부 또는 모순 전환 직전)
- [ ] **(모순형)** 단계 폭로 시간축 적용 (편당 4~7단계, 각 단계 60~90초)
- [ ] **(이미지강화형)** Scale Jump 다단 누적 사이클 4~6단 적용
- [ ] **(이미지강화형)** 5대 강제 도구 적용 (단일 인격 강화·이미지 결정타 슬로건·스케일 누적 환산·학술 풀이 누적 사이클·누적 거시 슬로건 결정타)
- [ ] **(과정추적형)** 결말 40% 역전 적용 (본문 약 40% 지점)
- [ ] **(과정추적형)** 자기 셋업 부정 적용 (결말 40% 역전과 연동)
- [ ] **(과정추적형)** 연표 누적 시그니처 적용 (편당 13단 결정타)
- [ ] **(과정추적형)** 5대 강제 도구 적용 (시간 마커 압도·연쇄 인과 사슬 마커·상태 변화 폭로 마커·과거형 서술 우세·전환점 명시 마커)
- [ ] **(과정추적형)** 외교·국제 주제 시 국제 정치 학교 싸움 의인화 100% 적용 (선빵·동네 형 부르기·찐따·뚝배기 깨다 등)
- [ ] **(과정추적형)** 시스템 한계·종말 위험 영역 시 자조적 종말론 결정타 적용

### B2-4. 무리 사용·금지 톤 검증

- [ ] 인터넷 슬랭 편당 10회 이내 (무리 사용 금지)
- [ ] 한 단락 안 동일 도구 반복 없음
- [ ] **권위적 강의자 자세·가르치는 톤·"제가 알려드리겠습니다" 톤 없음**
- [ ] 3인칭 객관화 톤 없음
- [ ] 욕설 수준 표현 없음 (CLAUDE.md 정합)
- [ ] 인터넷 슬랭 도입부 첫 30초 사용 없음
- [ ] 무거운 구간(추모·재난) 슬랭·위트 사용 없음

### B2-5. 타 유형 도구 차용 검증 (충돌 회피)

- [ ] **(모순형 대본)** 발명가·역사 인물 친근 호칭 차용 없음 (이미지강화형 분기)
- [ ] **(모순형 대본)** Scale Jump 다단 누적 사이클 차용 없음 (이미지강화형 분기)
- [ ] **(모순형 대본)** 단일 인격 강화 의인화 차용 없음 (이미지강화형 골격)
- [ ] **(모순형 대본)** 시간 마커 압도·과거형 서술 우세 차용 없음 (과정추적 골격)
- [ ] §4·§5 차용 검증은 B2-6 참조
- [ ] **(이미지강화형 대본)** 시간 척도 환산·사료 친근 인용 차용 없음 (과정추적형 분기)
- [ ] **(이미지강화형 대본)** 결말 40% 역전·연표 누적 시그니처 차용 없음 (과정추적형 분기)
- [ ] **(이미지강화형 대본)** 양면 인격 의인화 차용 없음 (모순형 골격)
- [ ] **(이미지강화형 대본)** 단계 폭로 시간축·양보 후 반박 차용 없음 (모순형 분기)
- [ ] **(과정추적형 대본)** 게임 용어 매핑 차용 없음 (모순형 분기)
- [ ] **(과정추적형 대본)** 양면 인격 의인화·단일 인격 강화 의인화 차용 없음 (다른 유형 골격)
- [ ] **(과정추적형 대본)** 단계 폭로 시간축·양보 후 반박 차용 없음 (모순형 분기)

### B2-6. 비-모순형 공통 도구·이미지+과정 공유 도구 검증

- [ ] **(이미지강화·과정추적)** 자문자답 압축형 적용 — 이미지 편당 3~6회 / 과정 편당 13회+
- [ ] **(이미지강화·과정추적)** 단계 진입 평행 구조 적용 — 이미지 편당 1~5회 / 과정 편당 5~8회
- [ ] **(이미지강화·과정추적)** 반사실 가정 적용 — 이미지 편당 1~3회 / 과정 편당 0~2회
- [ ] **(이미지강화·과정추적)** 가정법 시뮬레이션 적용 — 추상·미래 주제 시 활용
- [ ] **(모순형)** 위 4개 도구 차용 금지 — SL 골격 어긋남 (현재 폭로 vs 가정 충돌)

### B2-7. 채널 부재 도구 차용 금지 검증

채널 25편 모두 사용하지 않는 도구. 우리 대본에 도입 시 채널 톤 어긋남 위험.

- [ ] **반전 질문 침묵여백** 사용 없음
- [ ] **반전 질문 유머전환** 사용 없음
- [ ] **머뭇 회피 후 폭로** 사용 없음
- [ ] **"알고 보면"·"실은" 마커** 사용 없음 (대신 "사실(상)"·"실제로" 사용)

| C10 | **썸네일 약속(L1) 단어 본문 직접 인용 검사 (공통 §0.3 #2 자발적 발견 정합):** 썸네일 **메인 문구**의 핵심 단어(2~4개)를 추출하여 대본 전체에 grep 검색을 수행한다. 직접 인용 발견 시 → 동의어 치환 필수. **썸네일 서브 문구는 검사 대상 외 (§0.3 #2 예외 정합 — 직접 인용 허용)**. 모순 공존형(공통 §A.1) 채택 시 두 단어 동시 검사(예: "영원의 거짓말" → "영원" + "거짓말"). P3 §1 모순 공존형 사전 동의어 정의 박스에서 미리 정의된 후보로 치환. |
| C11 | **단락 간 사실 중복 점검:** 같은 사실(인물·기관·동작·정제 내용)이 두 단락에 걸쳐 단어만 다르게 재인용됐는가? "X였다 → (다른 단락) → 양반이 X를 했다" 형태 발견 시 → 한 번만 호명하고 다음 단락은 결과·재해석으로 연결하도록 수정 제안. 모순형 변주 ⑥ 2단 아이러니 순환형 채택 시 특히 발생 — 1차·2차 단락이 같은 사실 호명할 위험. §0 D7 자체 퇴고 항목과 보완 관계. |
| C12 | **결론부 헤더 표기 검증:** 결론부 헤더가 `## 결론부 ({어법} / {엔딩 유형})` 형식인가? `## 엔딩` 옛 표기·`## 결론부 (엔딩)` 동의어 표기 발견 시 위반. (CLAUDE.md "대본 본문 섹션 헤더 표기 규칙" + P4 §D-1 정합) |
