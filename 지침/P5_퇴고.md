## 검증 에이전트 분리 원칙

**P5는 반드시 별도 서브 에이전트(Agent tool)로 실행한다.** P4를 집필한 에이전트가 직접 퇴고하지 않는다.

**이유:** 자기 작성물을 자기가 검증하면 확증 편향이 발생한다. 서브 에이전트는 대본을 "처음 보는 텍스트"로 읽기 때문에 편향이 구조적으로 제거된다.

**메인 에이전트 → 검증 에이전트 전달 항목:**

```text
- 대본 파일 경로
- 팩트리포트 파일 경로
- 조감도 파일 경로
- 서사 유형
- 확정 제목/썸네일
```

**검증 에이전트 반환:** 위반 항목 목록 + 수정 제안. 메인 에이전트가 수정 후 유저에게 보고한다.

## 낭독 스크립트 파일 생성

P5 전 항목 통과 후, 대본 원문을 그대로 복사한 스크립트 파일을 생성한다.

### 파일명

```text
[낭독대본] 주제명.md
```

### 파일 구조

```text
<!-- 발음 변환 목록 (TTS/녹음 시 참고)
영어 단어 전체를 한 음절씩 읽지 않고, 하나의 단어처럼 발음하는 것만 표기.
(WTO→떠블유티오, UN→유엔 처럼 알파벳을 나열해 읽는 것은 제외)

SWIFT → 스위프트
BRICS → 브릭스
[대본에 등장하는 해당 단어 목록]
-->

[대본 원문 전체 — 변형 없이 그대로]
```

### 판단 기준

- **목록에 포함:** 영어 철자와 무관하게 하나의 단어로 발음하는 것 (SWIFT→스위프트, OPEC→오펙, NATO→나토)
- **목록에서 제외:** 알파벳을 순서대로 읽는 것 (WTO, IMF, UN, GDP, CEO 등)

## 할루시네이션 방지 — 퇴고 전용

- 퇴고 시 "통과"를 위해 실제로 고치지 않고 체크만 하는 행위 금지
- 문제를 발견하면 반드시 수정 후 해당 항목 재검증
