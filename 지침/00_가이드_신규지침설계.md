### 2.3 모듈 로드 트리거 메커니즘

Phase 본체가 모듈을 호출하는 방식 정의.

**원칙:**
- Phase 본체는 모듈 로드 시점을 명시적으로 안내한다
- 집필 에이전트는 Phase 본체 진입 시 본체만 로드. 모듈은 본체 안내 시점에 Read 호출로 추가 로드
- 한 번에 한 Phase 본체 + 필요한 모듈만 로드 (워크플로우 "한 번에 한 Phase 지침" 원칙 정합)

**트리거 형식 (Phase 본체 안에 명시):**

```markdown
> **모듈 로드 시점 (이 Phase에서 진입 시 필요):**
>
> - 본 Phase 진입 직후: `지침/모듈/모듈_이론.md` §A·§B 참조 (해당 절만)
> - 본 채널 3유형 확정 후: `지침/모듈/모듈_엔진.md` §N (결정된 엔진 절만) 참조
> - 집필 진입 직전: `지침/모듈/모듈_어법.md` 전체 + `지침/모듈/모듈_이문호공.md` §4 (4역할) 참조
```

**호출 방식:**
- 집필 에이전트는 `Read` tool로 모듈 파일을 호출. 본문 인용 형식 아님.
- Phase 본체 내 "→ 모듈_엔진 §N 참조" 같은 한 줄 위임 표기로 표현.

**Phase별 모듈 로드 매핑:**

| Phase | 필요 모듈 |
| --- | --- |
| P0 | (없음 — Phase 본체만) |
| P1 | 모듈_이론(썸네일 3분류·도입부 전략 A-H) + 모듈_어법(도입부 예시 카탈로그) |
| P2 | (없음) |
| P3 | 모듈_이론 + 모듈_엔진 (3유형 결정 → 매칭 엔진 §N 진입) + 모듈_이문호공 §1 (10요인 강세 정보유형 권장 범위) |

### 2.4 본 채널 분류 관계 (분석 결과 자연 도출 영역)

**원칙:** 본 채널의 서사 유형·정보 분류·서사 엔진 분류는 **분석 결과로 자연 도출**. 분석 작업 시 기존 분류를 전제로 사용하지 않는다 (§4.7 백지 분석 원칙 정합).

**모듈 재설계 시 활용:** 분석 결과 자연 도출된 분류 + §4.6 3단계 자가 점검 적용 후 폐기·통합·유지 결정.

**점검 대상:** 신규 분류·카탈로그·매트릭스·소분류 작성 전 적용.

**위반 패턴:** A라는 자연 집필 가능 영역을 A, B, C, D ... N으로 분류하거나 A-1, A-2, A-3 으로 세분화하여 추론 과부하 발생 + 부자연 집필 유발.

**위반의 함정 — "채널 고유" 라벨로 정당화:**

- 분류 작성자가 "이건 채널 고유 분류"라고 라벨링하면 §4.1 ②로 통과
- 실제로는 자연 추론 가능 영역인데 라벨만 붙은 경우 다수
- "채널 고유"는 무엇이고 무엇이 아닌지 엄밀히 점검 필요

**자가 점검 질문 (3단계):**

1. **이 분류가 없으면 AI가 자연 집필 불가능한가?**
   - 답이 "분류 없어도 자연 집필 가능" → **분류 폐기** (자연 집필 영역)
   - 답이 "분류 있어야 본 채널 운영 결정 가능" → 2단계로

2. **이 분류는 본 채널 운영 결정에 직접 사용되는가?**
   - 후킹·결론부 어조·서사 구조 등 자연 추론 가능 영역의 라벨링이면 분류 폐기
   - 본 채널 3유형 결정·매칭 엔진 결정·SL 배치 등 운영 결정에 직접 사용되면 3단계로

3. **분류 수가 최소화되었는가?**

**핵심 원칙:** 본 신규 지침 설계는 백지에서 시작합니다. 기존 지침과의 비교·차이 분석은 금지합니다.

**배경:**

- 기존 지침 파일(`아카이브_재구성_*` 등)은 **아카이브로 이동** = 폐기 영역
- 아카이브 이동 이유 = 새롭게 지침 md 파일을 만들기 위함
- 새 지침의 기초 = **분석 작업** (말맛 분석·제목/썸네일 도입부 연결 분석 등)
- 모듈 md 파일(이론·엔진·이문호공·어법)은 본 분석 결과로 작성·재구성

**금지 (분석 작업 영역):**

- 기존 지침의 분류(본 채널 3유형·8엔진·12 언어장치 등)를 **분석 작업 시 분석 전제·도출 결과로 사용 금지**
- 분석 결과를 기존 분류와 비교하여 일치·차이 분석 금지
- "기존 분류와 어떤 관계인지" 표기 금지

**원칙:**

- 분석은 백지에서 자연 도출
- 분석 결과 = 새 지침의 단독 기초
- 모듈 md 파일은 분석 결과 흡수 후 재구성 대상

**모듈 재설계 시 처리 (§4.6 정합):**

- 분석 결과 자연 도출된 분류 + §4.6 3단계 자가 점검 적용 후 폐기·통합·유지 결정
- 기존 분류와 비교 분석 자체는 금지이나, 분석 결과 자연 도출된 분류가 기존 분류와 일치하는 경우 명칭 차용 가능
