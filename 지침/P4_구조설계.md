## 1. 서사 유형별 구조 템플릿

Phase 3에서 확정된 서사 유형에 따라 해당 섹션만 사용한다.

### 모순형

```text
인트로 → 신뢰 상환 → ML → A극(기초SL→심화SL→정점SL) → 피벗BR → B극(What→How→Why→현재의미) → 엔딩
```

### 이미지 강화형

```text
인트로 → 신뢰 상환 → ML → 기초레이어(What) → How레이어 → Scale Jump → Why레이어 → 통찰 → 엔딩
```

### 과정 추적형 — 사건 서사형

```text
인트로 → 신뢰 상환 → ML("이 사건은 왜/어떻게 이 결말에 이르렀나")
→ 발단(배경+촉발) → 전개(확대+가속) → 위기(정점) → 결말 → 의미부여 → 엔딩
```

### 과정 추적형 — 역사 서사형

```text
인트로 → 신뢰 상환 → ML("이 흐름이 지금 우리에게 왜 중요한가")
→ 기원 → 전환점1 → 전환점2 → 전환점N → 현재 도달 → 미래 함의 → 엔딩
```

### 과정 추적형 — 비교 서사형

```text
인트로 → 신뢰 상환 → ML("이 둘/셋의 차이가 만드는 결과는?")
→ 공통 전제 → 사례A → 사례B → (사례C) → 교차 대비 → 귀납적 통찰 → 엔딩
```

## 2. 모순형 설계

### A극 설계

- **A극 정점 문장을 먼저 정한다** (박스 이미지)
- SL의 인지적 기능 구분:
  - **기초SL** (A극 초반) — 기본 전제 건설
  - **심화SL** (A극 중반) — 전제 위에 깊이 추가
  - **정점SL** (A극 끝) — 확신 최고점 도달
- 확신 스케일 확대 순서: 인지 → 놀람 → 확신 → 확신 최고점 → 피벗 추락
- A극의 모든 SL은 A극 정점 방향에 기여해야 한다
- B극 정보가 A극에 미리 노출되면 안 된다

### B극 설계

- **B극 임팩트를 먼저 정한다**
- B극 순서: **현상(What) → 메커니즘(How) → 근본원인(Why) → 현재의미**
- Why SL의 Aha-moment를 한 문장으로 정의한다
- How는 중반에 해소한다 (신뢰 확보 목적)
- Why는 최대한 늦게 배치한다 (텐션 유지 목적)

### 모순형 7가지 변주

| # | 변주 | 설명 |
|---|------|------|
| 1 | 정적 역설 | 하나의 대상이 가진 두 얼굴을 정지 상태에서 대비한다 |
| 2 | 시간축 전개 | 시간 순서에 따라 A극→B극이 자연스럽게 드러나는 구조 |
| 3 | 사례 순회 | 복수 사례를 돌며 공통 패턴(B극)을 귀납적으로 도출한다 |
| 4 | 누적 반복형 | 같은 논리를 층층이 쌓아 정점에서 한꺼번에 뒤집는다 |
| 5 | 양면 동시 건설형 | A극과 B극 정보를 교차 배치하여 동시에 쌓아 올린다 |
| 6 | 감정 피벗형 | 감정의 방향 전환 자체가 서사의 핵심 동력이 된다 |
| 7 | 원인 역추적형 | 결과(현상)에서 출발하여 원인을 역으로 추적한다 |

## 3. 이미지 강화형 설계

### 레이어별 역할

| 레이어 | 설계 목표 | 시청자 상태 |
|---|---|---|
| 기초레이어(What) | 시청자가 "안다고 생각하는 이미지" 확립 | "아 그거 알지" |
| How레이어 | 표면 아래 작동 원리 해부. 신뢰 확보 구간 | "오, 이렇게 돌아가는 거였어?" |
| Scale Jump | 스케일/맥락을 비약적으로 확장. 대상의 진짜 위상 폭로 | "이 정도였어?!" |
| Why레이어 | Scale Jump가 가능한 근본 원인/구조적 이유 해부 | "그래서 이게 이렇게 된 거구나" |
| 통찰 | 새 이미지를 한 문장으로 각인 (지적 전리품 전달) | "이제 다르게 보인다" |

### 텐션 메커니즘: 해상도 격차

- 기초레이어에서 시청자의 현재 해상도(흐릿한 이미지)를 확립
- ML이 "이 대상의 진짜 모습은 당신이 아는 것과 다르다"를 암시
- 레이어를 올릴 때마다 해상도가 높아지면서 "아직 안 보이는 것"에 대한 궁금증이 커짐
- Scale Jump에서 해상도가 급격히 올라가며 정점 도달
- Why레이어에서 "왜 이런 규모/위상이 가능한지" 근본 원인을 밝혀 Aha-moment 생성

### 핵심 전환점

**How → Scale Jump BR:**

- How레이어에서 "원리를 이해했다"는 안정감을 준 뒤
- "그런데 이 원리가 작동하는 스케일을 보면" → Scale Jump 진입
- 피벗처럼 방향이 뒤집히는 건 아니지만 **규모의 충격**으로 정점을 만든다

**Scale Jump → Why BR:**

- Scale Jump의 충격 직후 "왜 이게 가능한 건데?" 질문을 자연스럽게 유발
- Why레이어에서 근본 원인/구조적 배경을 밝히며 Aha-moment 도달
- Scale Jump가 "얼마나"를 보여줬다면, Why는 "왜 그 정도인지"를 설명한다

### SL 배열

| 위치 | SL 역할 |
|---|---|
| 기초→How 전환 | "표면은 알겠는데, 어떻게 이게 가능한 거지?" |
| How 내부 | 메커니즘 하나가 풀릴 때마다 다음 메커니즘에 대한 SL 생성 |
| How→Scale Jump 전환 | "그래서 이게 얼마나 큰 얘긴데?" (핵심 전환 BR) |
| Scale Jump→Why 전환 | "왜 이게 이 정도 규모가 된 거지?" |
| Why→통찰 | "이걸 한마디로 뭐라고 해야 하지?" |

## 4. 과정 추적형 설계

### 하위 유형별 텐션 메커니즘

| 하위 유형 | 텐션 엔진 | 설명 |
|---|---|---|
| 사건 서사형 | 불가피성의 축적 | "이러면 어떻게 되는지 뻔한데..." → 그래도 벌어짐 |
| 역사 서사형 | 현재와의 거리 좁히기 | 과거에서 출발해 "이게 지금 우리 얘기"로 수렴 |
| 비교 서사형 | 패턴 발견의 쾌감 | 사례가 쌓일수록 시청자가 스스로 공통점을 눈치챔 |

### 하위 유형별 SL 설계 원칙

- **사건 서사형:** 각 단계 전환점마다 "다음에 무슨 일이?" SL 자연 생성. 결말을 아는 사건이면 "어떻게 그 지경까지?"로 SL 방향 전환
- **역사 서사형:** 전환점마다 "왜 여기서 방향이 바뀌었나?" SL. 현재 시점에 가까워질수록 SL 밀도 증가
- **비교 서사형:** 사례 A 제시 후 "B는 같은 조건에서 왜 다른 결과?" SL. 마지막 사례 후 "이 차이의 진짜 원인은?" 핵심 SL

### SL 보충 전략

전환점만으로 SL이 부족할 경우 (2분당 1개 미달):

- 전환점 내부에서 "왜 이 선택을 했는가?" SL을 하위 분기로 생성
- **사건 서사형:** 각 단계에 "만약 다른 선택이었다면?" 반사실 SL 추가
- **역사 서사형:** 전환점 사이에 "이때 왜 다른 경로를 안 택했는가?" SL 삽입
- **비교 서사형:** 사례 내부에서 "이 조건이 왜 결정적인가?" SL 분기

## 5. 공통 SL 배열 규칙

- **전제 건설 체인:** 앞 SL의 해소가 뒤 SL의 전제가 된다
- **편당 SL 수:** 2분당 1개 기준
  - 10분 = 5개 / 13분 = 6~7개 / 16분 = 8개
- **2분 이상 SL 없는 구간 금지**
- SL로 포장할 정보와 SL 없이 서술할 정보를 구분한다

## 6. BR(브릿지) 5유형

| # | 유형 | 패턴 | 빈도 |
|---|------|------|------|
| 1 | 역접 전환형 | "하지만/그런데" | 최다 |
| 2 | 질문 전환형 | "~냐?/~까요?" | 중간 |
| 3 | 정산 전환형 | "자/이쯤 되면" | 중간 |
| 4 | 확장 전환형 | "~뿐만이 아니라" | 소수 |
| 5 | 감정 피벗형 | 감정 전환 | 편당 최대 1회 |

- **BR-SL 동시 작동 원칙:** 절반 가까이를 동시 작동으로 설계한다
- **핵심 피벗 BR:** 편당 1회, 모순형은 A극→B극 전환점에 배치

### 논리적 브릿지 (비 오픈루프) 사용 가이드

오픈루프(SL)를 열지 않고 정보를 정리하는 구간에서 사용한다. 전체 BR 중 20% 이하로 제한한다.

| 유형 | 기능 | 패턴 |
|---|---|---|
| 선형적 흐름 연결 | 시간/공간의 계보 나열 | "유대교가 → 기독교를 → 이슬람교를" |
| 인과적 흐름 연결 | 사건의 결과가 다음의 원인이 되는 꼬리물기 | "이러한 이유로", "때문에" |

### 루프 중첩 제약

- 서브 오픈루프 중첩은 1개 유지를 원칙으로 하되, 서사 전개상 필요한 경우에만 최대 2개까지 허용
- 3개 이상의 서브 오픈루프가 동시에 열려 있는 상태는 절대 불가
- 2개가 열려 있다면 새 루프를 열기 전에 반드시 기존 루프 중 하나를 해소한다

## 8. 서사 비유 프레임

주제의 핵심 메커니즘과 일치하는 비유 영역을 1개 선정한다. 이 프레임이 대본 전체를 관통하는 비유의 세계관이 된다.

| 주제 메커니즘 | 비유 영역 예시 |
|---|---|
| 경쟁·승패 | 스포츠·게임 |
| 관성·잠금 | 기계·교통 |
| 축적·붕괴 | 건축·지질 |
| 전환·변화 | 요리·화학 |
| 확산·전파 | 생태·전염 |

- 같은 단락 내 영역 혼용 금지
- 무거운 구간(전쟁 피해, 재난, 추모 등)에서는 절제
- 팩트 리포트 검토 후 조정 가능

## 10. 루프 해소와 보상 원칙

### 보상의 비례 원칙

루프를 유지(지연)시킨 시간만큼 보상의 가치(정보의 밀도)도 커야 한다.

- ML(영상 전체 관통 루프): 가장 오래 유지되므로 가장 큰 보상이 필요
- A극 전체를 지탱한 SL: 피벗의 낙차가 보상 역할
- 개별 SL(2분 단위): 박스 이미지(강) + 뒷받침(약)이 최소 보상 단위

### 지적 보상의 품질 테스트

> "이 해소를 들은 시청자가 다른 사람에게 설명하고 싶어지는가?" — 아니면 보상이 약한 것.

### 보상의 분할 제공

가장 큰 보상을 한 번에 던지지 않는다.

```text
결과 공개 → 비하인드/의미 부여 → 추가 팁(또는 현재 연결)
```

## 11. 정보 배치 원리

- 익숙한 것의 숨겨진 의미를 폭로한다
- 없는 세계의 고통으로 가치를 증명한다
- 시청자가 스스로 패턴을 발견하게 한다
- 아는 것을 게이트웨이로 활용한다

### 구조 흐름
| 순서 | 구간 | SL 열림 | 내용 요약 | SL 해소 지점 | BR 유형 |

| 6 | 결론부 | | | ML 해소 | |

### Aha-moment
- 위치: (몇 번째 구간)
- 내용: (한 문장)
- 예상 반응:

### 결론

### 메커니즘 요약

## 14. 할루시네이션 방지 규칙

- SL 개수를 억지로 맞추기 위해 불필요한 정보를 넣지 않는다
- 감정 곡선을 도식적으로 따르느라 부자연스러운 전환을 만들지 않는다
- 팩트 리포트에 없는 내용으로 구조를 채우지 않는다
