# 마스터 워크플로우 (전체 제작 컨트롤러)

> **이 파일의 역할:** 대본 제작의 전체 흐름을 제어한다. 각 Phase에서 해당 Phase의 지침 파일만 로드하여 작업한다. 이전 Phase의 지침은 참조하지 않는다.

---

## 지침 망각 방지 원칙

1. **반드시 한 번에 한 Phase만 로드한다.** 전체 지침을 동시에 읽지 않는다.
2. **각 Phase 완료 시 GATE CHECK를 통과해야 다음 Phase로 진행한다.**
3. **Phase 파일 내부의 규칙만 따른다.** 다른 Phase 파일의 규칙을 기억해서 적용하지 않는다.
4. **집필 직전 지침 재로드:** P4 집필에 들어가기 전, 반드시 `P4_집필.md`를 다시 읽는다. 조감도와 팩트 리포트만 보고 집필하지 않는다. — 지침을 설계한 시점과 집필 시점 사이에 맥락이 밀려 지침 적용이 누락된다

## 할루시네이션 방지 원칙

1. **지침의 예시 문구를 대본에 그대로 사용하지 않는다.** 원리를 이해한 뒤 새로운 표현을 생성한다.
2. **확인되지 않은 수치/사실을 단정하지 않는다.** 웹 검색으로 팩트를 확인한 뒤 사용한다.
3. **지침 내부 용어(역설엔진, A극/B극, SL, BR, ML 등)를 대본에 노출하지 않는다.**
4. **같은 비유/밈/감탄사를 한 편 내에서 반복 사용하지 않는다.**

---

## 참조 표기 처리 원칙 (모든 지침 공통 — 핵심)

지침에 cross-reference 표기 ("→ 공통 §X.Y 참조" / "유형 파일 §X 참조" / "P3_구조설계 §X 참조" 등)가 등장하면:

1. **반드시 해당 섹션을 펼쳐 정의·사례·세부 규칙을 확인한 뒤 적용한다.**
2. 참조 표기는 "선택 안내"가 아니라 **"이 위치의 정의가 다른 곳에 있다"는 강제 포인터**.
3. 참조 미확인 시 옛 정보·익숙한 정보로 누락·오용 위험 발생 (예: 엔딩 5유형만 사용하고 신규 3유형 누락).

→ 모든 cross-reference는 **단일 지침 문서 원칙**(중복 회피)을 위한 것이며, 정보 접근은 단계별 함께 로드 명세로 자동 보장된다. 참조를 가볍게 여기지 않는다.

## 단일 지침 문서 갱신 시 동기화 의무 (하이브리드 구조 유지)

본 워크플로우는 **하이브리드 구조** (단일 지침 문서 = 공통/유형 파일 + 핵심 식별 정보 인라인 = 단계 파일)를 채택한다. 따라서 다음 동기화 의무가 발생한다:

1. **공통원칙(`00_공통_원칙.md`) 또는 유형 파일(`NN_유형_*.md`) 정의 갱신 시:**
   - 단계 파일(`P3_구조설계.md` · `P4_집필.md` 등)의 인라인 부분이 갱신된 정의와 일치하는지 점검
   - 불일치 발견 시 단계 파일 인라인을 즉시 동기화

2. **갱신 추적:**
   - 정의 갱신은 `[추출] 대본 - 분석/00_지침서_수정이력.md`에 기록
   - 단계 파일 인라인 동기화도 함께 기록 (또는 후속 동기화 작업 발생)

3. **위반 시 결과:** 옛 분류 잔재 발생 (예: 엔딩 5유형 vs 8유형 불일치 사례 — 2026-04-28 발견)

→ 단계 파일 인라인은 "안정적 핵심 식별 정보"(이름·개수·1줄 정의)에 한정하고, 변동 가능한 상세(사례·25편 분석·궁합)는 참조로 유지하여 동기화 부담을 최소화한다.

## Phase 흐름

```text
[키워드 발굴 요청]
    ↓
Phase 0: 키워드 발굴 → 방식 선택 (A/B/A+B) → 주제 확정
    ↓ 유저 확정
Phase 1: 퀵서치 → 핵심 팩트 5개 + 서사 유형 방향 판단 + 제목/썸네일 후보 5개
    ↓ 유저 선택
Phase 2: 딥리서치 → 팩트 리포트
    ↓ GATE 2 통과
Phase 3: 구조설계 → 구조 보고 (전체 조감도)
    ↓ 유저 승인
Phase 4: 대본 집필 + 팩트 검증
    ↓ GATE 4 통과
Phase 5: 퇴고 + 최종 검증 ← **검증 에이전트(서브 에이전트)로 실행**
    ↓
Phase 6: 장면 소스 제작 (방식 A 애니메이션 / 방식 B 실사 혼합)
    ↓
[7단계] 영상 제작 (촬영/편집) — 유저 담당
    ↓
[8단계] 썸네일/제목 최종화
    ↓
[9단계] 업로드 + SEO
    ↓
[10단계] 배포/프로모션
```

| 5 | `P5_퇴고.md` | ✅ 공통 + 확정된 유형 | **별도 서브 에이전트**가 체크리스트 통과, 맥락 검증, 시뮬레이션 |
| 6 | `P6_장면소스.md` | ❌ (조감도 기반) | **별도 서브 에이전트**가 장면 분할 + 연출 가이드 + 소스 배정 |

- ✅ 공통원칙(`00_공통_원칙.md`) + 해당 유형 지침(`01·02·03_유형_*.md`) 함께 로드
- 🔵 공통원칙만 함께 로드 (유형 미확정·미적용 단계)
- ❌ 추가 지침 로드 불필요 (Phase 지침 + Phase 입력 자료만)

## 핵심 설계 원리: 제목이 약속하고, 팩트가 증명한다

```text
키워드 입력
→ 퀵서치 기반 제목/썸네일 확정 (시청자와의 약속)
→ 딥리서치로 팩트 확보 + 신뢰 상환 전략 결정 (약속 이행 재료 + 전략)
→ 팩트 기반 구조설계 → 집필
```

팩트 리포트 단계에서 제목 약속을 어떻게 증명할지 **신뢰 상환 전략**을 결정한다. 도입부 신뢰 상환·본문 이연 신뢰 상환 중 주제 성격에 맞는 전략을 선택한다. 세부는 P2 1. 참조.

## GATE CHECK 상세

### GATE 1 (퀵서치 → 딥리서치 진입 조건)

> **본격 체크리스트는 `P1_퀵서치_제목썸네일.md` §11 GATE 1 체크리스트 참조.** 마스터 워크플로우는 위임만 표기 (단일 지침 문서 원칙).

### GATE 2 (딥리서치 → 구조설계 진입 조건)

> **본격 체크리스트는 `P2_딥리서치.md` §7 GATE 2 체크리스트 참조.** 마스터 워크플로우는 위임만 표기 (단일 지침 문서 원칙).

### GATE 3 (구조설계 → 집필 진입 조건)

- [ ] 유저가 구조 보고를 승인했는가?
- [ ] SL 배열이 전제 건설 체인을 따르는가?
- [ ] 엔딩 유형이 선택되었는가?
- [ ] 영상 목표 길이가 10~20분인가?
- [ ] 각 SL의 보상이 유지 시간에 비례하는가?
- [ ] ML 해소가 "타인에게 설명하고 싶은" 수준의 통찰인가?

### GATE 4 (집필 → 퇴고 진입 조건)

> **본격 체크리스트는 `P4_집필.md` GATE 4 체크리스트 참조.** 마스터 워크플로우는 위임만 표기 (단일 지침 문서 원칙).

### GATE 5 (퇴고 → 장면소스 진입 조건)

> **본격 체크리스트는 `P5_퇴고.md` A~E 전항목 참조.** 마스터 워크플로우는 위임만 표기 (단일 지침 문서 원칙).
>
> P5는 별도 서브 에이전트(`.claude/agents/P5_퇴고검증.md`)가 검증 본체이며, 메인 에이전트는 위반 항목 수정 후 유저 최종본 승인 → P6 진입.

## 7~10단계 (영상 제작 → 배포)

> Phase 5 완료 후 진입. 7단계는 유저 주도, 8~10단계는 Claude + 유저 협업.

| 단계 | 담당 | Claude 역할 |
|---|---|---|
| 7. 영상 제작 | **유저** (촬영/녹음/편집/자막/BGM) | 연출 가이드 기반 시각 자료 리스트, 자막 텍스트 정리 |
| 8. 썸네일/제목 최종화 | Claude + **유저** 최종 선택 | P1 후보 기반 제목 변형안 3~5개 |
| 9. 업로드 + SEO | Claude + **유저** 업로드 | 영상 설명문, 태그, 챕터(타임스탬프), 고정 댓글 문구 생성 |
| 10. 배포/프로모션 | **유저** 게시 | 커뮤니티 요약 텍스트, 쇼츠 하이라이트 구간, SNS 홍보 문구 생성 |
