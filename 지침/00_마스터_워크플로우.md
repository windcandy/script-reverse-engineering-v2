# 마스터 워크플로우 (대본 제작 컨트롤러)

> **이 파일의 역할:** 대본 제작의 전체 흐름을 제어한다. 각 Phase에서 해당 Phase의 지침 파일만 로드하여 작업한다. 이전 Phase의 지침은 참조하지 않는다.

---

## 지침 망각 방지 원칙

1. **한 번에 한 Phase만 로드한다.** 전체 지침을 동시에 읽지 않는다.
2. **각 Phase 완료 시 GATE CHECK를 통과해야 다음 Phase로 진행한다.**
3. **Phase 파일 내부의 규칙만 따른다.** 다른 Phase 파일의 규칙을 기억해서 적용하지 않는다.

## 할루시네이션 방지 원칙

1. **지침의 예시 문구를 대본에 그대로 사용하지 않는다.** 원리를 이해한 뒤 새로운 표현을 생성한다.
2. **확인되지 않은 수치/사실을 단정하지 않는다.** 웹 검색으로 팩트를 확인한 뒤 사용한다.
3. **지침 내부 용어(역설엔진, A극/B극, SL, BR, ML 등)를 대본에 노출하지 않는다.**
4. **같은 비유/밈/감탄사를 한 편 내에서 반복 사용하지 않는다.**

---

## Phase 흐름

```
Phase 1: 주제 선정 + 핵심 질문
  ↓ GATE 1 통과
Phase 2: 구조설계 (서사 유형 + SL 배열)
  ↓ GATE 2 통과
Phase 3: 제목 + 썸네일
  ↓ GATE 3 통과
Phase 4: 대본 집필 (도입부 → 본론 → 엔딩)
  ↓ GATE 4 통과  
Phase 5: 퇴고 + 최종 검증
```

## Phase별 로드 파일

| Phase | 로드할 파일 | 핵심 임무 |
|---|---|---|
| 1 | `P1_주제_핵심질문.md` | 주제 확정, 핵심 질문 한 문장, 프레임 선택 |
| 2 | `P2_구조설계.md` | 서사 유형, A극/B극 설계, SL 배열, 감정곡선 |
| 3 | `P3_제목_썸네일.md` | 제목(박스이미지), 썸네일 문구 5개 이상 |
| 4 | `P4_집필.md` | 도입부→본론→엔딩 순서로 집필 |
| 5 | `P5_퇴고.md` | 체크리스트 통과, 맥락 검증, 시뮬레이션 |

## GATE CHECK 상세

### GATE 1 (주제 → 구조설계 진입 조건)
- [ ] 핵심 질문이 한 문장으로 정의되었는가?
- [ ] "에? 왜?" 반응 테스트를 통과하는가?
- [ ] 프레임(기술적/문화적/경제적/지정학적 등)이 선택되었는가?
- [ ] 최종 결론, 지적 전리품, 논리 엔진이 정의되었는가?

### GATE 2 (구조설계 → 제목/썸네일 진입 조건)
- [ ] 서사 유형(모순형/이미지강화형/과정추적형)이 결정되었는가?
- [ ] A극 정점 문장이 정의되었는가?
- [ ] B극 Aha-moment가 한 문장으로 정의되었는가?
- [ ] SL 배열이 전제 건설 체인을 따르는가?
- [ ] 피벗 BR이 설계되었는가?

### GATE 3 (제목/썸네일 → 집필 진입 조건)
- [ ] 제목이 "시청자가 소유할 지적 전리품"을 담고 있는가?
- [ ] 썸네일 문구가 이문호공 2요인 이상 결합되어 있는가?
- [ ] 썸네일만 보고 답을 유추할 수 없는가?
- [ ] 금지어("당신이 몰랐던", "놀라운 비밀" 등)가 없는가?

### GATE 4 (집필 → 퇴고 진입 조건)
- [ ] 대본 전문이 완성되었는가?
- [ ] 영상 길이가 13~17분 범위인가? (글자 수 기준 약 4,500~6,500자)
- [ ] 도입부 2분 이내에 ML이 열려 있는가?
- [ ] 박스 이미지이 최소 3개 있는가?
