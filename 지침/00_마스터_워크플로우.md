# 마스터 워크플로우 (전체 제작 컨트롤러)

> **이 파일의 역할:** 대본 제작의 전체 흐름을 제어한다. 각 Phase에서 해당 Phase의 지침 파일만 로드하여 작업한다. 이전 Phase의 지침은 참조하지 않는다.

---

## 지침 망각 방지 원칙

1. **반드시 한 번에 한 Phase 본체만 로드한다.** 전체 지침을 동시에 읽지 않는다. 모듈은 본체가 안내하는 시점에 한해 추가 로드한다.
2. **각 Phase 완료 시 GATE CHECK를 통과해야 다음 Phase로 진행한다.**
3. **Phase 본체 내부의 규칙만 따른다.** 다른 Phase 본체의 규칙을 기억해서 적용하지 않는다.
4. **집필 직전 지침 재로드:** P4 집필에 들어가기 전, 반드시 `P4_집필.md` + P4가 안내하는 모듈 전체를 다시 읽는다. 조감도와 팩트 리포트만 보고 집필하지 않는다.

## 참조 표기 처리 원칙 (모든 지침 공통 — 핵심)

지침에 상호 참조 표기 ("→ 모듈_레고이론 §0.4 참조" / "모듈_엔진 §N 참조" / "P3_구조설계 §3 참조" 등)가 등장하면:

1. **반드시 해당 섹션을 펼쳐 정의·사례·세부 규칙을 확인한 뒤 적용한다.**
2. 참조 표기는 "선택 안내"가 아니라 **"이 위치의 정의가 다른 곳에 있다"는 강제 포인터**.
3. 참조 미확인 시 옛 정보·익숙한 정보로 누락·오용 위험 발생.

→ 모든 상호 참조는 원본이 한 곳에만 있도록 하는 단일 출처 원칙을 위한 것이며, 정보 접근은 Phase별 모듈 로드 매핑으로 자동 보장됩니다. 참조를 가볍게 여기지 않습니다.

## 단일 출처 갱신 시 동기화 의무

본 지침 시스템은 **모듈 카탈로그(원본) + Phase 본체 인라인 식별 정보(안정적 핵심만)** 구조를 채택합니다. 다음 동기화 의무가 발생합니다:

1. **모듈 7파일 정의 갱신 시:**
   - Phase 본체의 인라인 부분이 갱신된 정의와 일치하는지 점검
   - 불일치 발견 시 Phase 본체 인라인을 즉시 동기화

2. **갱신 추적:**
   - 정의 갱신은 `00_용어정리.md` 업데이트 이력에 기록
   - Phase 본체 인라인 동기화도 함께 기록

3. **위반 시 결과:** 옛 분류 잔재 발생

→ Phase 본체 인라인은 "안정적 핵심 식별 정보"(이름·개수·1줄 정의)에 한정하고, 변동 가능한 상세(사례·세부 규칙·궁합)는 참조로 유지하여 동기화 부담을 최소화합니다.

## Phase 흐름

```text
[키워드 발굴 요청] (예: "키워드 발굴해줘" / "키워드 찾아줘")
    ↓
Phase 0: 키워드 발굴 → 방식 선택 (A/B/A+B/C) → 3개 후보 확정
    ↓ 유저 확정
Phase 1: 3개 후보 각각 퀵서치 → 핵심 팩트 5개 + 본 채널 3유형 방향 판단 + 제목/썸네일 후보 1-2개 (총 3-6개)
    ↓ 유저 선택
Phase 2: 딥리서치 → 팩트 리포트
    ↓ GATE 2 통과
Phase 3: 구조설계 → 본 채널 3유형 확정 + 매칭 엔진 결정 + 4위치 골격 + 조감도
    ↓ 유저 승인
Phase 4: 대본 집필 + 팩트 검증
    ↓ GATE 4 통과
Phase 5: 퇴고 + 최종 검증
    ↓
Phase 6: 장면 소스 제작 (방식 A 애니메이션 / 방식 B 실사 혼합)
    ↓
[후속 1] 영상 제작 (촬영/편집) — 유저 담당
    ↓
[후속 2] 썸네일/제목 최종화
    ↓
[후속 3] 업로드 + SEO
    ↓
[후속 4] 배포/프로모션
```

## 핵심 설계 원리: 제목이 약속하고, 팩트가 증명한다

```text
키워드 입력
→ 퀵서치 기반 제목/썸네일 확정 (시청자와의 약속)
→ 딥리서치로 팩트 확보 + 신뢰 상환 전략 결정 (약속 이행 재료 + 전략)
→ 팩트 기반 구조설계 → 집필
```

팩트 리포트 단계에서 제목 약속을 증명할 **신뢰 상환 가능성 팩트를 확보**합니다 (도입부 즉시 증명 / 본문 이연 증명 — 전략 확정은 P3). 세부는 `P2_딥리서치.md` §1 참조.
