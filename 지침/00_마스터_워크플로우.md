# 마스터 워크플로우 (전체 제작 컨트롤러)

> **이 파일의 역할:** 대본 제작의 전체 흐름을 제어한다. 각 Phase에서 해당 Phase의 지침 파일만 로드하여 작업한다. 이전 Phase의 지침은 참조하지 않는다.

---

## 지침 망각 방지 원칙

1. **한 번에 한 Phase만 로드한다.** 전체 지침을 동시에 읽지 않는다.
2. **각 Phase 완료 시 GATE CHECK를 통과해야 다음 Phase로 진행한다.**
3. **Phase 파일 내부의 규칙만 따른다.** 다른 Phase 파일의 규칙을 기억해서 적용하지 않는다.

## 할루시네이션 방지 원칙

1. **지침의 예시 문구를 대본에 그대로 사용하지 않는다.** 원리를 이해한 뒤 새로운 표현을 생성한다.
2. **확인되지 않은 수치/사실을 단정하지 않는다.** 웹 검색으로 팩트를 확인한 뒤 사용한다.
3. **지침 내부 용어(역설엔진, A극/B극, SL, BR, ML 등)를 대본에 노출하지 않는다.**
4. **같은 비유/밈/감탄사를 한 편 내에서 반복 사용하지 않는다.**

## Phase 흐름

```text
[채널 운영] 주제 풀 관리 (채널운영_주제풀.md) — 독립 운영
  ↓
Phase 1: 주제 선정 + 핵심 질문 + 프레임 + 결론
  ↓ GATE 1 통과
Phase 2: 심층 리서치 (서사 요소별 팩트 발굴)
  ↓ GATE 2 통과
Phase 3: 서사 유형 확정 + 제목 + 썸네일 + 도입부 전략
  ↓ GATE 3 통과
Phase 4: 구조설계 (SL/BR 배열)
  ↓ GATE 4 통과
Phase 5: 대본 집필 (도입부 → 본론 → 엔딩)
  ↓ GATE 5 통과
Phase 6: 퇴고 + 최종 검증
  ↓
Phase 7: 장면 소스 제작 (방식 A 애니메이션 / 방식 B 실사 혼합)
  ↓
[7단계] 영상 제작 (촬영/편집) — 유저 담당
  ↓
[8단계] 썸네일/제목 최종화
  ↓
[9단계] 업로드 + SEO
  ↓
[10단계] 배포/프로모션
```

## Phase별 로드 파일

| Phase | 로드할 파일 | 핵심 임무 |
|---|---|---|
| 1 | `P1_주제_핵심질문.md` | 주제 확정, 핵심 질문 한 문장, 프레임 선택, 결론/지적전리품/논리엔진 정의 |
| 2 | `P2_심층리서치.md` | 서사 요소별 팩트 발굴, 팩트 리포트 작성 |
| 3 | `P3_제목_썸네일.md` | 서사 유형 확정, 제목(박스이미지), 썸네일 문구 5개+, 도입부 전략 선택 |
| 4 | `P4_구조설계.md` | 서사 유형별 극/레이어/전환점 설계, SL/BR 배열 |
| 5 | `P5_집필.md` | 도입부→본론→엔딩 순서로 집필 |
| 6 | `P6_퇴고.md` | 체크리스트 통과, 맥락 검증, 시뮬레이션 |

## 핵심 설계 원리: 팩트가 모든 의사결정에 선행한다

```text
주제/질문 확정 → 팩트 발굴(심층 리서치)
→ 팩트 기반 서사 유형 확정 + 시청자와의 약속(제목/썸네일)
→ 팩트 기반 구조설계 → 집필
```

팩트를 모르는 상태에서 서사 유형을 확정하지 않는다. 팩트를 모르는 상태에서 제목/썸네일을 확정하지 않는다. 팩트를 모르는 상태에서 구조를 설계하지 않는다.

## GATE CHECK 상세

### GATE 1 (주제 → 심층 리서치 진입 조건)

- [ ] 핵심 질문이 한 문장으로 정의되었는가?
- [ ] "에? 왜?" 반응 테스트를 통과하는가?
- [ ] 프레임(기술적/문화적/경제적/지정학적 등)이 선택되었는가?
- [ ] 최종 결론, 지적 전리품, 논리 엔진이 정의되었는가?
- [ ] Q1~Q8 답변이 완료되었는가?
- [ ] 경쟁 환경 점검이 완료되었는가? (레드오션 여부 + 차별화 포인트 한 문장 정의)

### GATE 2 (심층 리서치 → 제목/썸네일 진입 조건)

- [ ] 가능성 있는 서사 유형의 필수 요소가 팩트로 뒷받침되는가?
  - 모순형 가능성: 역설의 양극(A극 재료 / B극 재료) 팩트 확보
  - 이미지 강화형 가능성: Scale Jump 팩트 확보
  - 과정 추적형 가능성: 핵심 전환점 팩트 확보
- [ ] Aha-moment 후보가 실제 데이터에 기반하는가?
- [ ] 결론/지적 전리품을 뒷받침하는 근거가 확보되었는가?
- [ ] 팩트 리포트에 출처가 기록되었는가?

### GATE 3 (제목/썸네일 → 구조설계 진입 조건)

- [ ] 서사 유형(모순형/이미지강화형/과정추적형)이 팩트에 근거하여 확정되었는가?
- [ ] 제목이 "시청자가 소유할 지적 전리품"을 담고 있는가?
- [ ] 썸네일 전략(이미지 강화형/조각 추출형)이 선택되었는가?
- [ ] 썸네일 문구에 이문호공 2요인 이상이 후킹 프레이밍으로 적용되었는가?
- [ ] 썸네일 후보가 5개 이상이고, 각각에 전략·비중·메커니즘이 기술되었는가?
- [ ] 후보 중 답을 유추 가능한 것이 없는가?
- [ ] 금지어("당신이 몰랐던", "놀라운 비밀" 등)가 없는가?
- [ ] 제목과 썸네일이 동일 정보를 반복하지 않는가?
- [ ] 도입부 전략(A~H)이 선택되었는가?

### GATE 4 (구조설계 → 집필 진입 조건)

**공통:**
- [ ] SL 배열이 전제 건설 체인을 따르는가?
- [ ] 엔딩 유형이 선택되었는가?
- [ ] 서사 비유 프레임이 주제 메커니즘과 일치하는가?
- [ ] 영상 목표 길이가 13~17분인가?
- [ ] 각 SL의 보상이 유지 시간에 비례하는가?

**모순형 추가:**
- [ ] A극 정점 문장이 팩트에 기반하여 정의되었는가?
- [ ] B극 Aha-moment가 한 문장으로 정의되었는가?
- [ ] 피벗 BR이 설계되었는가?

**이미지 강화형 추가:**
- [ ] 기초레이어의 "시청자가 안다고 생각하는 이미지"가 명확한가?
- [ ] Scale Jump 문장이 정의되었는가?
- [ ] Why레이어의 근본 원인이 Scale Jump를 설명하는가?
- [ ] 통찰 문장(지적 전리품)이 한 문장으로 정의되었는가?
- [ ] How→Scale Jump 전환 BR이 설계되었는가?
- [ ] Scale Jump→Why 전환 BR이 설계되었는가?

**과정 추적형 추가:**
- [ ] 하위 유형(사건/역사/비교)이 선택되었는가?
- [ ] 핵심 전환점(위기/분기/교차점)이 정의되었는가?
- [ ] ML이 "결말"이 아닌 "과정의 의미"를 묻고 있는가?

### GATE 5 (집필 → 퇴고 진입 조건)

- [ ] 대본 전문이 완성되었는가?
- [ ] 글자 수 약 4,500~6,500자 (13~17분)인가?
- [ ] 도입부 2분 내 ML이 열려 있는가?
- [ ] 박스 이미지(강) + 뒷받침(약)이 최소 3세트 있는가?
- [ ] SL 열림 유형이 교차 배치되었는가?
- [ ] 반전 질문이 5회+ 있는가?
- [ ] 2분+ SL 없는 구간이 없는가?
- [ ] "우리/한국/여러분" 배치가 후반부에 있는가?

## 7~10단계 (영상 제작 → 배포)

> Phase 6 완료 후 진입. 7단계는 유저 주도, 8~10단계는 Claude + 유저 협업.

| 단계 | 담당 | Claude 역할 |
|---|---|---|
| 7. 영상 제작 | **유저** (촬영/녹음/편집/자막/BGM) | 연출 가이드 기반 시각 자료 리스트, 자막 텍스트 정리 |
| 8. 썸네일/제목 최종화 | Claude + **유저** 최종 선택 | P3 후보 기반 제목 변형안 3~5개, YouTube API 제목 패턴 분석 |
| 9. 업로드 + SEO | Claude + **유저** 업로드 | 영상 설명문, 태그, 챕터(타임스탬프), 고정 댓글 문구 생성 |
| 10. 배포/프로모션 | **유저** 게시 | 커뮤니티 요약 텍스트, 쇼츠 하이라이트 구간, SNS 홍보 문구 생성 |
