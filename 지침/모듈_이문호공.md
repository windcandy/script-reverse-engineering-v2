## §0. 본 모듈 역할 + 활용 시점

### 0.1 모듈 역할

본 모듈은 본 채널의 **끌리는 이유 카탈로그**다. 두 축을 통합 제공한다.

- **이문호공 10요인** — 시청자가 콘텐츠에 끌리는 심리 동력 10종. 본문 ML·해소·썸네일·제목 모든 후킹 지점의 공통 재료.
- **4역할 카탈로그** — 본문 단락이 서사 안에서 수행하는 4가지 역할 (오픈루프·의도적 지연·보상 투하·충전·빌드업).

### 0.2 활용 시점

| 시점 | 활용 영역 |
| --- | --- |
| P1 (퀵서치·제목·썸네일) | 썸네일·제목 후킹 — 10요인 모두 (10 도덕적 공분 + 9·4 부정·감정 변주 포함) |
| P3 (구조설계) | 9유형 × 4위치 매핑으로 본문 단락 배치 검증 |
| P4 (집필) | 4역할 카탈로그로 본문 블록 작성 + ML·해소 이문호공 권장 조합 적용 |
| P5 (퇴고) | ML 영역 후킹 충분성 점검 + 해소 은유 정보 밀도 점검 |

### 0.3 단위 정의 (본 모듈 한정 용어 정합)

- **4위치:** [기] 도입 / [승] 빌드업 / [전] 전환 / [결] 결산
- **9유형 (정보 유형):** 기원·배경 / 정의·개념 / 통계·수치 / 인과 / 사례·일화 / 비교·대비 / 인용·증언 / 메커니즘 / 해석·결산
- **4역할 (블록 작동):** 오픈루프 / 의도적 지연 / 보상 투하 / 충전·빌드업
- **블록:** 본문 단락 단위

> **본 채널 3유형 ↔ 9유형 관계:** 본 채널 3유형(모순형·이미지 강화형·과정 추적형)은 거시 서사 골격이다. 9유형은 그 안의 블록별 정보 분류로, 본 모듈 §3 매핑 + 모듈_엔진 "강세 정보유형 권장 범위" 컬럼에서 사용한다. 3유형 ↔ 9유형 직접 매핑은 없다.

## §1. 이문호공 10요인 본체

### 1.1 체계 정의

본 채널 이문호공 = **9 기본 요인 + 1 신규 요인** 총 10요인.

- **9 기본 요인:** 지식채널 단순화 원본 추출. 본문 ML·해소·서사 후킹 본체.
- **1 신규 요인 (10 도덕적 공분):** 썸네일·제목 후킹 영역 보강용. 사회적 부당함·도덕 위반 감정 자극.

### 1.2 10요인 정의 + 작동

| 번호 | 요인 | 정의 + 작동 |
| ---: | --- | --- |
| 1 | **이익 제시형** | 통찰적 이득 제공. 시청자가 이면을 발견했다는 성취감을 느끼게 한다. 금전·물질이 아닌 지적 이득. |
| 2 | **문제 제기형** | 지적 허점을 날카롭게 파고든다. 무지를 꾸짖지 않고, 가려진 진실에 대한 결핍을 자극한다. |
| 3 | **호기심형** | 답을 바로 주지 않고 정보 공백을 만든다. 썸네일·도입부의 의문을 영상 후반부까지 끌고 가며 해결 욕구를 극대화한다. |
| 4 | **공감형** | 공통 인지·상식·경험·일상 장면·감각으로 공감대 형성. 지적 유대감을 만든다. |
| 5 | **상식 파괴형** | 시청자가 진리라 믿는 명제에 의문을 던진다. 인지 부조화 유도. |
| 6 | **반전형** | A라는 결론이 예상되는 상황에서 전혀 다른 B 맥락을 투척. 서사 흐름을 예측 불가능한 방향으로 급격히 전환. |
| 7 | **경고형** | 작은 균열·결핍이 거대한 결과를 초래하는 과정을 대조한다. 위기의식 자극. 지침 예시 문구를 그대로 쓰지 말고 주제에 맞는 새 사례를 생성한다. |
| 8 | **비교형** | A vs B 대조로 비판적 사고 가동. 단독 대상보다 두 대상을 대조할 때 판단 욕구가 날카로워진다. |
| 9 | **최신 정보형** | 기존 상식을 단숨에 구식으로 만드는 새 데이터·미래 변수를 투척. 정보 선점 도파민 + 지적 우월감 동시 자극. |
| 10 | **도덕적 공분형** | 사회적 부당함·도덕 위반 감정 자극. 분노·억울함 유발. 선동이 아닌 사실 기반 서술. |

### 1.3 요인 변주

#### 1.3.1 4 공감형 응용 변주 — 동질감

- 공명·위로 (감정 측면)
- 개인 서사·일상 소재에 적합
- 본체와 동일하게 9 기본 요인 영역에서 사용

#### 1.3.2 9 최신 정보형 응용 변주 — 부정 측면

| 변주 | 정의 | 적용 영역 |
| --- | --- | --- |
| **본체 (긍정)** | 정보 선점 도파민 · 우월감 | 본문 ML·해소·도입 후킹 |
| **부정·현재 (FOMO)** | 다들 아는데 나만 모름 소외감 | 썸네일·도입 후킹 |
| **부정·과거 (시간적 후회)** | 진작 몰랐던 데 대한 자책 | 썸네일·도입 후킹 |

> **변주 운영 규칙:** 부정 변주(FOMO·시간적 후회)는 **썸네일·도입 후킹 한정**. 본문에서는 9 본체로 통합 운영한다.

### 1.4 적용 범위

| 영역 | 사용 요인 |
| --- | --- |
| 본문 ML·해소 | 9 기본 요인 중심 (10번 도덕적 공분은 공분 소환 필요 지점에서만 제한적 사용) |
| 썸네일·제목 후킹 | 10요인 모두 + 9·4 부정·감정 변주 포함 |

### 1.5 문장 단위 배치 원칙

서사 흐름에 따라 두 배치 방식을 유연하게 조합한다.

| 구분 | 복합 배치 (Arrangement) | 중첩 설계 (Overlap / Layering) |
| --- | --- | --- |
| 방향 | 수평·순차 | 수직·동시 |
| 작용 | 여러 요인을 적절한 위치에 나열 | 한 문장·문맥에 여러 요인을 압축 |
| 집필 사고 | 이번 단계에 A도 넣고 B도 넣어야지 | 이 한 문장으로 A·B·C를 동시에 느끼게 해야지 |
| 시청자 경험 | 자극이 끊이지 않고 이어짐 | 한 순간에 강력한 충격 |

- **중첩 설계 (단일 문장 내 2요인 이상):** 정보 밀도 강화 지점에 사용. 해소 은유 적용 시 자연스럽게 발동 (→ 모듈_이론 §C.1 공식 참조)
- **복합 배치 (문장당 1요인):** 서사 연속성 유지 지점에 사용

## §2. 4역할 카탈로그

### 2.1 4역할 정의

| 역할 | 정의 + 작동 |
| --- | --- |
| **오픈루프** | 지적 공백 생성·질문 제시. 답을 차단하고 궁금증을 유발한다. 도입 후킹·메인 오픈루프·중간 미니 오픈루프 3종으로 발동. |
| **의도적 지연** | 예상 비틀기·추가 정보 투입. 단서만 공급하고 결정적 답은 늦춘다. 단독보다 보상·오픈루프 결합으로 작동. |
| **보상 투하** | Aha-moment·역전 팩트·통찰 회수. 누적된 공백을 한 번에 또는 다단으로 푼다. 통념 정정형·시각 증거형·친근 변환형 3종으로 발동. |
| **충전·빌드업** | 다음 보상을 위한 사전 누적. 정점 직전·영역 진입 직전·도입 후킹 토대 단계에서 집중. |

### 2.2 4역할 발동 운영 원칙

- 보상·충전·오픈루프 3역할이 본문에 골고루 분포하도록 배치. 의도적 지연은 단독보다 보상·오픈루프와 결합 발동.
- 권장 조합은 §4.4 종합표로 일원화.

> **양식별 변주 허용:** 다층분기수렴형 같은 누적 수렴 구조는 충전 비중이 자연스럽게 높아진다. 8엔진별 권장 비중 변주는 모듈_엔진 참조.

### 2.3 4역할 × 12 언어장치 결합

12 언어장치 × 4역할 결합 매트릭스는 **모듈_어법 본체 카탈로그**에 위임한다. 본 모듈에서는 4역할 정의·발동 원칙만 유지한다.

> 욕설 수준 표현은 전 역할 금지 (모듈_이론·CLAUDE.md 정합).

### 2.4 회수 다단 원칙

메인 오픈루프 한 질문을 [승]·[전]에 분산 회수하는 다단 패턴 권장. 매 회수마다 의미를 강화한다.

> **운영 가이드:** 메인 오픈루프가 오래 회수되지 않으면 중간 부분 회수를 배치한다.

## §3. 9유형 × 4위치 매핑

### 3.1 4위치 강세 정보유형 권장 골격

본문 블록 작성 시 각 위치별 권장 정보유형 배치. 주제·엔진별 변주 가능.

#### 3.1.1 [기] 도입 — 정의·개념 + 해석·결산 양강

- 권장 골격: 정의·개념 + 해석·결산 (도입 표준 골격)
- 보조: 통계·수치 / 사례·일화 / 기원·배경
- 회피: 인과·메커니즘·비교·대비 도입 진입 (도입 추론 부담 가중)

#### 3.1.2 [승] 빌드업 — 인과 + 메커니즘 + 사례·일화 3강

- 권장 골격: 인과 + 메커니즘 + 사례·일화 (빌드업 표준 골격)
- 보조: 기원·배경 / 통계·수치 / 비교·대비
- 자제: 해석·결산 — 빌드업 중에는 사실·작동 우선

#### 3.1.3 [전] 전환 — 인과·사례 유지 + 해석·결산 등장

- 권장 골격: 인과 + 사례·일화 (전환 추론 유지)
- 추가: 해석·결산 (전환점 의미 표면화) / 메커니즘 (보조)
- 핵심 신호 장치: 비교·대비 — [전]에서 새 관점 도입을 통한 전환 신호로 활용

#### 3.1.4 [결] 결산 — 해석·결산 절대 우위

- 권장 골격: 해석·결산 본체. 보조 통계·인과는 메시지 뒷받침 한정.
- 원칙: 결산은 새 정보 도입 없이 의미 응축에 집중.

### 3.2 9유형 정의 (정보 유형)

본문 블록의 정보 분류 단위.

| 유형 | 정의 |
| --- | --- |
| **기원·배경** | 대상이 시작된 시점·맥락 (역사·발생 경로) |
| **정의·개념** | 대상이 무엇인가 (개념·범위 규정) |
| **통계·수치** | 정량 데이터 (퍼센트·건수·금액) |
| **인과** | 원인→결과 사슬 (왜 이렇게 됐는가) |
| **사례·일화** | 구체 케이스 (실제 사건·인물 일화) |
| **비교·대비** | A vs B (두 대상 대조) |
| **인용·증언** | 권위 발언·역사 기록 인용 |
| **메커니즘** | 작동 원리 (어떻게 작동하는가) |
| **해석·결산** | 화자의 종합 판정 (의미·메시지) |

### 3.3 9유형별 권장 위치 + 회피 위치

| 유형 | 권장 위치 | 회피 위치 |

## §4. 본문 ML·해소 + 썸네일·제목 적용 가이드

### 4.1 본문 ML — 후킹 충분성 권장

ML(메인 루프)은 도입부 말미에서 본문이 답할 핵심 질문을 공식화하는 단위다.

| 항목 | 권장 운영 |
| --- | --- |
| **후킹 강도** | 시청자 궁금증이 강하게 발생할 만큼 여러 후킹 요인이 결합되어야 한다. 자연스러운 궁금증 유발이 요인 카운팅보다 우선. |
| **활용 요인 영역** | 9 기본 요인 중심. 10 도덕적 공분은 공분 소환 필요 지점에서만 제한적 사용 |
| **권장 조합** | §4.4 종합표 "메인 오픈루프" 행 참조 |
| **금지** | 통념 정정 + 반전 단독 의존, 요인 카운트 채우기식 무리한 삽입 |

### 4.2 해소 은유 — 정보 밀도 강화 영역 정의

본문 SL 해소 시점에서 한 줄 각인이 필요할 때, 해소 은유는 정보 밀도를 높여 한 줄로 각인되도록 작성한다. 통상 2요인 결합이 자연스러움.

| 항목 | 본 모듈 정의 영역 |
| --- | --- |
| **밀도** | 한 문장 내 다층 의미 압축 |
| **필수 요인** | 상식 파괴. 단, 과정추적형은 인과 완성·감정 압축으로 대체 가능 |
| **보조 요인** | 주제에 맞는 보조 요인 1개 결합 (모순형은 경고·이미지강화형은 비교·과정추적형은 공감이 자연스러운 경향이나 강제 아님) |

> **분기 안내:** 해소 은유 공식 본체(은유·직유 선택 기준·작동 지점·본체 형식·대체 각인 도구 등)는 → **모듈_이론 §C.1** 참조.

### 4.3 썸네일·제목 후킹

| 항목 | 운영 원칙 |
| --- | --- |
| **활용 요인 영역** | 10요인 모두 + 9 부정 변주(FOMO·시간적 후회) + 4 감정 변주(동질감) |
| **10 도덕적 공분 활용 시점** | 사회적 부당함·도덕 위반 주제. 분노·억울함 반응 유도 |
| **9 부정 변주 활용 시점** | 시청자 정보 비대칭 심리 자극 (소외감·자책) |
| **권장 조합** | §4.4 종합표 "도입 후킹" 행 참조 |

### 4.4 전략 결정 지점별 권장 조합 종합

본문 작성 시 5대 전략 결정 지점별 권장 조합. 의미 단어 표기로 일원화 (숫자 코드는 §1.2 번호로 환산).

| 전략 지점 | 권장 조합 (의미 단어) |
| --- | --- |
| **도입 후킹** ([기] 초입) | 통념 정정 + 호기심 / 반전 + 최신 정보형 / 최신 정보형 + 공감 + 통념 정정 |
| **메인 오픈루프** ([기] 말미) | 호기심 단독 또는 호기심 + 비교·통념 정정·경고·이익 결합 |
| **[전] 진입 (피벗·영역 전환점·시기 전환점)** | 통념 정정 + 반전 / 반전 + 경고 / 반전 + 비교 / 반전 + 도덕 공분 |
| **Aha-moment 정점** | 통념 정정 + 반전 (정점의 표준) / 반전 + 비교 / 반전 + 공감 |
| **엔딩** | 공감 + 반전 / 이익 + 통념 정정 / 도덕 공분 + 공감 |

> **운영 원칙:**
>
> - **통념 정정 + 반전 단독 의존 회피.** 정점의 표준이라 과의존 시 단조로움 발생. 호기심·이익·공감·최신 정보형 보조 요인으로 다양화 권장.
> - 편당 핵심 정점에 상식 파괴 1회 이상 발동 권장.

> **P5 퇴고 점검:** 본 모듈 적용 결과의 P5 검수는 P5 자체 검수 영역에서 진행한다.
