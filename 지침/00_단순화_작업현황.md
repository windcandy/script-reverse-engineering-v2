# 지침 단순화 작업현황

지침 단순화 작업을 이어 진행한다.

작업 목적: 추론 과부화로 인한 지침 망각 현상 해소

## 검토 원칙

**항목별 점검 시 아래 5축 + 3항목을 항상 동시에 분석한다.**

### 5축

- 축 1: 추론 과부화 vs AI 자연 발생
  — AI가 별도 지침 없이도 자연 수행하는 항목인가?
  → 해당 시 삭제 (삭제 A)
- 축 2: AI 자체 추론 해결 가능성
  — 다른 지침의 복잡성 때문에 생긴 보완용 지침인가?
  → 해당 시 근본 지침 단순화 + 함께 삭제 (삭제 B)
- 축 3: 두 축 결합 처리 방향
  — 삭제 A + 삭제 B가 동시에 해당하면 근본 원인부터 제거
- 축 4: 분량 감축 vs 추론 부하 감축 구분
  — 분량이 줄어도 추론 부하가 그대로면 의미 없음
  — 추론 부하가 실제로 줄어야 단순화로 인정
  - 축 4-2: 추론 과부화 영향 정도 (大·中·小)
  - 축 4-3: 파일 위치 정합성
    — 공통 / 유형별 / 04_언어_장치 / P3 / P4 / P5 중 올바른 위치에 있는가?
- 축 5: 협업 파일 동시 점검 필수
  — 수정 시 P4·P5·에이전트·스킬 연동 파일 동시 확인

### 추가 4항목 (B안 항목별 점검 시 항상 병행)

- **순서:** 지침 순서가 실제 집필 흐름과 맞는가?
- **그룹핑:** 연관 항목이 같이 묶여 있는가, 떨어져 있는가?
- **분산:** 같은 내용이 여러 파일에 흩어져 참조를 따라가야 하는 구조인가?
- **중복:** 다른 파일 또는 같은 파일 내 동일 정보가 반복되는가? (본체 단일화 검토)
